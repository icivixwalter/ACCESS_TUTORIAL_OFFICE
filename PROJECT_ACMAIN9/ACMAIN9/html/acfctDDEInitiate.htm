Funzione DDEInitiate

        

È possibile utilizzare la funzione DDEInitiate per iniziare una conversazione DDE (Dynamic Data Exchange) con un'altra applicazione. Tale funzione apre un canale DDE per trasferire i dati da un server DDE a un'applicazione client e viceversa.

Se si desidera ad esempio trasferire dati da un foglio di calcolo Microsoft Excel a un database di Microsoft Access, è possibile utilizzare la funzione DDEInitiate per aprire un canale tra le due applicazioni. In questo esempio Microsoft Access funge da client e Microsoft Excel da applicazione server.

Sintassi

DDEInitiate(applicazione, argomento)

La funzione DDEInitiate utilizza i seguenti argomenti.

Argomento Descrizione
applicazione Espressione stringa che identifica un'applicazione in grado di partecipare a una conversazione DDE. In genere, l'argomento applicazione è costituito dal nome di un file EXE (senza l'estensione EXE) di un'applicazione per Microsoft Windows, come Microsoft Excel.
argomento Espressione stringa costituita dal nome di un argomento riconosciuto dall'argomento dell'applicazione. Per un elenco degli argomenti, consultare la documentazione dell'applicazione.

Osservazioni

La funzione DDEInitiate, se riesce, inizia una conversazione DDE con l'applicazione e l'argomento specificati dai relativi argomenti, quindi restituisce un valore intero Lungo. Quest'ultimo rappresenta un numero di canale univoco che identifica un canale attraverso cui può aver luogo il trasferimento dei dati. Tale numero di canale viene utilizzato in seguito con altre funzioni e istruzioni DDE.

Se l'applicazione non è ancora in esecuzione oppure se non riconosce l'argomento o non supporta DDE, la funzione DDEInitiate restituisce un errore di run-time.

Il valore dell'argomento varia a seconda dell'applicazione specificata dall'argomento applicazione, in quanto i nomi di argomento validi per le applicazioni che utilizzano documenti o file di dati spesso includono il nome di tali file.

Nota   Il numero massimo di canali che è possibile aprire contemporaneamente è determinato da Microsoft Windows e dalla memoria e le risorse disponibili sul computer. Se un canale non è utilizzato, chiuderlo con l'istruzione DDETerminate o DDETerminateAll, per risparmiare le risorse.

Suggerimento   Se è necessario manipolare gli oggetti di un'altra applicazione da Microsoft Access, è possibile utilizzare l'Automazione.
