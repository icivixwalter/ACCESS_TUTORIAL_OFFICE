Inviare comandi a un database SQL utilizzando una query di tipo pass-through

Utilizzando le query di tipo pass-through è possibile inviare comandi direttamente a un server di database ODBC, ad esempio Microsoft FoxPro. Con le query di tipo pass-through vengono utilizzate direttamente le tabelle presenti sul server invece di stabilire un collegamento.

  1. Nella finestra del database scegliere Query dall'elenco Oggetti, quindi fare clic sul pulsante Nuovo sulla barra degli strumenti.

  2. Nella finestra di dialogo Nuova query selezionare Visualizzazione Struttura, quindi scegliere OK.

  3. Senza aggiungere tabelle o query, scegliere Chiudi nella finestra di dialogo Mostra tabella.

  4. Scegliere Specifica SQL dal menu Query, quindi Pass-Through.

  5. Sulla barra degli strumenti fare clic sul pulsante Proprietà per visualizzare la finestra delle proprietà della query.

  6. Nella finestra delle proprietà della query impostare la proprietà Stringa conness. ODBC per specificare informazioni sul database al quale si desidera collegarsi. È possibile digitare le informazioni relative alla connessione o fare clic sul pulsante dei generatori , quindi immettere le informazioni relative al server a cui si sta effettuando il collegamento.

    Per ulteriori informazioni sulla sintassi della query, consultare la documentazione relativa al server di database SQL al quale si sta inviando la query.

  7. Se la query non è del tipo che restituisce record, impostare la proprietà Restituisci record a No.

  8. Nella finestra Query SQL pass-through, digitare la query di tipo pass-through.

  9. Per eseguire la query, fare clic sul pulsante Esegui sulla barra degli strumenti. Per una query di tipo pass-through che restituisce record, è possibile invece fare clic sul pulsante delle visualizzazioni sulla barra degli strumenti.

    Se necessario, viene visualizzata una richiesta di informazioni sul database di Microsoft SQL Server.

    Attenzione   Se si converte una query di tipo pass-through in una query di tipo diverso, quale una query di selezione, l'istruzione SQL immessa andrà perduta.

Note

  • Le query di tipo pass-through sono utili per l'esecuzione di routine memorizzate su un server ODBC.

  • Se non viene specificata una stringa di connessione nella proprietà Stringa conness. ODBC o se viene eliminata una stringa esistente, viene utilizzata la stringa predefinita "ODBC;". Con questa impostazione, viene visualizzata una richiesta di informazioni relative alla connessione ogni volta che si esegue la query.

  • È possibile che alcune query di tipo pass-through restituiscano messaggi oltre a dati. Se la proprietà Registra messaggi della query viene impostata a Sì, viene creata una tabella che contiene i messaggi restituiti. Il nome della tabella è costituito dal nome utente concatenato con un trattino (-) a un numero sequenziale che inizia con 00. Se ad esempio il nome utente predefinito è ADMIN, le tabelle restituite saranno denominate "ADMIN - 00", "ADMIN - 01", e così via.
