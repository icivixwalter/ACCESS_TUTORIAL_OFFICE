Informazioni sulla limitazione o la convalida dei dati

In Microsoft Access vi sono diversi metodi per definire la modalità di immissione dei dati nel database. È possibile ad esempio limitare i dati che possono essere immessi in un campo definendo una regola di convalida per quel campo. Se i dati immessi non sono conformi alla regola, verrà visualizzato un messaggio che indica il tipo di immissioni consentite. Un altro modo per definire l'immissione dei dati è creare una maschera di input per limitare il tipo di valori che possono essere immessi in determinate posizioni del campo. È possibile applicare queste semplici forme di convalida e di limitazione impostando le proprietà dei campi delle tabelle o dei controlli delle maschere.

Nella maggior parte dei casi è preferibile definire la convalida e la limitazione dei dati impostando le proprietà di un campo in visualizzazione Struttura della tabella. In tal modo si risparmia tempo, in quanto ogni volta che viene utilizzato quel campo in una maschera, la relativa regola di convalida e le altre proprietà verranno applicate alle immissioni di dati eseguite utilizzando la maschera.

Se però i dati immessi in un controllo di una maschera non sono associati a un campo di una tabella e si desidera convalidare o limitare ugualmente l'immissione dei dati, è necessario definire quelle proprietà nella maschera. In alcuni casi inoltre è necessario utilizzare delle macro o il codice Visual Basic, Applications Edition insieme a una maschera per eseguire convalide più complesse. È possibile ad esempio decidere di ignorare la regola di convalida in uso o confrontare valori di tabelle diverse.
