Metodo Rename

                 

Il metodo Rename esegue l'azione Rinomina in Visual Basic. Per ulteriori informazioni sul funzionamento dell'azione e dei relativi argomenti, vedere l'argomento della Guida in linea relativo all'azione.

Sintassi

DoCmd.Rename nuovonome[, tipooggetto, nomeprecedente]

Il metodo Rename utilizza gli argomenti riportati di seguito.

Argomento Descrizione
nuovonome Espressione stringa che è il nuovo nome dell'oggetto che si desidera rinominare. Il nome deve seguire le regole per la denominazione degli oggetti di Microsoft Access.
tipooggetto Una delle seguenti costanti intrinseche:
  acDataAccessPage
acDefault (impostazione predefinita)
acDiagram
acForm
acMacro
acModule
acQuery
acReport
acServerView
acStoredProcedure
acTable
nomeprecedente Espressione stringa che è il nome valido di un oggetto del tipo specificato dall'argomento tipooggetto.
  Se si esegue del codice di Visual Basic contenente il metodo Rename in un database di libreria, Microsoft Access cerca l'oggetto con questo nome prima nel database di libreria, quindi nel database corrente.

Osservazioni

Se si omettono gli argomenti tipooggetto e nomeprecedente (la costante predefinita acDefault viene definita per tipooggetto), Microsoft Access rinomina l'oggetto selezionato nella finestra del database. Per selezionare un oggetto nella finestra del database, è possibile utilizzare l'azione SelezionaOggetto o il metodo SelectObject con l'argomento Finestra del database impostato a Sì (True).

Se si omettono gli argomenti tipooggetto e nomeprecedente, non utilizzare la virgola dopo l'argomento nuovonome.
