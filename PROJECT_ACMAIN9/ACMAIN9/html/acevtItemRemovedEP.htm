Evento RimozioneElemento - Routine evento

        

Per creare una routine evento che viene eseguita quando si verifica l'evento RimozioneElemento, seguire la procedura inclusa nella sezione Osservazioni di questo argomento.

Sintassi

Private Sub evtRiferimenti_ItemRemoved(riferimento)

La routine evento RimozioneElemento utilizza gli argomenti riportati di seguito.

Argomento Descrizione
evtRiferimenti Una variabile di oggetto che rappresenta l'insieme References. È necessario dichiarare tale variabile di oggetto in un modulo di classe utilizzando la parola chiave WithEvents.
riferimento L'oggetto Reference che è stato rimosso dall'insieme References.

Osservazioni

La routine evento RimozioneElemento può contenere il codice che si desidera eseguire quando dal progetto viene rimosso un riferimento a una libreria tipi. È possibile rimuovere un riferimento da Visual Basic utilizzando il metodo Remove per rimuovere un oggetto Reference dall'insieme References.

Per creare la routine evento RimozioneElemento:

  1. Utilizzando la parola chiave WithEvents, dichiarare una variabile di oggetto di tipo References nella sezione Dichiarazioni di un modulo di classe. Microsoft Access utilizza tale variabile di oggetto per definire la routine vento per l'evento RimozioneElemento. La riga di codice che segue mostra una tipica dichiarazione WithEvents: 
    Public WithEvents evtRiferimenti As References
  2. Creare una definizione di routine evento all'interno del modulo di classe selezionando il nome della variabile di oggetto References dalla casella Oggetto nella finestra del modulo. Se ad esempio si utilizza la dichiarazione mostrata nel passaggio precedente, nella casella a discesa Oggetto verrà visualizzato evtRiferimenti. Selezionare quindi l'evento RimozioneElemento nella casella Routine.

  3. Aggiungere il codice da eseguire quando si verifica l'evento. Se ad esempio si desidera visualizzare una finestra di messaggio che indica che dal progetto è stato rimosso un riferimento, aggiungere tale codice alla routine evento.

  4. Assegnare l'insieme References alla variabile di oggetto References dichiarata con la parola chiave WithEvents. Tale istruzione di assegnazione deve trovarsi in una routine che viene eseguita prima dell'evento RimozioneElemento.

    Si potrebbe includere ad esempio questa istruzione nell'evento Inizializzazione del modulo di classe in cui è stata dichiarata la variabile di oggetto. Quando quindi si crea un'istanza della classe, la variabile di oggetto viene inizializzata automaticamente. Se si sta utilizzando la variabile di oggetto evtRiferimenti, tale routine evento risulta come segue: 

    Private Sub Class_Initialize()
    Set evtRiferimenti = Application.References
End Sub

Nota   Se è già stata eseguita la procedura precedente per creare una routine evento per l'evento AggiuntaElemento, non è necessario ripeterla per l'evento RimozioneElemento. È possibile utilizzare la stessa variabile di oggetto References per entrambi gli eventi.

Al fine di attivare l'evento RimozioneElemento, è necessario rimuovere il riferimento esistente utilizzando la variabile di oggetto References dichiarata con la parola chiave WithEvents.

Per rimuovere un riferimento e attivare l'evento:

  1. All'interno di un modulo standard, creare una nuova istanza della classe in cui è stata dichiarata la variabile di oggetto References. Se ad esempio il nome della classe è RefEvents, è possibile creare una nuova istanza della classe e assegnarla a una variabile di oggetto con l'istruzione di dichiarazione riportata di seguito. Tale istruzione può esistere a livello di modulo o in una routine. 
    Dim objEventiRif As New RefEvents
  2. In una routine Sub o Function, creare un nuovo oggetto Reference utilizzando il metodo Remove della variabile di oggetto che rappresenta l'insieme References (evtRiferimenti). Poiché l'evento RimozioneElemento nel modulo di classe è definito in associazione a una determinata variabile di oggetto References, è necessario creare il nuovo oggetto Reference utilizzando tale variabile. In caso contrario, l'evento non si verifica.

Tenere presente che è necessario qualificare la variabile di oggetto References con il nome della classe in cui è dichiarata.

Dopo aver completato i passaggi riportati in precedenza, è possibile rimuovere un riferimento esistente con una routine quale quella che segue:

Function RimuoveRiferimento(strNomeRif As String)
    Dim ref As Reference
    ' Crea nuova istanza della classe RefEvents.
    Dim objEventiRif As New RefEvents

    ' Crea un riferimento sulla variabile dell'insieme
    ' References definito nella classe RefEvents.
    Set ref = objEventiRif.evtRiferimenti(strNomeRif)
    objEventiRif.evtRiferimenti.Remove ref
End Function

Note

  • La routine che inizializza la variabile di oggetto References deve avere la stessa area di validità della dichiarazione della variabile di oggetto. Se ad esempio la variabile di oggetto References è stata dichiarata come Private all'interno di un modulo di classe, è necessario inizializzare la variabile in una routine nello stesso modulo di classe.

  • È possibile includere la routine evento RimozioneElemento solo in un modulo di classe.

  • Una volta dichiarata una variabile a livello di modulo per rappresentare l'insieme References, è possibile utilizzare tale variabile per creare definizioni di routine evento per entrambi gli eventi RimozioneElemento e AggiuntaElemento.

  • Se all'interno di un singolo progetto sono presenti più routine evento RimozioneElemento, ognuna di queste verrà eseguita quando si verifica l'evento, ma non verrà seguito nessun ordine specifico. È consigliabile quindi creare in un progetto solo una routine evento RimozioneElemento e includere in tale routine tutto il codice da eseguire quando l'evento si verifica.

  • L'evento RimozioneElemento si verifica dopo che il riferimento è stato rimosso dal progetto.
