Proprietà ResyncCommand (Comando Resync)

                 

È possibile utilizzare la proprietà ResyncCommand per specificare o determinare l'istruzione SQL o la stored procedure che verrà utilizzata nello snapshot aggiornabile di una tabella.

Impostazione

La proprietà ResyncCommand è un'espressione stringa che rappresenta un'istruzione SQL o una stored procedure a cui vengono applicati dei parametri dalle colonne chiave della Tabella univoca del cursore di output, utilizzando i segni interrogativi (?) come indicatori di parametro.

È possibile impostare questa proprietà utilizzando la finestra delle proprietà o Visual Basic.

Osservazioni

I parametri devono corrispondere per numero e ordine al set di colonne chiave della tabella, identificato dalla proprietà UniqueTable. La proprietà ResyncCommand serve a inserire i valori "corretti" di una riga di set di record dopo che è stato eseguito un aggiornamento, che può essere anche un aggiornamento ad una colonna di join. Per ulteriori informazioni, vedere il punto 1 dell'esempio.

Per le pagine di accesso ai dati e le maschere basate su visualizzazioni o su query SQL senza parametri e contenenti un join, se la proprietà ResyncCommand ha valore null, Microsoft Access determina una query appropriata da usare per ripetere la sincronizzazione. Per le pagine di accesso ai dati e le maschere basate su stored procedure o istruzioni SQL con parametri, Access non è in grado di determinare una query di ripetizione di sincronizzazione appropriata in fase di esecuzione: in questo caso l'utente deve fornire la stringa ResyncCommand perché la correzione delle righe avvenga correttamente. Se la proprietà ResyncCommand è vuota ed Access non è in grado di determinare la query appropriata da usare, dopo un aggiornamento o un inserimento avrà luogo l'operazione predefinita di ripetizione della sincronizzazione ADO per visualizzare i valori correnti.
