Fare riferimento a un oggetto o alle relative proprietà nelle espressioni

È possibile utilizzare un identificatore in un'espressione per fare riferimento a un oggetto o alle relative proprietà. È possibile fare riferimento ad esempio a una maschera o a un report aperto, a un controllo di una maschera o di un report aperto o a una proprietà qualsiasi della maschera, del report o del controllo. L'identificatore riportato di seguito fa riferimento alla proprietà Visibile di un controllo:

Reports![Fattura]![Destinatario].Visible

L'identificatore completo di un oggetto o di una proprietà mostra la relazione fra gli elementi dell'identificatore. In questo identificatore:

  • Reports fa riferimento all'insieme dei report aperti del database. Vengono creati automaticamente gli insiemi Forms e Reports per ciascun database. L'insieme Forms è costituito da tutte le maschere aperte, l'insieme Reports da tutti i report aperti.

  • [Fattura] fa riferimento al report Fattura dell'insieme Reports.

  • [Destinatario] fa riferimento al controllo del report Fattura.

  • Visible fa riferimento alla proprietà Visibile del controllo Destinatario.

Spesso è utile fare riferimento a un oggetto o a una proprietà utilizzando il relativo identificatore completo, che in taluni casi viene richiesto in modo esplicito. Per fare riferimento ad esempio al controllo di una maschera o di un report che non sia la maschera o il report corrente, è necessario digitare il relativo identificatore completo. L'espressione riportata di seguito consente di visualizzare la somma dei valori contenuti nei controlli Subtotale e Trasporto della maschera Ordini in un controllo di una maschera diversa:

= Forms![Ordini]![Subtotale] + Forms![Ordini]![Trasporto]

In alcune circostanze è possibile tuttavia fare riferimento a un controllo o alle relative proprietà senza specificare l'identificatore completo:

  • Se si sta facendo riferimento al controllo di una maschera o di un report corrente, non è necessario specificare l'identificatore completo della maschera o del report. Per visualizzare ad esempio la somma dei valori contenuti nei controlli Subtotale e Trasporto in un altro controllo della stessa maschera, impostare la proprietà Origine controllo del controllo a:

    = [Subtotale] + [Trasporto]

  • Se si sta facendo riferimento al controllo di una sottomaschera o di un sottoreport, non è necessario specificare l'identificatore completo della maschera o del report, utilizzando la proprietà Form o Report. Ad esempio, è possibile utilizzare l'identificatore riportato di seguito per fare riferimento al controllo Quantità della Sottomaschera Ordini:

    Forms![Ordini]![Sottomaschera Ordini]![Quantità]

    L'identificatore completo del controllo Quantità sarebbe:

    Forms![Ordini]![Sottomaschera Ordini].Form![Quantità]

  • In un argomento della macro o dell'azione non è necessario specificare l'identificatore della maschera o del report da cui la macro è eseguita. Se si imposta ad esempio una proprietà di evento di una maschera al nome di una macro, è possibile fare riferimento ai controlli della maschera nella colonna Condizione o nell'argomento dell'azione della macro, senza specificare l'identificatore della maschera.

  • In una routine di Visual Basic, Applications Edition è possibile utilizzare la parola chiave Me, piuttosto che l'identificatore completo, per fare riferimento a un controllo della maschera o del report corrente. Per assegnare ad esempio la somma dei valori dei controlli Subtotale e Trasporto di una maschera alla variabile TotaleOrdine in una delle routine evento della maschera, aggiungere l'istruzione riportata di seguito alla routine evento: 
    TotaleOrdine = Me![Subtotale] + Me![Trasporto]

Note  

  • Quando si esegue una macro o del codice Visual Basic, Applications Edition contenente un'espressione che fa riferimento a una maschera o un report, la maschera o il report devono essere aperti.

  • In una routine di Visual Basic è possibile fare riferimento a un oggetto racchiudendo il suo nome fra parentesi tonde e fra virgolette ("), invece di utilizzare l'operatore punto esclamativo (!). La sintassi a parentesi tonde è necessaria se si desidera utilizzare una variabile in un identificatore. Gli identificatori riportati di seguito ad esempio sono equivalenti: 
    Forms![Ordini]![DataOrdine]
Forms("Ordini")("DataOrdine")
