Risolvere gli errori nel codice Visual Basic

L'esecuzione del debug è la routine utilizzata per rilevare e risolvere gli errori nel codice Visual Basic. Durante l'esecuzione del codice, è possibile che si verifichino tre tipi di errore:

  • Errori di compilazione, che si verificano se il codice è costruito in modo non corretto. È possibile che a un'istruzione non corrisponda l'equivalente istruzione di chiusura come If ed End If o For e Next oppure che sia stato commesso un errore di programmazione che viola le regole di Microsoft Visual Basic, come una parola digitata erroneamente, un separatore mancante o un errore di mancata corrispondenza tra tipi.

    Gli errori di compilazione includono inoltre gli errori di sintassi, ossia gli errori di grammatica o di punteggiatura, comprese parentesi di apertura a cui non corrispondono parentesi di chiusura o un numero errato di argomenti passati a una funzione.

  • Errori di run-time, che si verificano dopo l'inizio dell'esecuzione dell'applicazione. Gli esempi di errori di run-time includono il tentativo di eseguire un'operazione non valida, come la divisione per zero o la scrittura su un file non esistente.

  • Errori logici, che si verificano quando l'applicazione non funziona come si desidera e produce risultati non corretti.

Per contribuire all'individuazione dei tre tipi di errore e per controllare come procede l'esecuzione del codice, in Visual Basic Editor vengono forniti gli strumenti di debug che consentono di analizzare il codice una riga alla volta, di esaminare o controllare i valori delle espressioni e delle variabili e di tenere traccia delle chiamate di routine.
