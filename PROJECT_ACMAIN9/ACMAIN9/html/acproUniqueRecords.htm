Proprietà UniqueRecords (Record univoci)

                 

È possibile utilizzare la proprietà UniqueRecords per specificare se restituire solo record univoci basati su tutti i campi dell'origine dati sottostante e non solo quei campi presenti nella query stessa.

Nota   La proprietà UniqueRecords si applica solo a query di comando di accodamento e di creazione tabelle, e a query di selezione.

Impostazione

La proprietà UniqueRecords viene impostata come riportato di seguito.

Impostazione Descrizione
Sì Non restituisce i record duplicati.
No Impostazione predefinita. Restituisce i record duplicati.

È possibile impostare la proprietà UniqueRecords nella finestra delle proprietà della query o in visualizzazione SQL della finestra di query.

Nota   Tale proprietà viene impostata quando si crea una nuova query utilizzando un'istruzione SQL. Il predicato DISTINCTROW corrisponde all'impostazione della proprietà UniqueRecords. Il predicato DISTINCT corrisponde all'impostazione della proprietà UniqueValues.

Osservazioni

È possibile utilizzare la proprietà UniqueRecords quando si desidera omettere dati basati su interi record duplicati e non solo su campi duplicati. Un record viene considerato univoco a patto che il valore in un campo del record sia diverso dal valore dello stesso campo in un altro record.

La proprietà UniqueRecords funziona solo quando viene utilizzata più di una tabella nella query e vengono selezionati campi dalle tabelle utilizzate nella query. La proprietà UniqueRecords viene ignorata se la query include una sola tabella.

Le proprietà UniqueRecords e UniqueValues sono correlate, in quanto è possibile impostare a Sì solo una delle due proprietà alla volta. Quando si imposta a Sì la proprietà UniqueRecords, ad esempio, la proprietà UniqueValues viene automaticamente impostata a No. È tuttavia possibile impostare entrambe le proprietà a No. In tal caso vengono restituiti tutti i record.
