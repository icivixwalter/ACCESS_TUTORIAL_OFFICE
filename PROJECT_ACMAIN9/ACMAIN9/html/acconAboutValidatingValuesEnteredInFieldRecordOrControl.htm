Informazioni sulla convalida dei valori immessi in un campo, in un record o in un controllo

Per convalidare un valore immesso in un campo, in un record o in controllo, creare una regola di convalida, quindi definire il testo che si desidera visualizzare quando i dati immessi non soddisfano tale regola. È possibile ad esempio creare una regola per cui tutte le date immesse devono fare riferimento al 1998.

In Microsoft Access vi sono diversi metodi per definire le regole di convalida. In questo argomento verranno fornite informazioni di riferimento relative a:

Regole di convalida dei campi

Per essere certi che i dati vengano immessi correttamente in un campo, è necessario definire una regola di convalida dei campi. Questo tipo di regole verifica il valore immesso in un campo quando l'utente lo lascia. Se la regola non è stata rispettata, viene visualizzato un messaggio definito secondo le proprie necessità. La regola di convalida del campo viene definita in una tabella.

Regole di convalida dei record

Una regola di convalida dei record controlla quando è possibile salvare un record. A differenza delle regole di convalida dei campi, le regole di convalida dei record possono fare riferimento ad altri campi e, per questo motivo, risultano utili per confrontare valori di campi diversi nell'ambito di una tabella. La regola di convalida del record viene definita in una tabella.

Regole di convalida dei controlli

È inoltre possibile definire una regola di convalida in un controllo di una maschera. Nella maggior parte dei casi è consigliabile definire la regola di convalida per il campo nella tabella, in quanto qualsiasi regola di convalida di questo tipo viene applicata automaticamente a un controllo basato su quel campo, purché il controllo sia stato creato dopo aver definito la regola di convalida in visualizzazione Struttura della tabella. È necessario però definire la regola di convalida nel controllo se questo non è associato o è basato su dati contenuti in una tabella collegata (esterna). È necessario definire la regola di convalida nel controllo anche se questa fa riferimento a campi o a controlli di altre maschere oppure se contiene funzioni di aggregazione sui domini, funzioni di aggregazione, funzioni definite dall'utente oppure le funzioni CurrentUser o Eval.

Convalida a livello di maschera mediante macro o Visual Basic, Applications Edition

In alcuni casi è necessario utilizzare le macro o il codice Visual Basic, Applications Edition insieme a una maschera per eseguire convalide più complesse. È possibile ad esempio decidere di ignorare la regola di convalida in uso in determinate circostanze o di confrontare i valori di tabelle diverse.
