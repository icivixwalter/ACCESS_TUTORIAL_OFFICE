Importare o collegare tabelle da un altro database di Microsoft Access o da un progetto di Microsoft Access

Nota   È possibile collegare una tabella solo in un database di Microsoft Access, non in un progetto di Microsoft Access.

  1. Aprire un database oppure passare alla finestra del database aperto.

  2. Per importare delle tabelle, scegliere Carica dati esterni dal menu File, quindi Importa.

    Per collegare delle tabelle, scegliere Carica dati esterni dal menu File, quindi Collega tabelle.

  3. Assicurarsi che sia selezionato Microsoft Access (*.mdb;*.adp;*.mdw;*.mda;*.mde;*.ade) nella casella Tipo file della finestra di dialogo Importa o Collega.

  4. Fare clic sulla freccia alla destra della casella Cerca in, selezionare l'unità e la cartella in cui è memorizzato il database di Access con estensione mdb o il progetto di Access con estensione adp che si desidera importare o collegare, quindi fare doppio clic sull'icona del database.

    Importante   Se ci si collega a un file in una rete locale, assicurarsi di utilizzare un percorso UNC (Universal Naming Convention) invece di utilizzare la lettera identificativa di un'unità di rete mappata in Esplora risorse di Windows (Gestione risorse in Windows 95 e Windows NT 4.0). La lettera di un'unità può variare da computer a computer o non essere definita, mentre un percorso UNC è un modo affidabile e coerente per individuare l'origine dati che contiene la tabella collegata.

  5. Nella finestra di dialogo Importa oggetti selezionare le tabelle che si desidera importare o collegare.

    Se si sta effettuando un'importazione e si desidera importare soltanto le definizioni delle tabelle selezionate e non i dati in esse contenuti, scegliere Opzioni, quindi fare clic su Solo definizione nel riquadro Importa tabelle. Per ulteriori informazioni, vedere l'argomento relativo all'importazione di altri oggetti di database.

Note

  • Se si importa una tabella già collegata, Microsoft Access non importa i dati, ma collega la tabella alla relativa origine dati, ovvero copia le informazioni collegate.

  • Sebbene sia possibile importare tabelle da un progetto di Access in un database di Access, può risultare più efficiente importare la tabella da Microsoft SQL Server. Se si stanno importando tabelle da un progetto di Access a un altro, è consigliabile l'utilizzo di DTS (Data Transformation Services), un metodo più rapido e flessibile per l'importazione e la trasformazione dei dati. Per ulteriori informazioni, consultare la documentazione di Microsoft SQL Server.

  • Quando si importa una tabella in un progetto di Access, vengono importati solo i dati e le definizioni dei dati, non le proprietà, i vincoli, le relazioni e gli indici.

  • Se il database che si desidera importare o collegare ha una password di database, è necessario fornirla prima di continuare. Il collegamento delle tabelle di tale database può avere conseguenze non previste. Per ulteriori informazioni, fare clic su .

  • Se si collegano due tabelle dello stesso database di Access, qualsiasi relazione stabilita fra le due tabelle nell'altro database rimane attiva.

  • Se si importa una tabella contenente dei campi di ricerca in un database di Access, si consiglia di importare anche le tabelle o le query a cui fanno riferimento i campi di ricerca. Se non si effettua questa operazione, all'apertura della tabella importata in visualizzazione Foglio dati, verrà visualizzato un messaggio di errore per ciascuna tabella o query mancante. Per risolvere questo problema, è necessario importare la tabella o la query. Se non è possibile o non si desidera importare le tabelle o le query mancanti, aprire la tabella importata in visualizzazione Struttura, selezionare un campo di ricerca che fa riferimento a una tabella o a una query mancante, fare clic sulla scheda Ricerca, quindi impostare a Casella di testo la proprietà Visualizza controllo. Ripetere questa procedura per ciascun campo di ricerca che fa riferimento a una tabella o query mancante.
