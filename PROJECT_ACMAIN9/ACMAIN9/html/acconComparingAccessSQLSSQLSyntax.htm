Confronto tra la sintassi SQL di Microsoft Access e la sintassi SQL di SQL Server

Nella tabella riportata di seguito vengono riassunte le più importanti differenze tra la sintassi SQL di Microsoft Access e la sintassi SQL di Microsoft SQL Server e viene illustrato come vengono gestite queste differenze dall'Upsize guidato. Queste informazioni di riferimento sono utili anche per convertire manualmente la sintassi SQL da un'applicazione di database di Microsoft Access a un'applicazione di progetto di Microsoft Access.

Elemento di sintassi SQL Microsoft
Access		 Microsoft SQL
Server 		Upsize guidato
Identificatori Limite di 64 caratteri.

Consente l'utilizzo di parole chiave e caratteri speciali.

Può iniziare con qualsiasi carattere.

 SQL Server 6.5:
  • Limite di 30 caratteri.

  • Non consente l'utilizzo di parole chiave e caratteri speciali.

  • Deve iniziare con un carattere alfabetico.

SQL Server 7.0 è compatibile con gli identificatori di Access.

 Esegue la conversione nei nomi di SQL Server, se necessario.
Campi di output Consente l'utilizzo dello stesso nome per più campi di output. I nomi di campi di output multipli non sono supportati nelle visualizzazioni. Aggiunge un alias per i nomi di colonne doppi.
Separatore data Segno di cancelletto (#) Apostrofo (') Converte il separatore.
Costanti booleane True, False; On, Off; Yes, No Numeri interi: 1 (true), 0 (false) Converte la costante.
Concatenazione di stringhe E commerciale (&) Segno più (+) Converte il carattere.

Nota   In SQL Server il carattere E commerciale (&) viene utilizzato per le operazioni AND logiche bit per bit.
Caratteri jolly L'asterisco (*) corrisponde a zero o più caratteri.

Il punto interrogativo (?) corrisponde a un singolo carattere.

Il punto esclamativo (!) indica non in elenco

Il segno di cancelletto (#) indica una sola cifra.

 Il segno di percentuale (%) corrisponde a zero o più caratteri.

Il carattere di sottolineatura (_) corrisponde a un singolo carattere.

L'accento circonflesso (^) indica non in elenco.

Nessun equivalente al segno di cancelletto (#).

 Converte i caratteri.

Converte il segno di cancelletto (#) in [0-9] per indicare una singola cifra.
TOP Include automaticamente dei legami se è presente una proposizione ORDER BY. Non supportato da SQL Server 6.5.

SQL Server 7.0 richiede una proposizione WITH TIES esplicita.

 In SQL Server 6.5 vengono ignorate tutte le query TOP.

In SQL Server 7.0 viene aggiunta una proposizione WITH TIES alla fine di tutte le query TOP che contengono una proposizione ORDER BY.

CREATE INDEX Consente l'utilizzo di indici crescenti e decrescenti.

Consente di dichiarare valori primary, no nulls e ignore nulls.

   Converte l'indice.
DROP INDEX La sintassi è la seguente:

Drop Index <nome indice> ON <nome tabella>

 La sintassi è la seguente:

Drop Index <nome tabella>, <nome indice>

 Non esegue la conversione.
DISTINCTROW Supportato (consente la selezione di record univoci). Non supportato. Non esegue la conversione.
OWNERACCESS Supportato (controlla le autorizzazioni durante l'esecuzione). Non supportato. Rimuove dall'istruzione SQL.
Tabella in UNION Supportato. Consente di specificare le tabelle utilizzando la sintassi:

TABLE <nome tabella>

Non supportato. Non esegue la conversione.
ORDER BY nelle unioni Supportato. Consente più ordini per proposizioni nelle query di unione. Supportato. Consente un solo ordine per proposizione alla fine dell'istruzione. Non esegue la conversione.
TRANSFORM Supportato. Utilizzato per le query di tabelle incrociate Non supportato Non esegue la conversione.
PARAMETERS Supportato (Registrato in SQL). Non supportato. Rimuove i parametri espliciti dalla stringa SQL.
