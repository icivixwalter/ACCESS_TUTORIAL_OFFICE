Pubblicare file HTML generati dal server su un server Web

Per utilizzare file HTML generati dal server (ASP o IDC/HTX) su un server Web, attenersi alla seguente procedura:

  1. Installare il software appropriato nel computer che diventerà il server Web ed elaborerà i file HTML generati dal server.

  2. Creare sotto la cartella principale una cartella in cui memorizzare i file ASP o IDC/HTX. La cartella principale predefinita è \Webshare\Wwwroot per Personal Web Server e \Inetpub\Wwwroot per Microsoft Internet Information Server.

  3. Definire i privilegi appropriati per la cartella: Esegui per Windows NT 4.0 o versioni successive ed Esegui gli script per Windows 95 o versioni successive. Per ulteriori informazioni sulla condivisione di una cartella, vedere la Guida di Windows.

  4. Copiare nella cartella i file ASP o IDC/HTX, nonché i file correlati (quali oggetti grafici o file collegati) e le relative cartelle oppure assicurarsi che i file correlati possano essere individuati dal software del server Web. Per ulteriori informazioni, vedere l'argomento relativo alla gestione dei collegamenti e delle pagine Web.

  5. Copiare il database di Microsoft Access o il database di Microsoft SQL Server nella cartella oppure definirne il percorso di rete nell'origine dati ODBC definita nel passaggio 6.

  6. Definire l'origine dati ODBC come DSN di sistema sul server Web. Assicurarsi che il nome dell'origine dati corrisponda a quello immesso nella finestra di dialogo Opzioni di output appropriata al momento dell'invio dei file ASP o IDC/HTX come output. Per ulteriori informazioni, vedere l'argomento relativo alla definizione di un'origine dati ODBC.

  7. Se lo si desidera, definire la protezione del database per l'origine dati ODBC, ma assicurarsi che gli utenti possano accedere a quest'ultima.

    Per un database di Microsoft Access definire un nome utente e una password di protezione a livello utente per consentire agli utenti di accedere al database di Access dalla pagina Web. Se non si definisce un nome utente o una password di protezione a livello utente, verrà utilizzato il nome utente predefinito Admin senza password. Per ulteriori informazioni, vedere l'argomento relativo alla protezione a livello utente in un database di Access.

    Per un progetto di Microsoft Access definire nelle caselle Nome utente e Password un nome utente e una password di database per consentire agli utenti di accedere al database di Microsoft SQL Server dalla pagina Web. Se non si definisce un nome utente o una password, verrà utilizzato il nome utente predefinito è Sa senza password. Per ulteriori informazioni, vedere l'argomento relativo alla protezione dei database SQL Server.

    Il nome utente e la password devono corrispondere a quelli immessi nelle caselle Utente connesso come e Password utente della finestra di dialogo Opzioni di output per Microsoft Active Server Pages o Opzioni di output per HTX/IDC visualizzata quando si inviano i file ASP o IDC/HTX come output.

Nota   Se si desidera esportare più fogli dati, è possibile automatizzare l'operazione creando una macro o creando un programma con Visual Basic, Applications Edition. Tali soluzioni si rivelano particolarmente utili quando si devono esportare frequentemente fogli dati per verificare e creare un sito Web.
