Evento AggiuntaElemento - Routine evento

        

Per creare una routine evento che viene eseguita quando si verifica l'evento AggiuntaElemento, seguire la procedura inclusa nella sezione Osservazioni di questo argomento.

Sintassi

Private Sub evtRiferimenti_ItemAdded(riferimento)

La routine evento AggiuntaElemento utilizza gli argomenti riportati di seguito.

Argomento Descrizione
evtRiferimenti Una variabile di oggetto che rappresenta l'insieme References. È necessario dichiarare questa variabile di oggetto in un modulo di classe utilizzando la parola chiave WithEvents.
riferimento L'oggetto Reference che è stato aggiunto all'insieme References.

Osservazioni

La routine evento AggiuntaElemento può contenere il codice che si desidera eseguire quando al progetto viene aggiunto un riferimento a una libreria tipi. È possibile aggiungere un riferimento da Visual Basic utilizzando il metodo AddFromFile o AddFromGUID per aggiungere un oggetto Reference all'insieme References.

Per creare la routine evento AggiuntaElemento:

  1. Utilizzando la parola chiave WithEvents, dichiarare una variabile di oggetto di tipo References nella sezione Dichiarazioni di un modulo di classe. Microsoft Access utilizza questa variabile di oggetto per definire la routine evento per l'evento AggiuntaElemento. La riga di codice seguente mostra una tipica dichiarazione WithEvents: 
    Public WithEvents evtRiferimenti As References
  2. Creare una definizione di routine evento all'interno del modulo di classe selezionando il nome della variabile di oggetto References dalla casella Oggetto nella finestra del modulo. Se ad esempio si utilizza la dichiarazione mostrata nel passaggio precedente, nella casella a discesa Oggetto verrà visualizzato evtRiferimenti. Selezionare quindi l'evento AggiuntaElemento nella casella Routine.

  3. Aggiungere il codice da eseguire quando si verifica l'evento. Se ad esempio si desidera visualizzare una finestra di messaggio che indica che al progetto è stato aggiunto un riferimento, aggiungere tale codice alla routine evento.

  4. Assegnare l'insieme References alla variabile di oggetto References dichiarata con la parola chiave WithEvents. Tale istruzione di assegnazione deve trovarsi all'interno di una routine che viene eseguita prima che si verifichi l'evento AggiuntaElemento.

    Si potrebbe ad esempio includere tale istruzione nell'evento Inizializzazione del modulo di classe in cui è stata dichiarata la variabile di oggetto. Quando si crea quindi un'istanza della classe, la variabile di oggetto viene inizializzata automaticamente. Se si sta utilizzando la variabile di oggetto evtRiferimenti, questa routine evento risulta come riportato di seguito: 

    Private Sub Class_Initialize()
    Set evtRiferimenti = Application.References
End Sub

Nota   Se per creare una routine evento per l'evento RimozioneElemento, è già stata eseguita la procedura precedente non è necessario ripeterla per l'evento AggiuntaElemento. Per entrambi, è possibile utilizzare la stessa variabile di oggetto References.

Per attivare l'evento AggiuntaElemento, è necessario creare il nuovo riferimento utilizzando la variabile di oggetto References dichiarata con la parola chiave WithEvents.

Per aggiungere un riferimento e attivare l'evento:

  1. All'interno di un modulo standard, creare una nuova istanza della classe in cui è stata dichiarata la variabile di oggetto References. Se ad esempio il nome della classe è RefEvents, è possibile creare una nuova istanza della classe e assegnarla a una variabile di oggetto con l'istruzione di dichiarazione riportata di seguito. Tale istruzione può esistere a livello di modulo o all'interno di una routine. 
    Dim objEventiRif As New RefEvents
  2. In una routine Sub o Function, creare un nuovo oggetto Reference utilizzando il metodo AddFromFile o AddFromGUID della variabile di oggetto che rappresenta l'insieme References (evtRiferimenti). Poiché l'evento AggiuntaElemento nel modulo di classe è definito in associazione a una determinata variabile di oggetto References, è necessario creare il nuovo oggetto Reference utilizzando tale variabile. In caso contrario, l'evento non si verifica.

Tenere presente che è necessario qualificare la variabile di oggetto References con il nome della classe in cui è dichiarata.

Dopo aver completato i passaggi riportati in precedenza, è possibile aggiungere un nuovo riferimento con una routine quale quella riportata di seguito:

Function AggiungiRiferimento(strNomeFile As String)
    Dim ref As Reference
    ' Crea una nuova istanza della classe RefEvents.
    Dim objEventiRif As New RefEvents

    ' Crea un riferimento sulla variabile dell'insieme References
    ' definita nella classe RefEvents.
    Set ref = objEventiRif.evtRiferimenti.AddFromFile(strNomeFile)
End Function

Note

  • La routine che inizializza la variabile di oggetto References deve avere la stessa area di validità della dichiarazione della variabile di oggetto. Se ad esempio la variabile di oggetto References è stata dichiarata come Private all'interno di un modulo di classe, è necessario inizializzare la variabile in una routine all'interno dello stesso modulo di classe.

  • È possibile includere la routine evento AggiuntaElemento solo in un modulo di classe.

  • Una volta dichiarata una variabile a livello di modulo per rappresentare l'insieme References, è possibile utilizzare tale variabile per creare definizioni di routine evento per entrambi gli eventi AggiuntaElemento e RimozioneElemento.

  • Se all'interno di un singolo progetto sono presenti più routine evento AggiuntaElemento, queste verranno eseguite tutte quando si verifica l'evento, seguendo un ordine non definito. È consigliabile quindi creare in un progetto un'unica routine evento AggiuntaElemento e includere tutto il codice da eseguire quando l'evento si verifica in tale routine.

  • L'evento AggiuntaElemento si verifica dopo che il riferimento è stato aggiunto al progetto.
