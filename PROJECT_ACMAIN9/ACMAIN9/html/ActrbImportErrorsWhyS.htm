Errori relativi all'importazione di un foglio di calcolo o di un file di testo

In un database di Microsoft Access, per la maggior parte dei record che provocano un errore, viene aggiunta una riga alla tabella relativa agli errori di importazione. Per visualizzare l'elenco degli errori incontrati, aprire la tabella relativa agli errori di importazione nella finestra del database. Per ulteriori informazioni sui messaggi riguardanti la tabella relativa ad errori di importazione, fare clic su .

Nei progetti di Microsoft Access, non viene creata alcuna tabella relativa agli errori di importazione. È consigliabile l'utilizzo di DTS (Data Transformation Services), un metodo più rapido e flessibile per l' importazione e la trasformazione dei dati. Per ulteriori informazioni, consultare la documentazione di Microsoft SQL Server.

Vengono riportate di seguito le cause più comuni di errore:

  • Dei dati di un campo non possono essere memorizzati nel tipo di dati assegnato a quel campo particolare. Un valore di testo può essere stato incluso accidentalmente, per esempio, in un campo che dovrebbe contenere soltanto dei numeri o delle date, oppure una riga del file di testo o del foglio di calcolo contiene delle informazioni di riepilogo o dei caratteri estranei.

    Se si ritiene che il programma abbia assegnato un tipo di dati corretto per tale campo, modificare il file di testo o il foglio di calcolo per correggere gli errori, quindi effettuare nuovamente l'importazione. In caso contrario, effettuare nuovamente l'importazione e specificare il tipo di dati appropriato.

  • È stato assegnato il tipo di dati non corretto ad un campo. Il tipo di dati viene assegnato in ciascun campo in base ai dati nella prima riga importata. Se, ad esempio, un campo contenente principalmente dei valori di testo ha un numero nella prima riga, in un database e in un progetto di Access viene assegnato il tipo di dati Numerico e non è possibile importare i record rimanenti.

    Effettuare nuovamente l'importazione e specificare il tipo di dati appropriato durante l'importazione.

  • Una o più righe del file di testo o del foglio di calcolo contengono più campi rispetto alla prima riga. La seconda riga di un file, ad esempio, può avere un carattere di delimitazione di campo aggiuntivo seguito da un valore che non si riesce ad inserire nella nuova tabella.

    Modificare il file di testo o il foglio di calcolo, in modo che ciascuna riga abbia lo stesso numero di campi, quindi importare nuovamente.

  • I campi di tipo data importati da un foglio di calcolo di Microsoft Excel differiscono di quattro anni. Microsoft Excel per Windows utilizza il Sistema Data 1900, in cui i numeri seriali vanno da uno a 65.380 e corrispondono alle date dal 1 gennaio 1900 al 31 dicembre 2078, mentre Microsoft Excel per il Macintosh utilizza il Sistema Data 1904, in cui i numeri seriali vanno da zero a 63.918 e corrispondono alle date dal 1 gennaio 1904 al 31 dicembre 2078.

    Prima di importare i dati cambiare il sistema relativo alla data del foglio di calcolo di Microsoft Excel, oppure, dopo l'importazione dei dati, eseguire una query di aggiornamento utilizzando l'espressione [campo data] + 1462 per correggere le date. Per ulteriori informazioni sul sistema relativo alla data di Microsoft Excel, consultare la documentazione relativa a Microsoft Excel.
