Viene visualizzato un errore ODBC quando viene aperta una tabella o una maschera in un database convertito

L'errore "ODBC - connessione a nomeoriginedati non riuscita" viene visualizzato quando una tabella contenuta nel database Microsoft Access convertito versione 1.x o 2.0 è collegata a un'origine dati ODBC che utilizza un driver e un gestore di driver a 16 bit. Microsoft Access 2000 può aprire solo origini dati ODBC collegate che utilizzano le versioni a 32 bit del Gestore di driver ODBC (Odbc32.dll) e del driver ODBC appropriato, ad esempio la versione a 32 bit del driver ODBC Microsoft SQL Server (Sqlsrv32.dll).

Per correggere questo problema, è necessario creare un nuovo nome di origine dati a 32 bit avente il medesimo nome (DSN) per ciascuna origine dati ODBC collegata al database originale 1.x o 2.0.

  1. Fare clic sul pulsante Start (Avvio in Windows 95 e Windows NT 4.0), scegliere Impostazioni, quindi Pannello di controllo.

  2. Fare doppio clic sull'icona Fonti dati ODBC (32 bit) (in Windows 95 o versione successiva) oppure sull'icona ODBC (in Windows NT Workstation 4.0 o versione successiva).

  3. Creare il nuovo nome origine dati immettendo valori identici ai nome origine dati originale. Per Microsoft SQL Server è ad esempio necessario definire almeno il Nome e il Server per l'origine dati, ma potrebbe essere necessario definire ulteriori valori quali il Nome database. Ulteriori informazioni sulle origini dati.

Se non si è sicuri di quale fosse il nome dell'origine dati originale, aprire il database originale tramite la versione di Microsoft Access con cui è stato creato, aprire la tabella collegata in visualizzazione Struttura e visualizzare il foglio Proprietà tabella. La proprietà Descrizione della tabella contiene la definizione della stringa di connessione ODBC. Il parametro visualizzato dopo DSN= nella stringa di connessione è il nome dell'origine dati. Per visualizzare la parte restante della definizione, aprire il Pannello di controllo di Windows 95 o versione successiva, fare doppio clic sull'icona ODBC (non sull'icona ODBC 32 bit). Nell'elenco Fonti dati (Driver) fare clic sul nome dell'origine dati, quindi scegliere Impostazione per visualizzarne la definizione.
