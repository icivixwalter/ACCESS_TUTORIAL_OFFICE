Azione Salva

        

È possibile utilizzare l'azione Salva per salvare un determinato oggetto di Microsoft Access o l'oggetto attivo se non altrimenti specificato. In alcuni casi è inoltre possibile salvare l'oggetto attivo con un nuovo nome, come con il comando Salva con nome del menu File.

Impostazione

L'azione Salva utilizza gli argomenti riportati di seguito.

Argomento azione Descrizione
Tipo oggetto Il tipo di oggetto che si desidera salvare. Selezionare Tabella, Query, Maschera, Report, Macro, Modulo, Pagina di accesso ai dati, Visualizzazione server, Diagramma o Stored procedure nella casella Tipo oggetto della sezione Argomenti azione nella finestra della macro. Per selezionare l'oggetto attivo, lasciare vuoto questo argomento. Se si seleziona un tipo di oggetto in questo argomento, è necessario selezionare il nome di un oggetto presente nell'argomento Nome oggetto.
Nome oggetto Il nome dell'oggetto da salvare. La casella Nome oggetto visualizza tutti gli oggetti contenuti nel database del tipo selezionato dall'argomento Tipo oggetto. Se si lascia vuoto l'argomento Tipo oggetto, è possibile lasciare vuoto anche questo argomento per salvare l'oggetto attivo o, in alcuni casi, immettere un nuovo nome nell'argomento per salvare l'oggetto attivo con tale nome.

Se si immette un nuovo nome, è necessario che tale nome rispetti le regole per la denominazione degli oggetti di Microsoft Access.

Osservazioni

È possibile utilizzare l'azione Salva con tutti gli oggetti di database che è possibile aprire e salvare esplicitamente. È necessario aprire l'oggetto specificato per l'azione Salva per riscontrare qualsiasi effetto sull'oggetto stesso. L'effetto di questa azione equivale a selezionare un oggetto e a salvarlo scegliendo Salva dal menu File o facendo clic sul pulsante Salva sulla barra degli strumenti. Lasciando vuoto l'argomento Tipo oggetto e immettendo un nuovo nome nell'argomento Nome oggetto si ottiene lo stesso risultato che scegliendo Salva con nome dal menu File e immettendo un nuovo nome per l'oggetto attivo. L'utilizzo dell'azione Salva consente di specificare un oggetto da salvare e di eseguire il comando Salva con nome da una macro.

Nota   Non è possibile utilizzare l'azione Salva per salvare con un nuovo nome quanto segue:

  • Una maschera in visualizzazione Maschera o in visualizzazione Foglio dati.

  • Un report in Anteprima di stampa.

  • Un modulo.

  • Una visualizzazione del server in visualizzazione Foglio dati o in Anteprima di stampa.

  • Una pagina di accesso ai dati in visualizzazione Pagina.

  • Una tabella in visualizzazione Foglio dati o in Anteprima di stampa.

  • Una query in visualizzazione Foglio dati o in Anteprima di stampa.

  • Una stored procedure in visualizzazione Foglio dati o in Anteprima di stampa.

L'azione Salva, sia che venga eseguita tramite una macro nel database corrente, sia nel database di libreria, salva sempre l'oggetto specificato o l'oggetto attivo nel database in cui è stato creato l'oggetto stesso.

Se si salva l'oggetto attivo con un nome nuovo, ma il nome è uguale a quello di un oggetto esistente dello stesso tipo, è necessario specificare in una finestra di dialogo se si desidera sovrascrivere l'oggetto esistente. Se è stato impostato a No l'argomento Mostra avvisi dell'azione ImpostaAvvisi, la finestra di dialogo non viene visualizzata e l'oggetto esistente viene automaticamente sovrascritto.

Per eseguire l'azione Salva in Visual Basic, utilizzare il metodo Save dell'oggetto DoCmd.
