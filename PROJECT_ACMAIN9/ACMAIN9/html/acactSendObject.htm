Azione InviaOggetto

Azione InviaOggetto

        

È possibile utilizzare l'azione InviaOggetto per includere un determinato foglio dati, una maschera, un report, un modulo o una pagina di accesso ai dati di Microsoft Access in un messaggio di posta elettronica, dove è possibile visualizzarlo e inviarlo. È possibile includere gli oggetti in formato Microsoft Excel 2000 (*.xls), Testo MS-DOS (*.txt), Rich Text Format (*.rtf) o HTML (*.html) in messaggi di Microsoft Exchange, Microsoft Mail, posta di Microsoft Windows per Workgroup o di un'altra applicazione di posta elettronica che utilizza l'interfaccia Microsoft Mail Applications Programming Interface (MAPI).

Se si dispone di un'applicazione di posta elettronica che utilizza il protocollo Vendor Independent Mail (VIM) ed è stata installata una DLL (Mapivim.dll) che converte i messaggi di posta elettronica MAPI nel protocollo VIM, è possibile inviare gli oggetti di Microsoft Access all'applicazione di posta elettronica VIM.

Impostazione

L'azione InviaOggetto utilizza gli argomenti riportati di seguito.

Argomento azione Descrizione
Tipo oggetto Il tipo di oggetto da includere nel messaggio di posta elettronica. Selezionare Tabella per un foglio dati di tipo tabella, Query per un foglio dati di tipo query, Maschera per una maschera o un foglio dati di tipo maschera, Report, Modulo, Pagina di accesso ai dati, Visualizzazione server o Stored procedure nella casella Tipo oggetto della sezione Argomenti azione della finestra della macro. Non è possibile inviare una macro. Se si desidera includere l'oggetto attivo, selezionarne il tipo con questo argomento, ma lasciare vuoto l'argomento Nome oggetto.
Nome oggetto Il nome dell'oggetto da includere nel messaggio di posta elettronica. Nella casella Nome oggetto vengono visualizzati tutti gli oggetti del database corrispondenti al tipo selezionato tramite l'argomento Tipo oggetto. Se entrambi gli argomenti Tipo oggetto e Nome oggetto vengono lasciati vuoti, all'applicazione di posta elettronica viene inviato un messaggio senza alcun oggetto di database.
  Se si esegue una macro contenente l'azione InviaOggetto in un database di libreria, l'oggetto con questo nome viene ricercato prima nel database di libreria, quindi nel database corrente.
Formato output Il tipo di formato che si desidera utilizzare per l'oggetto incluso. Nella casella Formato output è possibile selezionare HTML (*.html), Microsoft Excel (*.xls), Testo MS-DOS (*.txt), Rich Text Format (*.rtf) o Pagina di accesso ai dati di Microsoft Access (*.html). I moduli possono essere inviati solo in formato testo MS-DOS. Le pagine di accesso ai dati possono essere inviate solo in formato HTML. I moduli possono essere inviati solo in formato MS-DOS. Se si lascia vuoto questo argomento, viene richiesto di specificare il formato di output.
  Nota   Non è possibile includere oggetti in formato Microsoft Active Server Pages (*.asp) o Microsoft IIS (*.htx, *.idc) anche se queste selezioni sono visualizzate nell'elenco.
A I destinatari del messaggio i cui nomi si desidera inserire nella riga A del messaggio di posta elettronica. Se si lascia vuoto questo argomento, viene richiesto di specificare i nomi dei destinatari.
  Separare con un punto e virgola (;) i nomi dei destinatari specificati in questo argomento e negli argomenti Cc e Ccn oppure utilizzare il separatore di elenco impostato sulla scheda Numeri della finestra di dialogo Proprietà - Impostazioni internazionali del Pannello di controllo di Windows. Se l'applicazione non riesce a identificare i nomi dei destinatari, il messaggio non viene inviato e si verifica un errore.
Cc I destinatari del messaggio i cui nomi si desidera inserire nella riga Cc ("copia per conoscenza") del messaggio di posta elettronica. Se si lascia vuoto questo argomento, la riga Cc del messaggio di posta elettronica rimane vuota.
Ccn I destinatari del messaggio i cui nomi si desidera inserire nella riga Ccn ("copia per conoscenza nascosta") del messaggio di posta elettronica. Se si lascia vuoto questo argomento, la riga Ccn del messaggio di posta elettronica rimane vuota.
Oggetto messaggio L'oggetto del messaggio. Il testo viene visualizzato sulla riga Oggetto del messaggio di posta elettronica. Se si lascia vuoto questo argomento, la riga Oggetto del messaggio di posta elettronica rimane vuota.
Testo messaggio Il testo che si desidera includere nel messaggio oltre all'oggetto di database. Tale testo viene visualizzato nel corpo principale del messaggio di posta elettronica, dopo l'oggetto. Se si lascia vuoto questo argomento, non viene aggiunto altro testo al messaggio. Se si lasciano vuoti gli argomenti Tipo oggetto e Nome oggetto, è possibile utilizzare questo argomento per inviare un messaggio senza un oggetto di database.
Modifica messaggio Specifica se il messaggio può essere modificato prima dell'invio. Se si seleziona Sì, l'applicazione di posta elettronica viene avviata automaticamente e il messaggio può essere modificato. Se si seleziona No, il messaggio viene inviato senza che sia possibile modificarlo. L'impostazione predefinita è No.
File Modello Il percorso e il nome del file che si desidera utilizzare come modello per un file HTML. Il file modello è un file contenente tag HTML.

Osservazioni

L'azione InviaOggetto è disponibile solo se sul computer è installata un'applicazione di posta elettronica compatibile MAPI oppure se si dispone di un'applicazione compatibile VIM ed è stata installata la DLL Mapivi32.dll. Per informazioni su come installare il supporto Microsoft Access per le applicazioni di posta elettronica VIM, consultare il Manuale del programmatore di Microsoft Office 2000/Visual Basic. Per informazioni su come ottenere tale pubblicazione, fare clic su .

L'oggetto del messaggio di posta elettronica è nel formato di output selezionato. Facendo doppio clic sull'oggetto, viene avviato il software appropriato con l'oggetto aperto. Microsoft Excel (per i file XLS), Blocco note di Microsoft Windows (per i file TXT) e Microsoft Word (per i file RTF) sono le applicazioni per ciascun tipo di formato di output. Per i file HTML, viene avviato il browser Internet predefinito.

Quando si utilizza l'azione InviaOggetto per includere un oggetto di database nel messaggio di posta elettronica si applicano le seguenti regole:

  • È possibile inviare fogli dati in formato tabella, query e maschera. Nell'oggetto incluso, tutti i campi del foglio dati vengono visualizzati come in Microsoft Access, tranne i campi contenenti oggetti OLE. Le colonne di tali campi sono incluse nell'oggetto ma i campi sono vuoti.

  • Per un controllo associato a un campo di tipo Sì/No (Interruttore, Pulsante di opzione o Casella di controllo), il file di output visualizza il valore -1 (Sì) o 0 (No).

  • Per una casella di testo associata a un campo Collegamento ipertestuale, il file di output visualizza il collegamento ipertestuale per tutti i formati di output tranne Testo MS-DOS, nel qual caso il collegamento ipertestuale viene visualizzato come testo normale.

  • Se si invia una maschera in visualizzazione Maschera, l'oggetto incluso contiene sempre la visualizzazione Foglio dati della maschera.

  • Se si invia un report, gli unici controlli inclusi nell'oggetto sono le caselle di testo per i file XLS oppure le caselle di testo e le etichette per i file RTF, TXT e HTML. Tutti gli altri controlli vengono ignorati. Inoltre, non vengono incluse informazioni sulle intestazioni e i piè di pagina. L'unica eccezione è costituita dal fatto che, quando si invia un report in formato Microsoft Excel, nell'oggetto viene inclusa una casella di testo in un piè di pagina di gruppo contenente un'espressione che utilizza la funzione Sum. Nell'oggetto non viene incluso nessun altro controllo in un'intestazione o in piè di pagina e nessun'altra funzione di aggregazione oltre a Sum.

  • I sottoreport vengono inclusi nell'oggetto, mentre le sottomaschere vengono incluse quando si invia l'output a file ASP, ma solo se si invia l'output come maschera e non come foglio dati.

  • Quando si invia un foglio dati, una maschera o una pagina di accesso ai dati in formato HTML, viene creato un solo file HTML. Quando si invia un report in formato HTML, viene creato un file HTML per ciascuna pagina del report.

Per ulteriori informazioni sulle regole e sulle limitazioni relative all'inclusione di oggetti in formato HTML, vedere Esportare un foglio dati in formato HTML statico e Esportare un report in formato HTML statico.

Suggerimento   Se si inviano i dati di un report e le colonne e le righe di dati dell'oggetto incluso non sono allineate come nel report, considerare i seguenti suggerimenti:

  • Scegliere Dimensione dal menu Formato e quindi Adatta per ridimensionare i controlli.

  • Evitare la sovrapposizione o l'eccessiva vicinanza dei controlli.

  • Scegliere Allinea dal menu Formato, quindi scegliere il comando appropriato per allineare i controlli nel report. I controlli non allineati nella stessa riga possono essere posizionati in righe separate nell'oggetto incluso, aumentando in tal modo la spaziatura.

L'azione InviaOggetto equivale a scegliere Invia dal menu File, mentre gli argomenti dell'azione corrispondono alle impostazioni presenti nelle finestre di dialogo Invia. Il comando Invia tuttavia è valido solo per l'oggetto attivo. Utilizzando l'azione InviaOggetto, è possibile specificare l'oggetto che si desidera inviare.

Nota   È possibile inviare i dati selezionati con il comando Invia. Questa funzione tuttavia non è disponibile con l'azione InviaOggetto.

Per eseguire l'azione InviaOggetto in Visual Basic, utilizzare il metodo SendObject dell'oggetto DoCmd.
