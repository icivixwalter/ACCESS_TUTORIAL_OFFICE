Proprietà Required (Richiesto)

                 

È possibile utilizzare la proprietà Required per determinare se un valore deve essere richiesto in un campo. Se la proprietà è impostata a Sì, quando si immettono dati in un record, è necessario immettere un valore nel campo o in un qualsiasi controllo associato e il valore non può essere Null. È possibile, ad esempio, che si desideri essere certi che il controllo Cognome abbia un valore in ciascun record. Quando si desidera consentire in un campo valori Null, non basta impostare la proprietà Required a No, ma in presenza dell'impostazione della proprietà ValidationRule è necessario che essa riporti esplicitamente "regolaconvalida Or Is Null".

Nota   La proprietà Required non si applica a campi di tipo Contatore.

Impostazione

La proprietà Required viene impostata come riportato di seguito.

Impostazione Visual Basic Descrizione
Sì True (-1) Il campo richiede un valore.
No False (0) Impostazione predefinita. Il campo non richiede un valore.

È possibile impostare questa proprietà per tutti i campi di una tabella, tranne i campi con tipo di dati Contatore, utilizzando la finestra delle proprietà della tabella o Visual Basic.

Nota   Per avere accesso alla proprietà Required di un campo in Visual Basic, utilizzare la proprietà DAO Required.

Osservazioni

La proprietà Required è rafforzata a livello di tabella dal modulo di gestione di database Microsoft Jet. Se si imposta a Sì la proprietà, il campo che ha lo stato attivo deve ricevere o contenere già un valore, quando si immettono dati in una tabella o in una maschera o in un foglio dati basati su una tabella, quando una macro o Visual Basic imposta il valore del campo oppure si importano dati nella tabella.

È possibile utilizzare le impostazioni delle proprietà Required e AllowZeroLength per diversificare le informazioni non esistenti, memorizzate nel campo come stringa a lunghezza zero (" ") e le informazioni probabilmente esistenti ma sconosciute, memorizzate nel campo come valore Null. Se si imposta la proprietà AllowZeroLength a Sì, una stringa a lunghezza zero costituisce un'immissione valida nel campo, senza tener conto dell'impostazione della proprietà Required. Se la proprietà Required viene impostata a Sì e la proprietà AllowZeroLength a No, è necessario immettere nel campo un valore. Una stringa a lunghezza zero non costituisce un'immissione valida.

Suggerimento   È possibile utilizzare una maschera di input quando si immettono dati in un campo, in modo da diversificare la visualizzazione di un valore Null e una stringa a lunghezza zero. È possibile, ad esempio, visualizzare la stringa "Nessuno" al momento dell'immissione di una stringa a lunghezza zero.

Se si imposta la proprietà Required a Sì per un campo di una tabella che contiene già dati, è possibile controllare se il campo ha un valore in tutti i record esistenti. È possibile, tuttavia, richiedere che un valore venga immesso in tale campo in tutti i nuovi record anche se sono presenti nel campo record con valori Null.

Nota   Per rafforzare una relazione tra tabelle correlate che non consentono valori Null, impostare a Sì la proprietà Required del campo chiave esterna della tabella correlata. Il modulo di gestione di database Jet assicura allora la presenza di un record correlato nella tabella principale prima di poter creare un record nella tabella secondaria. Se il campo chiave esterna fa parte della chiave primaria della tabella secondaria, tale operazione non risulta necessaria, poiché non è possibile che un campo chiave primaria contenga un valore Null.
