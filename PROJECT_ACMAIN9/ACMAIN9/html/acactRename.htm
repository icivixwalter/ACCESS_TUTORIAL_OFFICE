Azione Rinomina

        

È possibile utilizzare l'azione Rinomina per rinominare un oggetto di database specificato.

Impostazione

L'azione Rinomina utilizza gli argomenti riportati di seguito.

Argomento azione Descrizione
Nuovo nome Il nuovo nome per l'oggetto di database. Immettere il nome dell'oggetto nella casella Nuovo nome della sezione Argomenti azione nella finestra della macro. L'argomento è obbligatorio.
Tipo oggetto Il tipo di oggetto che si desidera rinominare. Selezionare Tabella, Query, Maschera, Report, Macro, Modulo, Pagina di accesso ai dati, Visualizzazione server, Diagramma o Stored procedure. Per rinominare l'oggetto selezionato nella finestra del database, non immettere alcun valore per questo argomento.
Nome precedente Il nome dell'oggetto da rinominare. La casella Nome precedente visualizza tutti gli oggetti nel database del tipo selezionato dall'argomento Tipo oggetto. Se non si immette alcun valore per l'argomento Tipo oggetto, fare altrettanto per questo.

Nota   Se si esegue una macro contenente l'azione Rinomina in un database di libreria, l'oggetto con tale nome viene cercato innanzitutto nel database di libreria, quindi nel database corrente.

Osservazioni

È necessario che il nuovo nome dell'oggetto di database segua le regole per la denominazione degli gli oggetti di Microsoft Access.

Non è possibile rinominare un oggetto aperto.

Se non si immette alcun valore per gli argomenti Tipo oggetto e Nome precedente, viene rinominato l'oggetto selezionato nella finestra del database. Per selezionare un oggetto nella finestra del database, è possibile utilizzare l'azione SelezionaOggetto con l'argomento Da finestra database impostato a Sì.

È inoltre possibile rinominare un oggetto facendo clic su di esso nella finestra del database, facendo di nuovo clic in modo che sia possibile modificarne il nome e immettendo un nuovo nome. Con l'azione Rinomina non è necessario selezionare innanzitutto l'oggetto nella finestra del database e non è neanche necessario arrestare la macro per immettere il nuovo nome.

Questa azione è diversa dall'azione CopiaOggetto, che crea una copia dell'oggetto con un nuovo nome.

Per eseguire l'azione Rinomina in Visual Basic, utilizzare il metodo Rename dell'oggetto DoCmd.
