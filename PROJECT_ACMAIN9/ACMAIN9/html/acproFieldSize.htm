Proprietà FieldSize (Dimensione campo)

                 

È possibile utilizzare la proprietà FieldSize per impostare la dimensione massima dei dati che possono essere memorizzati in un campo impostato al tipo di dati Testo, Numerico o Contatore.

Impostazione

Se la proprietà DataType è impostata a Testo, immettere un numero compreso tra zero e 255. L'impostazione predefinita è 50.

Se la proprietà DataType è impostata a Contatore, la proprietà FieldSize può essere impostata a Intero lungo o ID replica.

Se la proprietà DataType è impostata a Numerico, le impostazioni della proprietà FieldSize e i relativi valori risultano corrispondenti come indicato nella tabella riportata di seguito.

È possibile impostare la proprietà solo nella finestra delle proprietà della tabella.

Per impostare le dimensioni di un campo in Visual Basic, è necessario utilizzare la proprietà DAO Size per leggere e impostare le dimensioni massime dei campi di tipo Testo. Per tipi di dati diversi da Testo l'impostazione della proprietà ADO Type determina automaticamente l'impostazione della proprietà Size.

Nota   È possibile specificare le dimensioni predefinite di campo per i campi di tipo Testo o Numerico modificando i valori in Dimensioni campi predefinite nella scheda Tabelle/Query, disponibile scegliendo Opzioni dal menu Strumenti.

Osservazioni

Si consiglia di utilizzare l'impostazione della proprietà FieldSize più ridotta possibile, poiché dimensioni di dati più ridotte possono essere elaborate con più velocità e richiedono minore quantità di memoria.

Attenzione   Se si converte un'impostazione della proprietà FieldSize grande a una dimensione inferiore in un campo che contiene già dei dati, è possibile che si verifichi una perdita di dati del campo. Se si modifica, ad esempio, l'impostazione della proprietà FieldSize a un campo di tipo Testo da 255 a 50, i dati eccedenti la nuova impostazione a 50 caratteri verranno scartati.

Se i dati di un campo di tipo Numerico non vengono compresi nella nuova impostazione della proprietà FieldSize, i numeri frazionari potrebbero essere arrotondati o è possibile che si abbia un valore Null. Se, ad esempio, si effettua una modifica del tipo di dati da Precisione singola a Intero, si arrotondano i valori frazionari al numero intero più vicino e i valori superiori a 32.767 o inferiori a -32.768 danno campi contenenti valori Null.

Non è possibile annullare le modifiche apportate ai dati che sono il risultato di una modifica della proprietà FieldSize dopo aver salvato tali modifiche in visualizzazione Struttura della tabella.

Suggerimento   Se si prevede di eseguire molti calcoli in un campo dati contenente da una a quattro posizioni decimali, è possibile utilizzare il tipo di dati Valuta. I campi tipo di dati Precisione singola e Precisione doppia richiedono calcoli a virgola mobile. I campi tipo di dati Valuta utilizzano un calcolo più rapido a virgola fissa.
