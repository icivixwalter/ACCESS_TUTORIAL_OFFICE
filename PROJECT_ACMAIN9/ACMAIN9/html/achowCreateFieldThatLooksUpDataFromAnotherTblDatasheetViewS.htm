Creare un campo che cerca i dati di un'altra tabella in visualizzazione Foglio dati

  1. In visualizzazione Foglio dati aprire la tabella nella quale si desidera aggiungere un campo di ricerca.

  2. Per inserire il campo di ricerca all'interno della tabella, fare clic sulla colonna a destra della quale si desidera inserire il campo, quindi scegliere Ricerca guidata dal menu Inserisci.

  3. Nella prima finestra di dialogo della Ricerca guidata selezionare l'opzione che indica se si desidera cercare i valori in una tabella o query.

  4. Fare clic su Avanti e seguire le indicazioni delle altre finestre di dialogo della Ricerca guidata.

    Al termine della Ricerca guidata, il campo di ricerca verrà aggiunto al foglio dati. Come impostazione predefinita, a tale campo verrà assegnato un nome nel foglio dati, "Campo#", dove # rappresenta un numero. Se si desidera, è possibile cambiare questo nome. Per ulteriori informazioni su come modificare il nome di un campo in visualizzazione Foglio dati, fare clic su .

Note

  • Quando si utilizza la Ricerca guidata per creare un elenco di ricerca, vengono impostate alcune proprietà del campo in base alle scelte che sono state effettuate nella Ricerca guidata. Dopo aver creato un campo con un elenco di ricerca, se lo si aggiunge ad una maschera la relativa definizione verrà copiata nella maschera stessa. La casella combinata o di riepilogo viene creata automaticamente nella maschera. Se nella tabella la definizione di un campo viene modificata dopo aver aggiunto il campo ad una maschera, queste modifiche non avranno alcun effetto nella maschera. Per evitare ciò, eliminare il campo dalla maschera e poi aggiungerlo di nuovo. Per informazioni sulle proprietà che vengono impostate tramite la Ricerca guidata, fare clic su .

  • È inoltre possibile aggiungere un campo di ricerca a una tabella che visualizza i valori della stessa tabella contenente il campo di ricerca. Nella tabella Impiegati del database di esempio Northwind, il campo Superiore è un campo di ricerca che visualizza i dati dei campi Nome e Cognome cercando il corrispondente ID impiegato nella stessa tabella.
