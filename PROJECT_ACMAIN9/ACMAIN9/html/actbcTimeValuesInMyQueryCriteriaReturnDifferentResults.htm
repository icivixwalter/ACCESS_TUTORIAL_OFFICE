I valori di data e ora nei criteri di una query restituiscono risultati diversi

Quando si converte un database di Microsoft Access versione 2.0 in Microsoft Access 2000, le query che contengono criteri basati su determinati valori di ora in campi Data/ora potrebbero restituire risultati diversi rispetto alle versioni precedenti. Questo comportamento può verificarsi inoltre se si collegano tabelle da un database di Microsoft Access versione 2.0 a un database di Microsoft Access 2000. Viene influenzata solo la parte relativa all'ora dei campi Data/ora.

Le query contenenti criteri che specificano date fra il 1900 e il 1929 potrebbero restituire risultati diversi. Per esempio il criterio di data #01/01/15# in Microsoft Access 2.0 e 95 rappresenta il 1° gennaio 1915 mentre in Microsoft Access 97 e versioni successive rappresenta il 1° gennaio 2015. Per risolvere il problema, modificare la data nel criterio specificando il secolo, ad esempio #01/01/1915#.
