Suggerimenti per l'ottimizzazione delle prestazioni di Microsoft Access e del sistema

È possibile ottimizzare le prestazioni di Microsoft Access sul computer seguendo le istruzioni sottoelencate:

  • Se si utilizzano database da non condividere con altri utenti, installare Microsoft Access e tutti i database sull'unità disco rigido invece che su un server di rete.

  • Se si è l'unico utente a utilizzare un database, aprirlo in accesso esclusivo. Nella finestra di dialogo Apri, fare clic sulla freccia accanto al pulsante Apri, quindi scegliere Apertura esclusiva dal menu visualizzato.

  • Rendere disponibile una maggiore quantità di memoria chiudendo le applicazioni che non si stanno utilizzando.

  • Aumentare la RAM del computer. Se si sta utilizzando Microsoft Windows 95 o versione successiva, si consigliano almeno 24 MB di memoria (16 MB più altri 8 MB per Microsoft Access). Se invece si utilizza Windows NT 4.0 o versione successiva, si consigliano almeno 40 MB di memoria (32 MB più altri 8 MB per Microsoft Access).

  • Non utilizzare RAM per un disco RAM.

  • Eliminare periodicamente i file non necessari, svuotare il Cestino e compattare i database, quindi deframmentare il disco rigido con l'Utilità di deframmentazione dischi. Per eseguire l'Utilità di deframmentazione dischi, fare clic sul pulsante Start (Avvio in Windows 95 e Windows NT 4.0), scegliere Programmi, quindi Accessori. Scegliere Utilità di sistema, quindi Utilità di deframmentazione dischi.

  • Nella maggior parte dei casi, l'impostazione predefinita della memoria virtuale utilizzata in Windows consente di ottenere le migliori prestazioni. In alcune situazioni, tuttavia, modificando i parametri della memoria virtuale si possono migliorare le prestazioni. Se si è già tentato di eliminare i file non necessari e persistono problemi relativi alle prestazioni, provare a modificare l'impostazione predefinita della memoria virtuale nei casi riportati di seguito:
    • Non si dispone di spazio sufficiente sull'unità disco utilizzata correntemente per la memoria virtuale e si dispone di spazio necessario su un'altra unità locale.

    • È disponibile un'altra unità locale più rapida dell'unità corrente, a meno che tale disco non sia già utilizzato in modo eccessivo.

    In questi casi, è possibile ottenere migliori prestazioni specificando un'altra unità per la memoria virtuale.

    È inoltre possibile ottenere migliori prestazioni specificando lo spazio su disco disponibile per la memoria virtuale pari almeno a 25 MB meno la RAM disponibile. Se, ad esempio, il computer dispone di 12 MB di RAM, è necessario specificare almeno 13 MB di memoria virtuale. Se si eseguono applicazioni più grandi, è possibile che si desideri specificare una quantità maggiore.

    Per modificare i parametri della memoria virtuale di Windows, nel Pannello di controllo di Windows, fare doppio clic sull'icona Sistema, scegliere la scheda Prestazioni, scegliere Memoria virtuale, quindi scegliere Impostazione manuale della memoria virtuale. Specificare quindi un altro disco rigido oppure immettere un valore nella casella Minima che risulta pari almeno a 25 meno la RAM disponibile.

  • Se si dispone di una bitmap di sfondo a schermo intero sul desktop di Windows, sostituirla con una bitmap a tinta unita o a motivo continuo oppure con nessuna bitmap.

  • Se si utilizza uno screen saver, utilizzarne uno nero oppure nessuno.

  • Eliminare il software di compressione disco oppure spostare i database in un'unità decompressa.

Per ulteriori informazioni sull'ottimizzazione delle prestazioni del database, consultare il capitolo 10 "Ottimizzazione dell'applicazione" nel manuale Creazione di applicazioni con maschere e report.
