Proprietà MDE

                 

È possibile utilizzare la proprietà MDE per determinare se un database è stato salvato come file MDE.

Nota   Questa proprietà è disponibile solo con un database di Microsoft Access (MDB).

Impostazione

Quando un database è salvato come file MDE, Microsoft Access aggiunge la proprietà MDE al database e la imposta al valore Stringa "T".

La proprietà MDE può essere impostata solo utilizzando Visual Basic ed è di sola lettura in tutte le visualizzazioni.

Importante   Il tentativo di impostare la proprietà MDE per un database che non è stato salvato come file MDE genera un errore di run-time 3270 (proprietà non trovata). La proprietà MDE non esiste nel database finché questo non viene salvato come file MDE.

Osservazioni

È possibile convertire un database in un file MDE scegliendo Utilità database dal menu Strumenti, quindi Crea file MDE. Quando si salva un database come un file MDE, tutto il codice sorgente modificabile viene rimosso, tutti i moduli vengono compilati e il database compattato. Il codice Visual Basic continua a funzionare, ma non può essere visualizzato o modificato. Non è possibile aggiungere oggetti relativi ai report o alle maschere, né apportarvi delle modifiche.

Attenzione   Assicurarsi di conservare una copia del database originale. Se si desidera modificare del codice o la struttura degli oggetti in un database MDE, è necessario farlo nel database originale per poi creare un nuovo database MDE. Il database originale è anche necessario per poterlo convertire nel formato delle future versioni di Microsoft Access, in quanto non è possibile effettuare direttamente la conversione di file MDE.

Il salvataggio di un database come file MDE è consigliato quando si desidera evitare modifiche alle maschere o ai report, oppure nascondere il codice Visual Basic agli altri utenti del database. È inoltre possibile convertire un database in un file MDE quando si realizzano delle procedure guidate personalizzate oppure per la parte front end di un'applicazione frontend/backend.

Un database duplicato non può essere salvato come file MDE. Se tuttavia il database è stato salvato come file MDE, può essere duplicato.
