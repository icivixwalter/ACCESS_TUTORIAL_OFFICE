Informazioni su come aggiungere un'immagine non associata a una maschera, report o pagina di accesso ai dati

Per aggiungere un'immagine a una maschera, un report o una pagina di accesso ai dati si segue una procedura diversa a seconda di come deve essere utilizzata l'immagine e se si effettua l'operazione in una maschera, un report o una pagina di accesso ai dati.

In questo argomento verranno fornite informazioni di riferimento relative a:

Maschere e report

Pagine di accesso ai dati

Maschere e report

Il modo in cui un'immagine non associata viene aggiunta dipende da come si desidera visualizzarla, ovvero se come immagine di sfondo (filigrana) che ricopre l'intera finestra o come immagine visualizzata in un controllo. Se si desidera aggiungere un'immagine di sfondo, utilizzare la proprietà Immagine della maschera o del report. Se invece si desidera aggiungere un'immagine che venga visualizzata in un controllo, utilizzare un controllo Immagine o una cornice di oggetto non associato.

L'utilizzo di un controllo Immagine o di una cornice di oggetto non associato dipende dal fatto che si desideri o meno modificare l'immagine direttamente dalla maschera o dal report. Se non è necessario aggiornare l'immagine, questa può essere aggiunta utilizzando un controllo Immagine. Se si intende aggiornare l'immagine, è possibile aggiungerla utilizzando una cornice di oggetto non associato. In seguito, se si desidera modificare l'immagine, si può fare doppio clic sull'oggetto dalla maschera o dal report per aprire l'applicazione utilizzata per creare l'oggetto stesso. Dato che i controlli Immagine vengono caricati più velocemente rispetto alle cornici di oggetti non associati, è probabile che per la maggior parte delle immagini non associate si preferisca utilizzare un controllo Immagine.

È possibile inserire immagini in una maschera o in un report oppure collegarsi a esse. Quando viene inserita in una maschera o in un report, l'immagine viene memorizzata nel database. Un oggetto inserito è sempre disponibile. Se si modifica l'immagine dalla maschera o dal report, l'immagine viene modificata nel database.

Quando viene effettuato un collegamento a un'immagine, è possibile visualizzarla e modificarla dalla maschera o del report, ma le modifiche vengono memorizzate nel file dell'oggetto originale e non nel database. È possibile inoltre modificare il file dell'oggetto separatamente. In questo caso gli ultimi cambiamenti saranno visualizzati alla successiva apertura della maschera o del report. Il collegamento a un'immagine è utile nel caso di file molto grandi che non si intende includere nel database e nel caso di file che si intende utilizzare in più maschere e report. Si noti tuttavia che, se il file di un oggetto collegato viene spostato, sarà necessario ripristinare il collegamento.

Pagine di accesso ai dati

La procedura da seguire per inserire un'immagine non associata in una pagina di accesso ai dati dipende varia a seconda che si desideri che appaia come immagine di sfondo (filigrana) che copre l'intera pagina o come immagine visualizzata in un controllo. Se si desidera aggiungere un'immagine di sfondo in una pagina di accesso ai dati, è possibile farlo utilizzando le opzioni del menu Formato. Se si desidera invece aggiungere un'immagine che venga visualizzata in un controllo, utilizzare il controllo Immagine.

Dopo aver inserito l'immagine nella pagina di accesso ai dati, non sarà più possibile modificarla. Inserendo un'immagine in una pagina di accesso ai dati, l'immagine viene aggiunta ai file di supporto che accompagnano la pagina di accesso ai dati, ma non viene memorizzata direttamente nella pagina. Se pertanto si desidera spostare la pagina di accesso ai dati in una nuova posizione, non copiarla, ma salvarla nella nuova posizione. L'immagine verrà così salvata automaticamente insieme alla pagina di accesso ai dati.
