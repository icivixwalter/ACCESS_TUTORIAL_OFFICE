Informazioni su come utilizzare una password di database e la protezione a livello utente con un database di Microsoft Access salvato come file MDE

La creazione di un file MDE protegge le maschere, i report e il codice Microsoft Visual Basic del database di Microsoft Access, senza richiedere l'accesso agli utenti o senza che sia necessario creare e gestire gli account utente e le autorizzazioni necessarie alla protezione a livello utente. Se tuttavia prima di salvare un database di Access come file MDE si definiscono la password del database o la protezione a livello utente, queste funzioni sono disponibili anche per il file MDE creato da tale database. Nel caso siano state definite la password del database di Access o la protezione a livello utente, l'eventuale eliminazione di tali funzioni deve essere eseguita prima di salvare il database come file MDE. Per ulteriori informazioni sull'eliminazione della password di un database, fare clic su . Per ulteriori informazioni sull'eliminazione della protezione a livello utente, fare clic su .

Prima di procedere al salvataggio come file MDE di un database di Access protetto all'accesso e con protezione a livello utente devono essere soddisfatti i seguenti requisiti:

  • È necessario associarvi il file di informazioni sul gruppo di lavoro che definisce gli account utente utilizzati per accedere al database o che erano in uso al momento della creazione del database.

  • L'account utente deve avere le autorizzazioni Apertura/esecuzione e Apertura esclusiva per il database.

  • L'account utente deve avere le autorizzazioni Modifica struttura o Amministrazione per tutte le tabelle del database oppure è necessario essere il proprietario di tutte le tabelle del database.

  • L'account utente deve avere le autorizzazioni Lettura struttura per tutti gli oggetti del database.
