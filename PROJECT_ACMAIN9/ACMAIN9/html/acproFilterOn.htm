Proprietà FilterOn (Applica filtro)

                 

È possibile utilizzare la proprietà FilterOn per specificare o determinare se viene applicata la proprietà Filter per una maschera o un report.

Nota   Se è stato specificato un filtro su server all'interno di un progetto di Microsoft Access (ADP), utilizzare la proprietà ServerFilterByForm.

Impostazione

La proprietà FilterOn utilizza le impostazioni riportate di seguito.

Impostazione Visual Basic Descrizione
Sì True (-1) La proprietà Filter dell'oggetto viene applicata.
No False (0) Impostazione predefinita. La proprietà Filter dell'oggetto non viene applicata.

Per i report, è possibile impostare la proprietà FilterOn utilizzando la finestra delle proprietà del report oppure Visual Basic.

Per le maschere, è possibile impostare la proprietà FilterOn in una macro o utilizzando Visual Basic. È inoltre possibile impostare la proprietà facendo clic sul pulsante Applica filtro sulla barra degli strumenti della visualizzazione Maschera o sulla barra degli strumenti Filtro/ordinamento.

Osservazioni

Per applicare un filtro salvato, premere il pulsante FilterOn per le maschere o applicare il filtro utilizzando una macro o Visual Basic impostando la proprietà FilterOn a True per maschere o report. Per i report, è possibile impostare la proprietà FilterOn a Sì nella tavola delle proprietà del report.

Il pulsante FilterOn indica lo stato delle proprietà Filter e FilterOn. Il pulsante rimane disattivato fin quando è presente un filtro da applicare. Se è attualmente applicato un filtro esistente, il pulsante FilterOn viene visualizzato incassato. Per applicare automaticamente un filtro all'apertura di una maschera o di un report, nell'impostazione della proprietà di evento OnOpen della maschera, specificare una macro che utilizza l'azione FilterOn o una routine evento che utilizza il metodo ApplyFilter dell'oggetto DoCmd.

È possibile rimuovere un filtro facendo clic sul pulsante incassato Applica filtro, scegliendo Rimuovi filtro/ordinamento dal menu Record o utilizzando Visual Basic per impostare la proprietà FilterOn a False. Per i report, è possibile rimuovere un filtro impostando la proprietà FilterOn a No nella tavola delle proprietà del report.

Nota   Quando si crea un nuovo oggetto, questo eredita le proprietà RecordSource, Filter, ServerFilter, OrderBy e OrderByOn della tabella o della query da cui è stato creato. Per le maschere e i report, i filtri ereditati non vengono automaticamente applicati all'apertura di un oggetto.
