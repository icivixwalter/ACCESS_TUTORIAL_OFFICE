Password di database

Il primo tipo di password è definito "password di database". Se si imposta una password di database, tutti gli utenti devono immettere tale password per poter aprire il database. L'impostazione di una password di database è un modo semplice per impedire l'apertura del database da parte di utenti non autorizzati. Una volta aperto, il database non è però protetto da alcuna altra misura di sicurezza, a meno che non sia stata impostata la protezione a livello utente. Se per il database è stato impostato questo tipo di protezione, è possibile impedire ad altri utenti di impostare una password di database.

Password degli account di protezione

Il secondo tipo di password viene definito "password degli account di protezione" e viene utilizzato unicamente se è stata impostata la protezione a livello utente per un gruppo di lavoro. Con una password dell'account di protezione si impedisce ad altri utenti di connettersi utilizzando quel nome utente.

In base all'impostazione predefinita, viene assegnata una password vuota all'account Amministratore predefinito e a qualsiasi nuovo account utente creato nell'ambito del gruppo di lavoro. Per proteggere un database è importante aggiungere una password per:

  • l'account Amministratore (per attivare la finestra di dialogo Accesso);

  • l'account utente proprietario del database e delle relative tabelle, query, maschere, report e macro;

  • qualsiasi account utente aggiunto al gruppo Amministratori.

È inoltre possibile aggiungere password agli account creati per gli utenti o indicare a questi ultimi di aggiungere password proprie.

Gli utenti possono creare o modificare le loro password dell'account di protezione. Solo un account Amministratore tuttavia può cancellare una password se l'utente la dimentica.

Password di Visual Basic, Applications Edition (VBA)

Il terzo tipo di password viene definito "password di Visual Basic, Applications Edition (VBA)". Questa password può essere utilizzata una volta per sessione per proteggere il codice VBA nei moduli e nei moduli sottostanti alle maschere e ai report. Essa impedisce agli utenti non autorizzati di modificare, tagliare, incollare, copiare, esportare ed eliminare il codice VBA.
