Il tipo di dati e la verifica della convalida determinano l'inserimento di dati in visualizzazione Foglio dati o in visualizzazione Maschera

Quando ci si sposta al di fuori di un campo dopo averne modificato i dati, questi vengono convalidati per verificare che in quel campo sia consentito l'inserimento di tale valore. In caso contrario viene visualizzato un messaggio di avviso e per potersi spostare su un altro campo, è necessario modificare i dati inserendo un valore accettabile oppure annullare le modifiche.

Di seguito sono riportati i motivi per cui potrebbe non essere consentito l'inserimento di un valore in un campo:

  • Il valore non è compatibile con il tipo di dati del campo. Se viene inserito un valore che non corrisponde al tipo di dati, si tenta di convertire il valore nel tipo di dati corretto. Alcuni dati tuttavia non possono essere convertiti in altri tipi di dati. Non è possibile ad esempio memorizzare un nome in un campo con tipo di dati Numerico in un database di Microsoft Access o con tipo di dati smallint in un progetto di Microsoft Access.

  • I dati interrompono un set di regole incluso nella proprietà del campo Valido se.

  • Si è tentato di lasciare vuoto un campo in cui è richiesto l'inserimento di dati.

  • La macro di convalida specificata per la proprietà Prima di aggiornare ha annullato l'aggiornamento.

I dati vengono convalidati anche ci si sposta al di fuori del record. Prima di salvare un record, si verifica che i dati rispettino il set di regole di convalida incluso nella proprietà Valido se. Se inoltre la tabella che si sta modificando dispone di una chiave primaria o di un indice univoco, si verifica che nel campo o nei campi del record modificato non vi sia lo stesso valore di un altro record presente nella tabella. Se il record non può essere salvato, è necessario effettuare tutte le correzioni necessarie oppure fare clic su Annulla per annullare tutte le modifiche apportate al record.
