Definizione e funzionamento delle query di comando

Una query di comando apporta modifiche a molti record con una sola operazione. Esistono quattro tipi di query di comando: di eliminazione, di aggiornamento, di accodamento e di creazione tabella.

Query di eliminazione

Questo tipo di query elimina un gruppo di record da una o più tabelle. È possibile ad esempio utilizzare una query di eliminazione per rimuovere i prodotti di cui è stata interrotta la produzione o per i quali non esistono ordini. Con le query di eliminazione vengono eliminati sempre record interi, non solo determinati campi al loro interno.

Per informazioni sulla creazione di query di eliminazione, fare clic su .

Query di aggiornamento

Questo tipo di query apporta modifiche globali a un gruppo di record in una o più tabelle. È possibile ad esempio aumentare i prezzi del 10% per tutti i prodotti caseari o aumentare gli stipendi del 5% per le persone appartenenti a una determinata categoria lavorativa. Con una query di aggiornamento è possibile modificare dati in tabelle esistenti.

Query di accodamento

Questo tipo di query aggiunge un gruppo di record di una o più tabelle alla fine di una o più tabelle. Si supponga ad esempio che vengano acquisiti nuovi clienti e un database contenente una tabella di informazioni su di essi. Invece di digitare nuovamente tutte queste informazioni, si accodano alla propria tabella Clienti. Le query di accodamento sono inoltre utili per:

  • Accodare campi in base a dei criteri. È possibile ad esempio accodare solo i nomi e gli indirizzi di clienti con ordini di rilievo.

  • Accodare record nel caso in cui alcuni campi di una tabella non esistano in un'altra tabella. Nel database di esempio Northwind nella tabella Clienti esistono 11 campi. Si supponga di voler accodare record di un'altra tabella con campi corrispondenti a 9 degli 11 della tabella Clienti. Una query di accodamento accoderà i dati nei campi corrispondenti e ignorerà gli altri.

Per informazioni sulla creazione di query di accodamento, fare clic su .

Query di creazione tabella

Questo tipo di query crea una nuova tabella in base a tutti i dati o parte di essi contenuti in una o più tabelle. Le query di creazione tabella sono utili per:

  • Creare una tabella da esportare in altri database di Microsoft Access. È possibile ad esempio creare una tabella che contiene diversi campi della tabella Impiegati, quindi esportarla in un database utilizzato dall'ufficio del personale.

  • Creare pagine di accesso ai dati che visualizzano dati riferiti a un determinato momento. Si supponga ad esempio che il 15 maggio 1996 si desideri visualizzare una pagina di accesso ai dati nella quale vengano visualizzati i totali vendite del primo trimestre in base ai dati contenuti nelle tabelle sottostanti al 1° aprile alle ore 9.00. Una pagina di accesso ai dati basata su una query o un'istruzione SQL estrae dalle tabelle i dati più aggiornati (ossia i dati al 15 maggio 1996), piuttosto che quelli riferiti a un determinato momento. Per conservare i dati esattamente come erano alle 9.00 del 1° aprile 1996 è necessario definire una query di creazione tabella riferita esattamente a quel momento che recuperi i record necessari e li memorizzi in una nuova tabella, quindi utilizzare come base della pagina di accesso ai dati questa tabella anziché una query.

  • Creare una copia di backup di una tabella.

  • Creare una tabella cronologica che contiene i vecchi record. È possibile ad esempio creare una tabella che memorizza tutti i vecchi ordini prima di eliminarli dalla tabella Ordini corrente.

  • Migliorare le prestazioni di maschere, report e pagine di accesso ai dati basati su query a tabella multipla o su istruzioni SQL. Si supponga ad esempio di voler stampare più record basati su una query a cinque tabelle con i totali. È possibile rendere più veloce questa procedura definendo una query di creazione tabella che recupera i record necessari e li memorizza in una sola tabella. Quindi basare i report su questa tabella o porre la stessa in un'istruzione SQL come origine record di una maschera, di un report o di una pagina di accesso ai dati in modo che non sia necessario rieseguire la query per ogni report. I dati nella tabella tuttavia sono congelati al momento in cui si esegue la query di creazione tabella.

Per informazioni sulla creazione di query di creazione tabella, fare clic su .
