Oggetto Property, insieme Properties (MDB)

In Microsoft Access sono presenti diversi tipi di proprietà. È possibile rappresentare ciascuna di tali proprietà nel codice di Visual Basic con una variabile di oggetto Property e ciascuna proprietà fa parte di un insieme Properties.

Proprietà applicabili a oggetti DAO

  • Le proprietà incorporate vengono definite dal modulo di gestione di database Microsoft Jet per tutti gli oggetti DAO.

  • È possibile aggiungere proprietà definite dall'utente ad alcuni oggetti DAO. Tali oggetti DAO includono gli oggetti Database, Index, QueryDef e TableDef e gli oggetti Field nell'insieme Fields di un oggetto QueryDef o TableDef.

  • Alcune proprietà definite da Microsoft Access vengono applicate a oggetti DAO. È possibile in genere impostare tali proprietà nella finestra di Microsoft Access o da Visual Basic. Il Modulo di gestione di database Microsoft Jet non è in grado di riconoscere tali proprietà fino al momento in cui vengono appositamente creati oggetti Property corrispondenti e accodati all'insieme Properties. Gli oggetti DAO a cui è possibile applicare tali proprietà sono QueryDef e TableDef e gli oggetti Field contenuti nell'insieme Fields di un oggetto QueryDef o TableDef. Per gli elenchi di tali proprietà definite da Microsoft Access, vedere l'argomento Specifiche di Microsoft Access per oggetti TableDef, QueryDef e Field.

Le proprietà definite da Microsoft Access che vengono applicate a oggetti DAO si differenziano in più modi dalle proprietà definite dal modulo di gestione di database Microsoft Jet.

Per fare riferimento a una proprietà definita dall'utente o a una proprietà definita da Microsoft Access, è necessario fare esplicitamente riferimento all'insieme Properties. Per fare riferimento più rapidamente a una proprietà di Microsoft Access, utilizzare la sintassi riportata di seguito:

oggetto.Properties!nomeproprietà

Nell'esempio precedente, oggetto è un oggetto DAO e nomeproprietà è il nome della proprietà di Microsoft Access.

È anche possibile utilizzare la seguente sintassi, anche se è più lenta rispetto alla precedente:

oggetto.Properties("nome")

Per fare invece riferimento alle proprietà definite dal modulo di gestione di database Microsoft Jet, è sufficiente utilizzare la sintassi oggetto.nome.

Se si imposta per la prima volta il valore di una proprietà definita da Microsoft Access, è necessario creare prima tale proprietà utilizzando il metodo CreateProperty. La proprietà Caption di un oggetto Field, ad esempio, è una proprietà definita da Microsoft Access. Se tale proprietà non è stata precedentemente impostata in visualizzazione Struttura tabella, ma si imposta la proprietà per la prima volta dal codice di Visual Basic, prima di poterne impostare il valore è necessario creare la proprietà utilizzando il metodo CreateProperty e accodarla all'insieme Properties.

Una proprietà definita da Microsoft Access viene automaticamente accodata all'insieme Properties quando viene impostata per la prima volta nella finestra di Microsoft Access. Se una proprietà è già stata impostata nell'interfaccia utente, non è necessario creare e accodare tale proprietà nel codice. È possibile, ad esempio, impostare la proprietà Caption per una tabella in visualizzazione Foglio dati scegliendo Carattere dal menu Formato. Tale proprietà viene quindi inclusa nell'insieme Properties di un oggetto TableDef che punta alla tabella.

La proprietà non viene visualizzata nell'insieme Properties fino a quando non viene impostata una proprietà definita da Microsoft Access in visualizzazione Struttura tabella o dal codice di Visual Basic. Quando si impostano tali proprietà da Visual Basic, si consiglia di includere un codice di gestione degli errori che verifica la presenza della proprietà nell'insieme Properties e, nel caso non sia presente, la crea e la accoda.

Proprietà applicabili a oggetti di Microsoft Access

Ogni oggetto di Microsoft Access, così come gli oggetti DAO, contiene un insieme Properties con oggetti Property incorporati. Gli oggetti Property applicabili a una maschera, ad esempio, sono membri dell'insieme Properties dell'oggetto Form.

È anche possibile creare proprietà definite dall'utente per oggetti di Microsoft Access. È possibile, ad esempio, creare una proprietà chiamata Tipo testo che venga applicata a una casella di testo.

Gli oggetti Property contenuti negli insiemi Properties di oggetti Form, Report e Control sono diversi dagli oggetti DAO Property in quanto non hanno una proprietà Inherited.

È possibile elencare gli oggetti Property contenuti nell'insieme Properties di oggetti Form, Report e Control. L'oggetto Application e l'oggetto Screen di Microsoft Access dispongono di insiemi Properties che non è possibile elencare. Le proprietà di tali oggetti, inoltre, sono di sola lettura.
