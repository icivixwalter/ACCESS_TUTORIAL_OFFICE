Metodo CurrentDb

        

Il metodo CurrentDb restituisce una variabile di oggetto di tipo Database che rappresenta il database attualmente aperto nella finestra di Microsoft Access.

Nota   In Microsoft Access 2000, il metodo CurrentDb stabilisce un riferimento nascosto alla libreria di oggetti DAO 3.6 Microsoft in un database di Microsoft Access (mdb). Per utilizzare il metodo CurrentDb in un progetto di Access (adp) occorre impostare un riferimento permanente alla libreria di oggetti DAO 3.6 Object in Visual Basic Editor.

Sintassi

CurrentDb

Osservazioni

Per manipolare la struttura del database e i relativi dati da Visual Basic, è necessario utilizzare oggetti DAO. Il metodo CurrentDb consente di accedere al database corrente tramite il codice di Visual Basic senza dover essere a conoscenza del nome del database. Una volta che si dispone di una variabile che punta al database corrente, è possibile inoltre accedere e manipolare altri oggetti ed insiemi nella gerarchia di oggetti DAO.

È possibile utilizzare il metodo CurrentDb per creare più variabili di oggetto che fanno riferimento al database corrente. Nell'esempio riportato di seguito, entrambe le variabili dbsA e dbsB fanno riferimento al database corrente.

Dim dbsA As Database, dbsB As Database
Set dbsA = CurrentDb
Set dbsB = CurrentDb

Nota   Nelle versioni precedenti di Microsoft Access, per restituire un puntatore al database corrente, era possibile utilizzare la sintassi DBEngine.Workspaces(0).Databases(0) o DBEngine(0)(0). In Microsoft Access 2000 viene invece utilizzato il metodo CurrentDb. Tale metodo crea un'altra istanza del database corrente, mentre la sintassi DBEngine(0)(0) fa riferimento alla copia aperta del database corrente. Il metodo CurrentDb consente di creare più di una variabile di tipo Database che fa riferimento al database corrente. La sintassi DBEngine(0)(0) è ancora supportata, ma considerare di effettuare questa modifica al codice al fine di evitare possibili conflitti in un database multiutente.

Se è necessario utilizzare un altro database contemporaneamente a quello aperto nella finestra di Microsoft Access, utilizzare il metodo OpenDatabase di un oggetto Workspace. Il metodo OpenDatabase non apre effettivamente il secondo database nella finestra di Microsoft Access, bensì restituisce semplicemente una variabile Database che rappresenta tale database. Nell'esempio riportato di seguito viene restituito un puntatore al database corrente e a un database denominato Contatti.mdb.

Dim dbsCurrent As Database, dbsContacts As Database
Set dbsCurrent = CurrentDb
Set dbsContacts = DBEngine.Workspaces(0).OpenDatabase("Contatti.mdb")
