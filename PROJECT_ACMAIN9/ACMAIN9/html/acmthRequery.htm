Metodo Requery (oggetto Control o Form)

                 

Il metodo Requery aggiorna i dati sottostanti una determinata maschera o un controllo contenuto nella maschera attiva rieseguendo una query sull'origine dei dati relativa alla maschera o al controllo.

Sintassi

[oggetto.]Requery

Il metodo Requery utilizza l'argomento riportato di seguito.

Argomento Descrizione
oggetto Oggetto Form o Control che rappresenta la maschera o il controllo che si desidera aggiornare. Omettere questo argomento per rieseguire la query sull'origine dei dati relativa all'oggetto attivo.

Osservazioni

È possibile utilizzare tale metodo per assicurarsi che una maschera o un controllo visualizzino i dati più recenti.

Il metodo Requery:

  • Esegue nuovamente la query su cui si basa la maschera o il controllo.

  • Visualizza qualsiasi record nuovo o modificato oppure rimuove i record eliminati dalla tabella su cui si basa la maschera o il controllo.

  • Aggiorna i record visualizzati in base a qualsiasi modifica apportata alla proprietà Filter della maschera.

I controlli basati su una query o su una tabella comprendono:

  • Caselle di riepilogo e caselle combinate.

  • Controlli di sottomaschere.

  • Oggetti OLE, ad esempio grafici.

  • Controlli per cui l'impostazione della proprietà ControlSource include funzioni di aggregazione sui domini o funzioni di aggregazione SQL.

Se si specifica qualsiasi altro tipo di controllo per l'argomento oggetto, viene rieseguita la query sull'origine record della maschera.

Se il controllo specificato dall'argomento oggetto non è associato a un campo di una tabella o di una query, il metodo Requery forza un ricalcolo del controllo.

Se si omette l'argomento oggetto, il metodo Requery riesegue la query sull'origine dati sottostante relativa alla maschera o al controllo che acquisisce lo stato attivo. Se il controllo che acquisisce lo stato attivo dispone di un'origine dati o di un'origine riga, la query viene ripetuta; in caso contrario i dati del controllo vengono semplicemente aggiornati.

Se una sottomaschera acquista lo stato attivo, tale metodo riesegue la query solo sull'origine record relativa a tale sottomaschera e non sulla maschera principale.

Note

  • Il metodo Requery aggiorna i dati sottostanti una maschera o un controllo in modo da riflettere i record nuovi o eliminati dall'origine record dal momento dell'ultima riesecuzione della query. Il metodo Refresh indica solo le modifiche apportate al set di record corrente, non riflette i record nuovi o quelli eliminati nell'origine record. Il metodo Repaint si limita ad aggiornare la maschera specificata e i relativi controlli.

  • Il metodo Requery non passa il controllo al sistema operativo in modo da consentire a Windows di continuare l'elaborazione dei messaggi. Utilizzare la funzione DoEvents qualora si renda necessario cedere il controllo temporaneo al sistema operativo.

  • Il metodo Requery è più veloce dell'azione RieseguiQuery. Quando si utilizza l'azione RieseguiQuery, Microsoft Access chiude la query e la carica di nuovo dal database. Quando si utilizza il metodo Requery Microsoft Access esegue di nuovo la query senza chiuderla e caricarla di nuovo.
