Istruzione DDEExecute

        

È possibile utilizzare l'istruzione DDEExecute per inviare un comando da un'applicazione client ad una server attraverso un canale DDE (Dynamic Data Exchange).

Si supponga, ad esempio, di aver aperto un canale DDE in Microsoft Access per trasferire dati di tipo testo da un foglio di calcolo di Microsoft Excel ad un database Microsoft Access. Utilizzare l'istruzione DDEExecute per inviare a Microsoft Excel il comando Nuovo, in modo da specificare che si desidera aprire un nuovo foglio di calcolo. In questo esempio, Microsoft Access funge da applicazione client, mentre Microsoft Excel da applicazione server.

Sintassi

DDEExecute numcan, comando

L'istruzione DDEExecute utilizza i seguenti argomenti.

Argomento Descrizione
numcan Un numero canale, l'intero lungo restituito dalla funzione DDEInitiate.
comando Un'espressione stringa che specifica un comando riconosciuto dall'applicazione server. Per un elenco di tali comandi, consultare la documentazione dell'applicazione server.

Osservazioni

Il valore dell'argomento comando dipende dall'applicazione e dall'argomento specificato quando viene aperto il canale indicato dall'argomento numcan. Se quest'ultimo non è costituito da un numero intero corrispondente ad un canale aperto o se l'applicazione non è in grado di eseguire il comando specificato, si verifica un errore.

Da Visual Basic, è possibile utilizzare l'istruzione DDEExecute solo per inviare comandi ad un'altra applicazione. Per informazioni sull'invio di comandi a Microsoft Access da un'altra applicazione, vedere Utilizzare Microsoft Access come server DDE.

Suggerimento   Se è necessario manipolare gli oggetti di un'altra applicazione da Microsoft Access, è possibile utilizzare l'automazione.
