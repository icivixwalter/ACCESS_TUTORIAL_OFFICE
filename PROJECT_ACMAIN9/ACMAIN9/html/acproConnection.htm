Proprietà Connection

                 

È possibile utilizzare la proprietà Connection per restituire un riferimento all'oggetto Connection ActiveX Data Objects (ADO) corrente e alle relative proprietà.

Impostazione

[oggetto.]Connection[.metodo o proprietà]

La proprietà Connection utilizza le impostazioni seguenti.

Impostazione Descrizione
oggetto Oggetto CurrentProject o CodeProject.
Connection Obbligatorio. Se utilizzato senza specificare un metodo o una proprietà, restituisce la proprietà ConnectionString dell'oggetto Connection.
metodo o proprietà Metodo o proprietà dell'oggetto Connection.

La proprietà Connection è disponibile solo utilizzando Visual Basic ed è di sola lettura.

Osservazioni

Utilizzare la proprietà Connection dell'oggetto CurrentProject per fare riferimento all'oggetto Connection dell'oggetto del progetto di Microsoft Access (adp) o del database di Microsoft Access (mdb) corrente. Utilizzare la proprietà Connection dell'oggetto CodeProject per fare riferimento all'oggetto Connection dell'oggetto di database codice del progetto o del database di Microsoft Access. È possibile utilizzare la proprietà Connection per richiamare metodi per l'oggetto Connection quali BeginTrans e CommitTrans.

Note   La proprietà Connection restituisce un riferimento a una copia della connessione ActiveX Data Object (ADO) per il database attivo. In questo modo, l'applicazione del metodo Close o il tentativo di modificare la connessione tramite i metodi o le proprietà dell'oggetto Connection non influenzerà in alcun modo l'oggetto di connessione reale utilizzato da Microsoft Access per mantenere una connessione con il database corrente. Poiché la proprietà Connection rappresenta la connessione al provider Shape principale, per utilizzare questa proprietà risultano necessarie le informazioni seguenti.

  1. MSDataShape utilizza Recordset.CursorLocation = adUseClient. Non impostare CursorLocation prima di assegnare un set di record a CurrentProject.Connect.

  2. MSDataShape utilizza Recordset.CursorType = adOpenStatic. Non impostare CursorType prima di assegnare un set di record a CurrentProject.Connection.

  3. MSDataShape accetta Recordset.LockType = adLockOptimistic, adLockBatchOptimistic o adLockReadOnly (predefinito). Se impostato a adLockPessimistic, viene cambiato in adLockOptimistic.

  4. La connessione shape non supporta completamente ADOX, in particolare l'insieme Columns.Properties

  5. Per assicurare il corretto funzionamento di una connessione shape, impostare Command.CommandType a adCmdTable.
