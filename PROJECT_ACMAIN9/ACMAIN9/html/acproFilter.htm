Proprietà Filter (Filtro)

                 

È possibile utilizzare la proprietà Filter per specificare un sottoinsieme di record da visualizzare quando si applica un filtro a una maschera, a un report, a una query o a una tabella.

Nota   Se si desidera specificare un filtro di server in un progetto di Microsoft Access (ADP) per i dati che si trovano in un server, utilizzare la proprietà ServerFilter.

Impostazione

La proprietà Filter è un'espressione stringa composta da una proposizione WHERE priva della parola chiave WHERE. Nel codice di Visual Basic riportato di seguito, ad esempio, si definisce ed applica un filtro per visualizzare solo i clienti degli USA.

Me.Filter = "Stato = 'USA'"
Me.FilterOn = True

È possibile impostare la proprietà utilizzando una finestra delle proprietà di tabella o di maschera, una macro oppure Visual Basic.

È inoltre possibile impostare la proprietà in visualizzazione Maschera o in visualizzazione Foglio dati scegliendo Filtro dal menu Record e scegliendo uno dei comandi dal sottomenu.

Nota   L'impostazione della proprietà Filter non ha effetto sulla proprietà ADO Filter.

Osservazioni

È possibile utilizzare la proprietà Filter per salvare un filtro e applicarlo successivamente. I filtri vengono salvati con gli oggetti in cui sono stati creati. I filtri vengono caricati automaticamente all'apertura dell'oggetto, ma non vengono automaticamente applicati.

Quando si crea un nuovo oggetto, questo eredita le proprietà RecordSource, Filter, OrderBy e OrderByOn della tabella o della query da cui è stato creato.

Per applicare un filtro salvato a una maschera, a una query o a una tabella, è possibile fare clic sul pulsante Applica filtro sulla barra degli strumenti, scegliere Applica filtro/ordinamento dal menu Record oppure utilizzare una macro o Visual Basic per impostare la proprietà FilterOn a True (-1). Per i report, è possibile applicare un filtro, impostando la proprietà FilterOn a Sì nella finestra delle proprietà del report.

Il pulsante Applica filtro indica lo stato delle proprietà Filter e FilterOn. Il pulsante rimane disattivato fino a quando è presente un filtro da applicare. Se è attualmente applicato un filtro esistente, il pulsante Applica filtro viene visualizzato incassato.

Per applicare automaticamente un filtro all'apertura di una maschera, specificare nell'impostazione della proprietà di evento SuApertura della maschera una macro che utilizza l'azione ApplicaFiltro o una routine evento che utilizza il metodo ApplyFilter dell'oggetto DoCmd.

È possibile rimuovere un filtro facendo clic sul pulsante incassato Applica filtro, scegliendo Rimuovi filtro/ordinamento dal menu Record oppure utilizzando Visual Basic per impostare la proprietà FilterOn a False (0).

Nota   È possibile salvare un filtro come una query, scegliendo Salva come query dal menu File quando ci si trova nella finestra Filtro in base a maschera o Ordinamento/filtro avanzato.

Quando la proprietà Filter viene impostata in visualizzazione Struttura della maschera, Microsoft Access non compie un tentativo di convalidare l'espressione SQL. Se l'espressione SQL non è valida, si verifica un errore quando si applica il filtro.
