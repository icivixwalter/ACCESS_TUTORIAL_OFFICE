Metodo Save

                 

Il metodo Save esegue l'azione Salva in Visual Basic. Per ulteriori informazioni sul funzionamento dell'azione e dei relativi argomenti, vedere l'argomento della Guida in linea relativo all'azione.

Sintassi

DoCmd.Save [tipooggetto, nomeoggetto]

Il metodo Save utilizza gli argomenti riportati di seguito.

Argomento Descrizione
tipooggetto Una delle seguenti costanti intrinseche:
  acDataAccessPage
acDefault (impostazione predefinita)
acDiagram
acForm
acMacro
acModule
acQuery
acReport
acServerView
acStoredProcedure
acTable
  Nota   Se si chiude un modulo in VBE (Visual Basic Editor), è necessario utilizzare acModule nell'argomento tipooggetto.
nomeoggetto Espressione stringa che è il nome valido di un oggetto del tipo selezionato dall'argomento tipooggetto.

Osservazioni

Se si omettono gli argomenti tipooggetto e nomeoggetto (per l'argomento tipooggetto viene utilizzata la costante predefinita acDefault), Microsoft Access salva l'oggetto attivo. Se si omette l'argomento tipooggetto ma si immette un nome nell'argomento nomeoggetto, Microsoft Access salva l'oggetto attivo con il nome specificato. Se si immette un tipo di oggetto nell'argomento tipooggetto, è necessario immettere il nome di un oggetto esistente nell'argomento nomeoggetto.

Se si omette l'argomento tipooggetto ma si immette un nome nell'argomento nomeoggetto, è necessario includere la virgola dell'argomento tipooggetto.

Nota   Non è possibile utilizzare il metodo Save per salvare con un nuovo nome gli elementi riportati di seguito:

  • Una maschera in visualizzazione Maschera o in visualizzazione Foglio dati.

  • Un report in Anteprima di stampa.

  • Un modulo.

  • Una pagina di accesso ai dati in visualizzazione Pagina.

  • Una visualizzazione del server in visualizzazione Foglio dati o in Anteprima di stampa.

  • Una tabella in visualizzazione Foglio dati o in Anteprima di stampa.

  • Una query in visualizzazione Foglio dati o in Anteprima di stampa.

  • Una stored procedure in visualizzazione Foglio dati o in Anteprima di stampa.

Il metodo Save, sia che venga eseguito nel database corrente sia in un database di libreria, salva sempre l'oggetto specificato o l'oggetto attivo nel database in cui tale oggetto è stato creato.
