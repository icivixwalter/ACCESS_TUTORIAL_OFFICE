Utilizzare un'istruzione SQL come origine riga per una casella di riepilogo o una casella combinata in una maschera

È possibile che si desideri utilizzare un'istruzione SQL invece di una query salvata per l'origine riga di una casella di riepilogo o di una casella combinata in una maschera in modo da non visualizzare la query nella finestra del database. Le caselle di riepilogo e le caselle combinate basate su istruzioni SQL sono tuttavia più lente di quelle basate su query salvate.

  1. Aprire una maschera in visualizzazione Struttura.

  2. Assicurarsi che la casella di riepilogo o la casella combinata sia selezionata e fare clic sul pulsante Proprietà sulla barra degli strumenti per aprire la finestra delle proprietà per il controllo.

  3. Fare clic sul pulsante dei generatori accanto alla proprietà Origine riga per aprire il Generatore di query.

  4. Nel Generatore di query, scegliere SQL dal menu Visualizza.

  5. Nella finestra SQL selezionare l'intera istruzione SQL, quindi premere CTRL+C per copiarla. Chiudere il Generatore di query.

  6. Fare clic sulla casella della proprietà Origine riga e premere CTRL+V per incollare l'istruzione SQL nella casella della proprietà.

  7. Eliminare la query originale nella finestra del database.

Note

  • Per informazioni sulle proprietà che è possibile utilizzare per personalizzare una casella di riepilogo o una casella combinata, fare clic su .

  • Se si crea una casella di riepilogo o una casella combinata in una maschera utilizzando una creazione guidata, in Microsoft Access viene utilizzata un'istruzione SQL come origine riga per la casella di riepilogo o la casella combinata.
