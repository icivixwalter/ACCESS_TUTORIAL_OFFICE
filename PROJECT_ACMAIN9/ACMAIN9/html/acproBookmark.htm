Proprietà Bookmark

                 

È possibile utilizzare la proprietà Bookmark con maschere per impostare un segnalibro che identifica in modo univoco un determinato record nella tabella sottostante della maschera, nella query o nell'istruzione SQL.

Impostazione

La proprietà Bookmark contiene un'espressione stringa creata da Microsoft Access.

È possibile impostare tale proprietà utilizzando una macro o Visual Basic.

Nota   La proprietà Bookmark della maschera si ottiene o si imposta in modo distinto dalla proprietà ADO Bookmark o DAO Bookmark della tabella sottostante o della query.

Osservazioni

Quando si apre una maschera associata in visualizzazione Maschera, si assegna un segnalibro univoco a ciascun record. In Visual Basic è possibile salvare il segnalibro per il record corrente assegnando il valore della proprietà Bookmark della maschera a una variabile di tipo stringa. Per tornare a un record salvato dopo lo spostamento a un altro record, impostare la proprietà Bookmark della maschera al valore della variabile di tipo stringa salvata. È possibile utilizzare la funzione StrComp per confrontare una Variante o una variabile di stringa con un segnalibro oppure per confrontare un segnalibro con un altro. Il terzo argomento della funzione StrComp deve essere impostato a zero.

Nota   I segnalibri non vengono salvati con i record che essi rappresentano e sono validi solo quando la maschera è aperta. Essi vengono ricreati da Microsoft Access ogni volta che si apre una maschera associata.

Non esiste limite al numero di segnalibri che è possibile salvare, purché ciascuno di essi venga salvato con una variabile di tipo stringa univoca.

La proprietà Bookmark è disponibile solo per il record corrente della maschera. Per salvare un segnalibro per un record diverso da quello corrente, spostarsi al record desiderato e assegnare il valore della proprietà Bookmark a una variabile di tipo stringa che identifichi tale record.

È possibile utilizzare segnalibri in qualsiasi maschera che si basi interamente su tabelle di Microsoft Access. È possibile, tuttavia, che altri prodotti di database non supportino i segnalibri. Non è possibile, ad esempio, utilizzare segnalibri in una maschera basata su una tabella collegata priva di indice primario.

La riesecuzione di una query su una maschera invalida qualsiasi segnalibro impostato sui record della maschera. Scegliendo Aggiorna dal menu Record, tuttavia, i segnalibri non vengono toccati.

Poiché viene creato un segnalibro univoco per ciascun record contenuto in un set di record di una maschera quando questa viene aperta, il segnalibro di una maschera non funzionerà in un altro set di record, anche quando i due set di record sono basati sulla stessa tabella, query o istruzione SQL. Si supponga, ad esempio, di aprire una maschera associata alla tabella Clienti. Se si apre, quindi, la tabella Clienti utilizzando Visual Basic e si utilizza il metodo ADO Seek o DAO Seek per individuare un determinato record della tabella, non sarà possibile impostare la proprietà Bookmark della maschera al record corrente della tabella. Per eseguire questo tipo di operazione si può utilizzare il metodo ADO Find o DAO Find con la proprietà RecordsetClone della maschera.

Se si imposta la proprietà Bookmark a una variabile di tipo stringa e quindi si prova a tornare a un record dopo che questo è stato cancellato, si verifica un errore.

Il valore della proprietà Bookmark non è uguale al numero del record.
