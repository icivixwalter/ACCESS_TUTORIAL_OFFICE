Impossibile aprire o aggiornare una tabella collegata di Paradox

  • Se si collega una tabella di Paradox che ha una chiave primaria, il file indice con estensione px associato sarà necessario per l'apertura della tabella collegata. Se si collega una tabella di Paradox contenente un campo Memo, il file memo con estensione mb associato sarà necessario per l'apertura della tabella collegata. Se si eliminano o si spostano questi file, non sarà possibile aprire la tabella collegata.

  • Se si collega una tabella di Paradox che non ha una chiave primaria, non si potranno aggiornare i dati della tabella utilizzando Microsoft Access. Se si desidera rendere la tabella aggiornabile, definire una chiave primaria per la tabella di Paradox.

  • Degli oggetti OLE memorizzati in una tabella di Paradox collegata o importata non possono essere aperti dall'interno di Microsoft Access.

  • Non è possibile aggiornare una tabella, se il relativo file di dati con estensione dbf o i file indice con estensione idx, cdx, ndx o mdx sono di sola lettura. Utilizzare Risorse del computer o Esplora risorse (Gestione risorse in Windows 95 e Windows NT 4.0) per visualizzare la cartella in cui si trovano i file, fare clic con il pulsante destro del mouse su un file, scegliere Proprietà, quindi scegliere la scheda Generale. Se la casella di controllo Sola lettura è selezionata, deselezionarla e scegliere OK.

  • Non è possibile aggiornare una tabella di Paradox, se il relativo file di dati con estensione db o file indice con estensione px è di sola lettura. Utilizzare Risorse del computer o Esplora risorse (Gestione risorse in Windows 95 e Windows NT 4.0) per visualizzare la cartella in cui si trovano i file, fare clic con il pulsante destro del mouse su un file, scegliere Proprietà, quindi scegliere la scheda Generale. Se la casella di controllo Sola lettura è selezionata, deselezionarla e scegliere OK.

  • Per collegare (in lettura/scrittura) una tabella di Paradox 8.0, è necessario avere installato nel computer Borland Database Engine (BDE) versione 4.x o successiva. Se non è installato BDE 4.x o versione successiva, è possibile ottenerlo installando Paradox 8.0 o Delphi. Se è installata una versione precedente di BDE che non supporta il formato di file di Paradox 8, è possibile aggiornare il modulo di gestione dal sito Web di Borland. In alternativa, se BDE non è installato nel computer, è possibile comunque effettuare collegamenti (sola lettura) a tabelle di Paradox 3.x, 4.x e 5.0.

  • Quando si collega una tabella di Paradox residente su un server e condivisa da più utenti, impostare la chiave ParadoxNetPath nel registro di configurazione di Windows al percorso del file Paradox.net o Pdoxusrs.net. Se inoltre si utilizza la versione 4.x di Paradox per condividere dati all'interno del gruppo di lavoro, è necessario impostare a "4.x." la chiave ParadoxNetStyle. Per ulteriori informazioni su queste impostazioni, fare clic su .

  • Se si utilizza Microsoft Access per collegare i dati di Paradox a una rete mentre gli utenti di Paradox li stanno utilizzando, è necessario definire tutte e tre le impostazioni ParadoxUserName, ParadoxNetPath e ParadoxNetStyle nel registro di configurazione. Utilizzare l'impostazione ParadoxUserName per specificare un nome utente univoco per accedere ai dati di Paradox condivisi. Utilizzare l'impostazione ParadoxNetPath per specificare il percorso completo della cartella contenente il file Paradox.net, per Paradox 3.x, o il file Pdoxusrs.net, per Paradox 4.x e 5.0. L'impostazione ParadoxNetPath, che comprende la lettera dell'unità, deve essere coerente per tutti gli utenti che condividono un determinato database. Utilizzare l'impostazione ParadoxNetStyle per specificare lo stile di blocco di rete da utilizzare quando si accede ai dati di Paradox. Valori possibili sono "3.x" per Paradox 3.x e "4.x" per Paradox 4.x e 5.0. Per ulteriori informazioni, consultare la guida dell'amministratore di rete Paradox. Per ulteriori informazioni su queste impostazioni, fare clic su .
