Impostare le proprietà di avvio da Visual Basic

In un database di Microsoft Access (mdb) le proprietà di avvio sono proprietà di un oggetto Database. Un oggetto Database è un oggetto DAO fornito dal gestore di database Microsoft Jet, ma poiché le proprietà di avvio vengono definite da Microsoft Access, queste non vengono riconosciute automaticamente dal gestore di database Jet. Se una proprietà di avvio non è stata impostata in precedenza, è necessario crearla e aggiungerla all'insieme Properties dell'oggetto Database.

In un progetto di Microsoft Access (adp) le proprietà di avvio sono proprietà di un oggetto CurrentProject e, come l'oggetto Database in un database di Access (mdb), non vengono riconosciute automaticamente dal gestore di database Microsoft Jet. Se una proprietà di avvio non è stata impostata in precedenza, è necessario crearla e aggiungerla all'insieme AccessObjectProperties dell'oggetto CurrentProject.

Quando si impostano le proprietà di avvio da Visual Basic, è opportuno includere il codice di gestione degli errori per verificare la presenza della proprietà nell'insieme Properties o nell'insieme AccessObjectProperties. Per ulteriori informazioni sull'impostazione delle proprietà definite da Microsoft Access, vedere Impostare le proprietà di oggetti DAO in Visual Basic o Impostare le proprietà di oggetti ADO (ActiveX Data Object) in Visual Basic.
