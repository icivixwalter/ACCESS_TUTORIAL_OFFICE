Crittografare o decrittografare un database

La crittografia di un database compatta il file e lo rende indecifrabile per un programma di utilità o un elaboratore di testi. La decrittografia di un database è il processo inverso alla crittografia.

  1. Avviare Microsoft Access senza aprire un database.

    Importante   Non è possibile crittografare o decrittografare un database quando questo è aperto. In un ambiente multiutente, l'operazione di crittografia o di decrittografia non riesce se un altro utente ha aperto il database.

  2. Scegliere Protezione dal menu Strumenti, quindi Crittografa/Decrittografa database.

  3. Specificare il database che si desidera crittografare o decrittografare, quindi scegliere OK.

  4. Specificare un nome, un'unità ed una cartella per il database crittografato o decrittografato, quindi scegliere OK.

    Se viene utilizzato lo stesso nome, unità e cartella del database di origine e se l'operazione di crittografia o di decrittografia del database è riuscita, il file originale viene automaticamente sostituito con la versione crittografata o decrittografata. Se si verifica un errore, il file originale non viene eliminato.

Note

  • Se è stata definita la protezione a livello utente per crittografare o decrittografare un database è necessario essere il proprietario del database o un membro del gruppo Amministratori e disporre dell'autorizzazione Apertura esclusiva per il database. In caso contrario l'operazione di crittografia o di decrittografia non riesce.

  • Se il nome di un oggetto contenuto in un database versione 1.x include l'apice inverso (`), non sarà possibile crittografare il database. Utilizzare la versione 1.x di Microsoft Access per rinominarlo, quindi modificare i riferimenti a quell'oggetto nelle query, nelle maschere, nei report, nelle macro e nel codice Access Basic.

  • La crittografia di un database non limita l'accesso agli oggetti. Per ulteriori informazioni sui metodi di protezione, fare clic su .

  • L'operazione di crittografia o di decrittografia non riesce se sul disco non si dispone dello spazio sufficiente per la memorizzazione della versione originale del database e di quella crittografata o decrittografata.
