Proprietà ProjectName

                 

È possibile utilizzare la proprietà ProjectName per specificare o determinare il nome dell'oggetto Application di Microsoft Access.

Impostazione

La proprietà ProjectName è un'espressione stringa che corrisponde al nome dell'oggetto Application di Microsoft Access. Non è possibile impostare la proprietà ProjectName al nome di una classe esistente di oggetti, ad esempio "applicazione", "maschera" o "report". Se si converte un database con un nome uguale a quello di una classe esistente di oggetti, Microsoft Access accoderà un carattere di sottolineatura al nome per evitare che si verifichino conflitti. Ad esempio, se il database da convertire si chiama "Applicazione", il database avrà come nome di progetto "Applicazione_" in Microsoft Access 2000.

È possibile impostare la proprietà ProjectName nella scheda Avanzate della finestra di dialogo Opzioni, disponibile scegliendo Opzioni dal menu Strumenti, o utilizzando il metodo SetOption in Visual Basic. Ad esempio:

Application.SetOption "Nome progetto", "MioProgettoAccess"

Osservazioni

Quando si crea un nuovo database, Microsoft Access imposta la proprietà ProjectName al nome del file di database senza l'estensione "mdb". La modifica del nome del file di database non ha alcun effetto sull'impostazione esistente della proprietà ProjectName.

Quando si imposta la proprietà ProjectName, viene visualizzato il nome dell'oggetto Application nella finestra di dialogo Riferimenti, disponibile nella finestra Modulo, se si sceglie Riferimenti dal menu Strumenti.

Quando si modifica l'impostazione della proprietà Nome Progetto, il progetto non si troverà più in forma compilata. Per ricompilare il progetto, è necessario chiudere e riaprire il database, quindi scegliere Compila e salva tutti i moduli dal menu Debug.

È possibile utilizzare il nome del progetto da un'altra applicazione, ad esempio Microsoft Excel, come parte del nome completo di un elemento in un progetto di Microsoft Access, utilizzando la sintassi nomeprogetto.Application.Forms!nomemaschera. Per fare riferimento, ad esempio, al campo Cognome della maschera Impiegati nel database di esempio Northwind, impostare un riferimento da Microsoft Excel per il database Northwind, quindi utilizzare il riferimento riportato di seguito:

Dim strCognome As String
strCognome = Northwind.Application.Forms!Impiegati!Cognome

È possibile creare un riferimento per un database di Microsoft Access, scegliendo il nome di progetto, l'oggetto Application, nella finestra di dialogo Riferimenti, disponibile nella finestra Modulo se si sceglie Riferimenti dal menu Strumenti. Dopo aver selezionato il nome di progetto, esso viene visualizzato anche nell'elenco Progetto/libreria nel Visualizzatore oggetti.
