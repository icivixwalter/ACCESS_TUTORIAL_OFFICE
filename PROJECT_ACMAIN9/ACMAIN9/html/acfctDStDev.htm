Funzioni DStDev, DStDevP

Funzioni DStDev, DStDevP

        

È possibile utilizzare le funzioni DStDev e DStDevP per stimare la deviazione standard relativa a un set di valori di un determinato set di record, un dominio. Utilizzare le funzioni DStDev e DStDevP in Visual Basic, in una macro, in un'espressione nella query o in un controllo calcolato di una maschera o di un report.

Utilizzare la funzione DStDevP per valutare una popolazione intera e la funzione DStDev per valutare un campione di popolazione.

È possibile ad esempio utilizzare la funzione DStDev in un modulo per calcolare la deviazione standard relativa a un set di votazioni riportate da studenti nei compiti in classe.

Sintassi

DStDev(espr, dominio[, criteri])

DStDevP(espr, dominio[, criteri])

Le funzioni DStDev e DStDevP dispongono degli argomenti riportati di seguito.

Argomento Descrizione
espr Un'espressione che identifica il campo numerico in cui si desidera trovare la deviazione standard. È possibile che si tratti di un'espressione stringa che identifica un campo in una tabella o in una query oppure un'espressione che esegue un calcolo dei campi. In espr è possibile includere il nome di un campo in una tabella, un controllo in una maschera, una costante o una funzione. Se espr include una funzione, è possibile che sia incorporata o definita dall'utente, ma non è possibile che si tratti di un'altra funzione di aggregazione sui domini o di aggregazione SQL.
dominio Un'espressione stringa che identifica il set di record che costituisce il dominio. È possibile che sia un nome di tabella o di query.
criteri Una stringa opzionale utilizzata per limitare l'intervallo di dati sui cui viene eseguita la funzione DStDev o DStDevP. L'argomento criteri ad esempio equivale spesso alla proposizione WHERE priva della parola WHERE in un'espressione SQL. Se si omette l'argomento criteri, le funzioni DStDev e DStDevP valutano espr confrontandolo con l'intero dominio. È necessario che qualsiasi campo incluso in criteri appartenga anche all'argomento dominio. In caso contrario, le funzioni DStDev e DStDevP restituiscono il valore Null.

Osservazioni

Se dominio fa riferimento a meno di due record o se meno di due record soddisfano criteri, le funzioni DStDev e DStDevP restituiranno un valore Null, indicando che non è possibile calcolare una deviazione standard.

Sia che la funzione DStDev o DStDevP venga utilizzata in una macro, in un modulo, in un'espressione nella query o in un controllo calcolato, è necessario creare l'argomento criteri con attenzione per assicurarsi che venga valutato correttamente.

È possibile utilizzare le funzioni DStDev e DStDevP per specificare i criteri nella riga Criteri di una query di selezione. È possibile ad esempio creare una query in una tabella Ordini e in una tabella Prodotti per visualizzare tutti i prodotti per cui le spese di trasporto si collocano attorno alla media aggiungendo la deviazione standard relativa alle spese di trasporto. La riga Criteri sottostante al campo Spese trasporto contiene la seguente espressione:

>(DStDev("[SpeseTrasporto]", "Ordini") + DAvg_("[SpeseTrasporto]", "Ordini"))

È possibile utilizzare le funzioni DStDev e DStDevP all'interno di un'espressione di campo calcolato in una query o nella riga Aggiorna a di una query di aggiornamento.

Nota   È possibile utilizzare le funzioni DStDev e DStDevP o le funzioni StDev e StDevP in un'espressione di campo calcolato di una query di totalizzazione. Se si utilizza la funzione DStDev o DStDevP, i valori vengono calcolati prima del raggruppamento dei dati. Se si utilizza la funzione DevSt o StDevP, i dati vengono raggruppati prima che vengano valutati i valori contenuti nell'espressione di campo.

Utilizzare le funzioni DStDev e DStDevP in un controllo calcolato quando si rende necessario specificare criteri per limitare l'intervallo di dati su cui viene eseguita la funzione. Per visualizzare ad esempio la deviazione standard relativa agli ordini da inoltrare in California, impostare la proprietà Origine controllo di una casella di testo alla seguente espressione:

=DStDev("[SpeseTrasporto]", "Ordini", "[ZonaDestinatario] = 'CA'")

Se si desidera semplicemente trovare la deviazione standard in tutti i record contenuti in dominio, utilizzare la funzione DevSt o StDevP.

Suggerimento   Se il tipo di dati del campo da cui deriva espr è un numero, le funzioni DStDev e DStDevP restituiranno un tipo di dati Precisione doppia. Se si utilizza la funzione DStDev o DStDevP in un controllo calcolato, includere una funzione di conversione tra tipi di dati nell'espressione per migliorare le prestazioni.

Nota   Le modifiche ai record che non vengono salvate in dominio non vengono incluse quando si utilizzano queste funzioni. Se si desidera che la funzione DStDev o DStDevP sia basata sui valori modificati, è necessario prima salvare le modifiche scegliendo il comando Salva record dal menu File, spostando lo stato attivo su un altro record oppure utilizzando il metodo Update.
