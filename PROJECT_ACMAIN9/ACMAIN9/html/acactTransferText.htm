Azione TrasferisciTesto

                 

È possibile utilizzare l'azione TrasferisciTesto per importare o esportare del testo tra il database di Microsoft Access (MDB) o il progetto di Microsoft Access (ADP) corrente e un file di testo. È anche possibile collegare i dati di un file di testo al database corrente di Access. È possibile visualizzare in Microsoft Access i dati di un file di testo collegato e nelle stesso tempo avere accesso completo a tali dati dal programma di elaborazione testi. È anche possibile importare da, esportare in e collegarsi a una tabella o a un elenco di un file HTML file (con estensione HTML).

Nota   Se ci si collega ai dati di un file di testo o di un file HTML, tali dati sono accessibili in sola lettura in Microsoft Access.

Impostazione

L'azione TrasferisciTesto utilizza gli argomenti riportati di seguito.

Argomento azione Descrizione
Tipo trasferimento Il tipo di trasferimento che si desidera effettuare. È possibile importare, esportare o collegare dati in file di testo delimitato o a larghezza fissa oppure in file HTML. È inoltre possibile esportare dati in un file di stampa unione di Microsoft Word che sarà possibile utilizzare per creare documenti uniti come lettere tipo e etichette postali.
  Selezionare Importa Delimitato, Importa Larghezza Fissa, Importa HTML, Esporta Delimitato, Esporta Larghezza Fissa, Esporta HTML, Esp. stampa unione Word, Collegamento Delimitato, Collegamento Larg. Fissa o Collegamento HTML nella casella Tipo trasferimento della sezione Argomenti azione della finestra delle macro. L'impostazione predefinita è Importa Delimitato.
  Nota  In un progetto di Microsoft Access (ADP) sono supportati solo i tipi di trasferimento Importa Delimitato, Importa Larghezza Fissa, Esporta Delimitato, Esporta Larghezza Fissa o Esp. stampa unione Word.
Nome file specifiche Il nome del file delle specifiche per il set di opzioni che determina il modo in cui un file di testo viene importato, esportato o collegato. Per i file di testo a larghezza fissa è necessario specificare un argomento o utilizzare un file schema.ini, che deve essere memorizzato nella stessa cartella del file di testo importato, collegato o esportato.
  È possibile utilizzare il sottocomando Importa o Collega tabelle del comando Carica dati esterni o Esporta del menu File per creare una specifica per un determinato tipo di file di testo, ad esempio un file di testo delimitato in cui si utilizzano le tabulazioni per separare i campi e il cui formato per le date è MGA. Quando si sceglie uno di questi comandi e si seleziona un tipo di file di testo da importare, esportare o collegare, viene eseguita Creazione guidata Importazione testo, Esportazione testo o Collegamento testo. È possibile scegliere il pulsante Avanzate nella creazione guidata e definire e salvare una specifica nella finestra di dialogo visualizzata. È quindi possibile digitare il nome della specifica in questo argomento ogni volta che si desidera importare o esportare lo stesso tipo di file di testo.
  È possibile importare, esportare o collegare i file di testo delimitato senza digitare un nome di specifica per questo argomento. In questo caso, vengono utilizzate le impostazioni predefinite della finestra di dialogo della creazione guidata. Per i file di stampa unione viene utilizzato un formato predeterminato, pertanto non è necessario digitare un nome di specifica per questo argomento quando si esportano questi tipi di file. È possibile utilizzare le specifiche di importazione o esportazione con i file HTML ma l'unica parte applicabile della specifica è quella relativa alla formattazione del tipo di dati.
Nome tabella Il nome della tabella di Microsoft Access in cui importare, da cui esportare o a cui collegare dati di testo. È anche possibile digitare il nome della query di Microsoft Access dalla quale si desidera esportare i dati. Questo argomento è richiesto.
  Se si seleziona Importa Delimitato, Importa Larghezza Fissa o Importa HTML nella casella Tipo trasferimento, i dati di testo vengono accodati a questa tabella se la tabella esiste già. Altrimenti, viene creata una nuova tabella contenente i dati di testo.
  In Microsoft Access 2000 non è possibile utilizzare un'istruzione SQL per specificare i dati da esportare quando si utilizza l'azione TrasferisciTesto ma è necessario prima creare una query e quindi specificarne il nome nell'argomento Nome tabella.
Nome file Il nome del file di testo da cui importare, in cui esportare o a cui collegarsi. Includere il percorso completo. Questo argomento è richiesto.
  Quando si esportano i dati da Microsoft Access, viene creato un nuovo file di testo. Se il nome del file è uguale a quello di un file di testo esistente, quest'ultimo viene sostituito.
  Se si desidera importare o collegare una determinata tabella o un determinato elenco in un file HTML, è possibile utilizzare l'argomento Nome tabella HTML.
Nomi campo Specifica se la prima riga del file di testo contiene i nomi dei campi. Se si seleziona Sì, vengono utilizzati i nomi di questa riga come nomi dei campi della tabella di Microsoft Access quando vengono importati o collegati dati di testo. Se si seleziona No, la prima riga viene considerata come una normale riga di dati. L'impostazione predefinita è No.
  Questo argomento viene ignorato per i file di stampa unione di Microsoft Word per Windows in quanto la prima riga deve contenere i nomi dei campi.
  Quando si esporta una tabella o una query di selezione di Microsoft Access in un file di testo delimitato o a larghezza fissa, i nomi dei campi della tabella o della query di selezione vengono inseriti nella prima riga del file di testo se per questo argomento è stato selezionato Sì.
  Se si sta importando o collegando un file di testo a larghezza fissa e si seleziona Sì in questa casella, i nomi dei campi contenuti nella prima riga devono essere separati mediante il delimitatore di campo impostato nella specifica di importazione o esportazione. Se si sta esportando in un file di testo a larghezza fissa e si seleziona Sì per questo argomento, i nomi dei campi vengono inseriti automaticamente nella prima riga del file di testo con questo delimitatore.
Nome tabella HTML Il nome della tabella o dell'elenco contenuto nel file HTML che si desidera importare o collegare. Questo argomento viene ignorato a meno che l'argomento Tipo trasferimento non sia impostato a Importa HTML o Collegamento HTML. Se si lascia vuoto questo argomento, vengono importati o collegati la prima tabella o il primo elenco del file HTML.
  Il nome della tabella o dell'elenco del file HTML è determinato dal testo specificato dal tag <CAPTION>, se ne esiste una, altrimenti dal testo specificato dal tag <TITLE>. Se più tabelle o elenchi hanno lo stesso nome, alla fine di ogni nome viene aggiunto un numero, ad esempio Impiegati1 e Impiegati2.
Tabella codici Nome del set di caratteri utilizzato con la tabella codici.

Osservazioni

È possibile esportare i dati delle query di selezione di Microsoft Access in file di testo. Il set di risultati della query viene esportato come se fosse una tabella.

I dati di testo che si accodano a una tabella di Microsoft Access esistente devono essere compatibili con la struttura della tabella.

  • Ciascun campo del testo deve essere dello stesso tipo di dati del campo corrispondente della tabella.

  • I campi devono essere nello stesso ordine, a meno che non si imposti l'argomento Nomi campo a Sì, nel qual caso i nomi dei campi del testo devono corrispondere a quelli della tabella.

Questa azione equivale a scegliere Carica dati esterni o Esporta dal menu File della finestra del database e quindi Importa o Collega tabelle. È possibile utilizzare questi comandi per selezionare un'origine dati, ad esempio Microsoft Access oppure un tipo di database, di foglio di calcolo o di file di testo. Se si seleziona un file di testo delimitato o a larghezza fissa oppure un file HTML, viene visualizzata una creazione guidata in cui si richiede di selezionare il nome del file e altre opzioni. Gli argomenti dell'azione TrasferisciTesto riflettono le opzioni presenti in queste creazioni guidate.

Suggerimento   Una specifica di importazione o esportazione memorizza le informazioni necessarie per l'importazione, l'esportazione o il collegamento di un file di testo. È possibile utilizzare specifiche memorizzate per importare, esportare o collegare dati di testo da o in file di testo simili. Se ad esempio si ricevono in un file di testo le cifre relative alle vendite settimanali provenienti da un computer mainframe, è possibile creare e salvare una specifica per questo tipo di dati e quindi utilizzarla ogni volta che questi dati vengono aggiunti al database di Microsoft Access.

Nota Se si esegue una query o si applica un filtro al foglio di calcolo collegato, la query o il filtro distinguono tra maiuscole e minuscole.

Per eseguire l'azione TrasferisciTesto in Visual Basic, utilizzare il metodo TransferText dell'oggetto DoCmd.
