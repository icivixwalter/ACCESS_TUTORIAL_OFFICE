Proprietà SourceConnectStr, SourceDatabase (Stringa conness. origine, Database origine)

                 

È possibile utilizzare tali proprietà per accedere a dati esterni quando non è possibile collegare le tabelle esterne al proprio database.

  • La proprietà SourceConnectStr specifica il nome dell'applicazione utilizzata per creare un database esterno.

  • La proprietà SourceDatabase specifica il database esterno in cui risiedono le tabelle origine o le query per una query.

Nota   Le proprietà SourceConnectStr e SourceDatabase si applicano a tutte le query tranne alle query di definizione dati, di tipo pass-through e di unione.

Impostazione

Utilizzare una espressione stringa per impostare il valore delle proprietà SourceConnectStr e SourceDatabase.

È possibile impostare tali proprietà nella finestra delle proprietà della query o in visualizzazione SQL della finestra della query. Nell'istruzione SQL, le proprietà corrispondono alla proposizione IN.

Nota   Se si ha accesso a più origini database, utilizzare la proprietà Source invece delle proprietà SourceConnectStr e SourceDatabase.

Osservazioni

È necessario utilizzare le proprietà SourceConnectStr e SourceDatabase per accedere a tabelle di database esterni create in applicazioni che non utilizzano tabelle collegate.

Vengono riportati di seguito esempi delle impostazioni di tali proprietà.

  • Per un database di Microsoft Access, l'impostazione della proprietà SourceDatabase è il percorso e il nome del database, ad esempio C:\Conti\Clienti. L'estensione del nome file viene aggiunta automaticamente. La proprietà SourceConnectStr non dispone di un valore per un database Microsoft Access.

  • Per un database creato da un prodotto come Paradox, l'impostazione della proprietà SourceDatabase è il percorso, ad esempio C:\DatiParadox. L'impostazione della proprietà SourceConnectStr è la specificazione del tipo di database, ad esempio Paradox 3.x. Per un elenco di specificazioni, vedere la proprietà ADO Connect.

  • Nell'esempio riportato di seguito vengono utilizzate tabelle di dBASE IV nella directory C:\DATIDB come origine di dati per la query. 
    SELECT Clienti.COMPANYNAM, Ordini.ORDERID, Ordini.ORDERDATE
FROM Cliente INNER JOIN Ordini
ON Cliente.CUSTOMERID = Ordini.CUSTOMERID
IN 'C:\Dbdata'[dBASE IV;];

  • Per un database Open Database Connectivity (ODBC) l'impostazione della proprietà SourceConnectStr è il nome del database origine e qualsiasi informazione aggiuntiva richiesta dal prodotto, come un identificativo di accesso (ID) e una password. L'impostazione per un database Microsoft SQL Server, ad esempio, potrebbe essere la seguente: 
    ODBC;DSN=server_vendite;UID=giulia;PWD=password;DATABASE=vendite;

La proprietà SourceDatabase non dispone di un valore per un database ODBC.
