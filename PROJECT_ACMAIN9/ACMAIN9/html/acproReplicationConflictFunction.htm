Proprietà ReplicationConflictFunction

                 

La proprietà ReplicationConflictFunction consente di utilizzare del codice di risoluzione dei conflitti personalizzato per risolvere i conflitti di dati che si verificano fra repliche al momento della sincronizzazione. Quando si imposta tale proprietà, Microsoft Access chiama la funzione di risoluzione dei conflitti invece del Risolutore dei conflitti incorporato, quando l'utente richiede di risolvere tali conflitti.

Impostazione

Impostare la proprietà ReplicationConflictFunction a una stringa di testo che è il nome della routine Function che si desidera chiamare. Tenere presente che l'impostazione deve essere il nome di una routine Function; non può essere il nome di una routine Sub. A meno che questa proprietà non sia stata impostata, Microsoft Access chiama il Risolutore dei conflitti.

Per impostare il valore di questa proprietà per la prima volta da Visual Basic, è necessario creare la proprietà e accodarla all'insieme Properties dell'oggetto Document definito dall'utente. È necessario includere del codice di gestione degli errori nella routine da provare nel caso in cui la proprietà non esista ancora nell'insieme Properties. Per ulteriori informazioni, vedere il metodo CreateProperty.

Quando si crea e si accoda la proprietà ReplicationConflictFunction, è necessario inoltre fornire un valore True (-1) per l'argomento DDL facoltativo del metodo CreateProperty. Tale argomento impedisce di modificare o eliminare il nuovo oggetto Property a meno che non si disponga delle autorizzazioni dbSecWriteDef. Nell'eventualità non venga fornito un valore True per questo argomento, il codice di risoluzione dei conflitti personalizzato non verrà richiamato.

Per restituire nuovamente il controllo al Risolutore dei conflitti incorporato, rimuovere tale proprietà dall'insieme Properties.

È possibile inoltre aggiungere o eliminare questa proprietà sulla scheda Personalizza della finestra di dialogo Proprietà - NomeDatabase, disponibile scegliendo Proprietà database del menu File.

Osservazioni

Se lo stesso record di un database replicato è stato modificato in una o più repliche, quando si sincronizza una replica con il set di repliche si verificano dei conflitti. Quando si sceglie di risolvere tali conflitti, Microsoft Access chiama il Risolutore dei conflitti incorporato. Tale procedura guidata mostra ciascun conflitto all'utente, il quale deve determinare manualmente quali record modificati contengono i dati corretti.

Se si preferisce rendere automatica la risoluzione dei conflitti per l'applicazione, è possibile scrivere routine personalizzate e quindi dare loro la precedenza sulla funzionalità incorporata impostando la proprietà ReplicationConflictFunction. È necessario creare una sola funzione che agisca come punto di immissione nel codice di risoluzione dei conflitti, quindi impostare la proprietà ReplicationConflictFunction al nome di tale funzione.

Se si scrive del codice di risoluzione dei conflitti personalizzato, è necessario che questo gestisca sia gli errori della struttura di replica sia gli errori dei dati di replica. Per un esempio, aprire il database Wzcnf80.mda della procedura guidata nella cartella Office ed esaminare il codice di origine.
