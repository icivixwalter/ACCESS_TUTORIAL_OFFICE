Rimuovere la protezione a livello utente

Questa procedura comprende due fasi. Connettersi innanzitutto come amministratore del gruppo di lavoro e assegnare al gruppo Utenti le autorizzazioni per tutte le tabelle, le query, le maschere, le macro e per tutti i report. Restituire quindi la proprietà del database e di questi oggetti all'Amministratore predefinito uscendo e connettendosi come Amministratore, creando un database vuoto e importando tutti gli oggetti dal database originale in quello nuovo.

  1. Avviare Microsoft Access.

  2. Aprire il database.

  3. Connettersi come amministratore del gruppo di lavoro (membro del gruppo Amministratori).

  4. Assegnare al gruppo Utenti autorizzazioni complete per tutte le tabelle, le query, le maschere, le macro e per tutti i report del database.

    Procedura

  5. Uscire e riavviare Microsoft Access, aprire un database, quindi connettersi come Amministratore.

  6. Creare un nuovo database vuoto e lasciarlo aperto.

  7. Importare tutti gli oggetti dal database originale in quello nuovo.

    Procedura

  8. Se all'apertura del database gli utenti utilizzeranno il file di informazioni sul gruppo di lavoro corrente, è necessario cancellare la password dell'Amministratore in modo da disattivare la finestra di dialogo Accesso del gruppo di lavoro corrente. Questa operazione non è necessaria se gli utenti utilizzeranno il file di informazioni sul gruppo di lavoro predefinito creato al momento dell'installazione di Microsoft Access.

In questa fase il nuovo database è privo di qualsiasi protezione. Tutti gli utenti che sono in grado di aprire il nuovo database dispongono di autorizzazioni complete per tutte le tabelle, le query, le maschere, le macro e per tutti i report in esso contenuti. Queste autorizzazioni valgono per qualsiasi gruppo di lavoro, dato che l'account Amministratore è lo stesso in ciascun file di informazioni sul gruppo di lavoro. Il file di informazioni del gruppo di lavoro corrente al momento della creazione del nuovo database nel passaggio 6 definisce il gruppo Amministratori per il nuovo database.
