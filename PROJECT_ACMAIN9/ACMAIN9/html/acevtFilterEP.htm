Evento Filtro - Routine evento

        

Per creare una routine evento da eseguire quando si verifica l'evento Filtro, impostare la proprietà Su filtro a [Routine Evento] e fare clic sul pulsante dei generatori .

Sintassi

Private Sub Form_Filter(Cancel As Integer, FilterType As Integer)

La routine evento Filtro comprende gli argomenti riportati di seguito.

Argomento Descrizione
Cancel L'impostazione determina se aprire o meno la finestra del filtro. Impostando l'argomento Cancel a True (-1), si impedisce l'apertura della finestra del filtro. È inoltre possibile utilizzare il metodo CancelEvent dell'oggetto DoCmd per annullare l'apertura della finestra del filtro.
FilterType La finestra del filtro che si sta tentando di aprire. L'argomento FilterType può essere una delle costanti intrinseche riportate di seguito.
  Costante Valore
  acFilterByForm 0
  acFilterAdvanced 1
  acServerFilterByForm 2

Osservazioni

È possibile utilizzare la routine evento Filtro per disattivare o nascondere determinati controlli nella finestra Filtro in base a maschera o Filtro su server in base a maschera ed evitare che vengano utilizzati nei criteri del filtro. È possibile ad esempio fare in modo che la maschera Ordini mostri i record che hanno un valore nel controllo TotaleDovuto, indipendentemente dalle loro dimensioni. Tale controllo può essere disattivato o nascosto nella routine evento Filtro e attivato o mostrato nella routine evento ApplicazioneFiltro. Per disattivare o nascondere i controlli nella finestra Filtro in base a maschera o Filtro su server in base a maschera, è preferibile utilizzare la routine evento Filtro piuttosto che la macro Filtro, in quanto consente di verificare l'argomento FilterType per controllare se è stata aperta la finestra Filtro in base a maschera o Filtro su server in base a maschera. In un database di Microsoft Access (mdb) tutti i campi di una maschera vengono visualizzati nella finestra Ordinamento/filtro avanzato, anche se i controlli sono disattivati o nascosti.
