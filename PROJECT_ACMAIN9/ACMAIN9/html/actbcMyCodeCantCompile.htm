Il codice non viene compilato

Quando si converte un database di Microsoft Access versione 2.0 in Microsoft Access 2000, l'utilità di conversione converte il codice Access Basic in codice Visual Basic, Applications Edition (VBA). Se la compilazione di una parte di codice non riesce, potrebbe comparire un messaggio di errore dopo la conversione in Microsoft Access 2000.

Nel database di Access convertito, aprire un modulo in visualizzazione Struttura e scegliere Compila tutti i moduli dal menu Debug. Mentre Microsoft Access esegue la compilazione, si ferma ad ogni riga di codice che contiene un errore. È possibile quindi modificare la sintassi per risolvere l'errore di compilazione.

In rari casi l'utilità di conversione potrebbe non riuscire a convertire la sintassi. Potrebbe per esempio non riuscire a modificare un'istruzione DoCmd in un metodo DoCmd. Potrebbero inoltre comparire messaggi di errori di compilazione se il database di Microsoft Access versione 2.0 contiene errori di sintassi. Per questa ragione è preferibile compilare tutti i moduli di database di versioni precedenti prima di convertirli in Microsoft Access 2000.

Microsoft Access 2000 non supporta la libreria di compatibilità DAO 2.5/3.x. Se si tenta di convertire un database in cui il codice contiene versioni meno recenti di oggetti DAO e di metodi e proprietà che dipendono dalla libreria di compatibilità DAO 2.5/3.x, viene visualizzato un messaggio relativo a errori di compilazione durante la conversione del database. Prima di convertire il database, aggiornare il codice in modo che non dipenda dalla libreria di compatibilità DAO 2.5/3.x. Se il messaggio continua a essere visualizzato, aprire il database convertito in Microsoft Access 2000, rimuovere il riferimento alla libreria di compatibilità DAO 2.5/3.x mancante, quindi impostare un riferimento alla libreria di oggetti Microsoft DAO 3.6.
