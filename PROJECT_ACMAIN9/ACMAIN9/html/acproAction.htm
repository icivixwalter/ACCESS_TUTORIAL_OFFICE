Proprietà Action

Proprietà Action

                 

È possibile utilizzare la proprietà Action in Visual Basic per specificare l'operazione da eseguire su un oggetto OLE.

Impostazione

La proprietà Action viene impostata come riportato di seguito.

Costante Descrizione
acOLECreateEmbed (0) Crea un oggetto incorporato. Per utilizzare questa impostazione, è necessario innanzitutto impostare a acOLEEmbedded o acOLEEither la proprietà OLETypeAllowed del controllo. Impostare la proprietà Class al tipo di oggetto OLE che si desidera creare. È possibile utilizzare la proprietà SourceDoc per utilizzare come modello un file esistente.
acOLECreateLink (1) Crea un oggetto OLE collegato dal contenuto di un file. Per utilizzare questa impostazione, è necessario innanzitutto impostare le proprietà OLETypeAllowed e SourceDoc del controllo. Impostare a acOLELinked o acOLEEither la proprietà OLETypeAllowed. La proprietà SourceDoc specifica il file utilizzato per creare l'oggetto OLE. È inoltre possibile impostare la proprietà SourceItem del controllo per specificare, ad esempio, un intervallo di righe e colonne, se l'oggetto che si crea è un foglio di lavoro di Microsoft Excel. Quando si crea un oggetto OLE utilizzando questa impostazione, il controllo visualizza un metafile con immagine grafica relativo al file specificato dalla proprietà SourceDoc del controllo. Se si salva l'oggetto OLE, vengono salvate solo le informazioni di collegamento, come il nome dell'applicazione che ha fornito l'oggetto e il nome del file collegato, poiché il controllo contiene un'immagine dei dati, ma non i dati origine.
acOLECopy (4) Copia un oggetto negli Appunti. Quando si copia un oggetto OLE negli Appunti, anche tutti i dati e le informazioni di collegamento associati all'oggetto vengono copiati negli Appunti. È possibile copiare negli Appunti sia gli oggetti incorporati che quelli collegati. L'uso di questa impostazione equivale a scegliere Copia dal menu Modifica.
acOLEPaste (5) Incolla i dati dagli Appunti al controllo. Se l'operazione riesce, la proprietà OLEType del controllo viene impostata a acOLELinked o acOLEEmbedded. Se l'operazione non riesce, la proprietà OLEType viene impostata a acOLENone. L'uso di questa impostazione equivale a scegliere Incolla dal menu Modifica.
acOLEUpdate (6) Recupera i dati correnti dall'applicazione che fornisce l'oggetto e visualizza tali dati nel controllo come metafile grafico.
acOLEActivate (7) Apre un oggetto OLE per un'operazione, come la modifica. Per utilizzare questa impostazione è necessario innanzitutto impostare la proprietà Verb del controllo. La proprietà Verb specifica l'operazione da eseguire quando viene attivato l'oggetto OLE.
acOLEClose (9) Chiude un oggetto OLE e termina la connessione con l'applicazione che ha fornito l'oggetto. Questa impostazione si applica solo a oggetti incorporati. L'uso di questa impostazione equivale a scegliere Chiudi dal menu di controllo.
acOLEDelete (10) Elimina l'oggetto OLE specificato e libera la memoria associata. Questa impostazione consente di eliminare esplicitamente un oggetto OLE. Gli oggetti vengono automaticamente eliminati quando si chiude una maschera o quando si aggiorna l'oggetto con un nuovo oggetto. Non è possibile utilizzare la proprietà Action per eliminare un oggetto OLE associato dalla relativa tabella o query sottostante.
acOLEInsertObjDlg (14) Visualizza la finestra di dialogo Inserisci oggetto. Si visualizza tale finestra di dialogo in visualizzazione Maschera o Foglio dati per consentire di creare un nuovo oggetto o di collegare o incorporare un oggetto esistente. È possibile utilizzare la proprietà OLETypeAllowed del controllo per determinare il tipo di oggetto che è possibile creare con la costante acOLELinked, acOLEEmbedded o acOLEEither, utilizzando questa finestra di dialogo.
acOLEPasteSpecialDlg (15) Visualizza la finestra di dialogo Incolla speciale. Si visualizza tale finestra di dialogo in visualizzazione Maschera o Foglio dati per consentire di incollare un oggetto dagli Appunti. La finestra di dialogo offre più opzioni, compresa l'opzione di incollare un oggetto collegato o incorporato. È possibile utilizzare la proprietà OLETypeAllowed del controllo per determinare il tipo di oggetto che è possibile incollare con la costante acOLELinked, acOLEEmbedded o acOLEEither, utilizzando questa finestra di dialogo.
acOLEFetchVerbs (17) Aggiorna l'elenco di verbi supportati da un oggetto OLE. Per visualizzare l'elenco di verbi, utilizzare le proprietà ObjectVerbs e ObjectVerbsCount.

È possibile impostare la proprietà Action solo utilizzando Visual Basic. L'impostazione della proprietà Action ha un valore di tipo di dati Intero.

La proprietà Action non è disponibile in visualizzazione Struttura ma è possibile leggerla o impostarla in altre visualizzazioni.

Osservazioni

Quando, relativamente a un controllo, la proprietà Enabled è impostata a No o la proprietà Locked è impostata a Sì, non è possibile utilizzare alcune delle impostazioni della proprietà Action.
