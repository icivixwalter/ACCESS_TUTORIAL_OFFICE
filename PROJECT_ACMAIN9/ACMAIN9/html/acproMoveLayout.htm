Proprietà MoveLayout, NextRecord, PrintSection

                 

In Visual Basic è possibile utilizzare varie combinazioni di impostazioni delle proprietà riportate di seguito per specificare o modificare il layout di un report durante l'anteprima di stampa, la stampa, o il salvataggio in un file.

  • MoveLayout. Questa proprietà determina se Microsoft Access deve spostarsi alla successiva posizione di stampa nella pagina.

  • NextRecord. Questa proprietà determina se una sezione deve passare al record successivo.

  • PrintSection. Questa proprietà determina se una sezione deve essere stampata.

Impostazione

La proprietà MoveLayout viene impostata come riportato di seguito.

Impostazione Descrizione
True (-1) Impostazione predefinita. Le proprietà Left e Top della sezione vengono spostate alla successiva posizione di stampa.
False (0) Le proprietà Left e Top della sezione rimangono invariate.

La proprietà NextRecord viene impostata come riportato di seguito.

Impostazione Descrizione
True Impostazione predefinita. La sezione viene spostata al record successivo.
False La sezione non viene spostata al record successivo.

La proprietà PrintSection viene impostata come riportato di seguito.

Impostazione Descrizione
True Impostazione predefinita. La sezione viene stampata.
False La sezione non viene stampata.

Per impostare tali proprietà, specificare una macro o una routine evento per la proprietà OnFormat di una sezione.

Microsoft Access imposta tali proprietà a True prima dell'evento Formattazione di ciascuna sezione.

Osservazioni

Tali proprietà risultano utili quando si desidera utilizzare un report come modello in cui scrivere dati di una macro o di Visual Basic durante la stampa.

La tabella riportata di seguito indica il risultato di diverse combinazioni delle impostazioni di tali proprietà.

MoveLayout NextRecord PrintSection Descrizione
True True True Impostazione predefinita. Si sposta alla posizione di stampa successiva, al record successivo e stampa i dati.
True False True Si sposta alla posizione di stampa successiva, non si sposta al record successivo, ma stampa i dati. Si tratta della combinazione in genere utilizzata quando i dati di una sezione richiedono più spazio di quello consentito dal layout e si desidera stampare i rimanenti dati nello spazio che verrebbe altrimenti occupato dalla sezione successiva.
False True False Salta un record senza lasciare uno spazio vuoto nella pagina.
True True False Salta un record e lascia uno spazio vuoto nella pagina.
True False False Lascia uno spazio vuoto senza saltare un record.
False True True Stampa il record corrente al di sopra dell'ultimo record, sovrapponendolo.
False False True Non consentita.
False False False Non consentita.
