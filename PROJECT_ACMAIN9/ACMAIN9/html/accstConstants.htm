Informazioni preliminari sulle costanti di Microsoft Access

Generale

Una costante rappresenta un valore numerico o una stringa che rimane invariata. È possibile utilizzare le costanti per migliorare la leggibilità del codice Visual Basic e per gestirlo con più facilità. L'utilizzo di costanti intrinseche assicura inoltre il funzionamento del codice anche quando i valori sottostanti rappresentati dalle costanti vengono modificati nelle successive versioni di Microsoft Access.

Microsoft Access supporta tre tipi di costanti:

  • Costanti simboliche, create tramite l'istruzione Const e utilizzate nei moduli.

  • Costanti intrinseche, che fanno parte di Microsoft Access o di una libreria a cui si fa riferimento.

  • Costanti definite dal sistema: True, False e Null.

Costanti simboliche

Spesso può accadere che nel codice vengono utilizzati ripetutamente gli stessi valori oppure si scopre che il codice dipende da numeri che non hanno un ovvio significato. In questi casi è possibile semplificare la leggibilità e la gestione del codice tramite costanti simboliche o definite dall'utente che consentono di utilizzare un nome significativo invece di un numero o di una stringa.

Dopo aver creato una costante tramite l'istruzione Const, non è possibile modificarla o assegnarle un nuovo valore. Non è possibile inoltre creare una costante con lo stesso nome di una costante intrinseca.

Gli esempi riportati di seguito illustrano alcune modalità di utilizzo dell'istruzione Const per dichiarare costanti numeriche e costanti stringhe.

Const conPI = 3.14159265 
' PI corrisponde a questo numero.
Const conPI2 = conPI *2 
' Costante utilizzata per creare un'altra costante.
Const conVersione = "Versione 7.0"
' Dichiara una costante stringa.

Costanti intrinseche

Oltre alle costanti dichiarate con l'istruzione Const, vengono dichiarate automaticamente un numero di costanti intrinseche e viene fornito l'accesso alle costanti di Visual Basic, Applications Edition (VBA) e alle costanti ADO. È possibile inoltre utilizzare le costanti in altre librerie di oggetti a cui si fa riferimento. Per ulteriori informazioni sull'aggiunta di riferimenti, vedere Impostazione di riferimenti a librerie dei tipi.

È possibile utilizzare le costanti intrinseche in una macro o in Visual Basic. Queste costanti sono sempre disponibili. Le costanti specifiche incorporate utilizzate con una funzione, un metodo o una proprietà particolare vengono descritte nell'argomento della Guida relativo alla funzione, al metodo o alla proprietà corrispondente.

Nota   È possibile utilizzare il Visualizzatore oggetti per visualizzare gli elenchi delle costanti intrinseche di tutte le librerie di oggetti disponibili.

Le costanti intrinseche hanno un prefisso di due lettere che identifica la libreria di oggetti che definisce la costante. Le costanti della libreria di Microsoft Access hanno il prefisso "ac", quelle della libreria ADO "ad" e quelle della libreria Visual Basic "vb". Ad esempio:

  • acForm

  • adAddNew

  • vbCurrency

Nota   Poiché i valori rappresentati dalle costanti intrinseche possono cambiare nelle versioni successive di Microsoft Access, è consigliabile utilizzare le costanti invece dei valori effettivi. È possibile tuttavia visualizzare il valore effettivo di una costante scegliendo la costante nel Visualizzatore oggetti oppure digitando nomecostante nella finestra Immediata.

Le costanti intrinseche possono essere utilizzate nei punti in cui è possibile utilizzare le costanti simboliche o quelle definite dall'utente, compreso nelle espressioni. L'esempio riportato di seguito illustra il modo in cui è possibile utilizzare la costante intrinseca vbCurrency per determinare se la variabile varNum è di tipo Variant in base alla quale la funzione VarType restituisce 6 (Valuta).

Dim varNum As Variant
If VarType(varNum) = vbCurrency Then
Debug.Print "varNum contiene dati Valuta."
Else
Debug.Print "varNum non contiene dati Valuta."
End If

Esistono numerose categorie di costanti intrinseche. Per visualizzare un elenco delle costanti intrinseche di una determinata categoria, fare clic su una delle categorie riportate di seguito:

  • Costanti di azione

  • Costanti ADO

  • Costanti DAO

  • Costanti di routine evento

  • Costanti di codicechiave

  • Costanti varie

  • Costanti del metodo RunCommand (si tratta di un sottoinsieme delle costanti di azione)

  • Costanti di protezione

  • Costanti di Visual Basic

  • Costanti della funzione VarType

Costanti definite dal sistema

Le costanti definite dal sistema True, False e Null possono essere utilizzate ovunque in Microsoft Access. È possibile ad esempio utilizzare True nella seguente espressione di condizione della macro. La condizione viene soddisfatta se l'impostazione della proprietà Visible per la maschera Impiegati corrisponde a True.

Forms!Impiegati.Visible = True

La costante Null può essere utilizzata ovunque in Microsoft Access. È possibile ad esempio utilizzare Null per impostare la proprietà DefaultValue per un controllo di maschera utilizzando la seguente espressione:

=Null
