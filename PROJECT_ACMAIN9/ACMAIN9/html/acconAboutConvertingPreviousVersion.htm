Informazioni sulla conversione o attivazione di un database protetto da una versione precedente di Microsoft Access

Prima di utilizzare un database protetto da una versione precedente di Microsoft Access è necessario effettuare dei passaggi aggiuntivi sulla base di alcune considerazioni fondamentali.

Importante   Per convertire o attivare un database protetto, devono essere soddisfatti i requisiti di seguito riportati, altrimenti la conversione non avrà luogo:

  • Nel corso dell'installazione di Microsoft Access 2000 viene creato un nuovo file di informazioni sul gruppo di lavoro, che diventa il file corrente. Prima di convertire il database protetto, è pertanto necessario entrare a far parte del file di informazioni sul gruppo di lavoro che definisce gli account utente utilizzati per accedere al database che si desidera convertire o che erano in uso quando il database è stato inizialmente protetto. Per informazioni su come entrare a far parte di un file di informazioni sul gruppo di lavoro, fare clic su .

  • L'account utente utilizzato per la connessione durante la conversione deve disporre delle seguenti autorizzazioni:

    Autorizzazioni di Apertura/esecuzione e Apertura esclusiva per il database.

    Autorizzazione di Modifica struttura o Amministrazione per tutte le tabelle del database, altrimenti è necessario essere il proprietario di tutte le tabelle del database.

    Autorizzazione di Lettura struttura per tutti gli oggetti del database.

Se tutti gli utenti di un database protetto di una versione precedente intendono eseguire l'aggiornamento a Microsoft Access 2000, è opportuno convertire il database e il relativo file di informazioni sul gruppo di lavoro. Le operazioni successive dipendono dalla versione di Microsoft Access da cui si esegue l'aggiornamento.

Se soltanto alcuni utenti di un database protetto intendono eseguire l'aggiornamento a Microsoft Access 2000, è possibile condividere il database e il file di informazioni sul gruppo di lavoro tra tutte le versioni di Microsoft Access in uso al momento. Vi sono diversi modi per condividere il database protetto e il relativo file di informazioni sul gruppo di lavoro.

Importante   Eseguire la conversione di un database protetto di una versione precedente solo se tutti gli utenti di tale database intendono eseguire l'aggiornamento a Microsoft Access 2000. Se sono solo alcuni gli utenti che intendono procedere in tal modo, è possibile condividere il database nel formato corrente.

Importante   Poiché non è possibile proteggere il codice Microsoft Visual Basic nei moduli e il codice di maschere e report con la protezione a livello utente di Microsoft Access 2000, il codice convertito da un database protetto di una versione precedente non sarà più protetto. Per ulteriori informazioni, vedere l'argomento relativo alla protezione del codice Visual Basic.
