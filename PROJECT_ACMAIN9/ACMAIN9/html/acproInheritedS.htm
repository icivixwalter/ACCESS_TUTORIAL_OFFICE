Proprietà Inherited (MDB)

Microsoft Access definisce un numero di proprietà che vengono applicate ad oggetti DAO. Poiché tali proprietà vengono definite da Microsoft Access, il modulo di gestione di database Microsoft Jet non le riconosce automaticamente. Per impostare una di queste proprietà utilizzando Visual Basic, è necessario prima creare la proprietà utilizzando il metodo CreateProperty, quindi accodare la proprietà all'insieme Properties dell'oggetto. Per ulteriori informazioni, vedere l'argomento relativo all'oggetto Property nell'indice della Guida.

Quando si crea un oggetto basato su un altro oggetto, l'oggetto derivato eredita le proprietà dell'oggetto di origine. È possibile utilizzare la proprietà Inherited per determinare se una proprietà è stata creata per l'oggetto a cui viene applicata o se la proprietà è stata ereditata da un altro oggetto.

Si supponga, ad esempio, di voler impostare la proprietà DatasheetFontName di Microsoft Access per un oggetto TableDef. Se si imposta tale proprietà per la prima volta, è necessario creare un oggetto Property corrispondente ed accodare l'oggetto all'insieme Properties dell'oggetto TableDef. La proprietà Inherited del nuovo oggetto Property restituisce False (0).

Se si crea, quindi, un nuovo oggetto QueryDef in base alla tabella corrispondente all'oggetto TableDef, la proprietà DatasheetFontName verrà inclusa nell'insieme Properties dell'oggetto QueryDef. La proprietà Inherited relativa a un oggetto Property corrispondente a tale proprietà restituisce True (-1).
