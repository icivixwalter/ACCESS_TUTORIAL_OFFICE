Ordine degli eventi per i report e le sezioni di report

Gli eventi per report e sezioni di report si verificano quando un report viene aperto per essere stampato o visualizzato in anteprima oppure quando viene chiuso.

Eventi per report

Quando un report viene aperto per essere stampato o visualizzato in anteprima e in seguito chiuso o quando ci si sposta in un'altra finestra di Microsoft Access, si verifica la sequenza di eventi riportata di seguito:

Apertura Þ Attivazione Þ Chiusura Þ Disattivazione

Quando si passa da un report aperto all'altro, l'evento Disattivazione si verifica per il primo report e l'evento Attivazione per il secondo report:

Disattivazione (report1) Þ Attivazione (report2)

L'evento Disattivazione di un report si verifica anche quando si passa dal report a un'altra finestra di Access. L'evento Disattivazione tuttavia non si verifica quando si passa a una finestra di dialogo, a una maschera la cui proprietà Popup è impostata a Sì o a una finestra di un'altra applicazione.

Quando viene aperto un report basato su una query, viene attivato l'evento Apertura prima che venga eseguita la query sottostante. Da ciò risulta che è possibile impostare i criteri per il report utilizzando una macro o una routine evento che risponde all'evento Apertura. La macro o la routine evento ad esempio potrebbe aprire una finestra di dialogo personalizzata nella quale immettere i criteri per il report.

Eventi per sezioni di report

Quando un report viene stampato o visualizzato in anteprima, gli eventi Formattazione e Stampa si verificano per le sezioni di report dopo gli eventi Apertura e Attivazione per il report e prima degli eventi Chiusura o Disattivazione del report:

Apertura (report) Þ Attivazione (report) Þ Formattazione (sezione di report) Þ Stampa (sezione di report) Þ Chiusura (report) Þ Disattivazione (report)

È possibile inoltre che gli eventi seguenti si verifichino durante o dopo la formattazione ma prima dell'evento Stampa:

  • L'evento Riesame si verifica quando si ritorna a una sezione precedente nel corso della formattazione di un report.

  • L'evento ReportVuoto si verifica se il report non visualizza alcun record.

  • L'evento Pagina si verifica dopo la formattazione ma prima della stampa. È possibile utilizzare questo evento per personalizzare l'aspetto del report stampato.
