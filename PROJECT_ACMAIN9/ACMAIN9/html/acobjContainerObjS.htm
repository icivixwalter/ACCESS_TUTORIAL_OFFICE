Oggetto Container, insieme Containers (MDB)

Oltre agli oggetti Container definiti dal modulo di gestione di database Jet, i cinque oggetti Container indicati di seguito vengono definiti da Microsoft Access.

Nome contenitore Contiene informazioni su
Forms Maschere salvate
Modules Moduli salvati
Reports Report salvati
DataAccessPages Pagine di accesso ai dati salvate
Scripts Macro salvate

Gli oggetti Container definiti da Microsoft Access costituiscono oggetti di database di Microsoft Access e non oggetti (DAO) e forniscono al modulo di gestione di database Jet le informazioni sugli oggetti di Microsoft Access. Il modulo di gestione di database Jet usa tali informazioni per implementare la protezione per gli oggetti di database di Microsoft Access analogamente a quanto avviene per gli oggetti DAO.

Gli oggetti Container definiti da Microsoft Access e gli oggetti Container definiti dal modulo di gestione di database Jet sono compresi tutti nell'insieme Containers di Microsoft Access.

L'insieme Documents di un oggetto Container contiene gli oggetti Document che rappresentano i singoli oggetti del tipo descritto dall'oggetto Container. Ad esempio, l'oggetto Container Forms ha un insieme Documents che potrebbe comprendere un oggetto Document corrispondente a una maschera denominata Ordini.

Esclusi i contenitori Modules, gli oggetti Container possono essere utilizzati per stabilire e rendere operative le autorizzazioni per oggetti di database Microsoft Access. Per impostare le autorizzazioni per un oggetto Container, impostare la proprietà UserName dell'oggetto Container sul nome di un oggetto User o Group esistente, quindi impostare la proprietà Permissions dell'oggetto Container.

Nota   Fare attenzione a non confondere i vari tipi di oggetto Container con i tipi di insieme che hanno lo stesso nome. Gli oggetti Container Forms e Reports contengono insiemi Documents, che comprendono singoli oggetti Document rappresentanti ogni maschera o report salvato. Gli insiemi Forms e Reports fanno riferimento solo a maschere o report aperti.

Il contenitore Modules è ancora presente in Microsoft Access 2000 per garantire la compatibilità con le versioni precedenti. Siccome in questa nuova versione la protezione dei moduli viene gestita all'interno dell'ambiente Visual Basic dalla funzione Protezione, le autorizzazioni e le proprietà relative agli utenti dei singoli moduli non sono più supportate.
