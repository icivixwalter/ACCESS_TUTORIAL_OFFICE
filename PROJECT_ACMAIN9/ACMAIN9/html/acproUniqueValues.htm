Proprietà UniqueValues (Valori univoci)

                 

È possibile utilizzare la proprietà UniqueValues quando si desidera omettere i record contenenti dati duplicati nei campi visualizzati in visualizzazione Foglio dati. Se ad esempio l'output di una query include più di un campo, la combinazione dei valori di tutti i campi deve essere univoca perché un dato record venga incluso nel risultato.

Nota   La proprietà UniqueValues si applica solo a query di comando di accodamento e di creazione tabella, e a query di selezione.

Impostazione

La proprietà UniqueValues viene impostata come riportato di seguito.

Impostazione Descrizione
Sì Vengono visualizzati solo i record in cui i valori di tutti i campi visualizzati in visualizzazione Foglio dati sono univoci.
No Impostazione predefinita. Vengono visualizzati tutti i record.

È possibile impostare la proprietà UniqueValues nella finestra delle proprietà della query o in visualizzazione SQL della finestra di query.

Nota   È possibile impostare tale proprietà quando si crea una nuova query utilizzando un'istruzione SQL. Il predicato DISTINCT corrisponde all'impostazione della proprietà UniqueValues. Il predicato DISTINCTROW corrisponde all'impostazione della proprietà UniqueRecords.

Osservazioni

Quando si imposta a Sì la proprietà UniqueValues, i risultati della query non possono essere aggiornati e non riflettono le modifiche successive effettuate da altri utenti.

Le proprietà UniqueValues e UniqueRecords sono correlate, in quanto è possibile impostare a Sì solo una delle due proprietà alla volta. Quando si imposta a Sì la proprietà UniqueValues, ad esempio, la proprietà UniqueRecords viene automaticamente impostata a No. È tuttavia possibile impostare entrambe le proprietà a No. In tal caso vengono restituiti tutti i record.

Suggerimento   Se si desidera contare il numero di esempi di un valore in un campo, creare una query Totali.
