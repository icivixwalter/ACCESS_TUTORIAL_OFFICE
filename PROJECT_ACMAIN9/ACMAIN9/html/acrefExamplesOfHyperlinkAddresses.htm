Esempi di indirizzi di collegamenti ipertestuali

La seguente tabella contiene alcuni esempi di indirizzi di collegamenti ipertestuali. Per gli URL vengono utilizzate le barre, mentre per i percorsi UNC vengono utilizzate le barre rovesciate.

Microsoft Corporation#http://www.microsoft.com# La Home Page Microsoft sul Web, visualizzata come "Microsoft Corporation".
#http://www.microsoft.com# La Home Page Microsoft sul Web.
#http://www.microsoft.com#esempio# Tag Name chiamato "esempio" nel documento html della Home Page Microsoft.
#\\northwind\public\prodotti.doc##Elenco di prodotti scontati File di Microsoft Word chiamato Prodotti.doc nella directory Public del server Northwind. La descrizione è "Elenco di prodotti scontati".
Curriculum#c:\windows\personal\curricul.doc# File di Word chiamato Curricul.doc nella cartella \Windows\Personal. Visualizzato come "Curriculum."

Per facilitare l'immissione di un indirizzo di collegamento ipertestuale, è possibile omettere alcune parti dell'indirizzo. Ad esempio:

  • Se si digita http://esempio.microsoft.com, viene riconosciuto il protocollo "http:" e l'indirizzo viene tradotto come #http://esempio.microsoft.com#. Non è necessario immettere i simboli di cancelletto (#).

  • Se si desidera fare riferimento a un oggetto di Microsoft Access del database corrente, è sufficiente specificare il nome dell'oggetto che si desidera collegare, senza il percorso UNC. Se esistono più oggetti con lo stesso nome, viene utilizzato il seguente ordine per determinare a quale fare riferimento: maschere, report, query, tabelle, pagine di accesso ai dati, macro, moduli, visualizzazioni, schemi, stored procedure, tabelle collegate SQL, tabelle collegate e trigger. Se, ad esempio, esiste una maschera Ordini e una tabella Ordini, il riferimento sarà alla maschera Ordini. È possibile specificare un determinato tipo di oggetto immettendo tipo oggetto nome oggetto. Per fare riferimento alla tabella Ordini anziché alla maschera Ordini, ad esempio, immettere Tabella Ordini come sottoindirizzo. Per i tipi di oggetto comprendenti più parole, omettere gli spazi tra le parole, ad esempio inserendo paginadiaccessoaidati Ordini.

  • Se il testo immesso non include il simbolo di cancelletto (#), un protocollo, ad esempio "http:", o un riferimento a un oggetto del database corrente, il testo immesso viene automaticamente interpretato come testo da visualizzare e lo stesso testo, preceduto da "http://", viene considerato l'indirizzo. Se, ad esempio, si digita esempio.microsoft.com, l'indirizzo viene tradotto in esempio.microsoft.com#http://esempio.microsoft.com#.
