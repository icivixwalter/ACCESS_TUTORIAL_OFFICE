Evento Termine - Routine evento

        

Per creare la routine evento per l'evento Termine, aprire il modulo di classe, selezionare Class dalla casella Oggetto, quindi Terminate dalla casella Routine.

Sintassi

Private Sub Class_Terminate( )

Osservazioni

Per rimuovere dalla memoria un riferimento a un'istanza di una classe, è possibile impostare a Nothing la variabile alla quale è assegnato il riferimento. Si supponga ad esempio di aver creato un'istanza di una classe denominata OggettoPersonalizzato e di averla assegnata alla variabile di un oggetto denominata obj. Quando obj viene impostata a Nothing, il riferimento viene rimosso dalla memoria. Quando tutti i riferimenti all'istanza della classe sono stati rimossi dalla memoria, si verifica l'evento Termine.

Un riferimento a un'istanza di una classe viene rimosso dalla memoria anche quando la variabile dell'oggetto alla quale è assegnato il riferimento diviene non più valida. Se ad esempio la variabile viene definita all'interno di una routine, dopo l'esecuzione della routine tale variabile non sarà più valida. Se la variabile viene definita a livello di modulo in un modulo standard, la variabile non sarà più valida quando il codice viene reimpostato. L'evento Termine si verifica anche se tutti i riferimenti a un'istanza di una classe vengono rimossi in questo modo.

L'evento Termine non viene attivato se le istanze di una classe sono state rimosse dalla memoria perché Microsoft Access è stato chiuso in modo anomalo. Se ad esempio il codice richiama il metodo Quit dell'oggetto Application di Microsoft Access prima che tutte le istanze esistenti della classe siano rimosse dalla memoria, l'evento Termine non viene attivato per quella classe.
