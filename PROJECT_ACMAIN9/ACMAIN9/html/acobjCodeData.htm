Oggetto CodeData

                                   



L'oggetto CodeData fa riferimento agli oggetti memorizzati nel database di codice utilizzando l'applicazione di origine (server), che è Jet o SQL.

Osservazioni

L'oggetto CodeData dispone di numerosi insiemi contenenti tipi di oggetti specifici all'interno del database di codice. Nella tabella riportata di seguito vengono elencati i nomi di ogni insieme definito dal database e dei tipi di oggetto in esso contenuti.

Insiemi Tipo di oggetto
AllTables Tutte le tabelle
AllQueries Tutte le query (il conteggio delle query in un progetto di Microsoft Access (adp) darà zero).
AllViews Tutte le visualizzazioni (il conteggio delle visualizzazioni in un database di Access (mdb) darà zero).
AllStoredProcedures Tutte le stored procedure (il conteggio delle stored procedure in un database MDB darà zero).
AllDatabaseDiagrams Tutti i diagrammi di database (il conteggio dei diagrammi di database in un database MDB darà zero).

Nota Gli insiemi della tabella precedente contengono tutti i rispettivi oggetti nel database, indipendentemente dal fatto che siano aperti o chiusi.

Ad esempio, un oggetto AccessObject che rappresenta una tabella fa parte dell'insieme AllTables, che è uno degli insiemi di oggetti AccessObject all'interno del database corrente. Nell'insieme AllTables, le singole tabelle vengono indicizzate a partire da zero. È possibile fare riferimento a un singolo oggetto AccessObject dell'insieme AllTables utilizzando il nome della tabella oppurte il suo indice all'interno dell'insieme. Se si desidera fare riferimento a un elemento specifico dell'insieme AllTables, è preferibile utilizzare il nome, poiché l'indice dell'elemento può variare. Se il nome dell'oggetto include uno spazio, è necessario mettere il nome tra parentesi quadre ([ ]).

Sintassi Esempio
AllTables!nometabella AllTables!TabellaOrdini
AllTables![nome tabella] AllTables![Tabella Ordini]
AllTables("nometabella") AllTables("TabellaOrdini")
AllTables(indice) AllTables(0)
