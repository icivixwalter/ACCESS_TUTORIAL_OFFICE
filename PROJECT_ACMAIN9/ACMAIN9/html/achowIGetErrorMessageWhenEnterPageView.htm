Restituzione di un messaggio quando si tenta di immettere dati in visualizzazione Pagina

Quando si tenta di uscire da un campo dopo aver modificato i dati, questi vengono sottoposti a convalida per verificare che il valore immesso sia consentito per quel tipo di campo. Se il valore non è consentito, verrà visualizzato un messaggio. Per spostarsi dal campo, è necessario modificare i dati immettendo un valore accettabile oppure annullare le modifiche apportate.

Un valore può non essere valido per un campo per i seguenti motivi:

  • Il valore non è compatibile con il tipo di dati del campo. Se si immette un valore che non corrisponde al tipo di dati, viene effettuato un tentativo di convertire il tipo di dati errato in un tipo valido. Alcuni tipi di dati però non possono essere convertiti in altri tipi di dati. Non è possibile ad esempio memorizzare un nome in un campo di tipo Numerico in un database di Microsoft Access o dati di tipo smallint in un progetto di Microsoft Access.

  • I dati violano una regola impostata per la proprietà Valido se del campo.

  • L'immissione dei dati è obbligatoria, ma si è tentato di lasciare il campo vuoto.

  • La macro di convalida specificata per la proprietà Prima di aggiornare ha annullato l'aggiornamento.

I dati vengono convalidati anche quando si esce da un record. Prima di salvare un record, viene infatti verificato che le modifiche apportate non violino le regole di convalida impostate per la proprietà Valido se del record. Se inoltre la tabella che si sta modificando ha una chiave primaria o un indice univoco, verrà verificato che il record che si sta modificando non abbia lo stesso valore di uno o più campi di un altro record nella tabella. Se il record non può essere salvato, sarà necessario apportare le correzioni necessarie o fare clic sul pulsante Annulla record.
