I file HTML generati dal server non vengono visualizzati in un browser Web nel modo previsto

Quando un foglio dati viene esportato in formato di output ASP o IDC/HTX, i dati che compaiono quando il file asp o htx viene visualizzato in un browser Web sono gli effettivi dati del set di record alla base del foglio dati. Per impostazione predefinita i campi Testo e Memo vengono salvati in Microsoft Access con allineamento sinistro e tutti gli altri tipi di dati supportati con allineamento destro.

Quando si esporta un foglio dati, in Microsoft Access non vengono esportati gli oggetti OLE o i valori collegati da un campo di ricerca e vengono ignorate le impostazioni delle proprietà OrderBy, Filter, Format e InputMask.

Se si desidera che i file HTML generati dal server presentino l'aspetto di un foglio dati in un database di Microsoft Access, è possibile creare ed esportare una query che presenta lo stesso set di record del foglio dati. Quando si crea la query, attenersi alla seguente procedura:

  • Utilizzare la cella Ordinamento nella griglia di progettazione query per ordinare i dati. Ulteriori informazioni sull'ordinamento di una query.

  • Utilizzare la cella Criteri della griglia di progettazione query per filtrare i dati. Ulteriori informazioni su aggiunta e modifica dei criteri in una query.

  • Utilizzare la funzione Formato per formattare i campi di tipo Valuta, Data/ora e Sì/No. Ulteriori informazioni sull'utilizzo della funzione Formato.

  • Creare una query di ricerca automatica per visualizzare i valori di un elenco di ricerca. Ulteriori informazioni sulle query di ricerca automatica.
