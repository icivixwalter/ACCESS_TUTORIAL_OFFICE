Informazioni sull'upsize di un database di Microsoft Access

Con il termine "upsize" si indica il processo di migrazione di alcuni o tutti gli oggetti di database da un database di Microsoft Access (mdb) a un database di Microsoft SQL Server nuovo o esistente oppure a un progetto di Microsoft Access (adp). Procedure di upsize

L'Upsize guidato consente di effettuare l'upsize di un database di Access in un database di Microsoft SQL Server 6.5 e 7.0 nuovo o esistente oppure in un nuovo progetto di Access effettuando l'upsize dei dati e delle relative definizioni ed eseguendo la migrazione degli oggetti di database. Vi sono tre modi per utilizzare l'Upsize guidato:

  • Effettuare l'upsize solo dei dati e delle relative definizioni da un database di Access in un database di SQL Server.

  • Creare un database di Access front-end da un database di SQL Server back-end in modo da ottenere un'applicazione front-end/back-end.

  • Effettuare l'upsize di tutti gli oggetti di database da un database di Access a un progetto di Access in modo da ottenere un'applicazioneclient/server.

Report Upsize guidato

L'Upsize guidato crea un report contenente una descrizione dettagliata di tutti gli oggetti creati e di tutti gli errori che si sono verificati durante il processo. L'Upsize guidato crea automaticamente questo report come uno snapshot con lo stesso nome del database di Access, ad esempio Northwind.snp, e lo memorizza nella stessa cartella.

Il Report Upsize guidato contiene:

  • Dettagli sul database, tra cui la dimensione, il file di registro delle transazioni e i nomi e le dimensioni delle periferiche per un database di SQL Server 6.5.

  • Parametri di upsize, inclusi gli attributi di tabella scelti per effettuare l'upsize e la modalità di esecuzione.

  • Informazioni relative alle tabelle, incluso un confronto tra i valori di Access e di SQL Server per i nomi, i tipi di dati, gli indici, le regole di convalida, i valori predefiniti, i trigger e se sono stati aggiunti o meno degli indicatori di data e ora.

  • Gli eventuali errori riscontrati, tra cui l'esaurimento dello spazio nel file di registro delle transazioni o del database, autorizzazioni inadeguate, la mancata creazione di database o periferiche, il mancato rispetto delle regole di convalida, dei valori di tabella o di quelli predefiniti, la non attuazione delle relazioni e l'inosservanza delle query (poiché la conversione nella sintassi di SQL Server non è possibile) e errori di conversione dei controlli e delle origini record nelle maschere e nei report.

Conversione del codice dell'applicazione

La progettazione e la creazione di un'applicazione client/server ottimizzata sono diverse dalla progettazione e dalla creazione di un'applicazione file server ottimizzata. Pertanto, è probabilmente necessario adeguare l'applicazione per sfruttare al meglio le funzionalità di SQL Server e di un progetto di Access. Dopo aver effettuato l'upsize, è necessario convertire manualmente il codice da Data Access Objects (DAO) a ActiveX Data Objects (ADO) nei moduli e convertire le query non sottoposte a upsize, ad esempio quelle nidificate a un livello troppo profondo. Per ulteriori informazioni sullo sviluppo di applicazioni client/server ottimizzate, consultare il Manuale del programmatore di Microsoft Office 2000/Visual Basic.
