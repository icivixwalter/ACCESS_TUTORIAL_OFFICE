Informazioni sulle query di Ricerca automatica che immettono dati automaticamente

È possibile strutturare una query a più tabelle per inserire automaticamente alcuni valori di campo per un nuovo record. Quando si immette un valore nel campo di join contenuto nella query o in una maschera, un report o una pagina di accesso ai dati basata sulla query, in Microsoft Access vengono ricercate e inserite le informazioni esistenti correlate a quel valore. Se ad esempio si conosce il valore contenuto nel campo di join tra una tabella Clienti e una tabella Ordini (generalmente un identificatore clienti come ID cliente), l'utente può immettere l'ID cliente, mentre le informazioni restanti relative a quel cliente vengono immesse dal programma. Se non vengono trovate informazioni corrispondenti, viene visualizzato un messaggio di errore quando il record non è più nello stato attivo sulla query.

Per consentire il funzionamento della Ricerca automatica, è necessario che siano soddisfatte alcune condizioni:

  • La query deve essere basata su più di una tabella e le tabelle devono avere una relazione uno-a-molti. L'integrità referenziale non deve essere applicata.

  • Il campo di join sul lato "uno" della relazione deve avere un indice univoco. Un indice univoco significa che il campo è una chiave primaria o che la relativa proprietà Indicizzato in visualizzazione Struttura della tabella è impostata a Sì (Duplicati non ammessi).

  • Il campo di join aggiunto alla griglia di struttura deve provenire dalla tabella sul lato "molti" della relazione uno-a-molti. In altre parole, deve essere la chiave esterna di quella tabella. Per fare in modo che la Ricerca automatica funzioni, questo campo non può essere una chiave primaria e la relativa proprietà Indicizzato non può essere impostata a Sì (Duplicati non ammessi). In una query che ad esempio include le tabelle Clienti e Ordini trascinare sulla griglia di struttura il campo ID cliente della tabella Ordini e non quello della tabella Clienti.

  • Il valore immesso nel campo di join dal lato "molti" deve essere già esistente nel campo di join dal lato "uno".

Quando il valore del campo di join dal lato "molti" della relazione viene aggiunto o modificato in un record, in Microsoft Access vengono trovati e visualizzati automaticamente i valori associati dalla tabella sul lato "uno" della relazione.

Mentre il campo di join dal alto "molti" della relazione può essere aggiornato in qualsiasi momento, il campo di join dal lato "uno" può essere aggiornato solo se è stata attivata l'opzione per gli aggiornamenti a catena durante la definizione della relazione tra le tabelle. In ogni caso, quando si aggiornano dei dati viene automaticamente eseguito il ricalcolo di qualsiasi formula o espressione contenuta nella query che dipenda dai dati aggiornati.
