Quando attivare un database di Access

È opportuno attivare un database di Microsoft Access in un ambiente multiutente quando non tutti gli utenti sono in grado di eseguire contemporaneamente l'aggiornamento a Microsoft Access 2000. In questa situazione è necessario utilizzare simultaneamente un database con versioni diverse di Microsoft Access e gli utenti che hanno eseguito l'aggiornamento a Microsoft Access 2000 possono attivare un database nel formato di una versione precedente. Quando un utente di Microsoft Access 2000 attiva il database, viene mantenuto il formato originale in modo da consentire agli utenti delle versioni precedenti di Microsoft Access di continuare a utilizzare il database. Se ad esempio viene creato un database condiviso in Microsoft Access 2.0, questo database potrà essere utilizzato con Microsoft Access 2.0 e attivato in Microsoft Access 2000.

Compatibilità con le versioni successive

È possibile attivare un database di Microsoft Access 2.0, di Access 95 o di Access 97 in modo da poterlo utilizzare in Microsoft Access 2000. Quando si utilizza Microsoft Access 2000 per attivare un database di Access di una versione precedente, è possibile visualizzare gli oggetti di database e aggiungere, eliminare o modificare record. Non è tuttavia possibile modificare la struttura degli oggetti. Per modificare la struttura di oggetti esistenti o per aggiungerne di nuovi è necessario aprire il database con la versione di Microsoft Access utilizzata per crearlo. Non è possibile aprire un database di Microsoft Access 2000 con una versione precedente di Microsoft Access.

Anziché attivare le applicazioni front-end/back-end, è possibile condividerle tra Microsoft Access 2.0, Access 95, Access 97 e Access 2000, purché il database back-end venga mantenuto nella versione meno recente di Microsoft Access utilizzata. Per ulteriori informazioni sull'utilizzo di un database di Microsoft Access con versioni diverse di Microsoft Access, fare clic su .

Struttura delle tabelle

Non è possibile collegare o importare una tabella di Access 2000 in un database di Access di una versione precedente. È tuttavia possibile esportare una tabella di Access 2000 in un database di Access di una versione precedente. È anche possibile tagliare, copiare e incollare i dati da una tabella di Access 2000 in una tabella di una versione precedente di Microsoft Access.

Nuove barre degli strumenti e barre dei menu

In Microsoft Access 97 o versioni successive è supportato un nuovo stile di barre degli strumenti e barre dei menu. Quando si attiva un database di Microsoft Access 95 in Microsoft Access 2000, le barre degli strumenti personalizzate vengono convertite nel nuovo stile, ma la conversione non viene salvata. Le barre dei menu personalizzate vengono interpretate come le nuove barre dei menu, ma le relative macro non vengono convertite, anche se continuano a essere supportate.

Attivazione di un database di Access protetto

Se il database di Access è protetto, è possibile mantenerne la protezione ricreando il file di informazioni sul gruppo di lavoro di Access 2000. Qualora non fosse possibile ricreare il file di informazioni sul gruppo di lavoro, utilizzare l'Amministratore gruppi di lavoro per aderire al file di informazioni sul gruppo di lavoro della versione precedente del database protetto. È tuttavia consigliabile convertire il file di informazioni sul gruppo di lavoro in Access 2000 prima di aderire al file. Se si utilizza il database con le informazioni sul gruppo di lavoro originali, verrà mantenuta tutta la protezione, a eccezione delle autorizzazioni per i moduli. In Access 2000 è possibile proteggere il codice Visual Basic, Applications Edition (VBA) con una password. Per ulteriori informazioni, vedere l'argomento relativo all'attivazione di un database protetto.

Dimensioni di un database di Access attivato

Quando si attiva un database di Access, le relative dimensioni possono aumentare. Se un database che è già stato attivato in una versione precedente di Microsoft Access viene nuovamente attivato in Access 2000, le dimensioni del progetto di Visual Basic aumenteranno ulteriormente. In rari casi è possibile che un database di Microsoft Access versione 2.0 sia stato attivato in Microsoft Access 95 per poi essere nuovamente attivato in Microsoft Access 97 e in Microsoft Access 2000. In questo caso è necessario che nel progetto di Visual Basic siano memorizzate le informazioni nel formato di ciascuna versione.

Compatibilità del codice

In Microsoft Access 2000 non è previsto il supporto per la libreria di compatibilità DAO 2.5/3.x. Se si tenta di attivare un database di Access il cui codice contiene versioni precedenti di oggetti, metodi e proprietà DAO che dipendono dalla libreria di compatibilità DAO 2.5/3.x, viene visualizzato un messaggio di errore di compilazione durante l'attivazione del database. Prima di attivare il database di Access, aggiornare il codice in modo che non dipenda dalla libreria di compatibilità DAO 2.5/3.x. Per informazioni sulla compatibilità tra Visual Basic di versioni precedenti di Microsoft Access e Visual Basic di Microsoft Access 2000, fare clic su .
