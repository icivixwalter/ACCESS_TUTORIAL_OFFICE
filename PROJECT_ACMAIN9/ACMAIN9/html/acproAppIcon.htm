Proprietà AppIcon

                 

È possibile utilizzare la proprietà AppIcon per specificare il nome del file bitmap (bmp) o icona (ico) che contiene l'icona dell'applicazione. È possibile, ad esempio, utilizzare la proprietà AppIcon per specificare un file BMP contenente l'immagine di un'automobile per rappresentare un'applicazione relativa a parti di ricambio di un'automobile.

Impostazione

La proprietà AppIcon è un'espressione stringa che rappresenta il nome di un file o di icona valida (incluso il percorso).

Il modo più semplice per impostare questa proprietà consiste nell'utilizzare l'opzione Icona applicazione della finestra Avvio, disponibile scegliendo Avvio dal menu Strumenti. In alternativa, è possibile utilizzare una macro o Visual Basic.

Per impostare la proprietà AppIcon utilizzando una macro o Visual Basic, è necessario prima impostarla una volta nella finestra di dialogo Avvio oppure crearla come segue:

  • In un database di Microsoft Access (MDB) è possibile aggiungerla utilizzando il metodo CreateProperty e accodandola all'insieme Properties dell'oggetto Database.

  • In un progetto di Microsoft Access (ADP) è possibile aggiungerla all'insieme AccessObjectProperties dell'oggetto CurrentProject utilizzando il metodo Add.

Per rendere le modifiche immediatamente visibili, è necessario usare anche il metodo RefreshTitleBar.

Osservazioni

Se l'applicazione viene distribuita, si raccomanda di fare in modo che il file BMP o ICO contenente l'icona risieda nella stessa directory dell'applicazione Microsoft Access.

Se la proprietà AppIcon non è stata impostata oppure non è valida, viene visualizzata l'icona di Microsoft Access.

L'impostazione di questa proprietà ha effetto subito dopo che viene impostata nel codice, purché questo includa il metodo RefreshTitleBar, oppure dopo che è stata chiusa la finestra Avvio.
