Modalità di upsize dell'applicazione

È possibile effettuare l'upsize dell'applicazione di database di Microsoft Access seguendo diverse modalità. In questo argomento verranno fornite informazioni di riferimento relative a ciascuna opzione di modifica dell'applicazione:

Nessuna

Collega tabelle di SQL Server all'applicazione esistente

Crea una nuova applicazione client/server di Access

Salva password e ID utente

Nessuna

Selezionare Nessuna per effettuare l'upsize dei dati senza apportare altre modifiche all'applicazione di database di Access esistente.

Collega tabelle di SQL Server all'applicazione esistente

Se si seleziona Collega tabelle di SQL Server all'applicazione esistente, l'Upsize guidato modifica il database di Access in modo che le query, le maschere, i report e le pagine di accesso ai dati utilizzino i dati del nuovo database di Microsoft SQL Server anziché quelli del database di Microsoft Access.

L'Upsize guidato rinomina le tabelle di Microsoft Access sottoposte all'upsize aggiungendo il suffisso "_local". Se ad esempio si effettua l'upsize di una tabella denominata Impiegati, questa verrà rinominata Impiegati_local nel database di Access. A questo punto l'Upsize guidato creerà una tabella di SQL Server collegata denominata Impiegati.

Le query, le maschere, i report e le pagine di accesso ai dati basati sulle tabelle Impiegati originali utilizzeranno ora la tabella Impiegati di SQL Server collegata. Molte delle proprietà dei campi della tabella locale originale derivano dalla nuova tabella locale, tra cui Descrizione, Etichetta, Formato, Maschera di input e Posizioni decimali. Per ulteriori informazioni, vedere l'argomento relativo alle proprietà della tabella collegata.

Crea una nuova applicazione client/server di Access

Se si seleziona Crea una nuova applicazione client/server di Access, l'Upsize guidato crea un nuovo progetto di Microsoft Access. L'Upsize guidato richiede di immettere un nome, che in base all'impostazione predefinita corrisponde al nome del database di Access corrente, quindi aggiunge un suffisso "CS" e lo memorizza nello stesso percorso del database di Access esistente.

L'Upsize guidato crea un file di progetto di Access, visualizza la finestra di dialogo Proprietà di Data Link in modo da potersi connettere a un database di SQL Server, quindi effettua l'upsize di tutti gli oggetti di database dal database di Access al progetto di Access:

  • Query   L'Upsize guidato modifica le query convertendole in visualizzazioni o in stored procedure ed effettuando l'upsize della sintassi SQL di Access affinché venga utilizzata la sintassi SQL di SQL Server. Le query di selezione vengono convertite in visualizzazioni. Le query ordinate vengono convertite in una combinazione di visualizzazioni e stored procedure per consentire la nidificazione e l'ordinamento. Tenere presente che le visualizzazioni possono essere nidificate, ma non possono contenere proposizioni ORDER BY, mentre le stored procedure possono contenere proposizioni ORDER BY, ma non possono essere nidificate. Le query con parametri, le query che dipendono da una query con parametri e le query di comando vengono convertite in stored procedure. Potrebbe essere necessario convertire manualmente le query non sottoposte all'upsize, ad esempio quelle nidificate a un livello troppo profondo. Le query SQL di tipo pass-through, le query di definizione dati e le query di unione non vengono sottoposte all'upsize.

  • Maschere, report e controlli   L'Upsize guidato modifica le proprietà Origine record, Origine controllo e Origine riga cambiando i nomi delle tabelle, delle query, delle istruzioni SQL e dei campi nei nomi equivalenti delle tabelle, delle visualizzazioni, delle stored procedure o dei campi di SQL Server.

  • Pagine di accesso ai dati   L'Upsize guidato modifica le informazioni relative alla connessione OLE DB e all'associazione dei dati nel controllo dell'origine dati di Microsoft Office affinché sia compatibile con il nuovo database di SQL Server, quindi copia il file HTML corrispondente della pagina nello stesso percorso del progetto di Access, rinominando il file HTML con un suffisso "_CS". Le nuove pagine del progetto di Access conservano il nome originale in modo che i collegamenti ipertestuali tra le pagine di SQL Server rimangano funzionanti.

  • Macro e moduli   L'Upsize guidato non apporta alcuna modifica a questi oggetti di database. Dopo avere effettuato l'upsize, è necessario convertire manualmente il codice che utilizza recordset di oggetti DAO (Data Access Object) in oggetti ADO (ActiveX Data Object) nei moduli e rivedere il codice di progettazione delle tabelle e delle query. L'Upsize guidato non effettua l'upsize del linguaggio di definizione dei dati (DDL) SQL. Per ulteriori informazioni sullo sviluppo di applicazioni client/server ottimizzate, vedere il Manuale del programmatore di Microsoft Office 2000/Visual Basic.

Salva password e ID utente

In base all'impostazione predefinita, la creazione guidata crea delle tabelle collegate nell'applicazione esistente oppure crea un progetto di Access senza salvare il nome utente e la password. Viene richiesto pertanto di immettere un nome e una password ogni volta che si effettua la connessione a un database di SQL Server.

Se si seleziona Salva password e ID utente, gli utenti possono accedere a un database di SQL Server senza effettuare la connessione. Se si seleziona Crea una nuova applicazione client/server di Access, il progetto di Access memorizza il nome utente e la password nella stringa di connessione OLE DB.

Nota   Questa opzione è disattivata per l'opzione Nessuna se una tabella di SQL Server collegata è configurata con una tabella MSysConf per impedire che le password vengano salvate.
