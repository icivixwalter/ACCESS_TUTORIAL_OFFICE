Metodo OpenForm

                 

Il metodo OpenForm esegue l'azione ApriMaschera in Visual Basic. Per ulteriori informazioni sul funzionamento dell'azione e dei relativi argomenti, vedere l'argomento della Guida in linea relativo all'azione.

Sintassi

DoCmd.OpenForm nomemaschera[, visualizzazione][, nomefiltro][, condizioneWHERE][, modalitàdati][, modalitàfinestra][, apriarg]

Il metodo OpenForm utilizza gli argomenti riportati di seguito.

Argomento Descrizione
nomemaschera Espressione stringa che è il nome valido di una maschera contenuta nel database corrente.
  Se si esegue del codice di Visual Basic contenente il metodo OpenForm in un database di libreria, Microsoft Access cerca la maschera con questo nome prima nel database di libreria, quindi nel database corrente.
visualizzazione Una delle seguenti costanti intrinseche:
  acDesign
acFormDS
acNormal (impostazione predefinita)
acPreview
  acNormal apre la maschera in visualizzazione Maschera.
  Se si omette questo argomento, viene utilizzata la costante predefinita (acNormal).
nomefiltro Espressione stringa che è il nome valido di una query contenuta nel database corrente.
condizioneWHERE Espressione stringa che è una proposizione SQL WHERE valida senza la parola WHERE.
modalitàdati Una delle seguenti costanti intrinseche:
  acFormAdd
acFormEdit
acFormPropertySettings (impostazione predefinita)
acFormReadOnly
  Se si omette questo argomento (viene utilizzata la costante predefinita acFormPropertySettings), Microsoft Access apre la maschera nella modalità dati impostata dalle proprietà AllowsEdits, AllowsDeletions, AllowsAdditions e DataEntry.
modalitàfinestra Una delle seguenti costanti intrinseche:
  acDialog
acHidden
acIcon
acWindowNormal (impostazione predefinita)
  Se si omette questo argomento, viene utilizzata la costante predefinita (acWindowNormal).
apriarg Espressione stringa. Questa espressione viene utilizzata per impostare la proprietà OpenArgs della maschera. Questa impostazione può quindi essere utilizzata dal codice in un modulo di maschera, come la routine evento Apri. È possibile fare riferimento alla proprietà OpenArgs anche nelle macro e nelle espressioni.
  Si supponga, ad esempio, che la maschera che si apre sia un elenco a modulo continuo di clienti. Se si desidera spostare lo stato attivo sul record di un determinato cliente all'apertura della maschera, è possibile specificare il nome del cliente con l'argomento apriarg e quindi utilizzare il metodo FindRecord per spostare lo stato attivo sul record del cliente del quale è stato specificato il nome.
  Questo argomento è disponibile soltanto in Visual Basic.

Osservazioni

L'argomento condizioneWHERE può avere una lunghezza massima di 32.768 caratteri, a differenza dell'argomento dell'azione condizione WHERE nella finestra della macro, che può avere una lunghezza massima di 256 caratteri.

È possibile omettere un argomento facoltativo all'interno della sintassi, ma è necessario includere la virgola dell'argomento stesso. Se si omette un argomento finale, non utilizzare la virgola dopo l'ultimo argomento specificato.
