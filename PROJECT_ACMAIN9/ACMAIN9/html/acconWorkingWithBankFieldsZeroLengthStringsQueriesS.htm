Utilizzare campi vuoti nelle query

Quando un campo non contiene valori, contiene un valore Null oppure, nel caso di campi di tipo Testo, Memo o Collegamento ipertestuale, un valore Null o una stringa a lunghezza zero. Se un campo contiene valori Null, è possibile che questi influenzino i risultati della query. Di seguito sono riportate alcune istruzioni per l'utilizzo di valori Null e stringhe a lunghezza zero nelle query. Per informazioni sulle differenze tra valori Null e stringhe a lunghezza zero, fare clic su .

Collegamento con join di campi che contengono valori Null

Quando si collegano con join delle tabelle in una query, il risultato include soltanto i record che non contengono valori Null nei campi corrispondenti. Per elencare ad esempio fornitori e clienti che vivono nella stessa zona, creare una query che include le tabelle Fornitori e Clienti che saranno collegate con join nel campo Zona. Quando vengono visualizzati i risultati, vengono mostrati soltanto i valori dei record che contengono un valore contenuto nel campo Zona in entrambe le tabelle.

Ricerca di valori Null o di stringhe a lunghezza zero

Se si utilizza una query per cercare valori Null o stringhe a lunghezza zero, digitare Is Null nella cella Criteri per cercare valori Null oppure digitare due volte le virgolette (" ") senza interporre spazi nella cella Criteri per cercare stringhe a lunghezza zero.

Influenza dei valori Null sui calcoli numerici

Se si utilizza una funzione di aggregazione per calcolare la somma, la media, il conteggio o un altro calcolo sui valori di un campo, i record contenenti valori Null in quel campo non verranno inclusi nel calcolo. Questo è vero sia che la funzione di aggregazione venga calcolata utilizzando la riga Formula nella griglia di struttura della query, la Creazione guidata Query semplice o un'espressione personalizzata. Se ad esempio viene utilizzata la funzione Conteggio per contare il numero di valori contenuti in un campo, questa restituirà un conteggio di tutti i record con valori non Null. Se si desidera trovare il numero totale di record inclusi quelli contenenti valori Null, utilizzare Conteggio con il carattere jolly asterisco (*).

Se si utilizza un operatore aritmetico (ad esempio +, -, *, /) in un'espressione quale [Scorte]+[Quantità Ordinata] e uno dei campi nell'espressione contiene un valore Null, il risultato dell'intera espressione sarà Null.

Conversione di valori Null a zero

Quando si hanno dei campi che contengono valori Null, è possibile creare un'espressione che converte i valori Null a zero. È possibile effettuare questa operazione se si desidera includere i record che contengono valori Null nel calcolo di una funzione di aggregazione oppure se si desidera impedire che un'espressione risulti in un valore Null quando un campo a cui si fa riferimento contiene valori Null. Utilizzare ad esempio la funzione Nz per convertire i valori Null a zero.

Nz([Subtotale],0)+Nz([Trasporto],0)

Per ulteriori informazioni sulla funzione Nz, fare clic su .

Combinazione di campi di tipo Testo che contengono valori Null

Se si utilizza un'espressione per combinare due campi che contengono valori di testo e uno o entrambi i campi includono valori Null, utilizzare l'operatore & invece dell'operatore + per combinare i valori. L'operatore & combina i valori anche se questi contengono valori Null, mentre l'operatore + restituisce un valore Null se uno dei due valori è Null. Ad esempio:

Nome: [Cognome] & " " & [Nome]

Ordinamento su campi che contengono valori Null e stringhe a lunghezza zero

Quando si esegue l'ordinamento di un campo in ordine crescente, vengono elencati per primi i record in cui quel campo contiene un valore Null. Se un campo contiene sia valori Null che stringhe a lunghezza zero, i valori Null vengono considerati per primi nel criterio di ordinamento, subito seguiti dalle stringhe a lunghezza zero.

Utilizzo del carattere jolly asterisco (*) per restituire valori non Null

Se si utilizza l'espressione Like "*" quando vengono definiti dei criteri della query per un campo, i risultati della query includeranno in quel campo stringhe a lunghezza zero ma non valori Null.
