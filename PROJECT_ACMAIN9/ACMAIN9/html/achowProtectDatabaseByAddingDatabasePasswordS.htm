Proteggere un database aggiungendo una password

  1. Chiudere il database. Se questo è condiviso su una rete, assicurarsi che anche tutti gli altri utenti lo abbiano chiuso.

  2. Fare una copia di backup del database e conservarla in un posto sicuro.

  3. Scegliere Apri dal menu File.

  4. Fare clic sulla freccia a destra del pulsante Apri, quindi scegliere Apertura esclusiva.

  5. Scegliere Protezione dal menu Strumenti, quindi Imposta password database.

  6. Nella casella Password digitare la password, tenendo presente che esiste la distinzione tra maiuscole e minuscole.

  7. Nella casella Verifica confermare la password digitandola di nuovo, quindi scegliere OK.

    La password è quindi impostata. Alla successiva apertura del database da parte di un utente qualsiasi verrà visualizzata una finestra di dialogo in cui viene richiesta una password.

Attenzione

  • Se non si ricorda la password, non è possibile recuperarla né aprire il database.

  • Non utilizzare una password se si desidera replicare un database. I database replicati non possono essere sincronizzati se sono state impostate delle password.

Note

  • La password di un database viene memorizzata insieme al database e non insieme al file di informazioni sul gruppo di lavoro.

  • Non è possibile impostare una password di database se è stata definita la protezione a livello utente e non si dispone dell'autorizzazione Amministrazione per il database. Una password di database viene impostata in aggiunta alla protezione a livello utente. Se quest'ultima è stata definita, qualsiasi limitazione basata sulle autorizzazioni della protezione a livello utente rimarrà attiva.

  • Se una tabella di un database protetto da password è collegata, la password viene salvata nella memoria cache del database al quale è collegata quando viene stabilito il collegamento. Questa operazione può avere delle conseguenze inaspettate. Per ulteriori informazioni, fare clic su .
