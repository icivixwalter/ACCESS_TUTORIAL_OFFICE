Metodo DefaultWorkspaceClone

                 

È possibile utilizzare il metodo DefaultWorkspaceClone per creare un nuovo oggetto Workspace senza richiedere all'utente di connettersi di nuovo. Se, ad esempio, è necessario eseguire simultaneamente due set di transazioni in aree di lavoro separate, è possibile utilizzare il metodo DefaultWorkspaceClone per creare un secondo oggetto Workspace con lo stesso nome dell'utente e la stessa password senza richiedere di nuovo tali informazioni.

Nota   Il metodo DefaultWorkspaceClone è incluso in questa versione di Microsoft Access per ragioni di compatibilità con le versioni precedenti che utilizzano il linguaggio di oggetti DAO.

Sintassi

applicazione.DefaultWorkspaceClone

Il metodo DefaultWorkspaceClone dispone dell'argomento riportato di seguito.

Argomento Descrizione
applicazione Oggetto Application.

Osservazioni

Il metodo DefaultWorkspaceClone crea un clone dell'oggetto Workspace predefinito in Microsoft Access. Le proprietà dell'oggetto Workspace hanno impostazioni identiche a quelle dell'oggetto Workspace predefinito, tranne l'impostazione della proprietà Name. Per l'oggetto Workspace predefinito, il valore della proprietà Name è sempre #Area di lavoro predefinita #. Per l'oggetto Workspace clonato, il valore è #CloneAccess#.

La proprietà UserName dell'oggetto Workspace predefinito indica il nome con il quale l'utente ha effettuato la connessione. L'oggetto Workspace clone equivale all'oggetto Workspace creato qualora lo stesso utente effettuasse di nuovo la connessione con la stessa password.
