Azione Esci

        

È possibile utilizzare l'azione Esci per uscire da Microsoft Access. Tale azione può inoltre specificare una delle diverse opzioni per salvare oggetti di database prima di uscire da Microsoft Access.

Impostazione

L'azione Esci utilizza l'argomento riportato di seguito.

Argomento azione Descrizione
Opzioni Specifica ciò che accade agli oggetti non salvati al momento dell'uscita da Microsoft Access. Nella casella Opzioni della sezione Argomenti azione nella finestra della macro selezionare Prompt per visualizzare finestre di dialogo che richiedono se salvare tutti gli oggetti, Salva tutto per salvare tutti gli oggetti senza che venga richiesta la conferma o Esci per uscire senza salvare alcun oggetto. L'impostazione predefinita è Salva tutto.

Osservazioni

Microsoft Access non esegue alcuna azione che in una macro segue l'azione Esci.

È possibile utilizzare tale azione, per uscire da Microsoft Access senza che vengano richieste conferme tramite le finestre di dialogo Salva, per mezzo di un comando di menu personalizzato o di un pulsante di una maschera. È possibile ad esempio che si abbia una maschera master utilizzata per visualizzare gli oggetti contenuti nella propria area di lavoro personalizzata. Tale maschera potrebbe avere un pulsante Esci che serve a eseguire una macro contenente l'azione Esci con l'argomento Opzioni impostato a Salva tutto.

L'effetto di tale azione equivale a quello del comando Esci del menu File. Se non è stato salvato alcun oggetto quando si sceglie tale comando, le finestre di dialogo visualizzate sono le stesse che appaiono quando si utilizza Prompt come argomento Opzioni dell'azione Esci.

È possibile utilizzare l'azione Salva in una macro per salvare un determinato oggetto senza dover uscire da Microsoft Access o chiudere l'oggetto.

Per eseguire l'azione Esci in Visual Basic, utilizzare il metodo Quit dell'oggetto DoCmd.
