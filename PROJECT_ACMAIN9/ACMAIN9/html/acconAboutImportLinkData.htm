Differenza tra importazione e collegamento

In Microsoft Access vi sono due modi per utilizzare i dati di un'origine dati esterna. È possibile:

  • Importare i dati in una nuova tabella di Microsoft Access per convertire i dati da un formato diverso e copiarli in Microsoft Access. È anche possibile importare oggetti di database nel database di Microsoft Access o nel progetto di Microsoft Access corrente.

  • Collegare i dati per connettersi a essi da un'altra applicazione senza importarli. In questo modo, è possibile visualizzare e modificare i dati sia nell'applicazione originale che in un database di Access. Nelle versioni precedenti di Microsoft Access questo processo veniva definito con il nome di allegazione.

È possibile importare o collegare i dati da una varietà di database, programmi e formati di file supportati.

Importazione di dati

Quando si importano dei dati, se ne crea una copia in una nuova tabella nel database o nel progetto di Access. In questo processo la tabella o il file di origine non viene alterato.

Quando si importano dei dati, non è possibile accodarli a tabelle esistenti, tranne quando si importano file di testo o fogli di calcolo. Una volta importata una tabella tuttavia, in un database di Access è possibile eseguire una query di accodamento o in un progetto di Access è possibile utilizzare una stored procedure per aggiungere i dati della tabella in un'altra.

È anche possibile importare oggetti di database che non siano tabelle, ad esempio maschere e report, da un altro database o progetto di Access.

Collegamento di dati

In un database di Access collegare i dati significa leggerli e, nella maggior parte dei casi, aggiornarli nell'origine dati esterna senza eseguire l'importazione. Il formato dell'origine dati esterna non viene alterato in modo che l'utente possa continuare a utilizzare il file con il programma con cui è stato creato. È tuttavia possibile aggiungere, eliminare o modificare i relativi dati utilizzando Microsoft Access. Una tabella può essere collegata solo in un database di Access, non in un progetto.

In Microsoft Access vengono utilizzate icone diverse per rappresentare le tabelle collegate e le tabelle memorizzate nel database corrente. Se si elimina l'icona corrispondente a una tabella collegata, si elimina il collegamento alla tabella, non la tabella esterna vera e propria.

Modi per importare e collegare i dati

Vi sono due modi per importare o collegare i dati:

  • Aprire o creare un database di Access oppure creare un progetto di Access in cui inserire le tabelle importate o le tabelle collegate nel caso di un database di Access, quindi utilizzare il comando Importa o Collega tabelle del sottomenu Carica dati esterni del menu File.

  • In un database di Access aprire il database esterno direttamente, ad esempio dBASE, Paradox o Exchange, e qualsiasi origine dati ODBC, tra cui Microsoft SQL Server e Microsoft Visual FoxPro. A questo punto verrà creato automaticamente un nuovo database di Access nella stessa cartella del database esterno e verranno aggiunti dei collegamenti a ciascuna tabella del database esterno.

Utilizzo delle origini dati ODBC

È possibile importare o collegare dati da database ODBC, ad esempio Microsoft SQL Server e Visual FoxPro, nonché da altri programmi dotati di driver conformi a ODBC Level 1 per accedere ai relativi file di dati.

Programmi non supportati

Se si utilizza un programma i cui dati non sono memorizzati in uno dei formati di file o di database esterni supportati, ma che è comunque in grado di esportare, convertire o salvare i dati in uno di questi formati, è possibile importare tali dati. Inoltre, quando si importa o collega un formato di database esterno, quale dBASE o Paradox, in genere vengono mantenuti gli indici. Non è possibile ad esempio importare o collegare direttamente un database di Microsoft Works, ma è possibile esportare i dati da Microsoft Works in un file di database dBASE IV (dbf), quindi importare i dati in Microsoft Access per mantenere i nomi dei campi e gli indici.

In generale la maggior parte dei programmi non supportati, anche quelli su sistemi operativi diversi, è in grado di esportare i dati in un file di testo delimitato o a larghezza fissa, che potrà poi essere importato o collegato da Microsoft Access. Gli indici non verranno comunque mantenuti.

Automatizzazione delle operazioni di importazione

Le operazioni di importazione più frequenti possono essere eseguite più facilmente se le si rende automatiche creando una macro o creando una routine Visual Basic, Applications Edition. Ciò è utile ad esempio per importare i dati con scadenze regolari o quando l'importazione dei dati deve essere eseguita secondo requisiti particolarmente complessi o insoliti.
