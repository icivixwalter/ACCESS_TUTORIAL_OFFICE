Utilizzare un formato di visualizzazione dati o una maschera di input

In Microsoft Access esistono due proprietà che producono risultati simili: la proprietà Formato e la proprietà Maschera di input.

Utilizzare la proprietà Formato per visualizzare i dati in formato congruente. Se ad esempio viene impostata la proprietà Formato di un campo Data/ora al formato Data breve, tutte le date immesse verranno visualizzate in questo formato: 12-gen-96. Se un utente del database immette una data nel formato 01/12/96 (o qualsiasi altro formato data valido), la data verrà visualizzata nel formato Data breve quando viene salvato il record.

La proprietà Formato influisce solo sulla modalità di visualizzazione di un valore, non sulla modalità di memorizzazione del valore stesso nella tabella. Un formato inoltre non viene applicato fino a quando i dati immessi non vengono salvati: nel campo non viene visualizzato alcun elemento che suggerisca o controlli il formato in cui vengono immessi i dati. Se si desidera controllare la modalità di immissione dei dati, utilizzare una maschera di input in aggiunta o invece di un formato di visualizzazione dei dati. Se si desidera che i dati vengano visualizzati esattamente nello stesso modo in cui sono stati immessi, non impostare la proprietà Formato.

Per i campi di tipo Numerico, Valuta, Data/ora, Contatore e Sì/No sono disponibili formati di visualizzazione predefiniti e possono essere definiti anche formati personalizzati. Non esistono formati predefiniti per i campi di tipo Testo, Memo o Collegamento ipertestuale, ma è possibile definire formati personalizzati. Non è possibile definire un formato di visualizzazione per i campi di tipo Oggetto OLE.

Se per un campo vengono definiti sia un formato di visualizzazione che una maschera di input, la maschera di input viene utilizzata durante l'aggiunta o la modifica dei dati, mentre con l'impostazione della proprietà Formato viene determinata la modalità di visualizzazione dei dati quando viene salvato il record. Quando si utilizzano sia la proprietà Formato che la proprietà Maschera di input, i risultati corrispondenti non devono essere in conflitto.
