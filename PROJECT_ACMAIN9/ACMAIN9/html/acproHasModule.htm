Proprietà HasModule (Possiede modulo)

                 

È possibile utilizzare la proprietà HasModule per specificare o determinare se una maschera o un report debba contenere un modulo di classe. Se si imposta la proprietà a No è possibile migliorare le prestazioni e ridurre le dimensioni del database.

Impostazione

La proprietà HasModule viene impostata come riportato di seguito.

Impostazione Visual Basic Descrizione
Sì True (-1) La maschera o il report contiene un modulo di classe.
No False (0) Impostazione predefinita. La maschera o il report non contiene un modulo di classe.

È possibile impostare la proprietà HasModule utilizzando la finestra delle proprietà della maschera o del report, una macro o Visual Basic.

È possibile impostare la proprietà HasModule solo in visualizzazione Struttura maschera o report ed è possibile leggerla in tutte le altre visualizzazioni.

Osservazioni

Le maschere o i report con la proprietà HasModule impostata a No vengono considerati oggetti leggeri. Gli oggetti leggeri sono più piccoli degli oggetti che hanno moduli di classe associati e vengono caricati e visualizzati più velocemente. In molti casi maschere o report non è utilizzano le routine evento né un modulo di classe.

Se l'applicazione utilizza un pannello principale per passare alle altre maschere, anziché i pulsanti di comando con le routine evento, è possibile utilizzare un pulsante di comando con una macro o un collegamento ipertestuale. Per aprire la maschera Impiegati con un pulsante di comando su un pannello principale, ad esempio, è possibile impostare la proprietà HyperlinkSubAddress del controllo a "Maschera Impiegati".

Gli oggetti leggeri non vengono visualizzati in Visualizzatore oggetti e non è possibile utilizzare la parola chiave New per creare una nuova instanza dell'oggetto. È possibile utilizzare una maschera o un report leggero come sottomaschera o sottoreport e visualizzarlo nell'insieme Forms o Reports. Gli oggetti leggeri supportano l'uso di macro e routine pubbliche esistenti in moduli standard quando vengono richiamati dalla finestra delle proprietà dell'oggetto.

Microsoft Access imposta la proprietà HasModule a True quando si tenta di visualizzare il modulo di un oggetto, anche se non è stato aggiunto alcun codice al modulo. Se, ad esempio, si sceglie Codice dal menu Visualizzazione per una maschera in visualizzazione Struttura, Microsoft Access aggiunge un modulo di classe all'oggetto Form e imposta la relativa proprietà HasModule a True. È possibile aggiungere un modulo di classe a un oggetto nello stesso modo, impostando la proprietà HasModule a Sì nella finestra delle proprietà dell'oggetto.

Avviso   Se si imposta la proprietà HasModule a No utilizzando la finestra delle proprietà di un oggetto o la si imposta a False utilizzando Visual Basic, Microsoft Access elimina il modulo di classe dell'oggetto e gli eventuali codici contenuti.

Quando si utilizza un metodo dell'oggetto Module o si fa riferimento alla proprietà Module per una maschera o un report in visualizzazione Struttura, Microsoft Access crea il modulo associato e imposta la proprietà HasModule dell'oggetto a True. Se si fa riferimento alla proprietà Module di una maschera o di un report in fase di esecuzione e l'oggetto ha la proprietà HasModule impostata a False, si verificherà un errore.

Gli oggetti creati utilizzando le funzioni CreateForm o CreateReport sono leggeri per impostazione predefinita.
