Importare gli oggetti di database da un altro database di Microsoft Access o da un progetto di Microsoft Access

Importare gli oggetti di database da un altro database di Microsoft Access o da un progetto di Microsoft Access

  1. Aprire il database oppure passare alla finestra del database aperto.

  2. Scegliere Carica dati esterni dal menu File, quindi Importa.

  3. Verificare che nella casella Tipo file sia selezionato Microsoft Access (*.mdb; *.adp; *.mdw; *.mda; *.mde; *.ade).

  4. Fare clic sulla freccia a destra della casella Cerca in, selezionare l'unità e la cartella in cui è memorizzato il database di Microsoft Access (mdb) o il progetto di Microsoft Access (adp) da cui si desidera effettuare l'importazione, quindi fare doppio clic sull'icona del database.

  5. Nella finestra di dialogo Importa oggetti fare clic sulla scheda del tipo di oggetto che si desidera importare e su ciascun oggetto da importare oppure scegliere il pulsante Seleziona tutto per importare tutti gli oggetti del tipo desiderato. Ripetere questo passaggio per ogni tipo di oggetto da importare.

    Se si desidera importare soltanto le definizioni delle tabelle e non i dati in esse contenuti, scegliere il pulsante Opzioni, quindi selezionare il pulsante di opzione Solo definizione nell'area Importa tabelle.

    Nel caso di un database di Access, se si desidera importare delle query di selezione come tabelle, ad esempio per creare un database di sola lettura, scegliere il pulsante Opzioni e selezionare il pulsante di opzione Come tabelle nell'area Importa query. Le query di comando verranno importate come query indipendentemente dal pulsante di opzione selezionato.

    Se si desidera includere relazioni (solo per database di Access), barre degli strumenti e menu personalizzati oppure specifiche di importazione/esportazione (solo per database di Access), scegliere il pulsante Opzioni, quindi selezionare le caselle di controllo relative agli elementi da includere nell'area Importa. L'importazione di barre degli strumenti, barre dei menu o menu di scelta rapida non verrà effettuata se tali elementi hanno lo stesso nome di quelli che si trovano all'interno del database o del progetto di Access di destinazione.

Note

  • Non è possibile importare visualizzazioni, stored procedure o diagrammi di database da un progetto di Access in un database di Access, così come non è possibile importare query o relazioni da un database di Access in un progetto di Access.

  • Quando si importa una pagina di accesso ai dati, in realtà si importa solo il collegamento al file HTML corrispondente. Tale file rimane nella stessa posizione, il che significa che due pagine fanno riferimento allo stesso file HTML corrispondente. Per ovviare a questo inconveniente, salvare una delle pagine con un nome diverso scegliendo Salva con nome dal menu File, immettere un altro percorso per il nuovo file HTML corrispondente nella finestra di dialogo Salva come pagina di accesso ai dati, quindi eliminare la pagina appena salvata.

  • Quando si importa un oggetto da un altro database, in genere si importano anche gli oggetti correlati per permetterne il corretto funzionamento. Un report può avere ad esempio una query come origine record e tale query può essere basata su una relazione tra due tabelle.

  • Sebbene sia possibile importare tabelle da un progetto di Access in un database di Access, può risultare più efficiente importare la tabella da Microsoft SQL Server. Se si stanno importando tabelle da un progetto di Access a un altro, è consigliabile l'utilizzo di DTS (Data Transformation Services), un metodo più rapido e flessibile per l'importazione e la trasformazione dei dati. Per ulteriori informazioni, consultare la documentazione di Microsoft SQL Server.

  • Quando si importa una tabella in un progetto di Access, vengono importati solo i dati e le definizioni dei dati, non le proprietà, i vincoli, le relazioni e gli indici.

  • Sebbene si utilizzi normalmente il comando Converti database del sottomenu Utilità database del menu Strumenti per effettuare la conversione a una versione precedente di un database di Microsoft Access, è anche possibile ottenere gli stessi risultati importando gli oggetti di database. Ciò è particolarmente utile quando è necessario convertire solo alcuni degli oggetti di database oppure si desidera automatizzare il processo di conversione in Visual Basic mediante i metodi CreateDatabase e TransferDatabase.
