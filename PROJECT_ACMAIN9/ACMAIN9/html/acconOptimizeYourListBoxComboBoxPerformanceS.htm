Ottimizzare le prestazioni di una casella di riepilogo, di una casella di riepilogo a discesa o di una casella combinata

È possibile effettuare diverse operazioni per far funzionare le caselle di riepilogo, le caselle di riepilogo a discesa e le caselle combinate in modo più efficiente:

  • Utilizzare una query salvata invece di un'istruzione SQL come base della casella di riepilogo, della casella di riepilogo a discesa o della casella combinata. Se si utilizza una creazione guidata per creare la casella di riepilogo o la casella combinata in una maschera, la proprietà Origine riga del controllo viene impostata a un'istruzione SQL. Per modificare la proprietà Origine riga su una query salvata, fare clic sul pulsante dei generatori accanto alla casella della proprietà Origine riga. Con la finestra Generatore di query aperta, scegliere Salva dal menu File, quindi digitare un nome per la query. Quando si chiude la finestra Generatore di query e un messaggio chiede se si desidera aggiornare la proprietà, scegliere Sì.

  • Utilizzare una query salvata invece di una tabella come base della casella di riepilogo a discesa. Se si utilizza una creazione guidata per creare una casella di riepilogo a discesa in una pagina di accesso ai dati, la proprietà ListRowSource del controllo verrà impostata automaticamente alla tabella specificata nella creazione guidata. Per recuperare le righe da una query invece che da una tabella, selezionare il nome della query dalla casella delle proprietà ListRowSource.

  • Nella query specificata nella casella della proprietà Origine riga o ListRowSource includere soltanto i campi assolutamente necessari. I campi aggiuntivi possono ridurre le prestazioni.

  • Se i campi sono differenti, indicizzare sia il primo campo visualizzato nella casella combinata, nella casella di riepilogo o nella casella di riepilogo a discesa che il campo associato.

  • Nelle caselle combinate in una maschera impostare la proprietà Espansione automatica a No se non è necessaria la funzione di selezione di una voce di elenco man mano che se ne digitano le lettere.

  • Se in una maschera la proprietà Espansione automatica per una casella combinata è impostata a Sì, il tipo di dati del primo campo visualizzato dovrebbe essere Testo e non Numerico. Per trovare una corrispondenza nell'elenco, un valore numerico viene convertito in testo. Se i dati sono di tipo testo, questa conversione non è necessaria.

  • Se in una maschera il campo associato in una casella combinata di ricerca non è il campo visualizzato:
    • Non utilizzare espressioni per il campo associato o per il campo visualizzato.

    • Non utilizzare limitazioni nell'origine riga.

    • Se possibile, utilizzare origini riga a una sola tabella (o query) piuttosto che a più tabelle.
  • Se i dati non verranno modificati, non creare caselle di riepilogo, caselle di riepilogo a discesa o caselle combinate basate su dati di una tabella collegata. In questo caso è preferibile importare i dati nel database.

Nota   I generatori non sono disponibili nelle pagine di accesso ai dati. Le creazioni guidate non sono disponibili nelle pagine di accesso ai dati autonome o in quelle che fanno riferimento a un database diverso da quello attualmente aperto. In entrambi i casi è necessario creare una casella di riepilogo o una casella di riepilogo personalizzata.
