Metodo CodeDb

        

È possibile utilizzare il metodo CodeDb in un modulo di codice per determinare il nome dell'oggetto Database che fa riferimento al database in cui il codice è attualmente in esecuzione. Utilizzare il metodo CodeDb per accedere a oggetti DAO che fanno parte di un database di libreria.

È possibile, ad esempio, utilizzare il metodo CodeDb in un modulo di un database di libreria per creare un oggetto Database che fa riferimento al database di libreria. È quindi possibile aprire un recordset basato su una tabella contenuta nel database di libreria.

Sintassi

Set database = CodeDb

Il metodo CodeDb dispone dell'argomento riportato di seguito.

Argomento Descrizione
database Variabile di oggetto Database.

Osservazioni

Il metodo CodeDb restituisce un oggetto Database per cui la proprietà Name corrisponde al percorso e al nome completo del database da cui viene richiamato. Tale metodo è utile quando si rende necessario manipolare gli oggetti DAO nel database di libreria.

Quando si richiama un metodo in un database di libreria, il database dal quale il metodo è stato richiamato rimane quello corrente, anche quando il codice è in esecuzione in un modulo nel database di libreria. Per fare riferimento agli oggetti DAO nel database di libreria, è necessario conoscere il nome dell'oggetto Database che rappresenta il database di libreria.

Si supponga, ad esempio, di avere in un database di libreria una tabella che elenca i messaggi di errore. Per manipolare dal codice i dati contenuti nella tabella, è possibile utilizzare il metodoCodeDb per determinare il nome dell'oggetto Database che fa riferimento al database di libreria contenente la tabella.

Se il metodo CodeDb viene eseguito dal database corrente, restituisce il nome dello stesso database, che è lo stesso valore restituito dal metodo CurrentDb.
