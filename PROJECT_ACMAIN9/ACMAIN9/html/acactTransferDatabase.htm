Azione TrasferisciDatabase

        

È possibile utilizzare l'azione TrasferisciDatabase per importare o esportare dati tra il database di Microsoft Access (MDB) o il progetto di Microsoft Access (ADP) corrente e un altro database. Per i database di Microsoft Access (MDB) è anche possibile collegare una tabella di un altro database al database corrente. È possibile accedere ai dati di una tabella collegata mentre questa rimane nell'altro database.

Impostazione

L'azione TrasferisciDatabase utilizza gli argomenti riportati di seguito.

Argomento azione Descrizione
Tipo trasferimento Il tipo di trasferimento che si desidera effettuare. Selezionare Importa, Esporta o Collega nella casella Tipo trasferimento della sezione Argomenti azione della finestra della macro. L'impostazione predefinita è Importa.
  Nota   Il tipo di trasferimento Collega non è supportato nei progetti di Microsoft Access (ADP).
Tipo database Il tipo di database da cui importare, in cui esportare o a cui collegarsi. È possibile selezionare Microsoft Access o uno degli altri tipi di database riportati nella casella Tipo database. L'impostazione predefinita è Microsoft Access.
  I tipi di database visualizzati nella casella Tipo database dipendono dal modo in cui è stata eseguita l'installazione di Microsoft Access. L'installazione predefinita infatti non include tutti i tipi di database. Se il tipo di database che si desidera utilizzare per l'importazione, l'esportazione o il collegamento non è disponibile, eseguire il programma di installazione di Microsoft Access, scegliere il pulsante Aggiungi/Rimuovi, selezionare l'opzione Accesso ai dati e Controlli ActiveX nella finestra di dialogo, infine scegliere il pulsante Modifica opzione. Selezionare la casella di controllo Driver di database e scegliere il pulsante Modifica opzione. Quindi selezionare il tipo di database che si desidera installare e scegliere OK.
Nome database Il nome del database da cui importare, in cui esportare o a cui collegarsi. Includere il percorso completo. Questo argomento è richiesto.
  Per i tipi di database che utilizzano file differenti per ciascuna tabella, ad esempio FoxPro, Paradox e dBASE, immettere la directory contenente il file. Immettere il nome del file nell'argomento Origine (importazione o collegamento) oppure nell'argomento Destinazione (esportazione).
  Per i database ODBC, digitare la stringa di connessione ODBC (Open Database Connectivity) completa. Per visualizzare un esempio di stringa di connessione, collegare una tabella esterna a Microsoft Access scegliendo Carica dati esterni dal menu File e quindi Collega tabelle. Aprire la tabella in visualizzazione Struttura e visualizzare le proprietà della tabella. Il testo riportato nell'impostazione della proprietà Descrizione è la stringa di connessione per questa tabella.
  Per ulteriori informazioni sulle stringhe di connessione ODBC, consultare il file della Guida o altra documentazione relativa al driver ODBC di questo tipo di database ODBC.
Tipo oggetto Il tipo di oggetto da importare o esportare. Se si seleziona Microsoft Access per l'argomento Tipo database, è possibile selezionare Tabella, Query, Maschera, Report, Macro, Modulo, Pagina di accesso ai dati, Visualizzazione server, Diagramma o Stored procedure nella casella Tipo oggetto. Se invece si seleziona un altro tipo di database oppure si seleziona Collega nella casella Tipo trasferimento, questo argomento viene ignorato. L'impostazione predefinita è Tabella.
  Se si sta esportando una query di selezione in un database di Microsoft Access, selezionare Tabella in questo argomento per esportare il set di risultati di questa query, quindi selezionare Query per esportare la query stessa. Se si sta esportando una query di selezione in un altro tipo di database, questo argomento viene ignorato e il set di risultati della query viene esportato.
Origine Il nome della tabella, della query di selezione o dell'oggetto di Microsoft Access che si desidera importare, esportare o collegare. Per alcuni tipi di database, ad esempio FoxPro, Paradox o dBASE, tale nome è il nome di un file ed è necessario includere l'estensione, ad esempio dbf. Questo argomento è richiesto.
Destinazione Il nome della tabella, della query di selezione o dell'oggetto di Microsoft Access importato, esportato o collegato nel database di destinazione. Per alcuni tipi di database, ad esempio FoxPro, Paradox o dBASE, tale nome è il nome di un file ed è necessario includere l'estensione, ad esempio dbf. Questo argomento è richiesto.
  Se si seleziona Importa nell'argomento Tipo trasferimento e Tabella nell'argomento Tipo oggetto, viene creata una nuova tabella contenente i dati della tabella importata.
  Se si importa una tabella o un altro oggetto il cui nome è in conflitto con un nome esistente, al nome della tabella o dell'oggetto importato viene aggiunto un numero. Se ad esempio si importa Impiegati e tale nome esiste già, la tabella o l'altro oggetto importato viene rinominato Impiegati1.
  Se si esportano dati in un database di Microsoft Access o in un altro database, le tabelle o gli altri oggetti esistenti con lo stesso nome vengono automaticamente sostituiti.
Solo struttura Specifica se importare o esportare solo la struttura di una tabella di database senza i relativi dati. Selezionare Sì oppure No. L'impostazione predefinita è No.

Osservazioni

È possibile importare ed esportare tabelle tra Microsoft Access e altri tipi di database. È inoltre possibile esportare query di selezione di Microsoft Access in altri tipi di database. Il set di risultati della query viene esportato nella maschera di una tabella. È possibile importare ed esportare qualsiasi oggetto di database di Microsoft Access se entrambi i database sono database di Microsoft Access.

Se in Microsoft Access 2000 viene importata una tabella da un altro database di Microsoft Access (MDB) che costituisce una tabella collegata di quel database, essa rimarrà collegata dopo l'importazione. In altri termini, viene importato il collegamento e non la tabella stessa.

Se il database a cui si accede richiede una password, quando si esegue la macro viene visualizzata una finestra di dialogo in cui è possibile digitarla.

Nota   Tra i database di Microsoft Access e quelli di FoxPro versione 3.0 è solo possibile importare ed esportare dati. Se si dispone di un driver ODBC FoxPro, è possibile collegarsi alle tabelle dei database di questa versione di FoxPro.

L'azione TrasferisciDatabase equivale a scegliere Carica dati esterni o Esporta dal menu File della finestra del database e quindi Importa o Collega tabelle. È possibile utilizzare questi comandi per selezionare un'origine dati, ad esempio Microsoft Access o un tipo di database, di foglio di calcolo o di file di testo. Se si seleziona un database, vengono visualizzate una o più finestre di dialogo in cui è possibile selezionare il tipo di oggetto da importare o da esportare (per i database Microsoft Access), il nome dell'oggetto e altre opzioni che variano a seconda del database da cui si sta importando, in cui si sta esportando o a cui ci si sta collegando. Gli argomenti per l'azione TrasferisciDatabase riflettono le opzioni presenti in queste finestre di dialogo.

Se si desidera fornire informazioni sugli indici di una tabella FoxPro o dBASE collegata, collegare innanzitutto la tabella scegliendo Collega tabelle dal comando Carica dati esterni del menu File, quindi specificare gli indici nelle finestre di dialogo relative a questo comando. Le informazioni sugli indici vengono memorizzate in un file di informazioni speciale (INF) posizionato nella cartella Office (il percorso è C:\Programmi\Microsoft Office\Office). È quindi possibile eliminare il collegamento alla tabella collegata. La volta successiva che si utilizza l'azione TrasferisciDatabase per collegare questa tabella FoxPro o dBASE, verranno utilizzate le informazioni sugli indici specificate.

Nota   Se si esegue una query o si applica un filtro a una tabella collegata, la query o il filtro distinguono tra maiuscole e minuscole.

Per eseguire l'azione TrasferisciDatabase in Visual Basic, utilizzare il metodo TransferDatabase dell'oggetto DoCmd.
