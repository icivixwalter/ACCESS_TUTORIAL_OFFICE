Informazioni sulla copia di un oggetto di database

È possibile personalizzare con facilità un oggetto di database esistente di un database di Microsoft Access o di un progetto di Microsoft Access creandone una copia e modificandola.

In genere la copia di un oggetto di database eredita le proprietà associate all'oggetto originale. Se però si crea una copia di una tabella di un progetto di Access, non verranno ereditati gli indici, i vincoli e i trigger della tabella originale, pertanto sarà necessario ricreare queste proprietà nella copia. Se inoltre si crea una copia di una tabella o di una query di un database di Access e si desidera che anche nella copia sia presente un foglio dati secondario, è necessario aprire la copia e inserire tale foglio.

Se si desidera copiare unicamente la struttura di una tabella, non la tabella vera e propria con i relativi dati, o si desidera accodare i dati di una tabella a una tabella esistente, utilizzare il comando Copia per copiare la tabella.

È possibile trascinare un oggetto di database dalla finestra del database di un database o di un progetto di Access in un altro database o progetto già aperto. È possibile copiare una tabella, una query o un report trascinandolo dalla finestra del database in un'altra applicazione Microsoft.

È inoltre possibile salvare alcuni oggetti di database come oggetti di altro tipo.
