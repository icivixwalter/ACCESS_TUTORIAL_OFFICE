Proprietà Recordset

                 

È possibile utilizzare la proprietà Recordset per specificare o recuperare l'oggetto ADO Recordset o l'oggetto DAO Recordset che rappresenta l'origine record di una maschera.

Nota   Questa proprietà non può essere utilizzata con i tipi di set di record ODBCDirect in DAO.

Impostazione

La proprietà Recordset restituisce l'oggetto recordset che fornisce i dati sfogliati in una maschera. Se la maschera si basa su una query, ad esempio, fare riferimento alla proprietà Recordset equivale a clonare un oggetto Recordset utilizzando la stessa query. Tuttavia, diversamente da quando si utilizza la proprietà RecordsetClone, le modifiche di valuta apportate alla proprietà Form.Recordset si riflettono automaticamente nel record corrente della maschera.

Questa proprietà è disponibile solo utilizzando Visual Basic.

Osservazioni

Quando viene richiesto un set di record in un database di Microsoft Access (MDB), viene restituito un set di record DAO, mentre in un progetto di Microsoft Access (ADP), viene restituito un set di record ADO.

Il funzionamento in lettura/scrittura di una maschera la cui proprietà Recordset sia stata impostata utilizzando Visual Basic è determinato dal tipo di set di record (ADO o DAO) e dal tipo di dati (Jet o SQL) contenuti nel set di record identificato dalla proprietà.

Tipo di set di record In base ai dati SQL In base ai dati Jet
ADO Lettura/Scrittura (1) Sola lettura
DAO N/A Lettura/Scrittura

(1) Nota   La proprietà ADO Recordset.CursorLocation deve essere impostata a adUseClient. La proprietà UniqueTable è disponibile nel client/server nella finestra delle proprietà, ma non per i database di Access (MDB). Questa proprietà deve essere impostata utilizzando il codice di Visual Basic.

Global rstFornitori As ADODB.Recordset
Sub MakeRW()
  DoCmd.OpenForm "Fornitori"
  Set rstFornitori = New ADODB.Recordset
  rstFornitori.CursorLocation = adUseClient
  rstFornitori.Open "Select * From Fornitori", CurrentProject.Connection, adOpenKeyset, adLockOptimistic
  Set Forms("Fornitori").Recordset = rstFornitori
  Forms("Fornitori").UniqueTable = "Fornitori"
End Sub

La proprietà Recordset viene utilizzata:

  • Per associare più maschere ad un set di dati comune; ciò consente di sincronizzare più maschere. Ad esempio:
  Set Me.Recordset = Forms!Form1.Recordset

  • Per utilizzare sull'oggetto Recordset metodi non direttamente supportati sulle maschere. Ad esempio, si può ricorrere alla proprietà Recordset quando si desidera usare il metodo ADO Find o il metodo DAO Find per utilizzare una finestra di dialogo personalizzata che consenta di trovare un record.

  • Per disporre una transazione (che può poi essere riportata allo stato precedente) intorno ad un set di modifiche che interessano più maschere.

Quando viene aperto un nuovo oggetto Recordset, il primo record sarà il record corrente.

Metodi e proprietà ADO Metodi e proprietà DAO
Find, Move o altri metodi Move. FindFirst, FindNext, FindLast, FindPrevious, Move o altri metodi Move.

Nota   Se si modifica la proprietà RecordSource della maschera, occorre utilizzare l'istruzione Set. La modifica della proprietà Recordset di una maschera può anche modificare le proprietà RecordSource, RecordsetType e RecordLocks. Inoltre, è possibile escludere alcune proprietà correlate ai dati, ad esempio le proprietà Filter, FilterOn, OrderBy e OrderByOn.
