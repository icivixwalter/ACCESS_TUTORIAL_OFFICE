Ridenominazione e impostazione delle proprietà delle tabelle collegate in un database di Microsoft Access

Nota   È possibile collegare una tabella solo in un database di Microsoft Access e non in un progetto di Microsoft Access.

Impostazione delle proprietà delle tabelle collegate

Quando si apre una tabella collegata in visualizzazione Struttura, essa appare come una normale tabella di Microsoft Access. Sebbene non sia possibile modificare la definizione della tabella collegata e dei relativi campi nel database esterno, è possibile impostare le proprietà che controllano la modalità di visualizzazione dei campi nel programma. Le modifiche apportate alle proprietà delle tabelle collegate hanno effetto soltanto sulla gestione e visualizzazione dei dati di queste tabelle; non viene apportata alcuna modifica alla tabella di origine. Le proprietà dei campi, che si possono impostare per delle tabelle collegate, sono elencate nella tabella sottostante.

Proprietà Consente di
Formato Controlla il modo in cui dei dati vengono visualizzati in un campo
Posizioni decimali Controlla il numero di posizioni decimali visualizzate
Maschera di input Utilizzato per creare una maschera di input di dati, che presenti dei caratteri di separazione e degli spazi vuoti da riempire
Etichetta Modifica il nome utilizzato per l'intestazione di colonna del foglio dati di un campo, e consente di specificare un nome predefinito da utilizzare come etichetta quando si aggiunge un campo ad una maschera.

Per ulteriori informazioni sull'impostazione di proprietà di campo, fare clic su .

Non è possibile modificare altre proprietà di campo di tabelle collegate. Per rendere l'immissione dati più efficace e affidabile, è possibile, tuttavia, creare delle maschere per aggiungere o modificare dei dati nelle tabelle collegate o impostare delle proprietà per i controlli associati a dei campi delle tabelle collegate. È possibile, ad esempio, impostare le proprietà Valore predefinito, Valido se e Messaggio errore per i controlli di quelle maschere.

Nota   Se si collegano i dati di tabelle di un altro database di Microsoft Access, le tabelle utilizzano le impostazioni delle proprietà del database in cui sono memorizzate. Se, ad esempio, una tabella dispone di regole di convalida nel database originale, anche i dati immessi nella tabella collegata devono seguirle. Se è necessario modificare queste proprietà, è necessario aprire la tabella all'interno del database in cui è memorizzata.

Ridenominazione delle tabelle collegate

Si possono rinominare delle tabelle collegate. Dal momento che i nomi di tabella possono contenere fino a 64 caratteri e degli spazi, è possibile assegnare un nome più descrittivo ad una tabella, dopo averla collegata. Se si collega, ad esempio, la tabella SLSDATA di dBASE, la si potrebbe rinominare "Dati sulle vendite del 1995 (da dBASE)". Si noti che in questo modo non verrà rinominata la tabella stessa, ma verrà modificato il nome da utilizzare in riferimento al collegamento di quella tabella.

Rinominare una tabella collegata utilizzando la stessa procedura adottata per rinominare altri oggetti di database. Per ulteriori informazioni, fare clic su .
