Proprietà ControlSource (Origine controllo)

                 

È possibile utilizzare la proprietà ControlSource per specificare quali dati vengono visualizzati in un controllo. È possibile visualizzare e modificare dati associati ad un campo di una tabella, di una query o di un'istruzione SQL. È anche possibile visualizzare il risultato di un'espressione. Per un livello di gruppo di report, la proprietà ControlSource determina il campo o l'espressione in base a cui effettuare il raggruppamento.

Nota   La proprietà ControlSource non si applica a caselle di controllo, pulsanti di opzione o interruttori di un gruppo di opzioni, ma solo al gruppo di opzioni stesso.

Per i report, la proprietà ControlSource si applica solo ai livelli di gruppo di report.

Impostazione

La proprietà ControlSource viene impostata come riportato di seguito.

Impostazione Descrizione
Un nome di campo Il controllo è associato ad un campo di una tabella, di una query o di un'istruzione SQL. I dati di un campo vengono visualizzati nel controllo. Le modifiche apportate ai dati contenuti nel controllo modificano i dati corrispondenti del campo. Per rendere il controllo di sola lettura, impostare a Sì la proprietà Locked. Se si seleziona un controllo associato ad un campo contenente un tipo di dati con collegamento ipertestuale, si passa alla destinazione specificata nell'indirizzo del collegamento ipertestuale.
un'espressione Il controllo visualizza i dati generati da un'espressione. Questi dati possono essere modificati dall'utente ma non vengono salvati nel database.

È possibile impostare la proprietà ControlSource per un controllo utilizzando la finestra delle proprietà di un controllo, una macro o Visual Basic.

È inoltre possibile impostare la proprietà ControlSource per una casella di testo digitando un nome di campo o un'espressione direttamente nella casella di testo in visualizzazione Struttura della maschera o in visualizzazione Struttura del report.

Per un report è possibile impostare tale proprietà selezionando un campo o digitando un'espressione nella colonna Campo/Espressione della casella Ordinamento e raggruppamento. Per ulteriori informazioni, vedere la proprietà GroupLevel.

Per impostare il valore di tale proprietà in Visual Basic, utilizzare un'espressione stringa.

Osservazioni

Maschere e report funzionano nel database come "finestre". L'origine primaria dei dati per una maschera o per un report viene specificata impostando la relativa proprietà RecordSource a una tabella, una query o un'istruzione SQL. È quindi possibile impostare la proprietà ControlSource a un campo dell'origine dei dati o ad un'espressione. Se l'impostazione della proprietà ControlSource è un'espressione, il valore visualizzato è di sola lettura e non viene salvato nel database. È possibile utilizzare, ad esempio, le seguenti impostazioni:

Impostazione di esempio Descrizione
Cognome Per un controllo i dati del campo Cognome vengono visualizzati nel controllo. Per un livello di gruppo di report i dati vengono raggruppati secondo il cognome.
=Data( ) + 7 Per un controllo l'espressione visualizza nel controllo una data di sette giorni successiva a quella odierna.
=ParteData("q",DataSpedizione) Per un controllo l'espressione visualizza il trimestre relativo alla data di spedizione. Per un livello di gruppo di report i dati vengono raggruppati nel trimestre relativo alla data di spedizione.
