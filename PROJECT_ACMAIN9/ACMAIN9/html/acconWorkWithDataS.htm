Informazioni sull'utilizzo di dati e oggetti di database con il codice Visual Basic

Oltre all'utilizzo di dati e oggetti come maschere e report nell'interfaccia di Microsoft Access, è possibile utilizzare Microsoft Visual Basic per scrivere le routine che creano, eliminano e modificano in modo dinamico dati e oggetti. È possibile ad esempio scrivere una routine che colloca una didascalia in una maschera o che modifica il colore del campo di una maschera quando vengono eseguite determinate azioni.

È possibile fare riferimento a dati e oggetti direttamente nel codice oppure dichiarare variabili di oggetto che li rappresentino. Una volta dichiarata e assegnata una variabile di oggetto, è possibile utilizzarla come se fosse il nome dell'oggetto e modificarne il valore così come avviene per qualsiasi variabile.

La seguente routine Sub ad esempio crea una tabella Fatture scadute con un solo campo IDOrdine:

Sub Crea Tabella ()

    ' Dichiara variabili.
    Dim dbs As Database, tbl As TableDef, fld As Field

    ' Assegna il database corrente alla variabile
    ' di database.
    Set dbs = CurrentDB

    ' Crea una nuova tabella e un nuovo campo e li assegna
    ' alle variabili di tabella e campo.
    Set tbl = dbs.CreateTableDef("Fatture scadute")
    Set fld = tbl.CreateField("IDOrdine", dbText)

    ' Aggiunge il campo alla tabella, quindi aggiunge
    ' la tabella al database.
    tbl.Fields.Append fld
    dbs.TableDefs.Append tbl
    dbs.TableDefs.Refresh


End Sub

Gli oggetti di accesso ai dati (DAO, Data Access Objects) Microsoft forniscono oggetti come tabelle, query, relazioni e indici, che svolgono le attività di gestione dei dati in un database di Microsoft Access. Questi oggetti vengono chiamati oggetti DAO. È possibile condividere il codice Visual Basic che utilizza oggetti DAO con altre applicazioni che utilizzano Microsoft DAO, come Microsoft Excel.

È anche possibile utilizzare gli oggetti Jet e Replication per accedere e manipolare i dati in un server di database tramite qualsiasi provider OLE DB.

Microsoft Access definisce inoltre una varietà di oggetti da utilizzare quando si lavora con i dati, come maschere, report e controlli.
