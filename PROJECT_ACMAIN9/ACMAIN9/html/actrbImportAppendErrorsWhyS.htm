Errori relativi all'accodamento di un foglio di calcolo o di un file di testo

In un database di Microsoft Access, per la maggior parte dei record che provocano un errore, viene aggiunta una riga alla tabella relativa agli errori di importazione. Per visualizzare l'elenco completo degli errori riscontrati, aprire la tabella relativa agli errori di importazione nella finestra del database. Per ulteriori informazioni sui messaggi riguardanti la tabella relativa agli errori di importazione, fare clic su .

Nei progetti di Microsoft Access non viene creata alcuna tabella relativa agli errori di importazione. È consigliabile l'utilizzo di DTS (Data Transformation Services), un metodo più rapido e flessibile per l'importazione e la trasformazione dei dati. Per ulteriori informazioni, consultare la documentazione di Microsoft SQL Server.

I record accodati devono essere compatibili con la struttura della tabella esistente. Ciascun campo deve avere cioè lo stesso tipo di dati del campo corrispondente nella tabella di destinazione e i campi devono essere nello stesso ordine, a meno che non si stia utilizzando la prima riga del file come nomi di campo, nel qual caso essi devono coincidere.

Se si ritiene che il problema sia causato dai dati che si stanno accodando, modificare il foglio di calcolo o il file di testo, quindi effettuare nuovamente l'importazione. In alternativa può essere necessario modificare la stessa tabella di destinazione. È possibile riordinare i campi, modificare i tipi di dati, eseguire una query di aggiornamento, ad esempio per riformattare o per ricalcolare i dati, oppure eseguire una query di creazione tabelle, ad esempio per dividere un campo in due o per combinare diversi campi in uno solo.

Vengono riportate di seguito le cause più comuni di errore:

  • I nomi di campo nel foglio di calcolo o nel file di testo non corrispondono ai nomi di campo nella tabella a cui si stanno accodando i dati.

  • I dati in un campo non sono adatti al tipo di dati del campo di destinazione. Il campo di destinazione ha ad esempio il tipo di dati Data/ora in un database o un progetto di Microsoft Access, ma i dati contengono un valore di testo che non può essere riconosciuto come una data o un'ora.

  • I dati in un campo numerico sono troppo estesi per le dimensioni del campo di destinazione. Questo ha ad esempio la proprietà Dimensione campo impostata a Byte in un database di Access (a ? o un tipo di dati tinyint in un progetto di Access) ma i dati contengono un valore maggiore di 255.

  • I record che si stanno importando contengono dei valori duplicati che non possono essere memorizzati nella chiave primaria della tabella di destinazione o in uno qualsiasi dei campi della tabella la cui proprietà Indicizzato è impostata a Sì (Duplicati non ammessi) in un database di Access o che presenta l'opzione Ignora chiave duplicata in un progetto di Access.

  • Una o più righe del file di testo o del foglio di calcolo contengono un numero di campi superiore a quello della tabella di destinazione.

  • I campi di tipo data importati da un foglio di calcolo di Microsoft Excel differiscono di quattro anni. Microsoft Excel per Windows utilizza il Sistema Data 1900, in cui i numeri seriali vanno da uno a 65.380 e corrispondono alle date dal 1 gennaio 1900 al 31 dicembre 2078, mentre Microsoft Excel per il Macintosh utilizza il Sistema Data 1904, in cui i numeri seriali vanno da zero a 63.918 e corrispondono alle date dal 1 gennaio 1904 al 31 dicembre 2078.

    Prima di importare i dati cambiare il sistema relativo alla data del foglio di calcolo di Microsoft Excel, oppure, dopo l'importazione dei dati, eseguire una query di aggiornamento utilizzando l'espressione [campo data] + 1462 per correggere le date. Per ulteriori informazioni sul sistema relativo alla data di Microsoft Excel, consultare la documentazione relativa a Microsoft Excel.
