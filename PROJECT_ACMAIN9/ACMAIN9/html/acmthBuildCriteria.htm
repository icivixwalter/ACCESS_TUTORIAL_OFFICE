Metodo BuildCriteria

                 

Il metodo BuildCriteria restituisce una stringa di criteri analizzata, nel modo in cui viene visualizzata nella griglia di struttura della query, nella modalità filtro in base a maschera o nella modalità filtro su server in base a maschera. È possibile che si desideri, ad esempio, impostare la proprietà Filter o ServerFilter di una maschera in base a criteri variabili stabiliti dall'utente. È possibile utilizzare il metodo BuildCriteria per costruire l'argomento espressione stringa per la proprietà Filter o ServerFilter.

Sintassi

applicazione.BuildCriteria(campo, tipocampo, espressione)

Il metodo BuildCriteria dispone degli argomenti riportati di seguito.

Argomento Descrizione
applicazione Opzionale. Oggetto Application.
campo Espressione stringa che identifica il campo per cui si desidera definire i criteri.
tipocampo Costante intrinseca che denota il tipo di dati del campo. Per un elenco di tipi di dati di campo, vedere la proprietà ADO Type.
espressione Espressione stringa che identifica i criteri da analizzare.

Il metodo BuildCriteria restituisce una stringa.

Osservazioni

Il metodo BuildCriteria consente di costruire con facilità criteri per un filtro in base all'input dell'utente. Il metodo analizza l'argomento espressione come se immesso nella griglia di struttura della query, nella modalità filtro in base a maschera o filtro su server in base a maschera.

È possibile, ad esempio, che un utente che crea una query su una tabella Ordini limiti il set di risultati agli ordini inoltrati dopo il primo gennaio 1995, impostando i criteri in un campo DataOrdine. È possibile immettere un'espressione come quella riportata di seguito nella riga Criteria sottostante al campo DataOrdine.

>1-1-95

Microsoft Access analizza automaticamente tale espressione e restituisce l'espressione seguente:

>#01/01/95#

Il metodo BuildCriteria fornisce la stessa analisi di codice di Visual Basic. Per restituire, ad esempio, la stringa precedente analizzata correttamente, è possibile fornire al metodo BuildCriteria gli argomenti riportati di seguito.

Dim strCriteria As String
strCriteria = BuildCriteria("DataOrdine", dbDate, ">1-1-95")

È necessario fornire i criteri per la proprietà Filter nella forma analizzata corretta, è possibile a tale scopo utilizzare il metodo BuildCriteria per costruire una stringa analizzata in modo corretto.

È possibile utilizzare il metodo BuildCriteria per costruire una stringa con più criteri, se tali criteri fanno riferimento allo stesso campo. È possibile, ad esempio, utilizzare il metodo BuildCriteria con gli argomenti riportati di seguito per costruire una stringa con più criteri che fanno riferimento al campo DataOrdine.

strCriteria = BuildCriteria("DataOrdine", dbDate, ">1-1-95 e <5-1-95")

Questo esempio restituisce la stringa di criteri riportata di seguito.

DataOrdine>#01/01/95# e DataOrdine<#05/01/95#

Se, tuttavia, si desidera costruire una stringa di criteri che fa riferimento a più campi, è necessario creare le stringhe e concatenarle. Se, ad esempio, si desidera costruire dei criteri per un filtro per visualizzare i record relativi a ordini inoltrati dopo il 01/01/95 e per cui il costo di spedizione è inferiore a L. 80.000, è necessario utilizzare due volte il metodo BuildCriteria e concatenare le stringhe risultanti.
