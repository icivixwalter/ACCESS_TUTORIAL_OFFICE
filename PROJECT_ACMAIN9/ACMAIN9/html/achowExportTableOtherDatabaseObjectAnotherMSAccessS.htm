Esportare oggetti di database in un database di Microsoft Access o in un progetto di Microsoft Access esistente

  1. Nella finestra del database fare clic sul nome dell'oggetto da esportare, quindi scegliere Esporta dal menu File.

  2. Fare clic sulla freccia posizionata a destra della casella di riepilogo Salva in e selezionare l'unità o la cartella in cui si trova il database di Microsoft Access che si desidera esportare.

  3. Fare doppio clic sull'icona del database che si desidera esportare.

  4. Nella finestra di dialogo Esporta inserire un nome per il nuovo oggetto o accettare il nome corrente. Se si desidera esportare una tabella, nel riquadro Esporta tabelle selezionare se si desidera esportare la definizione e i dati della tabella o solo la definizione. Fare clic su OK per esportare l'oggetto.

Note  

  • È possibile esportare le tabelle a versioni precedenti di Microsoft Access.

  • Quando si esporta una tabella in un database di Access, vengono esportati solo i dati e le definizioni dei dati, non le proprietà, i vincoli, le relazioni e gli indici.

  • Poiché i dati in Microsoft Access 2000 vengono codificati in formato Unicode, possono essere convertiti e modificati quando si esporta una tabella a una precedente versione. Fare clic qui per informazioni sulla conversione dei database.

  • Non è possibile esportare visualizzazioni, stored procedure e diagrammi di database in un progetto di Microsoft Access o in un database di Access. Non è possibile esportare query o relazioni da un database di Access in un progetto di Access.

  • È possibile esportare solo un oggetto per volta. Se si desidera esportare più oggetti contemporaneamente è consigliabile aprire il database in cui si desidera esportarli e importare gli oggetti. Per ulteriori informazioni fare clic su .

  • Quando si esporta un oggetto in un altro database probabilmente si esporteranno anche gli oggetti correlati per permettere il corretto funzionamento dell'oggetto stesso. Ad esempio, un report può avere una query come origine record e tale query può essere basata su una relazione tra due tabelle.

  • Quando si esporta una pagina di accesso ai dati, si esporta solo il collegamento al file HTML corrispondente. Il file HTML rimane nella stessa posizione e pertanto due pagine di accesso ai dati faranno riferimento allo stesso file HTML corrispondente. Per evitare questa situazione, salvare una delle pagine di accesso ai dati con un nome diverso (scegliere Salva con nome dal menu File), immettere un percorso file diverso per il file HTML corrispondente nella finestra di dialogo Salva come pagina di accesso ai dati, quindi eliminare la pagina di accesso ai dati appena salvata.
