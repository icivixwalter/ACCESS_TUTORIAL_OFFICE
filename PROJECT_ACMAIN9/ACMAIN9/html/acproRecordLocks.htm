Proprietà RecordLocks (Blocco record)

                 

È possibile utilizzare la proprietà RecordLocks per determinare come i record sono bloccati e quel che succede quando due utenti cercano di modificare un file allo stesso momento. Durante la modifica un record può essere automaticamente bloccato, in modo da impedirne la modifica prima del termine da parte di altri utenti.

  • Maschere - Specifica il modo in cui i record vengono bloccati nella tabella o nella query sottostante durante l'aggiornamento di dati contenuti in un database multiutente.

  • Report - Specifica se i record contenuti in una tabella o in una query sottostante sono bloccati mentre un report viene visualizzato in anteprima o viene stampato.

  • Query - Specifica se i record contenuti in una query, in genere una query di comando in un database multiutente, sono bloccati durante l'esecuzione della query stessa.

Nota   La proprietà RecordLocks si applica soltanto alle maschere, ai report o alle query di un database Microsoft Access (mdb).

Impostazione

La proprietà RecordLocks viene impostata come riportato di seguito.

Impostazione Visual Basic Descrizione
Nessun blocco 0 Impostazione predefinita. Nelle maschere due o più utenti possono modificare lo stesso record contemporaneamente. Questo procedimento viene anche detto blocco ottimistico. Se due utenti provano a salvare le modifiche relative allo stesso record, viene visualizzato un messaggio all'utente che prova a salvare il record per secondo. Sarà quindi possibile eliminare il record, copiarlo negli Appunti o sostituire le modifiche effettuate dall'altro utente. Tale impostazione viene in genere utilizzata in maschere di sola lettura o in database per utente singolo. L'impostazione viene inoltre utilizzata in database multiutente per consentire a più di un utente la possibilità di effettuare modifiche nello stesso record contemporaneamente.

Nei report, i record non vengono bloccati durante la visualizzazione in anteprima o la stampa del report.

Nelle query i record non vengono bloccati durante l'esecuzione della query stessa.
Tutti i record 1 Tutti i record contenuti nella tabella o nella query sottostante sono bloccati mentre la maschera è aperta in visualizzazione Maschera o in visualizzazione Foglio dati, durante la visualizzazione in anteprima o la stampa del report o durante l'esecuzione della query. Pur essendo possibile la lettura dei record, non è possibile modificare, aggiungere o eliminare alcun record fino a quando la maschera viene chiusa, il report è stampato completamente o la query ha terminato l'esecuzione.
Record modificato 2 Solo maschere e query. Una pagina di record viene bloccata non appena un qualsiasi utente comincia a modificare un campo del record e rimane bloccata fino a quando l'utente si sposta su un altro record. Un record può essere quindi modificato da un solo utente alla volta. Questo procedimento viene detto anche blocco pessimistico.

È possibile impostare la proprietà utilizzando la finestra delle proprietà di una maschera, una macro o Visual Basic.

Nota   La modifica della proprietà RecordLocks di un report o di una maschera aperta ricrea automaticamente il set di record.

Osservazioni

È possibile utilizzare l'impostazione Nessun blocco per le maschere se una sola persona utilizza le tabelle o le query sottostanti o effettua tutte le modifiche ai dati.

In un database multiutente è possibile utilizzare l'impostazione Nessun blocco se si desidera utilizzare il blocco ottimistico e avvisare gli utenti che provano a modificare lo stesso record di una maschera. È possibile utilizzare l'impostazione Record modificato se si desidera impedire a due o più utenti di modificare i dati allo stesso momento.

È possibile utilizzare l'impostazione Tutti i record quando è necessario assicurare che nessuna modifica sia stata apportata ai dati dopo l'inizio dell'anteprima di stampa o della stampa di un report o dell'esecuzione di una query di accodamento, di eliminazione, di creazione tabella o di aggiornamento.

In visualizzazione Maschera o in visualizzazione Foglio dati, ciascun record bloccato è contraddistinto dall'indicatore di blocco nel relativo selettore di record.

Suggerimento   Per modificare l'impostazione predefinita della proprietà RecordLocks per le maschere, scegliere Opzioni dal menu Strumenti, fare clic sulla scheda Avanzate nella finestra di dialogo Opzioni e selezionare l'opzione desiderata sotto Blocco record predefinito.

I dati contenuti in una maschera, in un report o in una query di un database ODBC (Open Database Connectivity) vengono trattati come se fosse stata scelta l'impostazione Nessun record senza considerare l'impostazione della proprietà RecordLocks.
