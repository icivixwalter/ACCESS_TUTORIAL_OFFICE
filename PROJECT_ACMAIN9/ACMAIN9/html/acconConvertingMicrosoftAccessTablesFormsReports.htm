Convertire le tabelle, le maschere e i report di Microsoft Access

Numerose modifiche introdotte da Microsoft Access 2000 potrebbero influire sul funzionamento delle applicazioni nelle versioni 1.x o 2.0. Nelle sezioni che seguono vengono fornite ulteriori informazioni su tali modifiche.

Indici e relazioni

Una tabella di Microsoft Access può contenere fino a un massimo di 32 indici. Le tabelle molto complesse che fanno parte di più relazioni possono eccedere questo limite e il database che contiene tali tabelle non può essere convertito. La versione 3.5 del modulo di gestione di database Microsoft Jet crea indici per entrambi i lati della relazione tra le tabelle. Se la conversione del database non riesce, eliminare alcune relazioni e tentare di nuovo l'operazione.

Proprietà Solo in elenco delle caselle combinate

In Microsoft Access 2000 le caselle combinate accettano valori di tipo Null quando la proprietà Solo in elenco è impostata a Vero (-1), sia che l'elenco contenga valori di tipo Null o meno. Nella versione 2.0 una casella combinata con la proprietà Solo in elenco impostata a Vero non accetta un valore di tipo Null a meno che l'elenco non contenga un valore di questo tipo. Per impedire che gli utenti immettano un valore di tipo Null utilizzando una casella combinata è necessario impostare a Sì la proprietà Richiesto nel campo della tabella.

Menu e attivazione diretta di oggetti OLE

Per disporre di una maggiore funzionalità durante l'attivazione diretta di oggetti OLE, è possibile che alcuni comandi di menu siano stati spostati in un menu che non viene sostituito quando si attiva un server OLE.

Quando viene attivato un componente, le modifiche apportate non influenzano le macro dell'applicazione convertita che utilizzano l'azione EseguiVoceMenu per eseguire un comando di menu della versione 2.0. I comandi della versione 2.0 vengono associati ai comandi equivalenti in Microsoft Access 2000.

Creazione di un riferimento a un controllo in una maschera di sola lettura

In Microsoft Access 2000 non è possibile utilizzare un'espressione per fare riferimento al valore di un controllo contenuto in una maschera di sola lettura associata a un'origine record vuota. Nelle versioni precedenti l'espressione restituisce un valore di tipo Null. Prima di creare un riferimento a un controllo in una maschera di sola lettura è necessario assicurarsi che l'origine record della maschera contenga dei record.

Campi di tipo Data e immissione dati

Se in un campo di tipo Data in un foglio dati di una maschera o di una tabella si immette 3/3, Microsoft Access 2000 immette automaticamente l'anno corrente. Se tuttavia nello stesso campo si immette 3/3/, Microsoft Access visualizza un messaggio di errore. Se si omette l'ultimo separatore di data, viene eseguita la conversione della data nel formato appropriato.

Pulsanti creati con la Creazione guidata Pulsante di comando

Mentre nelle versioni 2.0 o 7.0 di Microsoft Access si utilizza la Creazione guidata Pulsante di comando per generare il codice che chiama un'altra applicazione, in Microsoft Access 2000 è necessario prima eliminare il pulsante, quindi crearlo di nuovo mediante l'apposita creazione guidata.

Moduli di classe di maschere e report

Nelle versioni precedenti di Microsoft Access agli oggetti Form e Report sono associati moduli di classe anche se dietro l'oggetto non esiste un codice. In Microsoft Access 2000 è possibile impostare a Falso la proprietà Possiede modulo di una maschera o di un report. Dopo aver impostato a Falso tale proprietà, la maschera o il report occuperanno meno spazio su disco e verranno caricati più velocemente poiché a essi non sarà più associato un modulo di classe.

Differenza di margini dei report della versione 2.0 convertiti

In Microsoft Access 2000 la stampa o l'anteprima di un report convertito da Microsoft Access 2.0 potrebbe presentare dei problemi se alcuni valori relativi ai margini sono impostati a 0. Quando si esegue la conversione di un report da Microsoft Access 2.0, i valori dei margini non sono impostati a 0, ma sul valore minimo valido per la stampante predefinita. Ciò impedisce che alcuni dati del report fuoriescano dall'area di stampa della stampante.

Per risolvere questo problema, ridurre la larghezza, la spaziatura o il numero delle colonne contenute nel report di modo che la somma della larghezza delle colonne e quella dei margini predefiniti sia inferiore o uguale alla larghezza del foglio.

Impossibilità di utilizzare la proprietà Formato per distinguere i valori di tipo Null e le stringhe a lunghezza zero

Nelle versioni 1.x e 2.0 è possibile utilizzare la proprietà Formato di un controllo per visualizzare valori differenti per valori di tipo Null e stringhe a lunghezza zero (" "). In Microsoft Access 2000 per fare distinzione tra i valori di tipo Null e le stringhe a lunghezza zero in un controllo contenuto in una maschera, è necessario impostare la proprietà Origine controllo a un'espressione che ricerchi eventuali valori di tipo Null. Per visualizzare ad esempio "Null" o "ZLS" in un controllo, impostare la relativa proprietà ControlSource all'espressione riportata di seguito.

=IIf(IsNull([MioControllo]), "Null", Format([MioControllo], "@;ZLS"))
