Espressioni ottimizzabili semplici e complesse e ottimizzazione mediante la tecnica Rushmore

Con Microsoft Access è possibile ottimizzare le espressioni semplici o complesse nella riga Criteri della griglia di struttura della query oppure in una proposizione WHERE di un'istruzione SQL SELECT. Per determinati tipi di espressioni complesse, Microsoft Access può utilizzare Rushmore, una tecnica di accesso ai dati utilizzata dal modulo di gestione di database Microsoft Jet per ottenere un maggiore livello di ottimizzazione.

Espressioni ottimizzabili semplici

Un'espressione ottimizzabile semplice può costituire un'intera espressione o esserne una parte e può assumere una delle seguenti forme:

Campo indicizzato Operatore di confronto Espressione

[Data ordine]=#15/09/96#

oppure

Espressione Operatore di confronto Campo indicizzato

#1/11/96#<[Data di assunzione]

In un'espressione ottimizzabile semplice:

  • Un campo indicizzato è un campo con un proprio indice oppure il primo campo di un indice multicampo.

  • L'operatore di confronto deve essere uno dei seguenti operatori: <, >, =, <=, >=, <>, BETWEEN…AND, LIKE, IN.

  • Un'espressione può essere una qualsiasi espressione valida, incluse costanti, funzioni e campi di altre tabelle.

Per visualizzare degli esempi di espressioni ottimizzabili semplici, fare clic su .

Espressioni complesse

Un'espressione complessa viene creata attraverso la combinazione di due espressioni semplici con l'operatore AND oppure OR e può assumere una delle seguenti forme:

Espressione semplice AND Espressione semplice

oppure

Espressione semplice OR Espressione semplice

Il livello di ottimizzazione (intero o parziale) di un'espressione complessa dipende dal fatto che solo una o entrambe le espressioni semplici siano ottimizzabili o meno e dall'operatore che è stato utilizzato per combinarle. È possibile ottimizzare un'espressione complessa tramite la tecnica Rushmore se tutte le seguenti proposizioni sono vere:

  • L' espressione utilizza AND oppure OR per unire i due criteri.

  • Entrambi i criteri sono costituiti da espressioni ottimizzabili semplici.

  • Entrambe le espressioni contengono campi indicizzati. I campi possono essere indicizzati singolarmente o far parte di un indice multicampo.

Nota   È possibile ottimizzare indici multicampo se la query dei campi indicizzati viene eseguita nell'ordine in cui essi appaiono nella finestra degli indici, a partire dal primo campo indicizzato e proseguendo con i campi successivi, fino a un massimo di 10. Se ad esempio si trova un indice multicampo che include i campi Cognome e Nome, è possibile ottimizzare una query eseguita su Cognome oppure su Cognome e Nome, ma non su Nome.

La tabella riportata di seguito indica il livello di ottimizzazione delle query per le diverse combinazioni di espressioni semplici in base ai criteri specificati per la query.

Espressione Operatore Espressione Risultato della query
Ottimizzabile AND Ottimizzabile Interamente ottimizzabile (utilizzando Rushmore)
Ottimizzabile OR Ottimizzabile Interamente ottimizzabile (utilizzando Rushmore)
Ottimizzabile AND Non ottimizzabile Parzialmente ottimizzabile (Rushmore non viene utilizzato)
Ottimizzabile OR Non ottimizzabile Non ottimizzabile
Non ottimizzabile AND Non ottimizzabile Non ottimizzabile
Non ottimizzabile OR Non ottimizzabile Non ottimizzabile
-- NOT Ottimizzabile Non ottimizzabile
-- NOT Non ottimizzabile Non ottimizzabile

È possibile inoltre utilizzare le parentesi per raggruppare combinazioni di espressioni semplici. Le regole elencate precedentemente valgono anche per combinazioni di espressioni raggruppate con le parentesi.

Dopo aver effettuato delle combinazioni di espressioni ottimizzabili semplici, creando così delle espressioni complesse, queste ultime possono, a loro volta, essere combinate per creare espressioni ancora più complesse che è possibile ottimizzare in base alle regole indicate in precedenza.

Per visualizzare degli esempi di espressioni ottimizzabili complesse, fare clic su .

Note

  • La funzione Count dispone di un alto livello di ottimizzazione per le query che utilizzano Rushmore. Per ulteriori informazioni sulla funzione Count, fare clic su .

  • Le query che utilizzano Rushmore operano con le tabelle di Microsoft Access, di Microsoft FoxPro e di dBASE (file DBF). Non è possibile utilizzare Rushmore con le origini dati ODBC, poiché Microsoft Access invia queste query all'origine dati ODBC invece di elaborarle a livello locale.
