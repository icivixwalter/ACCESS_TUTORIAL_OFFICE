Errore causato da un identificatore

In Microsoft Access 97 e versioni successive sono supportate alcune nuove parole chiave di Visual Basic che non è più possibile utilizzare come identificatori. Queste parole chiave sono AddressOf, Decimal, DefDec, Enum, Event, Friend, Implements, RaiseEvent, WithEvents. Quando si converte un database di una versione precedente di Access nel formato di Microsoft Access 2000, gli identificatori esistenti che corrispondono a nuove parole chiave di Visual Basic causeranno errori di compilazione. Per risolvere il problema, modificare gli identificatori esistenti affinché non corrispondano a parole chiave.
