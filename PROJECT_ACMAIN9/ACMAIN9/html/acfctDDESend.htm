Funzione DDESend

        

È possibile utilizzare la funzione DDESend per iniziare una conversazione DDE (Dynamic Data Exchange) con un'altra applicazione e inviare a questa un'informazione da un controllo contenuto in una maschera o in un report.

È possibile ad esempio utilizzare la funzione DDESend nella proprietà Origine controllo di una casella di testo per inviare i dati visualizzati in tale casella a una determinata cella di un foglio di calcolo di Microsoft Excel.

Sintassi

DDESend(applicazione, argomento, elemento, dati)

La funzione DDESend utilizza gli argomenti riportati di seguito.

Argomento Descrizione
applicazione Espressione stringa che identifica un'applicazione che può effettuare una conversazione DDE. applicazione è in genere il nome di un file EXE di un'applicazione per Microsoft Windows privo dell'estensione, ad esempio Microsoft Excel. Per iniziare una conversazione DDE con Microsoft Excel, digitare "Excel" per l'argomento applicazione.
argomento Espressione stringa corrispondente al nome di un argomento riconosciuto da applicazione. argomento è in genere un documento o un file di dati. Consultare la documentazione relativa all'altra applicazione per un elenco degli argomenti disponibili.
elemento Espressione stringa corrispondente al nome di un dato riconosciuto da applicazione. Consultare la documentazione relativa all'altra applicazione per un elenco degli elementi disponibili.
dati Stringa o espressione contenente i dati da inviare ad applicazione.

Osservazioni

La funzione DDESend inizia una conversazione DDE con applicazione e argomento e identifica elemento come il dato che riceve dati. Se ad esempio applicazione è Microsoft Excel, argomento potrebbe essere "Foglio1" ed elemento potrebbe essere un identificatore di riga e colonna, come "R1C1" o il nome di un intervallo di celle.

L'argomento dati specifica le informazioni che si desidera inviare. È possibile che si tratti di una stringa letterale, ad esempio "Report preparato da Marco" oppure un'espressione che include il risultato di una funzione che crea una stringa, ad esempio "Preparato il " & Date(). Se elemento fa riferimento a più informazioni, ad esempio a un intervallo denominato di un foglio di lavoro di Microsoft Excel contenente più celle, la funzione DDESend invia dati alla prima voce.

Nell'esempio riportato di seguito, la funzione DDESend invia la stringa "Testo" alla cella della riga 1, colonna 1 di un foglio di lavoro di Microsoft Excel. È possibile immettere tale espressione per un controllo casella di testo nella casella della proprietà Origine controllo nella finestra delle proprietà del controllo:

=DDESend("Excel", "Foglio1", "R1C1", "Testo")

Si supponga di voler inviare dei dati da un controllo associato di una maschera di Microsoft Access a una cella di un foglio di calcolo di Microsoft Excel. La proprietà Origine controllo del controllo associato contiene già un nome di campo o un'espressione. È possibile creare un'altra casella di testo o una casella combinata e impostare la relativa proprietà Origine controllo a un'espressione che includa la funzione DDESend, in cui dati è il nome del controllo associato. Se a esempio si dispone di una casella di testo associata chiamata Cognome, è possibile creare un'altra casella di testo e impostare la relativa proprietà Origine controllo come segue:

=DDESend("Excel", "Foglio1", "R1C1", [Cognome])

Tale controllo intermediario deve essere costituito da una casella di testo o da una casella combinata. Non è possibile utilizzare il nome di un controllo associato come argomento dati per una casella di controllo o per un gruppo di opzioni.

È possibile utilizzare la funzione DDESend solo nell'impostazione della proprietà Origine controllo di una casella di testo, di un gruppo di opzioni, di una casella di controllo o di una casella combinata di una maschera. Non è possibile richiamare la funzione DDESend da Visual Basic.

Quando si utilizza la funzione DDESend, il controllo diventa di sola lettura solo in visualizzazione Maschera e nell'Anteprima di stampa. Poiché anche la proprietà Origine controllo è di sola lettura in visualizzazione Maschera e nell'Anteprima di stampa, le modifiche al controllo devono essere effettuate in visualizzazione Struttura.

Il numero massimo di conversazioni DDE che possono essere aperte contemporaneamente è determinato da Microsoft Windows, la memoria e le risorse del computer. Se non è possibile iniziare la conversazione perché applicazione non è in esecuzione o non riconosce argomento o se è già stato raggiunto il numero massimo di conversazioni consentite, la funzione DDESend restituisce il valore Null.

Nota   L'altra applicazione potrebbe essere configurata in modo da ignorare la richiesta di conversazione DDE. In tal caso, la funzione DDESend restituisce il valore Null. È possibile analogamente impostare Microsoft Access in modo da ignorare le richieste di altre applicazioni. Scegliere Opzioni dal menu Strumenti e sulla scheda Avanzate della finestra di dialogo Opzioni, nel riquadro Operazioni DDE, selezionare Ignora richieste DDE.

Suggerimento   Se si rende necessario manipolare gli oggetti di un'altra applicazione da Microsoft Access, è possibile utilizzare l'Automazione.

Nella tabella riportata di seguito viene illustrato il comportamento della funzione DDESend quando viene utilizzata con ciascun controllo.

Controllo Osservazioni
Casella di testo o casella combinata Poiché la casella di testo o la casella combinata appare vuota in visualizzazione Maschera e nell'Anteprima di stampa, è possibile impostare la relativa proprietà Visibile a Falso.

L'argomento dati può fare riferimento a un altro controllo. Nell'esempio riportato di seguito viene illustrato come inviare il contenuto di un controllo chiamato Cognome a un foglio di lavoro di Microsoft Excel.

=DDESend("Excel", "Foglio1", "R1C1", [Cognome])
Gruppo di opzioni In visualizzazione Maschera e nell'Anteprima di stampa non è selezionato alcun pulsante di opzione del gruppo di opzioni. È possibile rendere il gruppo di opzioni e i relativi pulsanti non visibili, impostando la relativa proprietà Visibile a Falso.

L'argomento dati deve contenere dati numerici, ad esempio "2", altrimenti la funzione DDESend non invia le informazioni ed elemento non viene modificato.
Casella di controllo La casella di controllo è inattiva in visualizzazione Maschera e nell'Anteprima di stampa. È possibile renderla non visibile impostando la relativa proprietà Visibile a False.

L'argomento dati deve contenere dati numerici, ad esempio "2", altrimenti la funzione DDESend non invia le informazioni ed elemento non viene modificato.
