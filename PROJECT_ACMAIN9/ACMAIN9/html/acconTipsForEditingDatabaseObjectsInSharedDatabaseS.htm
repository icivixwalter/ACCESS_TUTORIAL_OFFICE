Suggerimenti per la modifica di oggetti del database di Access in un ambiente multiutente

  • È consigliabile creare la struttura degli oggetti prima di consentire ad altri utenti di aprire il database di Microsoft Access per un accesso condiviso. Quando si desidera apportare modifiche alla struttura, è possibile assicurarsi di essere l'unico utente a utilizzare il database di Access selezionando Apertura esclusiva dall'elenco che compare facendo clic sulla freccia accanto al pulsante Apri nella finestra di dialogo Apri quando si apre il database. Per ulteriori informazioni, vedere l'argomento relativo al salvataggio delle modifiche in un database di Microsoft Access condiviso.

  • Se un altro utente ha una tabella aperta o sta visualizzando dei dati nelle query, nelle maschere o nei report basati sulla tabella, non è possibile apportare modifiche alla struttura della tabella. Viene visualizzato un messaggio per indicare che la tabella è di sola lettura.

  • Se gli oggetti dipendono l'uno dall'altro, aggiornarli tutti contemporaneamente in modo che altri utenti non aprano versioni non congruenti degli oggetti. Se si desidera ad esempio aggiungere campi a una maschera, assicurarsi di aggiungere i campi alla query sottostante prima di aggiornare la maschera.

  • Se si modifica una query aperta da un altro utente, l'utente deve chiudere e riaprire l'oggetto modificato per utilizzare la versione più recente. Con i moduli di Visual Basic, Applications Edition gli altri utenti non possono eseguire le routine aggiornate Function o Sub fino a quando il database di Access non viene chiuso e riaperto.
