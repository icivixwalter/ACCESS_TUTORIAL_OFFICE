Elaborazione dei file IDC/HTX sul server Web

Quando si invia l'output di una tabella, di una query o di una maschera in formato file IDC/HTX, anziché come file html, vengono creati un file con estensione HTML (htx) e un file Internet Database Connector (idc). Internet Database Connector (httpodbc.dll) è un componente di Microsoft Internet Information Server.

Il file idc contiene una query sotto forma di istruzione SQL e le informazioni che Microsoft Internet Information Server utilizza per connettersi a un'origine dati ODBC, in questo caso un database di Microsoft Access o di Microsoft SQL server. Le informazioni relative alla connessione comprendono il nome dell'origine dati e spesso il nome utente e la password, entrambi facoltativi.

Il file htx è un file HTML che contiene i tag e le istruzioni per la formattazione e, anziché i dati, dei segnaposti che indicano dove inserire i valori restituiti dalla query nel file idc.

Dopo la pubblicazione dei file IDC/HTX su un server Web, su richiesta del browser Web, Microsoft Internet Information Server apre il database utilizzando il driver ODBC appropriato e le informazioni sulla connessione del file idc, esegue la query nel file idc per accedere ai dati, unisce i risultati e il file htx in un unico file html e infine invia di nuovo il file html creato dinamicamente al browser Web per visualizzare una pagina Web.

Elaborazione dei file ASP sul server Web

Quando si invia l'output di una tabella, di una query o di una maschera in formato file ASP, anziché come file html, viene creato un file Microsoft Active Server Page (asp). Active Server è un componente di Microsoft Internet Information Server 3.0 o versioni successive.

Il file asp contiene tag HTML inframmezzati da una o più query sotto forma di istruzioni SQL, direttive di modelli e codice Visual Basic Script con riferimenti ai controlli ActiveX Server. Il file asp inoltre contiene informazioni sulla connessione ODBC per connettersi a un'origine dati ODBC, in questo caso un database di Microsoft Access o di Microsoft SQL Server. Le informazioni relative alla connessione comprendono il nome dell'origine dati e spesso il nome utente e la password, entrambi facoltativi.

Dopo la pubblicazione dei file ASP su un server Web, su richiesta del browser Web, Microsoft Internet Information Server esegue il codice Visual Basic Scripting, chiama i controlli ActiveX Server, apre il database utilizzando il driver ODBC appropriato e le informazioni sulla connessione del file asp, esegue le query nel file asp per accedere ai dati, unisce i risultati e i tag HTML contenuti nel file asp in un unico file html e infine invia di nuovo il file html creato dinamicamente al browser Web per visualizzarlo come pagina Web.
