Risoluzione dei problemi relativi alla modifica di dati in un campo in visualizzazione Foglio dati o Maschera

Nelle maschere o nei fogli dati sono presenti diversi tipi di campi che non è possibile modificare. Nei campi elencati di seguito i valori vengono visualizzati come per i campi normali, ma se si immettere dati non accade nulla:

  • Campi Contatore. Se un campo in un database di Microsoft Access ha un tipo di dati Contatore o in un progetto di Microsoft Access ha un tipo di dati Numerico con la proprietà Identità impostata, al campo viene assegnato automaticamente un numero per ciascun record aggiunto. Questo tipo di campo viene spesso utilizzato come numero ID di record o come chiave primaria. Quando si aggiunge un nuovo record, il valore del campo viene inserito in modo automatico e non è possibile modificarlo.

  • Campi calcolati. In un campo calcolato di un database di Access vengono visualizzati valori calcolati da Microsoft Access. Questi valori in genere sono basati su altri campi presenti nelle tabelle, ma i campi calcolati non vengono memorizzati nelle tabelle e non possono essere modificati.

  • Campi bloccati o disattivati. Se la proprietà Bloccato di un controllo di una maschera è impostata a Sì oppure se la proprietà Abilitato è impostata a No, i dati contenuti nel campo non possono essere modificati.

  • Campi in un record bloccato. Se si utilizza un database di Microsoft Access in un ambiente multiutente e un record viene bloccato da un altro utente, i dati contenuti nel record non possono essere modificati. Quando ci si sposta su un record bloccato, nel selettore di record viene visualizzato l'indicatore del record bloccato .

  • Campi in snapshot. Mentre la maggior parte delle query restituiscono set di record che possono essere modificati, alcune di esse restituiscono snapshot, che invece non è possibile modificare.

  • Campi in una maschera di sola lettura o in un database bloccato. Se la proprietà Consenti modifiche della maschera in uso è impostata a No oppure se i dati sottostanti sono di sola lettura o bloccati da un altro utente, i dati contenuti nella maschera non possono essere modificati.

È possibile che si verifichino dei problemi durante l'immissione di date in un campo con tipo di dati Data/ora. Se si utilizza il formato a barre per immettere le date, non immettere una barra finale non seguita da un numero. È consentito immettere ad esempio 1/3 e non 1/3/.
