Azione CopiaOggetto

        

L'azione CopiaOggetto copia l'oggetto di database specificato in un altro database di Microsoft Access (MDB), nello stesso database o in un progetto di Microsoft Access (ADP) con un nuovo nome. È possibile ad esempio copiare o creare una copia di backup di un oggetto esistente in un altro database o creare rapidamente un oggetto simile con alcune modifiche.

Impostazione

L'azione CopiaOggetto utilizza gli argomenti riportati di seguito.

Argomenti azione Descrizione
Database destinazione Un percorso e un nome file validi per il database di destinazione. Immettere il percorso e il nome di file nella casella Database destinazione della sezione Argomenti azione della finestra della macro. Se si desidera selezionare il database corrente, lasciare vuota questa casella.

Nota   Questo argomento è disponibile solamente con i database di Microsoft Access (MDB). Quando si utilizza questa azione con un progetto di Microsoft Access (ADP), la casella Database destinazione deve essere lasciata vuota.

Se si esegue una macro contenente l'azione CopiaOggetto in un database di libreria e si lascia vuota questa casella, Microsoft Access copia l'oggetto nel database di libreria.
Nuovo nome Un nuovo nome per l'oggetto. Se viene effettuata una copia in un altro database, lasciare vuota questa casella per conservare lo stesso nome.
Tipo oggetto origine Il tipo di oggetto che si desidera copiare. Selezionare Tabella, Query, Maschera, Report, Macro, Modulo, Pagina di accesso ai dati, Visualizzazione server, Diagramma o Stored procedure. Per copiare l'oggetto selezionato nella finestra del database, lasciare vuota questa casella.
Nome oggetto origine Il nome dell'oggetto da copiare. La casella Nome oggetto origine visualizza tutti gli oggetti del tipo selezionato per l'argomento Tipo oggetto origine contenuti nel database. Nella casella Nome oggetto origine selezionare l'oggetto da copiare. Se si lascia vuota la casella Tipo oggetto origine, lasciare vuota anche questa casella.

Se viene eseguita una macro contenente l'azione CopiaOggetto in un database di libreria, Microsoft Access cerca l'oggetto con tale nome prima nel database di libreria, quindi in quello corrente.

Osservazioni

È necessario immettere un valore per uno o entrambi gli argomenti Database destinazione e Nuovo nome relativi all'azione CopiaOggetto.

Se non si immette alcun valore per Tipo oggetto origine e Nome oggetto origine, Microsoft Access copia l'oggetto selezionato nella finestra del database. Per selezionare un oggetto nella finestra del database, è possibile utilizzare l'azione SelezionaOggetto con l'argomento Da finestra database impostato a Sì.

L'azione CopiaOggetto equivale a selezionare un oggetto nella finestra del database, scegliere Copia dal menu Modifica, quindi Incolla. Per poter assegnare un nuovo nome all'oggetto, viene visualizzata la finestra di dialogo Incolla con nome. L'azione CopiaOggetto esegue automaticamente tutti i passaggi.

Nota   Quando vengono copiate le pagine di accesso ai dati, l'azione CopiaOggetto copia solo il collegamento al file con estensione htm associato e non il file vero e proprio.

È inoltre possibile copiare un oggetto selezionato nella finestra del database o un oggetto aperto nella finestra della tabella, nella finestra della query, nella finestra della maschera, nella finestra del report, nella finestra della macro o nella finestra del modulo scegliendo Salva con nome/Esporta dal menu File. La finestra di dialogo Salva con nome offre l'opzione di salvare una copia dell'oggetto nel database corrente con un nuovo nome oppure di salvare l'oggetto in un altro database. Se l'oggetto è già stato salvato e viene salvato nel database corrente con un nuovo nome, la versione originaria rimane contraddistinta dal vecchio nome.

È necessario che il percorso e il nome file del database di destinazione esistano prima che la macro esegua l'azione CopiaOggetto. Se non esistono, Microsoft Access visualizza un messaggio di errore.

Per eseguire l'azione CopiaOggetto in Visual Basic, utilizzare il metodo CopyObject dell'oggetto DoCmd.
