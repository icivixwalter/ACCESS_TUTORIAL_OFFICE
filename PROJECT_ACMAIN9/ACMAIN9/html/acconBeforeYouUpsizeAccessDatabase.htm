Prima di effettuare l'upsize di un database di Microsoft Access

Prima di effettuare l'upsize di un database di Microsoft Access in un database di Microsoft SQL Server o in un progetto di Microsoft Access, è opportuno effettuare le seguenti operazioni:

Eseguire un backup del database   Sebbene l'Upsize guidato non rimuova alcun dato o oggetto dal database di Access, è opportuno creare una copia di backup del database prima di effettuarne l'upsize.

Accertarsi di avere a disposizione una quantità di spazio su disco sufficiente sulla periferica che conterrà il database di SQL Server sottoposto a upsize. Le prestazioni dell'Upsize guidato sono migliori quando la quantità di spazio disponibile su disco è considerevole.

Se sul server è disponibile una grande quantità di spazio su disco disponibile, moltiplicare la dimensione del database di Access per due e riservare quello spazio. In questo modo si garantisce alla procedura guidata una quantità di spazio sufficiente per effettuare l'upsize del database e un considerevole margine per espansioni future. Se si prevede di aggiungere molti dati al database, moltiplicare per un multiplo superiore.

Attenzione   Se lo spazio su disco del server per l'Upsize guidato si esaurisce, la procedura si blocca, lasciando un database parziale e le eventuali periferiche create sul server. In questo caso, è possibile eliminare le periferiche, i database e le tabelle utilizzando Microsoft SQL Server Enterprise Manager.

Se si desidera, creare più periferiche a disco   Se si sta effettuando l'upsize in un database di SQL Server 6.5, è possibile creare delle periferiche prima di eseguire l'Upsize guidato. L'Upsize guidato crea tutte le nuove periferiche sullo stesso disco fisico in cui risiede il database master. Se il server dispone di più dischi, è possibile collocare il database su un disco e il file di registro delle transazioni su un altro disco. Nel caso di un problema del disco, sarà così possibile ripristinare il database. Accertarsi che le periferiche su entrambi i dischi siano grandi abbastanza. Se necessario, creare nuove periferiche.

In SQL Server i database e i file di registro delle transazioni possono estendersi su diverse periferiche. L'Upsize guidato consente tuttavia di specificare una sola periferica per il database e una per il file di registro delle transazioni. Per specificare più periferiche per un database o per un file di registro delle transazioni, rendere predefinite tali periferiche (e nessun'altra). A questo punto eseguire l'Upsize guidato e selezionare la periferica del file di registro delle transazioni o del database predefinita.

Nota   Se le dimensioni del nuovo database di SQL Server o le dimensioni del file di registro delle transazioni non richiedono l'utilizzo di tutte le periferiche impostate come predefinite, SQL Server utilizza unicamente le periferiche necessarie.

Impostare una stampante predefinita   È necessario impostare una stampante predefinita per l'Upsize guidato in quanto questa procedura guidata crea automaticamente uno snapshot del report dell'Upsize guidato.

Assegnarsi le autorizzazioni appropriate per il database di Access   Sono necessarie le autorizzazioni di Lettura struttura per tutti gli oggetti di database per sottoporli a upsize.

Assegnarsi le autorizzazioni appropriate per il database di SQL Server

  • Per effettuare l'upsize in un database esistente, è necessario essere in possesso delle autorizzazioni CREATE TABLE e CREATE DEFAULT.

  • Per creare un nuovo database, è necessario essere in possesso dell'autorizzazione CREATE DATABASE e delle autorizzazioni SELECT per le tabelle di sistema del database master.

  • Solo gli amministratori del sistema possono creare nuove periferiche.

Aderire fin dall'inizio alle regole di denominazione di SQL Server 6.5   Se si effettua l'upsize in un database di SQL Server 6.5, l'Upsize guidato converte i nomi non validi trasformando gli spazi e i caratteri non consentiti in caratteri di sottolineatura (_). È tuttavia opportuno aderire fin dall'inizio alle regole di denominazione di SQL Server 6.5, se possibile. I nomi di SQL Server 6.5 possono contenere al massimo 30 caratteri, di cui il primo deve essere una lettera o il simbolo @. I caratteri successivi possono essere numeri, lettere, il simbolo del dollaro ($), il simbolo di cancelletto (#) o il carattere di sottolineatura (_). Gli spazi non sono consentiti.

Nota   I nomi dei database di SQL Server 7.0 sono completamente compatibili con i nomi dei database di Access.
