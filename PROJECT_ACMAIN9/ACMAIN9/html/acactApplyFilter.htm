Azione ApplicaFiltro

Azione ApplicaFiltro

        

L'azione ApplicaFiltro applica un filtro, una query o una proposizione SQL WHERE a una tabella, una maschera o un report in modo da limitare o ordinare i record contenuti nella tabella oppure i record della tabella o della query sottostante alla maschera o al report. Per i report è possibile utilizzare tale azione solo in una macro specificata dalla proprietà di evento Su apertura del report.

Nota   È possibile utilizzare l'azione ApplicaFiltro solo per applicare una proposizione SQL WHERE quando si applica un filtro su server. Non è possibile applicare un filtro su server all'origine record di una stored procedure.

Impostazione

L'azione ApplicaFiltro utilizza gli argomenti riportati di seguito.

Argomenti azione Descrizione
Nome filtro Il nome di un filtro o di una query che limita o ordina i record contenuti nella tabella, nella maschera o nel report. È possibile immettere il nome di una query esistente o di un filtro salvato come query nella casella Nome filtro della sezione Argomenti azione della finestra della macro.
  Nota   Quando si utilizza questa azione per applicare un filtro su server, la casella Nome filtro deve essere lasciata vuota.
Condizione WHERE Una proposizione SQL WHERE valida, cioè priva della parola WHERE, o un'espressione che limita i record contenuti nella tabella nella maschera o nel report.
  Nota   Nell'espressione dell'argomento Condizione WHERE la parte sinistra dell'espressione contiene in genere un nome di campo della tabella o della query sottostante alla maschera o al report. La parte destra dell'espressione contiene in genere i criteri che si desidera applicare al campo in modo da limitare o ordinare i record. È possibile ad esempio che i criteri siano costituiti dal nome di un controllo di un'altra maschera contenente il valore a cui si desidera che corrispondano i record contenuti nella prima maschera. Il nome del controllo deve essere completo, ad esempio:

Forms!nomemaschera!nomecontrollo.

I nomi di campo devono essere racchiusi tra virgolette doppie e le variabili letterali tra virgolette singole.
  La lunghezza massima dell'argomento Condizione WHERE è di 256 caratteri. Se si rende necessario immettere una proposizione SQL WHERE più lunga, utilizzare il metodo ApplyFilter dell'oggetto DoCmd in Visual Basic. È possibile immettere istruzioni proposizione SQL WHERE contenenti fino a 32.768 caratteri in Visual Basic.
Tipo filtro Specifica se la ricerca comprende dati formattati. Selezionando Normale Microsoft Access ricerca i dati nel modo in cui sono stati formattati e visualizzati nel campo, mentre selezionando Server Microsoft Access ricerca i dati nel modo in cui sono stati memorizzati nel database, che non sempre corrisponde al modo in cui sono visualizzati. L'impostazione predefinita è Normale.

Nota   È possibile utilizzare l'argomento Nome filtro se è già stato definito un filtro che fornisce i dati appropriati. È possibile utilizzare l'argomento Condizione WHERE per immettere direttamente i criteri di limitazione. Se si utilizzano entrambi gli argomenti, Microsoft Access applica la proposizione WHERE ai risultati del filtro. È necessario utilizzare uno o entrambi gli argomenti.

Osservazioni

È possibile applicare un filtro o una query a una maschera in visualizzazione Maschera o in visualizzazione Foglio dati.

Il filtro e la condizione WHERE applicati sono i valori dell'impostazione della proprietà Filtro o Filtro su server della maschera o del report.

Per tabelle e maschere l'azione ApplicaFiltro equivale a scegliere Applica filtro/ordina o Applica filtro su server dal menu Record o a fare clic sul pulsante Applica filtro o Applica filtro su server della barra degli strumenti. Il comando del menu o il pulsante applica il filtro appena creato alla tabella o alla maschera, mentre l'azione ApplicaFiltro applica un determinato filtro o una query.

Se nel database di Microsoft Access (MDB) si sceglie Filtro dal menu Record, quindi Ordinamento/filtro avanzato dopo aver eseguito l'azione ApplicaFiltro, la finestra Ordinamento/filtro avanzato visualizzerà i criteri dei filtri selezionati tramite tale azione.

Per rimuovere un filtro e visualizzare tutti i record di una tabella o di una maschera di Access, è possibile utilizzare l'azione MostraOgniRecord, il comando Rimuovi filtro/ordina del menu Record o il pulsante Rimuovi filtro della barra degli strumenti. Per rimuovere un filtro da un progetto di Microsoft Access (ADP), è possibile tornare alla finestra Filtro su server in base a maschera, rimuovere tutti i criteri del filtro e quindi scegliere Applica filtro su server dal menu Record oppure fare clic sul pulsante Applica filtro su server della barra degli strumenti o ancora impostare la proprietà Filtro su server da maschera a Falso (0).

Quando si salva una tabella o una maschera, Microsoft Access salva qualsiasi filtro correntemente definito nell'oggetto, ma non applica automaticamente il filtro alla successiva apertura dell'oggetto, sebbene applichi automaticamente qualsiasi ordinamento relativo all'oggetto prima che venisse salvato. Se si desidera applicare automaticamente un filtro la prima volta che viene aperta una maschera, specificare una macro contenente l'azione ApplicaFiltro o una routine evento contenente il metodo ApplyFilter dell'oggetto DoCmd come impostazione della proprietà di evento Su apertura della maschera. È inoltre possibile applicare un filtro utilizzando l'azione ApriMaschera o ApriReport oppure i metodi corrispondenti. Per applicare automaticamente un filtro la prima volta che viene aperta una tabella, è possibile aprire la tabella utilizzando una macro contenente l'azione ApriTabella, quindi eseguire l'azione ApplicaFiltro.
