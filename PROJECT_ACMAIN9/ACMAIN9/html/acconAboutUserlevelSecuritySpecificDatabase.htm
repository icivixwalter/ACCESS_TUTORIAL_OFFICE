Informazioni sull'impostazione della protezione a livello utente per un database di Microsoft Access

Quando gli utenti aderiscono a un gruppo di lavoro di Microsoft Access utilizzando l'Amministratore gruppi di lavoro, la posizione del file di informazioni sul gruppo di lavoro viene indicata nel registro di configurazione di Windows. Ciò significa che in molti casi tutti i database di Access sul computer dell'utente utilizzeranno la protezione a livello utente in base a quel file di informazioni sul gruppo di lavoro.

È però possibile impostare questo tipo di protezione solo su un determinato database, non su tutti. A tal fine, è necessario che gli utenti non aderiscano a un gruppo di lavoro tramite l'Amministratore gruppi di lavoro. Attenersi invece alla seguente procedura:

  1. Creare un file di informazioni sul gruppo di lavoro e memorizzarlo in un percorso accessibile a tutti gli utenti di quel database.

  2. Creare un collegamento per avviare il database utilizzando l'opzione della riga di comando /wrkgrp per specificare la posizione del file di informazioni sul gruppo di lavoro. Ad esempio:

    "C:\Programmi\Microsoft Office\Office\MSACCESS.EXE \\SalesSystem\DBFiles\SalesS.mdb /wrkgrp \\SalesSystem\DBFiles\SalesS.mdw"

    Per ulteriori informazioni sulla creazione di collegamenti, vedere la Guida di Windows.
