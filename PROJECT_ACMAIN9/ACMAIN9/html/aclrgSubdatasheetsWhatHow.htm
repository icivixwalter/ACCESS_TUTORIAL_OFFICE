Definizione di foglio dati secondario

In un foglio dati secondario è possibile visualizzare o modificare dati correlati o collegati tramite join di un foglio dati di tabella, query, maschera o sottomaschera. Ad esempio, nel database di esempio Northwind la tabella Fornitori ha una relazione uno-a-molti con la tabella Prodotti, di conseguenza per ogni riga della tabella Fornitori in visualizzazione Foglio dati è possibile visualizzare e modificare le righe correlate della tabella Prodotti di un foglio dati secondario.

Modalità di utilizzo del foglio dati secondario

Access crea automaticamente un foglio dati secondario in una tabella che presenta una relazione uno-a-uno o che si trova sul lato "uno" di una relazione uno-a-molti, quando la proprietà Nome foglio dati second. della tabella è impostata ad Auto. La relazione si definisce specificando come corrispondenti i campi chiave primaria e chiave esterna nelle tabelle correlate. Per ulteriori informazioni, vedere Definizione di relazioni.

È tuttavia possibile aggiungere un foglio dati secondario a qualsiasi tabella, query o maschera. Un foglio dati secondario può avere come oggetto origine una tabella o una query, specificata nella proprietà SubdatasheetName. Un foglio dati secondario corrispondente a un controllo sottomaschera può avere come oggetto origine una tabella, una query o una maschera; tale oggetto origine viene specificato nella proprietà SourceObject del controllo sottomaschera.

Fogli dati secondari di tabella e di query

In visualizzazione Foglio dati sia la tabella sia la query possono contenere un foglio dati secondario. Se una tabella presenta una relazione uno-a-molti con più tabelle, è possibile scegliere la tabella da aggiungere come foglio dati secondario facendo clic sull'icona indicatore di espansione (+) e selezionando una tabella nella finestra di dialogo Inserisci foglio dati secondario.

Per impostazione predefinita, Access non visualizza la chiave esterna o i campi corrispondenti nel foglio dati secondario. È possibile visualizzare temporaneamente la chiave esterna o i campi corrispondenti scegliendo Scopri colonne dal menu Formato, ma alla successiva apertura del foglio dati contenente il foglio dati secondario, le colonne saranno di nuovo nascoste. Il fatto che questi campi vengano scoperti o nascosti modifica solo l'aspetto del foglio dati secondario, ma non modifica alcuna relazione o oggetto origine dello stesso.

Utilizzando il comando Foglio dati secondario del menu Inserisci è possibile aggiungere un foglio dati secondario a una tabella o a una query in visualizzazione Foglio dati oppure modificare un foglio dati secondario. I campi corrispondenti tra il foglio dati e il foglio dati secondario devono essere specificati nelle caselle Collega campi secondari e Collega campi master della finestra di dialogo Inserisci foglio dati secondario. Queste caselle corrispondono alle proprietà LinkChildFields e LinkMasterFields della tabella o della query contenente il foglio dati secondario. Se queste caselle rimangono vuote, vengono visualizzate tutte le righe del foglio dati secondario sotto ciascuna riga del foglio dati, non solo le righe del foglio dati secondario che presentano la corrispondenza.

Nidificazione dei fogli dati secondari

È possibile nidificare un foglio dati secondario all'interno di un foglio dati secondario fino a un massimo di otto livelli. Tuttavia, ogni foglio dati o foglio dati secondario può avere un solo foglio dati secondario nidificato. Ad esempio, nel database di esempio Northwind, la tabella Clienti può contenere un solo foglio dati secondario della tabella Origini, e questo a sua volta può contenere un solo foglio dati secondario Dettagli ordini. La tabella Clienti non può però contenere un foglio dati secondario della tabella Ordini e un foglio dati secondario della query Venditori.

Maschere, sottomaschere e fogli dati secondari

Anche una maschera in visualizzazione Maschera o in visualizzazione Foglio dati può avere un foglio dati secondario, ma esistono delle differenze tra il foglio dati secondario di una maschera e quello di una tabella o di una query:

  • Il foglio dati secondario viene inserito in una maschera aggiungendo un controllo sottomaschera e definendo un oggetto origine tramite l'impostazione della proprietà SourceObject del controllo della sottomaschera a una tabella, a una query o a una maschera. Il foglio dati secondario di una maschera corrisponde a una sottomaschera. Se l'oggetto origine della sottomaschera è una maschera, è possibile impostare la proprietà Visualizzazione predefinita della stessa a Foglio dati, Visualizzazione singola o Maschere continue. Di conseguenza, quando la maschera è in visualizzazione Foglio dati, è possibile vedere il foglio dati secondario come una maschera singola o continua. Nel foglio dati secondario è possibile passare dalla visualizzazione Foglio dati alla visualizzazione Maschera e viceversa scegliendo Foglio dati sottomaschera dal menu Visualizza. Come i fogli dati secondari di tabella e di query, se le proprietà LinkChildFields e LinkMasterFields della sottomaschera non sono definite, Access visualizza tutte le righe dell'oggetto origine, non solo quelle corrispondenti.

  • Sebbene una maschera possa avere più di una sottomaschera, ognuna delle quali può essere nidificata fino a un massimo di due livelli, in visualizzazione Foglio dati di una maschera viene visualizzato un solo foglio dati secondario. Access visualizza il foglio dati secondario corrispondente alla prima sottomaschera nell'ordine della struttura a schede della maschera. Tuttavia, se una sottomaschera ha una tabella o query come oggetto origine e la proprietà Visualizzazione predefinita impostata a Foglio dati, è possibile inserire un altro foglio dati secondario in quello che corrisponde alla sottomaschera e nidificare fino a otto livelli in visualizzazione Foglio dati o in visualizzazione Maschera.

  • La prima volta che si apre una tabella, una query o una maschera in visualizzazione Foglio dati, il relativo foglio dati secondario non viene espanso; quando invece una maschera viene aperta per la prima volta in visualizzazione Maschera, il foglio dati secondario corrispondente alla sottomaschera viene espanso.

  • Se l'oggetto origine della sottomaschera è una maschera, è possibile visualizzare in modo permanente la chiave esterna o i campi corrispondenti del foglio dati secondario corrispondente alla sottomaschera.

Confronto tra fogli dati secondari in un database e in un progetto di Microsoft Access

È possibile utilizzare un foglio dati secondario di tabella o di query solo in un database di Microsoft Access e non in un progetto di Microsoft Access. Tuttavia, è possibile utilizzare un foglio dati secondario in una sottomaschera o in visualizzazione Foglio dati di maschera sia in un database di Access sia in un progetto di Access.
