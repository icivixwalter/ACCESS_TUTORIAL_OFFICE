Metodo Requery (oggetto DoCmd)

                 

Il metodo Requery dell'oggetto DoCmd esegue l'azione RieseguiQuery in Visual Basic. Per ulteriori informazioni sul funzionamento dell'azione e del relativo argomento, vedere l'argomento della Guida in linea relativo all'azione.

Sintassi

DoCmd.Requery [nomecontrollo]

Il metodo Requery utilizza l'argomento riportato di seguito.

Argomento Descrizione
nomecontrollo Espressione stringa che è il nome di un controllo dell'oggetto attivo.

Osservazioni

Utilizzare solo il nome del controllo per l'argomento nomecontrollo e non l'intera sintassi.

È inoltre possibile utilizzare una variabile dichiarata per questo argomento come tipo di dati Controllo:

Dim ctlLstBox As Control
Set ctlLstBox = Forms!Form1!Field3
DoCmd.Requery ctlLstBox.Name

Il metodo Requery dell'oggetto DoCmd è diverso dal metodo Requery di Visual Basic. Il metodo Requery dell'oggetto DoCmd è stato aggiunto per ragioni di compatibilità con le versioni precedenti, in modo da rendere possibile l'esecuzione dell'azione RieseguiQuery nel codice di Visual Basic in Microsoft Access 95. Se si desidera rieseguire la query di un controllo che non appartiene all'oggetto attivo, è necessario utilizzare il metodo Requery di Visual Basic e non l'azione RieseguiQuery o il corrispondente metodo Requery dell'oggetto DoCmd. Il metodo Requery di una maschera o di un controllo in Visual Basic è più veloce dell'azione RieseguiQuery o del metodo DoCmd.Requery. Inoltre, quando si utilizza l'azione RieseguiQuery o il metodo DoCmd.Requery, Microsoft Access chiude la query e la carica nuovamente dal database. Al contrario, quando si utilizza il metodo Requery, Microsoft Access esegue nuovamente la query senza chiuderla e caricarla nuovamente.

Nota   Per eseguire nuovamente la query, è possibile utilizzare anche il metodo Requery per gli oggetti ADO.
