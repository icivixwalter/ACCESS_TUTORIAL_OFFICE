Azione EseguiComando

Azione EseguiComando

        

È possibile utilizzare l'azione EseguiComando per eseguire un comando incorporato di Microsoft Access. Il comando può essere visualizzato su una barra dei menu, una barra degli strumenti o un menu di scelta rapida di Microsoft Access.

Impostazione

L'azione EseguiComando utilizza l'argomento azione riportato di seguito.

Argomento azione Descrizione
Comando Nome del comando che si desidera eseguire. La casella Comando visualizza in ordine alfabetico i comandi incorporati di Microsoft Access disponibili. Si tratta di un argomento obbligatorio.

Osservazioni

In Microsoft Access 97 l'azione EseguiComando ha sostituito l'azione EseguiVoceMenu. Quando si apre e si salva una macro di una versione precedente di Microsoft Access contenente un'azione EseguiVoceMenu, l'azione e i relativi argomenti azione vengono convertiti automaticamente nell'equivalente azione EseguiComando. L'azione EseguiVoceMenu non viene più visualizzata nell'elenco di azioni della finestra della macro.

È possibile utilizzare l'azione EseguiComando per eseguire un comando di Microsoft Access da una barra dei menu personalizzata, una barra dei menu globale, un menu di scelta rapida personalizzato o un menu di scelta rapida globale. È però più semplice utilizzare la finestra di dialogo Personalizza, disponibile scegliendo Barre degli strumenti dal menu Visualizza e poi Personalizza, per aggiungere i comandi di Microsoft Access ai menu di scelta rapida e ai menu personalizzati. La scheda Comandi nella finestra di dialogo Personalizza elenca i comandi incorporati che vengono visualizzati sulle barre degli strumenti e sui menu di Microsoft Access.

È possibile utilizzare l'azione EseguiComando in una macro con espressioni condizionali per eseguire un comando in base ad alcune condizioni.

Nota   Il menu File di Microsoft Access 2000 visualizza i database utilizzati più di recente. È possibile scegliere uno di questi database invece di scegliere Apri database. Le voci di questi database non vengono visualizzate nella casella di riepilogo a discesa relativa all'argomento Comando e non sono disponibili utilizzando l'azione EseguiComando in una macro.

Quando si converte un database di una versione precedente di Microsoft Access, è possibile che alcuni comandi non siano più disponibili. È possibile che un comando sia stato rinominato, spostato in un menu diverso oppure che non sia più disponibile in Microsoft Access 2000. Le azioni EseguiVoceMenu relative a tali comandi non possono essere convertite in azioni EseguiComando. Quando si apre la macro, viene visualizzata un'azione EseguiComando con l'argomento Comando vuoto per tali comandi. È necessario modificare la macro e immettere un argomento comando valido oppure eliminare l'azione EseguiComando.

Per eseguire l'azione EseguiComando in Visual Basic, utilizzare il metodo RunCommand dell'oggetto Application, che equivale al metodo RunCommand dell'oggetto DoCmd.
