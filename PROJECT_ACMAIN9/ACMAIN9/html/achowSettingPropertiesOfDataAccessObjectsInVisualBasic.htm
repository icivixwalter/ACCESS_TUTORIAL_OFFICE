Impostare le proprietà di oggetti DAO in Visual Basic

Gli oggetti DAO (Data Access Object) consentono di manipolare la struttura del database e i dati in esso contenuti da Visual Basic. Molti oggetti DAO corrispondono a oggetti presenti nel database personale, ad esempio un oggetto TableDef corrisponde a una tabella di Microsoft Access e un oggetto Field corrisponde a un campo di una tabella.

La maggior parte delle proprietà che è possibile impostare per gli oggetti DAO è costituita da proprietà DAO. Tali proprietà sono definite dal modulo di gestione di database Microsoft Jet e sono impostate nello stesso modo in tutte le applicazioni che includono tale modulo. Alcune proprietà che è possibile impostare per gli oggetti DAO sono definite da Microsoft Access e non vengono riconosciute automaticamente dal modulo di gestione di database Jet. Le proprietà per gli oggetti DAO vengono impostate in modo diverso a seconda che una proprietà sia definita dal modulo di gestione di database Jet o da Microsoft Access.

Impostare le proprietà DAO per gli oggetti DAO

Per impostare una proprietà definita dal modulo di gestione di database Jet, fare riferimento all'oggetto nella gerarchia degli oggetti DAO. Il modo più semplice e veloce per eseguire questa operazione consiste nel creare variabili di oggetto che rappresentino i diversi oggetti che si desidera utilizzare e, successivamente, nel fare riferimento a tali variabili nel codice. Il codice riportato di seguito ad esempio indica come creare un nuovo oggetto TableDef e impostare la relativa proprietà Name.

Dim dbs As Database, tdf As TableDef
Set dbs = CurrentDb
Set tdf = dbs.CreateTableDef
tdf.Name = "Contatti"

Impostare le proprietà di Microsoft Access per oggetti DAO

Quando si imposta una proprietà definita da Microsoft Access ma che viene applicata a un oggetto DAO, il modulo di gestione di database Jet non riconosce automaticamente come valida la proprietà. La prima volta che si imposta la proprietà è necessario crearla e accodarla all'insieme Properties dell'oggetto a cui viene applicata. Quando la proprietà è compresa nell'insieme Properties può essere impostata nello stesso modo di qualsiasi proprietà DAO.

Se la proprietà viene impostata per la prima volta nell'interfaccia utente, viene aggiunta automaticamente all'insieme Properties ed è possibile impostarla normalmente.

Durante la scrittura delle routine per impostare le proprietà definite da Microsoft Access, è necessario includere il codice di gestione degli errori per verificare che la proprietà da impostare esista già nell'insieme Properties. Per ulteriori informazioni, consultare l'argomento della Guida in linea relativo al metodo CreateProperty o alla singola proprietà.

È necessario tenere presente che quando si crea una proprietà occorre specificare in modo corretto la relativa proprietà Type prima di accodarla all'insieme Properties. È possibile determinare la proprietà Type in base alle informazioni contenute nella sezione Impostazioni dell'argomento della Guida in linea relativo alla singola proprietà. La tabella riportata di seguito fornisce alcune indicazioni per determinare l'impostazione della proprietà Type.
