Metodo TransferSpreadsheet

                 

Il metodo TransferSpreadsheet esegue l'azione TrasferisciFoglioCalcolo in Visual Basic. Per ulteriori informazioni sul funzionamento dell'azione e dei relativi argomenti, vedere l'argomento della Guida in linea relativo all'azione.

Sintassi

DoCmd.TransferSpreadsheet [tipotrasferimento][, tipofogliocalcolo], nometabella, nomefile[, nomicampo][, intervallo]

Il metodo TransferSpreadsheet utilizza gli argomenti riportati di seguito

Argomento Descrizione
tipotrasferimento Una delle seguenti costanti intrinseche:
  acImport (impostazione predefinita)
acExport
acLink
  Se si omette questo argomento, viene utilizzata la costante predefinita (acImport).
tipofogliocalcolo Una delle seguenti costanti intrinseche o l'impostazione numerica equivalente:
  0 acSpreadsheetTypeExcel3 (impostazione predefinita)
6 acSpreadsheetTypeExcel4
5 acSpreadsheetTypeExcel5
5 acSpreadsheetTypeExcel7
8 acSpreadsheetTypeExcel8
8 acSpreadsheetTypeExcel9
2 acSpreadsheetTypeLotusWK1
3 acSpreadsheetTypeLotusWK3
7 acSpreadsheetTypeLotusWK4
4 acSpreadsheetTypeLotusWJ2 — solo per la versione giapponese
  Nota   È possibile collegarsi a dati contenuti in file di fogli di calcolo di Lotus 1-2-3 ma tali dati risultano di sola lettura in Microsoft Access. È possibile importare e collegare in sola lettura dati di file di Lotus con estensione WK4 ma non è possibile esportare i dati di Microsoft Access nel formato di quel foglio di calcolo. Inoltre, utilizzando questo metodo, Microsoft Access non supporta più l'importazione, l'esportazione o il collegamento di dati da fogli di calcolo di Lotus.WKS o Microsoft Excel versione 2.0.
  Se si omette questo argomento viene utilizzata la costante predefinita (acSpreadsheetTypeExcel8).
nometabella Espressione stringa che è il nome della tabella di Microsoft Access nella quale si desidera importare, dalla quale si desidera esportare o alla quale si desidera collegare dati di foglio di calcolo oppure della query di selezione di Microsoft Access i cui risultati si desidera esportare in un foglio di calcolo.
nomefile Espressione stringa che è il nome di file e il percorso del foglio di calcolo dal quale si desidera importare, nel quale si desidera esportare o al quale si desidera collegare dati.
nomicampo Utilizzare True (-1) per utilizzare la prima riga del foglio di calcolo come nomi di campo nell'importazione o nel collegamento. Utilizzare False (0) se si desidera che i dati della prima riga del foglio di calcolo siano considerati come dati normali. Se si omette questo argomento, viene utilizzata l'impostazione predefinita (False).
  Quando si esportano in un foglio di calcolo i dati di una tabella o di una query di selezione di Microsoft Access, i nomi dei campi vengono inseriti nella prima riga del foglio di calcolo, indipendentemente dal valore attribuito a questo argomento.
intervallo Espressione stringa che è un intervallo valido di celle o il nome di un intervallo del foglio di calcolo. Questo argomento si applica solo all'importazione. Omettere questo argomento per importare l'intero foglio di calcolo.
  Quando si esporta in un foglio di calcolo è necessario omettere questo argomento. Se si immette un intervallo l'importazione non viene eseguita.

Osservazioni

È possibile omettere un argomento facoltativo all'interno della sintassi, ma è necessario includere la virgola dell'argomento stesso. Se si omette un argomento finale, non utilizzare la virgola dopo l'ultimo argomento specificato.

Nota   È anche possibile utilizzare gli oggetti ADO per creare un collegamento tramite la proprietà ActiveConnection per l'oggetto Recordset.
