Note

  • Nei progetti di Access, il prefisso "n" significa "nazionale" e indica che il tipo di dati è abilitato per Unicode. Nei database di Access, tutte le colonne di testo sono abilitate per Unicode per impostazione predefinita.

  • L'equivalente in un progetto di Access del tipo di dati Contatore dei database di Access è il tipo di dati int, definito dalla proprietà Identity. Questo tipo di dati contiene valori sequenziali generati dal sistema che identificano in modo univoco ogni riga di una tabella (ad esempio gli identificativi dei dipendenti). Quando si inseriscono i valori in una tabella con una colonna Identity, il valore del campo successivo viene generato in modo automatico in base all'ultimo valore di Identity utilizzato (impostato nella proprietà Identity Seed) e al valore incrementale (impostato nella proprietà Identity Increment) specificato quando la colonna è stata creata. Per fare in modo che questo tipo di dati funzioni come il tipo Contatore, impostare a 1 il valore delle proprietà Identity Seed e Identity Increment.

  • Nel tipo di dati Sì/No di un database di Microsoft Access viene memorizzato il valore 0 o -1. 0 verrà interpretato come 0, No o Falso, mentre qualsiasi altro valore verrà interpretato come –1, Sì o Vero. Al contrario, in SQL Server nel tipo di dati bit viene memorizzato il valore 0 o +1. Nei progetti di Access 0 viene interpretato come Falso, mentre +1 come Vero.

  • Anche se il tipo di dati ntext non memorizza i collegamenti ipertestuali attivi, è possibile rendere attivo un collegamento ipertestuale all'interno di una maschera di un progetto di Access associando un controllo a qualsiasi campo di testo e impostando a Vero la proprietà IsHyperlink.
