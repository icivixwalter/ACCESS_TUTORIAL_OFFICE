Funzioni First, Last

In Microsoft Access è possibile utilizzare le funzioni First e Last nella griglia di struttura della query, in un'istruzione SQL in visualizzazione SQL della finestra della query o in un'istruzione SQL all'interno del codice di Visual Basic. È anche possibile utilizzare le funzioni First e Last in un controllo calcolato di una maschera o di un report.

Le funzioni First e Last risultano particolarmente utili nei controlli calcolati su un report. Se ad esempio si dispone di un report Ordini raggruppato secondo il campo PaeseDestinatario e ordinato in base al campo DataOrdine, è possibile utilizzare le funzioni First e Last nei controlli calcolati per visualizzare l'intervallo delle date relative a ordini a partire dal primo fino al più recente per ciascun raggruppamento. Per effettuare un raggruppamento in base al campo PaeseDestinatario, fare clic sul pulsante Ordinamento e raggruppamento sulla barra degli strumenti Struttura report. Scegliere PaeseDestinatario nella colonna Campo/Espressione e impostare a Sì le proprietà Intestazione (gruppo) e Piè di pagina (gruppo). In visualizzazione Struttura, creare due nuove caselle di testo nel piè di pagina PaeseDestinatario e impostare la proprietà Origine controllo per le due espressioni riportate di seguito:

=First([DataOrdine])
=Last([DataOrdine])

Quando si passa all'anteprima di stampa, verranno visualizzate le date del primo ordine e dell'ordine più recente per ciascun gruppo di ordini.

Note   Se si desidera la restituzione del primo o dell'ultimo record di un set di record in Access 2000, creare una query disposta sia in ordine crescente che decrescente e impostare a 1 la proprietà Primi valori. Per ulteriori informazioni, vedere l'argomento relativo alla proprietà TopValues. Da Visual Basic, è inoltre possibile creare un set di risultati ordinati per la query e utilizzare il metodo DAO MoveFirst o MoveLast per la restituzione del primo o dell'ultimo record di un set di record.

Nei progetti di Access (adp), le funzioni First e Last presuppongono la mancanza di ordinamento delle informazioni di origine. La funzione restituirà quindi la prima voce corrispondente ai criteri elencati nella funzione che potrebbe non coincidere con la prima o con l'ultima voce nelle informazioni di origine.
