Rendere disponibili agli utenti un'applicazione con protezione all'accesso e un gruppo di lavoro

Un'applicazione sicura dipende dagli utenti che hanno accesso a un database di applicazione in cui sono memorizzate le autorizzazioni e a un gruppo di lavoro in cui sono memorizzati gli account utente e gruppo e le password. A seconda della modalità di utilizzazione dell'applicazione, è possibile rendere disponibili agli utenti l'applicazione e il gruppo di lavoro in vari modi.

Copiare in un server di rete i file di applicazione e il file di informazioni che definisce il gruppo di lavoro

Utilizzando l'Amministratore gruppi di lavoro, gli utenti possono entrare nel gruppo di lavoro specificando il percorso del server di rete nel file di informazioni sul gruppo di lavoro, quindi eseguire l'applicazione direttamente dal server. È possibile impedire agli utenti l'apertura dell'applicazione con accesso esclusivo rimuovendo dal gruppo Utenti le autorizzazioni Apertura esclusiva per il database.

Fornire a ciascun utente una copia dei file applicativi e del file di informazioni che definisce il gruppo di lavoro

Utilizzando l'Amministratore gruppi di lavoro, gli utenti possono entrare nel gruppo di lavoro specificando il percorso del relativo file disponibile sul computer, quindi eseguire le copie dell'applicazione. Se tuttavia è necessario aggiornare l'applicazione, i relativi dati o il file di informazioni sul gruppo di lavoro, sarà necessario fornire a ciascun utente delle copie di tutti i file aggiornati.

In alternativa, può essere necessario dividere l'applicazione in un database di dati collocato su un server di rete e in un database di applicazione che contiene tutti gli altri oggetti di database presenti sul computer di ciascun utente. Con questo tipo di configurazione in genere si desidera mantenere sul server anche il file di informazioni sul gruppo di lavoro. Se quindi è necessario aggiornare l'applicazione, è sufficiente fornire agli utenti delle versioni aggiornate del database di applicazione. Per ulteriori informazioni su come eseguire questa operazione, fare clic su .

Fornire agli utenti di diversi gruppi di lavoro di Microsoft Access l'accesso all'applicazione creando lo stesso gruppo in ciascun gruppo di lavoro

È possibile creare un account gruppo in ciascun gruppo di lavoro con un nome e un ID personale (PID) identici, quindi aggiungere a quel gruppo gli utenti di ciascun gruppo di lavoro. Un utente che lavora in rete in un gruppo di lavoro di Microsoft Access a Honolulu può utilizzare ad esempio un'applicazione creata per un gruppo di lavoro di Microsoft Access a Melbourne se è membro di un account gruppo creato con un nome e un PID identici su entrambi i gruppi di lavoro.

Per eseguire l'applicazione, ciascun utente deve disporre di una copia di Microsoft Access. Per creare un'applicazione di Microsoft Access che non richieda una copia di questo programma è necessario utilizzare Microsoft Office 2000 Developer (MOD).
