Oggetto Document, insieme Documents

Oltre agli oggetti Document definiti dal modulo di gestione di database Microsoft Jet, vengono definiti da Microsoft Access gli oggetti Document riportati di seguito.

Oggetto Document Oggetto Container Tipo di informazioni
Form Forms Maschera salvata
Macro Scripts Macro salvata
Module Modules Modulo salvato
Report Reports Report salvato
DataAccessPage DataAccessPages PaginaDiAccessoAiDati salvata
SummaryInfo Databases Riepilogo del documento database
UserDefined Databases Proprietà definite dall'utente

Gli oggetti Document definiti da Microsoft Access sono oggetti di database di Microsoft Access e non oggetti DAO. Tali oggetti Document forniscono al modulo di gestione di database Microsoft Jet informazioni sugli oggetti di Microsoft Access. Il modulo di gestione utilizza tali informazioni per implementare la protezione per gli oggetti di database nello stesso modo in cui viene implementata per gli oggetti DAO.

L'insieme Documents di un oggetto Container contiene oggetti Document che rappresentano i singoli oggetti del tipo descritto dall'oggetto Container. L'oggetto Container Forms, ad esempio, dispone di un insieme Documents che potrebbe comprendere un oggetto Document corrispondente a una maschera chiamata Ordini.

Microsoft Access definisce due oggetti Document nell'oggetto Container Databases: SummaryInfo e UserDefined.

L'oggetto Document SummaryInfo consente l'accesso a livello di programmazione alle proprietà di riepilogo del documento, comprese le proprietà Title, Subject, Author, Keywords, Comments, Manager, Company, Category e Hyperlink Base. È anche possibile impostare tali proprietà nella scheda Riepilogo della finestra di dialogo Proprietà database, disponibile scegliendo Proprietà database dal menu File.

Per impostare tali proprietà in Visual Basic, è necessario crearle e accodarle all'insieme Properties dell'oggetto Document SummaryInfo, se non sono già state impostate nella finestra di dialogo Proprietà database. Una volta creata una proprietà, è necessario fare esplicito riferimento all'insieme Properties per impostarla. Nell'esempio riportato di seguito, doc è una variabile di oggetto che punta all'oggetto Document SummaryInfo:

doc.Properties!Title = "Commercianti Northwind"

Per ulteriori informazioni sulla creazione e l'impostazione di proprietà definite da Microsoft Access, ricercare "Oggetto Property" e "Metodo CreateProperty" nell'indice della Guida in linea.

L'oggetto Document UserDefined consente l'accesso a livello di programmazione alle proprietà definite dall'utente nella scheda Personalizza della finestra di dialogo Proprietà database. È anche possibile creare tali proprietà in Visual Basic e accodarle all'insieme Properties dell'oggetto Document UserDefined.

Nota   Non confondere le proprietà definite nell'insieme Properties di un documento Database con le proprietà definite nell'insieme Properties di un oggetto Database. È possibile creare proprietà definite dall'utente in entrambi gli insiemi Properties, ma è possibile impostare dalla finestra di dialogo Proprietà database solo le proprietà definite negli oggetti Document SummaryInfo o UserDefined.

È possibile utilizzare gli oggetti Document per stabilire e applicare autorizzazioni per singoli oggetti di database di Microsoft Access. Per impostare le autorizzazioni per un oggetto Document, impostare la proprietà UserName dell'oggetto Document al nome di un oggetto User o Group esistente. Impostare quindi la proprietà Permissions per l'oggetto Document.

Nota   Non confondere i tipi specifici di oggetti Container con i tipi di insieme che hanno lo stesso nome. Gli oggetti Container Forms e Reports contengono ciascuno insiemi Documents. Tali insiemi comprendono singoli oggetti Document che rappresentano ogni maschera o report salvato. Gli insiemi Forms e Reports fanno riferimento solo a maschere o report aperti.
