Usare ODBCDirect con Microsoft Access

Definizione di ODBCDirect

ODBCDirect è una tecnologia che consente di utilizzare i server di database ODBC senza caricare il modulo di gestione di database Jet di Microsoft. ODBCDirect si basa sul modello di oggetti Microsoft DAO 3.6 per modificare con semplicità il codice DAO esistente e godere dei vantaggi offerti da ODBCDirect. Microsoft DAO 3.6 include nuovi oggetti, metodi e proprietà per supportare ODBCDirect.

Vantaggi dell'accesso ai dati ODBC con ODBCDirect

ODBCDirect offre i seguenti vantaggi per le operazioni ODBC:

  • Con ODBCDirect è possibile eseguire il codice in maniera più veloce ed efficiente fornendo l'accesso diretto alle origini dati ODBC. Non dovendo caricare il modulo di gestione di database Microsoft Jet, ODBCDirect richiede al client una quantità limitata di risorse. Il server ODBC si occupa dell'intera elaborazione delle query.

  • ODBCDirect fornisce un accesso migliore a funzionalità specifiche del server che non sono disponibili utilizzando ODBC mediante Microsoft Jet. Per i server ad esempio che supportano la specifica del cursore, ODBCDirect consente di specificare la posizione del cursore, sia a livello locale che di server. Per interagire con le routine memorizzate a livello di server, inoltre, è possibile specificare i valori di input e verificare i valori restituiti, procedura che non è possibile eseguire con Microsoft Jet.

  • ODBCDirect inoltre supporta le query asincrone. Quando si esegue una query, non è necessario attenderne il termine dell'esecuzione prima di iniziare un'altra operazione. È possibile registrare l'esecuzione della query verificando la proprietà StillExecuting.

  • ODBCDirect supporta l'aggiornamento batch, consentendo di memorizzare nella cache le modifiche apportate all'oggetto Recordset a livello locale e di sottomettere tali modifiche al server in un singolo batch.

  • Con ODBCDirect è possibile creare set di risultati semplici e senza cursore o più complessi e con cursore. È inoltre possibile eseguire query che restituiscono qualsiasi numero di set di risultati, limitare il numero di righe restituite e monitorare tutti i messaggi e gli errori generati dall'origine dati remota, senza compromettere la prestazione dell'esecuzione della query.

Creazione di un'area di lavoro ODBCDirect

È possibile creare un'area di lavoro di ODBCDirect specificando la costante dbUseODBC per l'argomento tipo del metodo CreateWorkspace. È inoltre possibile specificare che venga creata un'area di lavoro ODBCDirect predefinita impostando la proprietà DefaultType dell'oggetto DBEngine.

Una volta creata un'area di lavoro ODBCDirect, è possibile utilizzare specifici oggetti DAO, proprietà e metodi per lavorare con i dati sul server del database ODBC.

Vantaggi dell'accesso ai dati ODBC con Microsoft Jet

Le aree di lavoro di Microsoft Jet e di ODBCDirect forniscono funzionalità diverse ma complementari. È consigliabile utilizzare le aree di lavoro di Microsoft Jet per accedere ai file MDB e ai formati di dati ISAM, ad esempio testo e fogli di calcolo. Microsoft Jet fornisce funzionalità uniche, non disponibili in ODBCDirect. È necessario utilizzare l'area di lavoro di Microsoft Jet nei seguenti casi:

  • Join aggiornabili. Per aggiornare i dati degli oggetti Recordset basati su join di più tabelle.

  • Join eterogenei. Per eseguire join di tabelle con diverse origini dati.

  • Operazioni DDL (Data Definition Language ). Per eseguire operazioni DDL utilizzando gli oggetti DAO. ODBCDirect non fornisce un oggetto TableDef e di conseguenza non è possibile creare o modificare tabelle utilizzando gli oggetti DAO. È tuttavia possibile eseguire operazioni DDL utilizzando ODBCDirect per eseguire le istruzioni SQL DDL.

  • Maschere e controlli associati. Per associare le maschere o i controlli a un'origine dati OBDC, se l'applicazione lo richiede. I dati a cui si accede all'interno di un'area di lavoro di ODBCDirect non possono essere associati alle maschere o ai controlli.

Se non si necessita delle funzionalità descritte, è possibile utilizzare un'area di lavoro ODBCDirect.

Nota   È possibile definire le aree di lavoro di Microsoft Jet e di ODBCDirect nelle applicazioni e quindi unirle in qualsiasi modo. Nella stessa funzione è possibile ad esempio definire un'area di lavoro di Microsoft Jet che esegue operazioni DDL utilizzando oggetti DAO o un'area di lavoro di ODBCDirect che esegue query asincrone.
