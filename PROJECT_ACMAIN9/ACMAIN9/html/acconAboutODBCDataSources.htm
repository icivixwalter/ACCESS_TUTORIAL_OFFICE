Definizione di origine dati

Un'origine dati è costituita dalla fonte dei dati e dalle informazioni sulla connessione necessarie per accedere a tali dati. Esempi di origini dati sono Microsoft Access, Microsoft SQL Server, Oracle RDBMS, un foglio di calcolo e un file di testo. Esempi di informazioni di connessione sono il percorso del server, il nome del database, l'ID di accesso, la password e varie opzioni di driver ODBC che definiscono la modalità di connessione all'origine dati.

Nell'architettura ODBC un'applicazione, quale Microsoft Access o un programma di Microsoft Visual Basic, si connette al Driver Manager ODBC che a sua volta utilizza un particolare driver ODBC, ad esempio il driver Microsoft SQL ODBC, per connettersi a un'origine dati, in questo caso un database di SQL Server. In Microsoft Access le origini dati ODBC vengono utilizzate per connettersi a dati esterni a Microsoft Access che non dispongono di driver incorporati.

Per connettersi a queste origini dati, attenersi alla seguente procedura:

  • Installare il driver ODBC appropriato nel computer contenente l'origine dati.

  • Definire un nome origine dati (DSN) utilizzando l'Amministratore origine dati ODBC per memorizzare le informazioni di connessione nel registro di configurazione di Windows o in un file DSN oppure una stringa di connessione in codice Visual Basic per passare le informazioni di connessione direttamente al Driver Manager ODBC.

Origini dati su macchina

Le origini dati su macchina memorizzano le informazioni di connessione nel registro di configurazione di Windows di un determinato computer con un nome definito dall'utente. È possibile utilizzare le origini dati su macchina soltanto sul computer sul quale sono state definite. Le origini dati su macchina sono di due tipi: utente e di sistema. Le origini dati utente possono essere utilizzate soltanto dall'utente corrente e sono visibili soltanto a quell'utente. Le origini dati di sistema possono essere utilizzate da tutti gli utenti di un computer e sono visibili a tutti gli utenti del computer e dei servizi a livello di sistema, quali i servizi di Microsoft Windows NT. Un'origine dati su macchina risulta particolarmente utile per elevare il livello di protezione poiché solo gli utenti connessi possono vederla e non può essere copiata da un utente remoto su un altro computer.

Origini dati su file

Le origini dati su file, chiamate anche DSN su file o file DSN, memorizzano le informazioni di connessione in un file di testo, non nel registro di configurazione di Windows, e sono generalmente più flessibili da utilizzare rispetto alle origini dati su macchina. È possibile ad esempio copiare un'origine dati su file su qualsiasi computer con il driver ODBC appropriato in modo che l'applicazione possa usufruire di informazioni di connessione precise e coerenti su tutti i computer che utilizza. In alternativa, è possibile collocare l'origine dati su file su un singolo server e condividerla con più computer in rete e gestire facilmente le informazioni di connessione da un'unica postazione.

Un'origine dati su file può anche non essere condivisibile. In tal caso, l'origine dati risiede su un singolo computer e punta a un'origine dati su macchina. È possibile utilizzare origini dati su file non condivisibili per accedere a origini dati su macchina esistenti da origini dati su file.

Stringhe di connessione

I programmatori possono definire una stringa di connessione formattata in codice Visual Basic che specifichi le informazioni sulla connessione. Utilizzando una stringa di connessione si evita di definire un'origine dati su macchina o su file e le informazioni sulla connessione vengono passate direttamente al Driver Manager ODBC. Ciò è particolarmente utile ad esempio se si desidera evitare che gli amministratori del sistema o gli utenti prima debbano creare un DSN oppure per semplificare l'installazione di un'applicazione.

Per ulteriori informazioni sulla programmazione ODBC, consultare ODBC 3.0 Programmer's Reference Manual.
