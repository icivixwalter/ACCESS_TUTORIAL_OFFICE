Impostare le proprietà delle pagine di accesso ai dati e dei controlli

Ciascuna pagina di accesso ai dati, sezione e relativo controllo dispongono di impostazioni delle proprietà che possono essere cambiate per modificare l'aspetto o la funzionalità di quel particolare elemento. Le proprietà possono essere visualizzate e cambiate utilizzando la finestra delle proprietà oppure Microsoft Visual Script Editor.

Per impostare le proprietà

  1. In visualizzazione Struttura selezionare la pagina di accesso ai dati, la sezione o il controllo di cui si desidera impostare la proprietà. È possibile selezionare:
    • un controllo, facendo clic sul controllo da selezionare

    • una sezione, facendo clic sul selettore della sezione che si desidera selezionare

    • l'intera pagina di accesso ai dati, facendo clic sulla barra del titolo della pagina di accesso ai dati
  2. Visualizzare la finestra delle proprietà selezionando l'oggetto e facendo clic sul pulsante Proprietà sulla barra degli strumenti. Nel caso di sezioni e controlli, è possibile fare clic con il pulsante destro del mouse e scegliere Proprietà dal menu di scelta rapida.

  3. Fare clic sulla proprietà di cui si desidera impostare il valore, quindi effettuare una delle seguenti operazioni:
    • nella casella delle proprietà digitare le impostazioni o l'espressione appropriata

    • se nella casella delle proprietà è contenuta una freccia, fare clic sulla freccia, quindi selezionare un valore dall'elenco

Se la finestra delle proprietà risulta vuota o non contiene le proprietà desiderate, vedere Risoluzione dei problemi relativi alla finestra delle proprietà.

Suggerimenti

  • In Microsoft Access è disponibile una finestra Zoom per immettere e visualizzare espressioni o altre impostazioni delle proprietà lunghe. Per visualizzare la finestra Zoom, fare clic su una casella delle proprietà all'interno della relativa finestra, quindi premere MAIUSC+F2 oppure fare clic con il pulsante destro del mouse e scegliere Zoom dal menu di scelta rapida.

  • Le impostazioni delle proprietà di un controllo associato a volte non corrispondono alle relative impostazioni del campo nella tabella, query o visualizzazione sottostante a cui è associato il controllo. Se le impostazioni sono diverse, in genere le impostazioni della maschera o del report hanno la priorità su quelle della tabella, della query o della visualizzazione. Per ulteriori informazioni su come vengono ereditate le proprietà, vedere le modalità di relazione tra le proprietà dei controlli e le proprietà dei campi sottostanti.
