Import, export, and link right-to-left tables in text formats

Informazioni specifiche sulla lingua

To enable the right-to-left features in Microsoft Access 2000, you must be running a 32-bit Microsoft Windows operating system that has right-to-left support — for example, the Arabic version of Microsoft Windows 95. Learn about system or special requirements for right-to-left languages.

Microsoft Access 2000 provides code page conversion of tables and worksheets with right-to-left characteristics for text files stored in logical text layout, with one of the following right-to-left code pages:

Language Code page
Arabic UTF-8 (default)
  ISO 8859-6
  DOS-720
  ASMO-708
Hebrew UTF-8 (default)
  ISO 8859-8
  DOS-862

To import, export, or link text files that are not recognized by Microsoft Access 2000, use ConvText, a stand-alone text-file conversion utility that provides both logical and visual text layout options and an expanded list of supported bidirectional code pages.

Note   A table/worksheet in Lotus, dBASE, Microsoft FoxPro, Paradox, or ODBC database format that requires code page and/or text layout conversion can still be imported into Access 2000 by saving the table/worksheet in text file format prior to importing it. The ability to save a table/worksheet in text file format is a standard feature of most database/spreadsheet products. When Access 2000 imports the text file, all field names and records from the original table/worksheet will be fully recreated in a new Access table.
