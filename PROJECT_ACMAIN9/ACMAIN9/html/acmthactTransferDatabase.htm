Metodo TransferDatabase

                 

Il metodo TransferDatabase esegue l'azione TrasferisciDatabase in Visual Basic. Per ulteriori informazioni sul funzionamento dell'azione e dei relativi argomenti, vedere l'argomento della Guida in linea relativo all'azione.

Sintassi

DoCmd.TransferDatabase [tipotrasferimento], tipodatabase, nomedatabase[, tipooggetto], origine, destinazione[, solostruttura][, salvaidconnessione]

Il metodo TransferDatabase utilizza gli argomenti riportati di seguito.

Argomento Descrizione
tipotrasferimento Una delle seguenti costanti intrinseche:
  acExport
acImport (impostazione predefinita)
acLink
  Se si omette questo argomento, viene utilizzata la costante predefinita (acImport).
  Nota   Il tipo di trasferimento acLink non è supportato in progetti di Microsoft Access (adp).
tipodatabase Una stringa espressione, che è il nome di uno dei tipi di database utilizzati per importare, esportare o collegare dati.

Microsoft Access (impostazione predefinita)
Jet 2.x
Jet 3.x
dBase III
dBbase IV
dBase 5
Paradox 3.x
Paradox 4.x
Paradox 5.x
Paradox 7.x
Database ODBC
  Nella finestra delle macro è possibile visualizzare i tipi di database contenuti nell'elenco dell'argomento dell'azione Tipo Database dell'azione TrasferisciDatabase.
nomedatabase Espressione stringa che è il nome completo, compreso il percorso, del database che si desidera utilizzare per importare, esportare o collegare dati.
tipooggetto Una delle seguenti costanti intrinseche:
  acTable (impostazione predefinita)
acQuery
acForm
acReport
acMacro
acModule
acDataAccessPage
acServerView
acDiagram
acStoredProcedure
  Rappresenta il tipo di oggetto del quale si desidera importare, esportare o collegare dati. È possibile specificare un oggetto diverso da acTable solo se si stanno importando o esportando dati tra due database di Microsoft Access. Se si stanno esportando i risultati di una query di selezione di Microsoft Access ad un altro tipo di database, specificare acTable per questo argomento.
  Se si omette questo argomento, l'impostazione predefinita è (acTable).
  Nota   La costante acDefault, che compare nell'elenco Elenco automatico membri relativo a questo argomento, non è valida. È necessario scegliere una delle costanti riportate sopra.
origine Espressione stringa che è il nome dell'oggetto del quale si desidera importare, esportare o collegare dati.
destinazione Espressione stringa che è il nome dell'oggetto importato, esportato o collegato nel database di destinazione.
solostruttura Utilizzare True (–1) per importare o esportare solo la struttura di una tabella di database. Utilizzare False (0) per importare o esportare la struttura della tabella ed i suoi dati.
  Se si omette questo argomento, viene utilizzata l'impostazione predefinita (False).
salvaidconnessione Utilizzare True per memorizzare l'identificatore (ID) e la password di connessione per un database ODBC nella stringa di connessione di una tabella collegata dal database. Se si esegue questa operazione, non è necessario connettersi ogni volta che si apre la tabella. Utilizzare False se non si desidera memorizzare l'ID e la password.
  Se si omette questo argomento, viene utilizzata l'impostazione predefinita (False).
  Questo argomento è disponibile solo in Visual Basic.

Osservazioni

È possibile omettere un argomento facoltativo all'interno della sintassi, ma è necessario includere la virgola dell'argomento stesso. Se si omette un argomento finale, non utilizzare la virgola dopo l'ultimo argomento specificato.

L'amministratore di un database ODBC può disattivare la funzione fornita dall'argomento salvaidconnessione, richiedendo a tutti gli utenti di immettere l'ID e la password ogni volta che si collegano al database ODBC.

Nota   È inoltre possibile utilizzare gli oggetti ADO per creare un collegamento utilizzando la proprietà ActiveConnection per l'oggetto Recordset.
