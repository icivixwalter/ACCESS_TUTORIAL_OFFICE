Proprietà AutoActivate (Attivazione automatica)

                 

È possibile utilizzare la proprietà AutoActivate per specificare come è possibile attivare un oggetto OLE.

Impostazione

La proprietà AutoActivate viene impostata come riportato di seguito.

Impostazione Costante Descrizione
Manuale acOLEActivateManual (0) L'oggetto OLE non è attivato quando riceve lo stato attivo o quando si fa doppio clic sul controllo. È possibile attivare un oggetto OLE solo tramite Visual Basic, impostando a acOLEActivate la proprietà Action del controllo.
Attivato acOLEActivateGetFocus (1) Solo per i controlli cornice oggetto non associato e grafico. Se il controllo contiene un oggetto OLE, viene attivata l'applicazione che fornisce l'oggetto quando il controllo riceve lo stato attivo.
Doppio clic acOLEActivateDoubleClick (2) Impostazione predefinita. Se il controllo contiene un oggetto OLE, viene attivata l'applicazione che fornisce l'oggetto quando si fa doppio clic sul controllo o si preme CTRL+INVIO quando il controllo possiede lo stato attivo.

È possibile impostare tale proprietà utilizzando la finestra delle proprietà del controllo, una macro o Visual Basic.

È possibile impostare la proprietà AutoActivate solo in visualizzazione Struttura.

Osservazioni

È possibile attivare alcuni oggetti OLE dall'interno del controllo. Quando si attiva un oggetto di questo tipo, è possibile modificarlo o eseguire altre operazioni su di esso dall'interno dei limiti del controllo. Tale caratteristica è chiamata attivazione diretta. Se un oggetto supporta l'attivazione diretta, per ulteriori informazioni sull'utilizzo di tale caratteristica, consultare la documentazione relativa all'applicazione utilizzata per creare l'oggetto.

Se si utilizza Visual Basic, è possibile determinare se un controllo contiene un oggetto, controllando l'impostazione della relativa proprietà OLEType.

Nota   Se si imposta a Doppio clic la proprietà Attivazione automatica di un controllo e si specifica l'evento DoppioClic per il controllo, l'evento DoppioClic si verifica prima che l'oggetto venga attivato.
