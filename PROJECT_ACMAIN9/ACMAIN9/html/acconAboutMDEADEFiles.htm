Informazioni sui file MDE

Se il database contiene del codice Microsoft Visual Basic, il salvataggio del database di Microsoft Access come MDE comporta la compilazione di tutti i moduli, la rimozione di tutto il codice sorgente modificabile e la compattazione del database di destinazione. Il codice Visual Basic continuerà a essere eseguito, ma non potrà essere visualizzato o modificato e la dimensione del database di Access si ridurrà a seguito della rimozione del codice. Verrà ottimizzato inoltre l'utilizzo della memoria con il conseguente miglioramento delle prestazioni.

Dopo avere salvato il database di Access come file MDE non sarà più possibile effettuare le seguenti operazioni:

  • Visualizzazione, modifica o creazione di maschere, report o moduli in visualizzazione Struttura.

  • Aggiunta, eliminazione o modifica dei riferimenti ai database o alle librerie di oggetti.

  • Modifica del codice utilizzando le proprietà o i metodi dei modelli di oggetti Microsoft Access o VBA. Un file MDE non contiene codice sorgente.

  • Importazione o esportazione di maschere, report o moduli. È tuttavia possibile importare o esportare tabelle, query, pagine di accesso ai dati e macro da o in database non MDE. Qualsiasi tabella, query, pagina di accesso ai dati o macro presente in un file MDE può essere importata in un altro database di Access, ma nessuna maschera, report o modulo può essere importato in un altro database di Access.

Attenzione   Salvare sempre una copia del database di Access originale. Per modificare la struttura di maschere, report o moduli di un database di Access salvato come file MDE, è necessario aprire il database originale, modificarlo, quindi salvarlo nuovamente come file MDE. Il salvataggio di un database di Access contenente tabelle come file MDE rende più difficoltosa la sincronizzazione delle diverse versioni dei dati qualora fosse necessario modificare in un secondo momento la struttura di maschere, report o moduli. Per questa ragione è più appropriato salvare un database di Access come file MDE per il database front end di un'applicazione back end/front end.

Attenzione   Nelle versioni future di Microsoft Access non sarà possibile aprire, convertire o eseguire il codice in un file MDE di Microsoft Access 2000. L'unico modo per convertire un file MDE di Microsoft Access 2000 consisterà nell'aprire il database di Access originale da cui è stato creato il file MDE, convertirlo, quindi salvarlo come file MDE.

Importante   È possibile che alcune limitazioni impediscano di salvare il database di Access come file MDE.

  • Se il database di Access è protetto a livello utente, è necessario soddisfare alcuni criteri. Per ulteriori informazioni, vedere l'argomento relativo all'utilizzo di altri tipi di protezione con un database di Access salvato come file MDE.

  • È necessario disporre dell'accesso tramite password al codice Visual Basic. Per ulteriori informazioni, vedere l'argomento relativo alla protezione del codice Visual Basic.

  • Se il database è stato replicato, prima di procedere è necessario rimuovere le proprietà e le tabelle del sistema di replica.

  • Se il database di Access contiene dei riferimenti a un altro database di Access o a un componente aggiuntivo, è necessario salvare tutti i database o i componenti aggiuntivi di Access nella sequenza di riferimenti come file MDE.
