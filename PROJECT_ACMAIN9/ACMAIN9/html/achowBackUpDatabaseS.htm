Fare una copia di backup di un database

  1. Chiudere il database. Se si è in un ambiente multiutente, assicurarsi che tutti gli utenti abbiano chiuso il database.

  2. Se si utilizzano Esplora risorse (Gestione risorse in Windows 95 o Windows NT 4.0), Risorse del computer, Microsoft Backup, il comando copy di MS-DOS oppure un altro programma software per l'esecuzione di backup, copiare il database (un file MDB) sul supporto di backup desiderato.

Note  

  • Se si sta effettuando una copia di backup su un disco floppy ed il file del database supera le dimensioni del disco, non è possibile utilizzare Esplora risorse o Risorse del computer per effettuare la copia di backup del database. È necessario utilizzare Microsoft Backup o un software di backup in modo da poter copiare il file su più dischi.

  • È inoltre consigliabile creare una copia di backup del file di informazioni sul gruppo di lavoro file. Se il file viene perduto o danneggiato, non sarà possibile avviare Microsoft Access fino a quando tale file non verrà ripristinato o generato di nuovo.

  • È possibile effettuare una copia di backup di singoli oggetti di database creando un database vuoto, quindi importando gli oggetti desiderati dal database originale.

  • Non è possibile utilizzare l'istruzione FileCopy in una seconda istanza di Microsoft Access per effettuare la copia di backup di un database aperto nella prima istanza di Microsoft Access.
