Immettere le periferiche e le dimensioni del database e del file di registro delle transazioni

Se si crea un nuovo database di Microsoft SQL Server 6.5, l'Upsize guidato richiede di immettere le periferiche e le dimensioni del database e del file di registro delle transazioni, nonché le dimensioni preallocate per il database e le transazioni. Immettere la periferica e la dimensione di un database

Tutti i database e i file di registro delle transazioni di SQL Server sono posizionati su periferiche. Con il termine periferica si indica sia la posizione logica dei database e dei file di registro delle transazioni sia il file fisico. Quando si crea una periferica, in SQL Server viene creato un file al quale viene assegnata una quantità fissa di spazio su disco.

Per gli amministratori di sistema, l'Upsize guidato visualizza la quantità di spazio libero disponibile sulle periferiche esistenti. Selezionare una periferica che disponga di una quantità di spazio libero sufficiente a contenere almeno la dimensione prevista per il database.

Se nessuna delle periferiche esistenti dispone di spazio libero sufficiente, è possibile crearne una nuova mediante l'Upsize guidato. Le nuove periferiche devono avere una dimensione equivalente almeno alla dimensione prevista del database. Se possibile, è consigliabile creare una periferica con dimensione maggiore del database, in modo da poter espandere quest'ultimo in un momento successivo o posizionare altri database o file di registro delle transazioni sulla stessa periferica.

Immettere la dimensione e la periferica del file di registro delle transazioni

Il file di registro delle transazioni è la registrazione cronologica di tutte le attività che si verificano sul server e viene utilizzato da SQL Server per il backup e il ripristino dei database.

La periferica del file di registro delle transazioni selezionata deve disporre di almeno un megabyte di spazio disponibile. È consigliabile posizionare il database e il file di registro delle transazioni corrispondente su periferiche in dischi fisici separati. È necessario creare queste periferiche prima dell'avvio dell'Upsize guidato, poiché in caso contrario tutte le nuove periferiche verrebbero create sullo stesso disco fisico in cui risiede la periferica del database Master.

Se si dispone di un solo disco fisico, è necessario collocare il database e il relativo file di registro delle transazioni su periferiche separate, in modo da poter ripristinare il database.

L'Upsize guidato crea il file di registro delle transazioni in base alla dimensione specificata. La dimensione del file di registro delle transazioni deve essere almeno equivalente alla dimensione di tutte le tabelle del database delle quali è stato eseguito l'upsize. La dimensione specificata non può eccedere lo spazio disponibile sulla periferica selezionata.

Nota   Per i nuovi database, l'Upsize guidato tronca il file di registro delle transazioni in corrispondenza del punto di arresto se si riempie prima del completamento dell'upsize. È tuttavia possibile che il file di registro di configurazione non venga contenuto da una sola tabella. In questo caso, non viene eseguito l'upsize della tabella e viene scritto un errore nel report dell'Upsize guidato.

Immettere la dimensione di un database e del file di registro delle transazioni

Quando si crea un database con Microsoft SQL Server 6.5, al database viene assegnata una quantità fissa di spazio su una o più periferiche. Tutto questo spazio non viene necessariamente utilizzato subito dal database. Per dimensione del database si intende la dimensione massima che il database può raggiungere prima che lo spazio si esaurisca.

Per valutare lo spazio necessario per il database, considerare la dimensione del database di Microsoft Access e valutare il rapporto in base al quale aumenterà il nuovo database di SQL Server. Per ciascun megabyte di dati di Microsoft Access si richiedono in genere almeno da 1,3 a 1,5 megabyte su SQL Server. È necessario considerare che nella dimensione di un file di database di Microsoft Access (mdb) sono inclusi tutti gli oggetti del database di Microsoft Access e non solo i dati.

Se si dispone di molto spazio su disco sul server, è possibile moltiplicare la dimensione del database di Access per due. Ciò assicura uno spazio sufficiente per eseguire l'upsize del database con l'Upsize guidato e ulteriore spazio per l'eventuale aumento della dimensione. Se si prevede di aggiungere molti dati al database, si consiglia di utilizzare un multiplo superiore.

La dimensione del file di registro delle transazioni deve essere pari al 25% della dimensione del database.
