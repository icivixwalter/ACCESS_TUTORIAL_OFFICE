Proprietà AllowZeroLength (Consenti lunghezza zero)

                 

È possibile utilizzare la proprietà AllowZeroLength per determinare se una stringa a lunghezza zero (" ") è un'immissione valida in un campo di tabella.

Nota   La proprietà AllowZeroLength viene applicata ai campi di tabella di tipo Testo, Memo e Collegamento ipertestuale.

Impostazione

La proprietà AllowZeroLength viene impostata come riportato di seguito.

Impostazione Visual Basic Descrizione
Sì True (-1) Una stringa a lunghezza zero è un'immissione valida.
No False (0) Impostazione predefinita. Una stringa a lunghezza zero non è un'immissione valida.

È possibile impostare questa proprietà utilizzando la finestra delle proprietà della tabella o tramite Visual Basic.

Nota   Per accedere alla proprietà AllowZeroLength di un campo tramite Visual Basic, utilizzare la proprietà DAO AllowZeroLength o la proprietà ADO Column.Properties ("Set OLEDB:Allow Zero Length").

Osservazioni

Se si desidera fare in modo che Microsoft Access memorizzi una stringa a lunghezza zero anziché un valore Null quando si lascia un campo vuoto, impostare le proprietà AllowZeroLength e Required a Sì.

Suggerimento   È possibile utilizzare la proprietà Format per fare una distinzione tra la visualizzazione di un valore Null e una stringa a lunghezza zero. Al momento dell'immissione di una stringa a lunghezza zero, ad esempio, è possibile visualizzare la stringa "Nessuno".

La proprietà AllowZeroLength agisce in modo indipendente dalla proprietà Required. La proprietà Required determina solo la validità di un valore Null rispetto a un campo. Se la proprietà AllowZeroLength è impostata a Sì, una stringa a lunghezza zero costituisce un valore valido per il campo senza tener conto dell'impostazione della proprietà Required.
