Funzione DAvg

        

È possibile utilizzare la funzione DAvg per calcolare la media di un set di valori contenuti in un determinato set di record, un dominio. Utilizzare la funzione DAvg nel codice Visual Basic oppure in una macro, in un'espressione nella query o in un controllo calcolato.

È possibile ad esempio utilizzare la funzione DAvg nella riga Criteri di una query di selezione sul costo di spedizione per limitare i risultati a quei record in cui i costi di spedizione superano la media. Oppure è possibile utilizzare un'espressione che includa la funzione DAvg in un controllo calcolato e visualizzare il valore medio relativo a ordini precedenti accanto al valore di un nuovo ordine.

Sintassi

DAvg(espr, dominio[, criteri])

La funzione DAvg dispone degli argomenti riportati di seguito.

Argomento Descrizione
espr Un'espressione che identifica il campo contenente i dati numerici di cui si desidera calcolare la media. È possibile che si tratti di un'espressione stringa che identifica un campo in una tabella o in una query oppure un'espressione che esegue un calcolo dei campi. In espr è possibile includere il nome di un campo in una tabella, un controllo in una maschera, una costante o una funzione. Se espr include una funzione, è possibile che sia incorporata o definita dall'utente, ma non è possibile che si tratti di un'altra funzione di aggregazione sui domini o di aggregazione SQL.
dominio Un'espressione stringa che identifica il set di record che costituisce il dominio. È possibile che sia un nome di tabella o di query.
criteri Una stringa opzionale utilizzata per limitare l'intervallo di dati sui cui viene eseguita la funzione DAvg. L'argomento criteri ad esempio equivale spesso alla proposizione WHERE priva della parola WHERE in un'espressione SQL. Se si omette l'argomento criteri, la funzione DAvg valuta espr confrontandolo con l'intero dominio. È necessario che qualsiasi campo incluso in criteri appartenga anche all'argomento dominio. In caso contrario, la funzione DAvg restituisce il valore Null.

Osservazioni

I record contenenti valori Null vengono esclusi dal calcolo della media.

Sia che la funzione DAvg venga utilizzata in una macro o in un modulo, in un'espressione nella query o in un controllo calcolato, è necessario creare con attenzione l'argomento criteri per assicurarsi che venga valutato correttamente.

È possibile utilizzare la funzione DAvg per specificare criteri nella riga Criteri di una query. Si supponga ad esempio che si desideri visualizzare un elenco di tutti i prodotti ordinati in quantità che superano la media. È possibile creare una query sulle tabelle Ordini, Dettagli ordini e Prodotti e includere il campo Nome del prodotto e il campo Quantità tramite l'espressione riportata di seguito nella riga Criteri sottostante il campo Quantità.

> DAvg("[Quantità]", "Ordini")

È inoltre possibile utilizzare la funzione DAvg all'interno di un'espressione in un campo calcolato di una query o nella riga Aggiorna a di una query di aggiornamento.

Nota   È possibile utilizzare sia la funzione DAvg che la funzione Media in un'espressione di campo calcolato in una query di totalizzazione. Se si utilizza la funzione DAvg, il calcolo della media dei valori viene eseguito prima del raggruppamento dei dati. Se si utilizza la funzione Media, i dati vengono raggruppati prima del calcolo della media dei valori nell'espressione di campo.

Utilizzare la funzione DAvg in un controllo calcolato quando si rende necessario specificare criteri per limitare l'intervallo di dati su cui viene eseguita la funzione DAvg. Per visualizzare ad esempio le spese medie di trasporto delle merci spedite in California, impostare la proprietà Origine controllo di una casella di testo alla seguente espressione:

=DAvg("[SpeseTrasporto]", "Ordini", "[ZonaDestinatario] = 'CA'")

Se si desidera semplicemente calcolare la media di tutti i record contenuti in dominio, utilizzare la funzione Media.

È possibile utilizzare la funzione DAvg in un modulo o in una macro o in un controllo calcolato di una maschera, se un campo da visualizzare non è nell'origine record su cui si basa la maschera. Si supponga ad esempio di avere una maschera in base alla tabella Ordini e di includere il campo Quantità dalla tabella Dettagli ordini in modo da visualizzare il numero medio di articoli ordinati da un determinato cliente. È possibile utilizzare la funzione DAvg per eseguire tale calcolo e visualizzare i dati nella maschera.

Suggerimenti

  • Se si utilizza la funzione DAvg in un controllo calcolato, è possibile che si desideri inserire il controllo nell'intestazione o nel piè di pagina della maschera, in modo che il valore di tale controllo non venga ricalcolato ogni volta che si passa a un nuovo record.

  • Se il tipo di dati del campo da cui deriva espr corrisponde a un numero, la funzione DAvg restituisce un tipo di dati Precisione doppia. Se si utilizza la funzione DAvg in un controllo calcolato, per migliorare le prestazioni includere una funzione di conversione tra tipi di dati nell'espressione.

  • Anche se è possibile utilizzare la funzione DAvg per stabilire la media dei valori in un campo di una tabella esterna, potrebbe risultare più efficiente creare una query contenente tutti i campi necessari, quindi basare su tale query la maschera o il report.

Nota   Le modifiche ai record che non vengono salvate in dominio non vengono incluse quando si utilizza la funzione. Se si desidera che la funzione DAvg sia basata sui valori modificati, è necessario prima salvare le modifiche scegliendo il comando Salva record dal menu File, spostando lo stato attivo su un altro record oppure utilizzando il metodo Update.
