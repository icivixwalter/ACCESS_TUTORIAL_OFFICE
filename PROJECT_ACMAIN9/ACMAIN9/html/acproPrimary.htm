Proprietà Primary (Primario)

                 

È possibile utilizzare la proprietà Primary per determinare il campo chiave primaria di una tabella. Un campo chiave primaria contiene i dati che identificano in modo univoco ciascun record di una tabella.

Impostazione

La proprietà Primary viene impostata come riportato di seguito.

Impostazione Visual Basic Descrizione
Sì True (-1) L'indice selezionato è la chiave primaria.
No False (0) L'indice selezionato non è la chiave primaria.

È possibile impostare la proprietà Primary in tre modi:

  • In visualizzazione Struttura della tabella selezionare il campo o i campi nell'ordine desiderato in base alla chiave primaria e facendo clic sul pulsante Chiave primaria sulla barra degli strumenti.

  • Nella finestra degli indici selezionare o immettere il nome di un indice nella colonna Nome indice e impostare la proprietà Primary a Sì nella sezione Proprietà indice.

  • In Visual Basic per accedere alla proprietà Primary di un indice, utilizzare la proprietà ADO PrimaryKey.

Osservazioni

Viene creato automaticamente un indice in base al campo chiave primaria di una tabella che viene utilizzata per trovare record e creare join tra tabelle. L'indice chiave primaria richiede un'immissione per ciascun campo chiave primaria e non consente duplicati. L'ordine dei campi in un campo chiave primaria multicampo determina il criterio di ordinamento predefinito per la tabella.

Se non è presente alcuna chiave primaria quando si salva una struttura di tabella, viene visualizzata una finestra di dialogo che chiede se si desidera creare una chiave primaria. Se si sceglie Sì verrà aggiunto alla tabella e verrà impostato come chiave primaria un campo con un tipo di dati Contatore con la relativa proprietà Nuovi valori impostata a Incremento. Se si sceglie No, non si crea alcuna chiave primaria.

Non è possibile utilizzare una tabella priva di chiave primaria in una relazione. È possibile che tale tabella renda più lenti l'ordinamento e la ricerca.
