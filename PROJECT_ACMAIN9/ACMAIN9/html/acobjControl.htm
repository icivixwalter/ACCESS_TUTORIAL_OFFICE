Oggetto Control

                                   




L'oggetto Control rappresenta un controllo di una maschera, di un report o di una sezione, contenuto o associato a un altro controllo. Il diagramma riportato di seguito mostra l'oggetto Control in relazione all'oggetto Form o Report.

Osservazioni

Tutti i controlli di una maschera o di un report appartengono all'insieme Controls relativo a quell'oggetto Form o Report. I controlli contenuti in una sezione specifica appartengono all'insieme Controls relativo a tale sezione. I controlli contenuti in un controllo Struttura a schede o in un controllo gruppo di opzioni appartengono all'insieme Controls relativo a quel controllo. Un controllo Etichetta associato a un altro controllo appartiene all'insieme Controls relativo a tale controllo.

Quando si fa riferimento a un singolo oggetto Control dell'insieme Controls, è possibile fare riferimento all'insieme Controls in modo implicito o esplicito.

' Fa riferimento implicitamente al controllo NuoviDati in 
' Controls.
Me!NuoviDati
' Utilizza se il nome di controllo contiene uno spazio.
Me![Nuovi Dati]
' Prestazioni poco più lente.
Me("NuoviDati")
' Fa riferimento a un controllo mediante il relativo indice dell'insieme Controls.
Me(0)
' Fa riferimento a un controllo NuoviDati utilizzando la sottomaschera insieme Controls.
Me.ctlSubForm.Controls!NuoviDati
' Fa riferimento esplicito al controllo NuoviDati nell'insieme Controls.
Me.Controls!NuoviDati
Me.Controls("NuoviDati")
Me.Controls(0)

Nota   È possibile utilizzare la parola chiave Me per rappresentare nel codice un oggetto Form o Report solo se si fa riferimento alla maschera o al report dal codice del modulo di classe. Se si fa riferimento a una maschera o a un report da un modulo standard o da un modulo di maschera o report diverso, è necessario utilizzare il riferimento completo alla maschera o al report.

Ciascun oggetto Control è caratterizzato da una determinata costante intrinseca. Ad esempio, la costante intrinseca di acTextBox è associata al controllo casella di testo e la costante acCommandButton è associata al pulsante di comando. Le costanti dei diversi controlli di Microsoft Access vengono impostate nella proprietà ControlType del controllo.

Per determinare di che tipo sia un controllo esistente, utilizzare la proprietà ControlType. Non è tuttavia necessario conoscere il tipo specifico di un controllo per poterlo utilizzare nel codice, basta rappresentarlo con una variabile di tipo dati Control.

Se il tipo di dati del controllo a cui si fa riferimento è noto e tale controllo è un controllo incorporato di Microsoft Access, è necessario rappresentarlo con una variabile di tipo specifico. Se, ad esempio, si sa che un particolare controllo è una casella di testo, per rappresentare tale controllo è necessario dichiarare una variabile di tipo TextBox, come è illustrato nel codice riportato di seguito.

Dim txt As TextBox
Set txt = Forms!Impiegati!Cognome

Nota   Se il controllo è un controllo ActiveX, per rappresentarlo è necessario dichiarare una variabile di tipo Control e non è possibile utilizzare un tipo specifico. Se non si è sicuri del tipo di controllo a cui punterà una variabile, dichiarare una variabile di tipo Control.

Il controllo Gruppo di opzioni può contenere altri controlli all'interno dell'insieme Controls, quali i controlli Pulsante di opzione, Casella di controllo, Interruttore ed Etichetta.

Il controllo Struttura a schede contiene un insieme Pages che è un tipo particolare di insieme Controls. L'insieme Pages contiene gli oggetti Page che sono dei controlli. A sua volta ogni oggetto Page contiene un insieme Controls che contiene tutti i controlli della pagina.

Altri oggetti Control presentano un insieme Controls che può contenere un'etichetta associata. Tra questi controlli vi sono i controlli Casella di testo, Gruppo di opzioni, Pulsante di opzione, Interruttore, Casella di controllo, Casella combinata, Casella di riepilogo, Pulsante di comando, Cornice oggetto associato e Cornice oggetto non associato.
