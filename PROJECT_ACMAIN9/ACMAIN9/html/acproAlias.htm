Proprietà Alias

                 

È possibile utilizzare la proprietà Alias per specificare un nome personalizzato per una tabella o query origine quando vengono utilizzate più di una volta nella stessa query.

Nota   La proprietà Alias si applica solo a tabelle o query quando vengono utilizzate come origine di dati di una query.

Impostazione

Utilizzando un'espressione stringa è possibile impostare la proprietà Alias per qualsiasi nome non correntemente utilizzato nel database. Questa proprietà viene impostata in visualizzazione Struttura query facendo clic col pulsante destro del mouse sull'elenco dei campi della griglia di struttura della query e scegliendo Proprietà dal menu di scelta rapida.

È possibile impostare la proprietà Alias in visualizzazione SQL della finestra di query utilizzando una proposizione AS dell'istruzione SQL.

Osservazioni

Quando si crea un self-join in un'istruzione SQL, è necessario utilizzare un alias.

Quando si aggiunge una tabella o una query a una query, Microsoft Access imposta automaticamente la proprietà Alias al nome della tabella o della query. Se si aggiunge una seconda istanza della stessa tabella o query, Microsoft Access imposta la proprietà Alias al nome dell'oggetto aggiungendo un trattino di sottolineatura seguito dal numero uno ("_1").

La creazione di un alias non modifica il nome della tabella o della query, ma specifica solo un altro nome che è possibile utilizzare per fare riferimento alla tabella o alla query. In una query contenente un self-join con gli impiegati e i relativi direttori, ad esempio, è possibile utilizzare due copie della tabella Impiegati. Microsoft Access chiama automaticamente la tabella duplicata Impiegati_1. Per assegnare un nome più significativo a tale tabella, è possibile impostarne la proprietà Alias a Direttori.
