Metodo CreateGroupLevel

        

È possibile utilizzare il metodo CreateGroupLevel per specificare un campo o un'espressione in cui raggruppare o ordinare i dati di un report.

Si supponga, ad esempio, di strutturare una Creazione guidata personalizzata che offra all'utente una scelta di campi in cui raggruppare i dati quando si progetta un report. Richiamare il metodo CreateGroupLevel dalla Creazione guidata per creare gli appositi gruppi secondo la scelta dell'utente.

Sintassi

CreateGroupLevel(report, espressione, intestazione, piè di pagina)

Il metodo CreateGroupLevel utilizza gli argomenti riportati di seguito.

Argomento Descrizione
report Espressione stringa che identifica il nome del report contenente il nuovo livello di gruppo.
espressione Espressione stringa che identifica il campo o l'espressione da riordinare o raggruppare.
intestazione, piè di pagina Valore Intero che indica se un campo o un'espressione avranno un'intestazione e/o un piè di pagina di gruppo associati. Se l'argomento intestazione o piè di pagina è True (-1), il campo o l'espressione avranno un'intestazione o un piè di pagina di gruppo. Se l'argomento intestazione o piè di pagina è False (0), il campo o l'espressione non avranno intestazione o piè di pagina. È possibile creare sia l'intestazione che il piè di pagina impostando entrambi gli argomenti a True.

Osservazioni

Utilizzare il metodo CreateGroupLevel quando si progetta una Creazione guidata che crea un report contenente gruppi o totali. Il metodo CreateGroupLevel raggruppa o ordina i dati contenuti nel campo o nell'espressione specificati e crea un'intestazione e/o un piè di pagina per il livello di gruppo.

Il metodo CreateGroupLevel è disponibile solo nella visualizzazione Struttura report.

Per tenere traccia dei livelli di gruppo creati per un report, Microsoft Access utilizza la matrice della proprietà GroupLevel. Il metodo CreateGroupLevel aggiunge alla matrice un nuovo livello di gruppo in base all'argomento espressione. Il metodo CreateGroupLevel restituisce quindi un valore di indice che rappresenta la posizione del nuovo livello di gruppo nella matrice. Il primo campo o la prima espressione ordinata o raggruppata corrisponde al livello zero, il secondo campo o la seconda espressione al livello uno e così via. È possibile avere fino a dieci livelli di gruppo in un report (da zero a nove).

Quando si specifica che l'argomento intestazione o piè di pagina o entrambi corrispondono a True, le proprietà GroupHeader e GroupFooter di un report vengono impostate a Sì e si crea un'intestazione e/o un piè di pagina per il livello di gruppo.

Una volta creati un'intestazione o un piè di pagina, è possibile impostare altre proprietà GroupLevel: GroupOn, GroupInterval e KeepTogether. È possibile impostare tali proprietà in Visual Basic o nella casella Ordinamento e raggruppamento, disponibile facendo clic su Ordinamento e raggruppamento sulla barra degli strumenti Struttura report.

Nota   Se la Creazione guidata crea livelli di gruppo in un report nuovo o esistente, è necessario che apra il report in visualizzazione Struttura.
