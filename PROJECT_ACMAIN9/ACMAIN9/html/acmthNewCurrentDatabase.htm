Metodo NewCurrentDatabase

                 

È possibile utilizzare il metodo NewCurrentDatabase per creare un nuovo database di Microsoft Access (mdb) nella finestra di Microsoft Access. È possibile utilizzare questo metodo per creare un nuovo database in un'altra applicazione che controlla Microsoft Access tramite Automazione, precedentemente nota come Automazione OLE. È possibile, ad esempio, utilizzare il metodo NewCurrentDatabase da Microsoft Excel per creare un nuovo database nella finestra di Microsoft Access.

Nota   È possibile utilizzare il metodo NewAccessProject per creare un nuovo progetto di Microsoft Access project (adp) nella finestra di Microsoft Access.

Sintassi

applicazione.NewCurrentDatabase nomedb

Il metodo NewCurrentDatabase prevede gli argomenti riportati di seguito.

Argomento Descrizione
applicazione Oggetto Application.
nomedb Espressione stringa uguale al nome di un nuovo file di database, compresi il percorso e l'estensione. È inoltre possibile specificare un percorso di rete, se supportato dalla rete, come:

\\Server\Nome condiviso\Cartella\Nomefile
  Nota   Se non viene fornita un'estensione di nomefile, al nomefile viene aggiunta l'estensione mdb.

Osservazioni

Il metodo NewCurrentDatabase consente di creare un nuovo database di Microsoft Access in un'altra applicazione tramite Automazione. Una volta creata un'istanza di Microsoft Access in un'altra applicazione, è necessario anche creare un nuovo database. Tale database viene aperto nella finestra di Microsoft Access.

Se il database identificato tramite l'argomento nomedb già esiste, si verifica un errore.

Il nuovo database viene aperto con l'account utente Amministratore.
