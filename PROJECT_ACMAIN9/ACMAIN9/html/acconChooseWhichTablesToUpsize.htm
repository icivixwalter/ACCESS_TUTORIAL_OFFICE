Scegliere le tabelle da sottoporre all'upsize

È possibile selezionare le tabelle del database di Microsoft Access di cui si desidera effettuare l'upsize in database di Microsoft SQL Server. In questo argomento verranno fornite informazioni di riferimento relative a:

Tabelle collegate

Query con alias

Tabelle collegate

Nella casella di riepilogo Tabelle disponibili sono incluse tutte le tabelle collegate, a eccezione delle tabelle di Microsoft SQL Server già contenute in un database di SQL Server. Le tabelle collegate che fanno riferimento a un database di SQL Server selezionato per l'upsize vengono automaticamente visualizzate nella casella di riepilogo Esporta in SQL Server e non possono essere rimosse. Le tabelle il cui nome termina con il suffisso "_local" vengono escluse dall'elenco delle tabelle disponibili per evitare che venga effettuato l'upsize di tabelle già sottoposte a questo processo. Se si desidera effettuare nuovamente l'upsize di queste tabelle, rinominarle prima di eseguire l'Upsize guidato rimuovendo il suffisso "_local". Vengono escluse anche le tabelle non correntemente visibili nella finestra del database, comprese le tabelle nascoste e quelle di sistema.

Query con alias

Se si decide di collegare delle tabelle di Microsoft SQL Server al database di Access, è possibile che l'Upsize guidato crei una query con alias per gestire le differenze tra Microsoft Access e Microsoft SQL Server 6.5, che non consente l'utilizzo di spazi o caratteri speciali nei nomi dei campi, fatta eccezione per il segno di cancelletto (#), il segno del dollaro ($) e il carattere di sottolineatura (_). L'Upsize guidato sostituisce automaticamente gli spazi e i caratteri non validi con il carattere "_". Se nel corso dell'upsize vengono modificati i nomi dei campi, l'Upsize guidato assegna alla tabella collegata il suffisso "_remote", ad esempio Impiegati_remote, e rinomina la tabella originale aggiungendo il suffisso "_local", ad esempio Impiegati_local. L'Upsize guidato quindi crea una query con alias denominata Impiegati affinché le maschere, i report e le query continuino a funzionare correttamente nella nuova tabella di SQL Server collegata.

Effetti dell'Integrità referenziale dichiarativa (DRI)

Se si intende effettuare l'upsize in SQL Server versione 6.5 ed è già stata selezionata l'Integrità referenziale dichiarativa (Declarative Referential Integrity, DRI) per l'upsize delle tabelle, la DRI impedisce di sovrascrivere le tabelle correlate già sottoposte all'upsize. È possibile eliminare un intero database di SQL Server sottoposto a un upsize utilizzando SQL Server Enterprise Manager per poi effettuare nuovamente l'upsize del database di Access. Se si desidera risottoporre all'upsize solo alcune tabelle, sarà necessario eliminarle insieme a qualsiasi altra tabella correlata partendo dalla tabella "molti" di una relazione uno-a-molti, ossia da una tabella che non contiene riferimenti alla relativa chiave primaria di un'altra tabella, quindi effettuare nuovamente l'upsize dell'intero set di tabelle. L'Upsize guidato è solo in grado di stabilire relazioni DRI tra tabelle correlate sottoposte contemporaneamente all'upsize, ma non è in grado di stabilire relazioni tra quelle tabelle e tabelle preesistenti sul server.
