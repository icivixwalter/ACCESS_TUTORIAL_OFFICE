Azione OutputSu

        

È possibile utilizzare l'azione OutputSu per inviare come output i dati contenuti in un determinato oggetto di database di Microsoft Access (foglio dati, maschera, report, modulo o pagina di accesso ai dati) in un file in formato Microsoft Excel 98 (*.xls), Testo MS-DOS (*.txt) o Rich Text Format (*.rtf). È inoltre possibile inviare i dati come output in un file in formato HTML (*.html), Microsoft Internet Information Server (*.htx, *.idc) o Microsoft Active Server (*.asp). I file in formato Microsoft Internet Information Server possono essere utilizzati anche con Microsoft Personal Web Server.

Impostazione

L'azione OutputSu utilizza gli argomenti riportati di seguito.

Argomento azione Descrizione
Tipo oggetto Il tipo di oggetto contenente i dati da inviare come output. Selezionare Tabella per un foglio dati di tipo tabella, Query per un foglio dati di tipo query, Maschera per una maschera o un foglio dati di tipo maschera, Report, Modulo, Pagina di accesso ai dati, Stored procedure o Visualizzazione nella casella Tipo oggetto della sezione Argomenti azione della finestra della macro. Non è possibile inviare una macro come output. Se si desidera inviare l'oggetto attivo, selezionarne il tipo con questo argomento, ma lasciare vuoto l'argomento Nome oggetto. Questo argomento è obbligatorio. L'impostazione predefinita è Tabella.
Nome oggetto Il nome dell'oggetto contenente i dati da inviare come output. Nella casella Nome oggetto vengono visualizzati tutti gli oggetti del database corrispondenti al tipo selezionato tramite l'argomento Tipo oggetto.
  Se si esegue una macro contenente l'azione OutputSu in un database di libreria, l'oggetto con questo nome viene ricercato prima nel database di libreria, quindi nel database corrente.
Formato output Il tipo di formato che si desidera utilizzare per inviare i dati come output. Nella casella è possibile selezionare HTML (*.html), Microsoft Active Server Pages (*.asp), Microsoft Excel (*.xls), Microsoft IIS (*.htx, *.idc), Testo MS-DOS (*.txt), Rich Text Format (*.rtf) o Microsoft Data Access Page (*.html). I moduli possono essere inviati come output solo in formato testo MS-DOS, mentre le pagine di accesso ai dati possono essere inviate come output solo in formato HTML. I formati Microsoft Internet Information Server e Microsoft Active Server Pages sono disponibili solo per le tabelle, le query e le maschere. Se si lascia vuoto questo argomento, viene richiesto di specificare il formato di output.
File output Il file a cui si desidera inviare i dati come output, compreso il percorso completo. È possibile includere l'estensione del nome di file standard (ASP, HTM o HTML, HTX, XLS, TXT o RTF) per il formato di output selezionato con l'argomento Formato output, ma non è obbligatorio. Se i dati vengono inviati in file di Microsoft Internet Information Server o Microsoft Active Server Pages, vengono sempre creati file con le estensioni standard HTX e IDC oppure ASP. Se si inviano come output pagine di accesso ai dati, vengono sempre creati file con estensione HTML. Se si lascia vuoto l'argomento File output, viene richiesto di specificare il nome di un file di output.
Avvio automatico Specifica se si desidera che dopo l'esecuzione dell'azione OutputSu venga immediatamente avviato il software appropriato con il file specificato tramite l'argomento File output aperto. Se si seleziona Sì, viene avviato uno dei seguenti software con il file di output dell'oggetto di Microsoft Access aperto: Microsoft Excel per i file XLS, Blocco note di Microsoft Windows per i file TXT oppure Microsoft Word per i file RTF. Per i file HTML, viene avviato il browser Internet predefinito..
File Modello Il percorso e il nome del file che si desidera utilizzare come modello per file HTML, HTX o ASP. Il file modello è un file contenente tag HTML.

Osservazioni

I dati di Microsoft Access vengono inviati come output nel formato selezionato e possono essere letti da qualsiasi applicazione che utilizza lo stesso formato. È possibile ad esempio inviare come output un report di Microsoft Access con la relativa formattazione a un documento in formato RTF e quindi aprire il documento in Microsoft Word.

Se l'oggetto di database viene inviato come output in formato HTML, viene creato un file in formato HTML contenente i dati dell'oggetto. È possibile utilizzare l'argomento File Modello per specificare un file da utilizzare come modello per questo file HTML.

Se l'oggetto viene inviato in formato Microsoft Internet Information Server, vengono creati due file:

  • Un file IDC che contiene informazioni su come connettersi a un'origine dati ODBC e un'istruzione SQL da eseguire su questa origine dati. In questo caso, l'oggetto di Microsoft Access che si sta inviando come output contiene i dati definiti dall'istruzione SQL e il database corrente di Microsoft Access è l'origine dati ODBC.

  • Un file HTX che specifica come formattare i dati restituiti dall'istruzione SQL specificata nel file IDC come documento HTML. È possibile utilizzare l'argomento File Modello per specificare un file HTML da utilizzare come modello per il file HTX.

Microsoft Internet Information Server utilizza i file XTH e IDC per creare un file HTML con i dati dell'oggetto di Microsoft Access inviato come output.

Se l'oggetto di database viene inviato come output in formato Microsoft Active Server Page, viene creato un file in formato ASP contenente informazioni su come accedere ai dati dell'oggetto e su come formattarli. Microsoft Internet Information Server utilizza il file ASP per creare un file HTML con i dati dell'oggetto di Microsoft Access inviato come output. È possibile utilizzare l'argomento File Modello per specificare un file HTML da utilizzare come modello per il file ASP.

Quando si utilizza l'azione OutputSu per inviare un oggetto di database in uno qualsiasi dei formati di output tranne Microsoft Active Server Pages, sono valide le seguenti regole (alcune delle quali applicabili anche al formato Microsoft Active Server Page):

  • È possibile inviare come output i dati dei fogli dati di tipo tabella, query e maschera. Nel file di output, tutti i campi del foglio dati vengono visualizzati come in Microsoft Access, a eccezione dei campi contenenti Oggetti OLE. Le colonne di tali campi sono incluse nel file di output ma i campi sono vuoti.

  • Per un controllo associato a un campo di tipo Sì/No (Interruttore, Pulsante di opzione o Casella di controllo), il file di output visualizza il valore 1 (Sì) o 0 (No).

  • Per una casella di testo associata a un campo Collegamento ipertestuale, il file di output visualizza il collegamento ipertestuale per tutti i formati di output, a eccezione di Testo MS-DOS, nel qual caso il collegamento ipertestuale viene visualizzato come testo normale.

  • Se si inviano i dati di una maschera in visualizzazione Maschera, il file di output contiene sempre la visualizzazione Foglio dati della maschera.

  • Se si inviano i dati di un report, gli unici controlli inclusi nel file di output sono le caselle di testo per i file di output XLS oppure le caselle di testo e le etichette per i file di output RTF, TXT e HTML. Tutti gli altri controlli vengono ignorati. Inoltre, nel file di output non vengono incluse informazioni sulle intestazioni e i piè di pagina. L'unica eccezione è costituita dal fatto che, quando si inviano come output i dati di un report in un file di Microsoft Excel, nel file di output viene inclusa una casella di testo in un piè di pagina di gruppo contenente un'espressione con la funzione Sum. Nel file di output non viene incluso nessun altro controllo in un'intestazione o in un piè di pagina e nessun'altra funzione di aggregazione oltre alla funzione Sum.

  • I sottoreport sono inclusi nel file di output, mentre le sottomaschere vengono incluse quando si inviano dati come output in un file ASP, ma solo quando si invia come output una maschera (e non un foglio dati).

  • Quando si invia come output un foglio dati, una maschera o una pagina di accesso ai dati in formato HTML, viene creato un unico file HTML. Quando si invia come output un report in formato HTML, viene creato un file HTML per ogni pagina del report.

Per ulteriori informazioni sulle regole e sulle limitazioni relative all'invio di dati come output in file HTML, vedere Esportare un foglio dati in formato HTML statico ed Esportare un report in formato HTML statico.

Per ulteriori informazioni sulle regole e sulle limitazioni relative all'invio di dati come output in file di Microsoft Internet Information Server, vedere Esportare un foglio dati in formato HTML generato da server ed Esportare una maschera in formato HTML generato da server.

Per ulteriori informazioni sulle regole e sulle limitazioni relative all'invio di dati come output in file di Microsoft Active Server Pages, vedere Esportare un foglio dati in formato HTML generato da server, Esportare una maschera in formato HTML generato da server e Controlli di maschera supportati e non supportati per il formato ASP dinamico.

Suggerimento   Se si inviano i dati di un report in un file di output e le colonne e le righe del file di output non sono allineate come nel report, considerare i seguenti suggerimenti:

  • Scegliere Dimensione dal menu Formato, quindi scegliere Adatta per ridimensionare i controlli.

  • Evitare la sovrapposizione e l'eccessiva vicinanza dei controlli.

  • Scegliere Allinea dal menu Formato, quindi scegliere il comando appropriato per allineare i controlli del report. I controlli non allineati nella stessa riga possono essere posizionati in righe separate nel file di output, aumentando in tal modo la spaziatura.

L'azione OutputSu equivale a scegliere Esporta dal menu File e a selezionare la casella di controllo Salva formattato nella finestra di dialogo visualizzata. Gli argomenti dell'azione corrispondono alle impostazioni presenti nella finestra di dialogo Esporta. Il comando Esporta tuttavia è valido solo per l'oggetto selezionato nella finestra del database. Utilizzando l'azione OutputSu è possibile specificare l'oggetto che si desidera inviare come output.

Nota   È possibile inviare i dati selezionati come output con il comando Esporta. Questa funzione tuttavia non è disponibile con l'azione OutputSu.

È anche possibile scegliere Collegamenti Office dal menu Strumenti, quindi Analizza con MS Excel o Pubblica con MS Word per inviare come output un oggetto di database e aprire immediatamente il file di output in Microsoft Excel o Microsoft Word.

Per eseguire l'azione OutputSu in Visual Basic, utilizzare il metodo OutputTo dell'oggetto DoCmd.
