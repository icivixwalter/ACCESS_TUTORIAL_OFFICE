Tipi di origine dati per una pagina di accesso ai dati

Una pagina di accesso ai dati ottiene i dati da un database di Microsoft Access o da un database di Microsoft SQL Server 6.5 o versioni successive. Per creare una pagina che utilizzi i dati di uno di questi database, è necessario che la pagina sia connessa al database. Se è già stato aperto un database o un progetto di Microsoft Access connesso a un database di Microsoft SQL Server, la pagina di accesso ai dati creata verrà connessa automaticamente al database corrente e il percorso verrà memorizzato nella proprietà ConnectionString della pagina. Quando la pagina viene aperta in Internet Explorer 5 o in visualizzazione Pagina, vengono visualizzati i dati correnti del database di origine utilizzando il percorso definito nella proprietà ConnectionString. Se la pagina di accesso ai dati viene creata quando il database si trova su un'unità locale, verrà utilizzato il percorso locale e pertanto i dati non saranno accessibili ad altri utenti. Per questo motivo è essenziale spostare o copiare il database in un percorso di rete accessibile a coloro che dovranno utilizzare questa pagina. Dopo averlo spostato o copiato in una condivisione di rete, aprire il database utilizzando l'indirizzo UNC. Se il database viene spostato o copiato dopo avere creato la pagina, sarà necessario aggiornare il percorso nella proprietà ConnectionString in base alla nuova posizione.

Quando la pagina di accesso ai dati è connessa a un database, nella scheda Database dell'Elenco campi compaiono le cartelle contenenti tutte le tabelle o le query disponibili nel database. I dati vengono aggiunti alla pagina di accesso ai dati da queste origini record. Se l'icona del database è contrassegnata da una X rossa e sotto di essa non compare alcuna cartella, è necessario connettere la pagina di accesso ai dati a un database per continuare la progettazione.

Tipi di origine dati per Microsoft Office Web Components su una pagina di accesso ai dati

Sebbene la pagina di accesso ai dati ottenga i dati da un database di Microsoft Access o di Microsoft SQL Server, i controlli di Microsoft Office Web Components sulla pagina possono visualizzare i dati di questi database o di altre origini. La pagina può ad esempio contenere un elenco di tabella pivot, un foglio di calcolo o un grafico con dati di un foglio di lavoro di Microsoft Excel o di un database non di Microsoft Access o Microsoft SQL Server. A seconda dell'origine, i dati in questi controlli possono essere uno snapshot dei dati originali o i dati veri e propri visualizzati mediante una connessione separata del controllo all'origine dati.

Per creare un componente Web con i dati di un'origine diversa dal database corrente, potrebbe essere necessario utilizzare un programma diverso da Microsoft Access per pubblicare prima i dati su una pagina Web. Per creare ad esempio un elenco di tabella pivot contenente i dati di un foglio di lavoro di Microsoft Excel, è necessario pubblicare i dati da Microsoft Excel. In Microsoft Access è quindi possibile aprire la pagina Web e trasformarla in una pagina di accesso ai dati aggiungendo i controlli associati a un database di Microsoft Access o di Microsoft SQL Server. La pagina può essere ulteriormente personalizzata con etichette, immagini o altre caratteristiche che ne migliorano l'aspetto. Da Microsoft Access è possibile creare un elenco di tabella pivot che utilizza i dati di un'origine esterna diversa da Microsoft Excel. Per eseguire questa operazione, è necessario specificare le informazioni di connessione nella sezione Origine dati della Casella delle proprietà Tabella pivot quando si crea il controllo.
