Proprietà RecordSource (Origine record)

                 

È possibile utilizzare la proprietà RecordSource per specificare l'origine dei dati di una maschera o report. È possibile visualizzare dati di una tabella, di una query o di un'istruzione SQL. Per visualizzare e modificare dati della tabella Impiegati di una maschera, ad esempio, impostare a Impiegati la proprietà RecordSource della maschera. È quindi possibile associare i controlli contenuti nella maschera o nel report a determinati campi della tabella Impiegati impostando la proprietà ControlSource del controllo al nome di un campo nella tabella. È possibile, ad esempio, associare un controllo al campo Cognome della tabella Impiegati impostando la proprietà ControlSource del controllo a Cognome.

Impostazione

L'impostazione della proprietà RecordSource può essere un nome di tabella, un nome di query o un'istruzione SQL. È possibile, ad esempio, utilizzare le impostazioni riportate di seguito.

Impostazione di esempio Descrizione
Impiegati Un nome di tabella che indica la tabella Impiegati come origine dei dati.
SELECT Ordini.DataOrdine FROM Ordini Un'istruzione SQL che indica il campo DataOrdine della tabella Ordini come origine dei dati. È possibile associare un controllo contenuto nella maschera o nel report al campo DataOrdine della tabella Ordini impostando la proprietà ControlSource del controllo a DataOrdine.

È possibile impostare la proprietà RecordSource utilizzando la finestra delle proprietà della maschera o del report, una macro o Visual Basic.

Per impostare tale proprietà in Visual Basic, utilizzare un'espressione stringa.

Nota   La modifica dell'origine record di una maschera aperta o di un report aperto provoca la ripetizione automatica della query dei dati sottostanti. Se la proprietà Recordset di una maschera viene impostata mentre la maschera è aperta, la proprietà RecordSource verrà aggiornata.

Osservazioni

Dopo aver creato una maschera o un report, è possibile modificare la relativa origine dati modificando la proprietà RecordSource. La proprietà RecordSource è inoltre utile se si desidera creare una maschera o un report utilizzabili più volte. È possibile, ad esempio, creare una maschera che include una struttura standard, copiarla e modificare la proprietà RecordSource in modo da visualizzare i dati di una tabella, di una query o di un'istruzione SQL diversa.

Limitando il numero di record contenuti nell'origine record di una maschera, è possibile migliorare le prestazioni, in particolare quando l'applicazione è in esecuzione in rete. È possibile, ad esempio, impostare la proprietà RecordSource di una maschera a un'istruzione SQL che restituisca un unico record e modificare l'origine record della maschera a seconda dei criteri selezionati.
