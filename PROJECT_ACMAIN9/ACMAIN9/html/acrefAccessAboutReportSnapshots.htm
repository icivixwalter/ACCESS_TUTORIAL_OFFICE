Definizione di snapshot di report

Lo snapshot di un report è un file con estensione snp che contiene una copia fedele di ogni pagina di un report di Microsoft Access e che conserva il layout bidimensionale, la grafica e altri oggetti incorporati del report.

Definizione di Snapshot Viewer

Snapshot Viewer è un programma per la visualizzazione, la stampa e l'invio tramite posta elettronica di snapshot, ad esempio di snapshot di report. Snapshot Viewer versione 9.0 comprende un programma eseguibile stand-alone, un controllo Snapshot Viewer (Snapview.ocx), un file della Guida in linea e altri file correlati. Per impostazione predefinita, Snapshot Viewer viene installato automaticamente quando viene creato per la prima volta uno snapshot di report. È possibile anche installare Snapshot Viewer dal programma di installazione o scaricandolo dalla pagina Web per i programmatori di Microsoft Access.

Il controllo Snapshot Viewer consente di visualizzare uno snapshot in Microsoft Internet Explorer versione 3.0 o successiva o in qualsiasi altra applicazione che supporti i controlli ActiveX, come Microsoft Access o Microsoft Visual Basic, Applications Edition.

Vantaggi degli snapshot di report e di Snapshot Viewer

Per stampare e distribuire i report a utenti che si trovano sia all'interno che all'esterno di un'organizzazione, l'uso degli snapshot di report può produrre un risparmio di tempo e di denaro. Invece di fotocopiare e di spedire i report stampati di Microsoft Access, è possibile distribuirli e pubblicarli elettronicamente utilizzando la posta elettronica o un browser Web, come Microsoft Internet Explorer. I destinatari potranno così visualizzare in linea i report in modo semplice e rapido e stamparne solo le pagine desiderate. Ciò si rivela particolarmente utile se i report contengono colori e immagini, ad esempio grafici e illustrazioni.

Utilizzo di snapshot di report

Per creare uno snapshot di report deve essere utilizzato Microsoft Access 2000. Per visualizzare, stampare, memorizzare, pubblicare, distribuire e archiviare uno snapshot di report non occorre disporre di una licenza di Access ma è sufficiente utilizzare Snapshot Viewer con un programma come Esplora risorse (Gestione risorse in Windows 95 e Windows NT 4.0), un programma di posta elettronica o un browser Web come Internet Explorer.

Utilizzo di snapshot di report sul Web

Vi sono due modi per pubblicare uno snapshot di report sul Web:

  • Creando un collegamento allo snapshot del report tramite un tag HTML di ancoraggio all'interno di una pagina Web. Ad esempio: 
    <A HREF="http://SitoWeb/NWTraders/catalogo.snp">Report Catalogo</A>

    Se il browser utilizzato è Internet Explorer versione 3.0 o successiva, facendo clic sul collegamento verranno attivati in maniera diretta lo snapshot del report e Snapshot Viewer. In altri browser, lo snapshot viene aperto in una finestra separata di Snapshot Viewer. In entrambi i casi è necessario che Snapshot Viewer sia installato nello stesso computer in cui è installato il browser. Per questo motivo è consigliabile inserire nella pagina anche un collegamento alla pagina Web che consente di scaricare Snapshot Viewer. Gli utenti che non hanno Snapshot Viewer installato nel computer potranno così scaricarlo e installarlo per visualizzare il report.

  • Incorporando il controllo Snapshot Viewer nella pagina Web tramite il tag Oggetto HTML. Ad esempio: 
    <OBJECT ID="SnapshotViewer" WIDTH=640 HEIGHT=480
 CLASSID="CLSID:F0E42D60-368C-11D0-AD81-00A0C90DC8D9">
    <PARAM NAME="_ExtentX" VALUE="16722">
    <PARAM NAME="_ExtentY" VALUE="11774">
    <PARAM NAME="_Version" VALUE="65536">
    <PARAM NAME="SnapshotPath"
 VALUE="http://SitoWeb/NWTraders/catalogo.snp">
    <PARAM NAME="Zoom" VALUE="0">
    <PARAM NAME="AllowContextMenu" VALUE="-1">
    <PARAM NAME="ShowNavigationButtons" VALUE="-1">
</OBJECT>

    Per incorporare il controllo Snapshot Viewer è possibile utilizzare l'applicazione Microsoft ActiveX Control Pad e impostare i parametri del controllo con Control Pad Object Editor.

    Questo metodo può essere utilizzato con Internet Explorer versione 3.0 e successive. In tal caso, se Snapshot Viewer non è installato nello stesso computer in cui è installato il browser, Internet Explorer scarica automaticamente il controllo ActiveX di Snapshot Viewer, ma non il programma Snapshot Viewer completo.

Invio di snapshot di report tramite posta elettronica

Uno snapshot di report può essere inviato tramite posta elettronica utilizzando Snapshot Viewer o Microsoft Access, oppure può essere incorporato come icona o come file in un messaggio utilizzando un programma di posta elettronica che supporti i controlli ActiveX, come ad esempio Microsoft Outlook o Microsoft Exchange. Se si inviano snapshot di report tramite posta elettronica, è utile inserire sempre un collegamento alla pagina Web dalla quale è possibile scaricare il software Snapshot Viewer.

Creazione e distribuzione di snapshot di report

Una tipica applicazione aziendale crea e distribuisce i report non solo su richiesta ma anche secondo una pianificazione sulla base di cicli regolari, ad esempio le quotazioni settimanali di borsa, i report mensili sulle vendite o le relazioni finanziarie trimestrali.

In Microsoft Access 2000 è possibile creare e distribuire snapshot di report programmati in più modi diversi. Ad esempio è possibile:

  • Creare una macro che utilizza l'azione OutputSu per esportare uno o più report in formato snapshot su un server Web utilizzato per pubblicare i report.

  • Creare una macro che utilizza l'azione InviaOggetto per spedire un report a un indirizzo di posta elettronica o a un elenco di distribuzione.

  • Creare un programma in Visual Basic, Applications Edition.
