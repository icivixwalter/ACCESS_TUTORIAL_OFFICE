Espressioni SQL

È possibile utilizzare molte funzioni di Visual Basic all'interno di stringhe SQL, tra cui quelle incluse nel codice di Visual Basic, in visualizzazione SQL della finestra della query o nella griglia di struttura della query. È anche possibile scrivere funzioni personalizzate di Visual Basic ed utilizzarle in stringhe SQL.

Le funzioni che è possibile includere nelle stringhe SQL devono restituire come valore una stringa o una Variante. Qualsiasi operazione eseguita dalla funzione, inoltre, può essere eseguita solo una volta. Non è possibile includere una funzione che deve essere eseguita su ogni record del set di dati, in quanto l'istruzione SQL viene passata solo una volta al modulo di gestione di database Microsoft Jet. Non è possibile, ad esempio, creare una funzione che utilizza come argomento un valore di campo.

È possibile includere funzioni di Visual Basic in un'istruzione SQL utilizzata per definire un oggetto QueryDef o un oggetto Recordset di tipo dynaset o snapshot nel codice. Nella griglia di struttura della query di Microsoft Access, è possibile includere funzioni di Visual Basic in un'istruzione SQL, in espressioni criterio o in espressioni campo calcolato.
