Metodo OpenConnection

                 

È possibile utilizzare il metodo OpenConnection per aprire una connessione ADO a un progetto di Microsoft Access (adp) esistente o a un database di Access (mdb) come progetto o database corrente nella finestra di Microsoft Access.

Sintassi

oggetto.OpenConnection [stringaconnessionebase][, idutente][, password]

Il metodo OpenConnection utilizza gli argomenti riportati di seguito.

Argomento Descrizione
oggetto L'oggetto CurrentProject o CodeProject.
stringaconnessionebase Un'espressione stringa che rappresenta la stringa di connessione di base del database.
idutente Un'espressione stringa che rappresenta il nome del progetto esistente di Access, compreso il nome del percorso e l'estensione del nome file. Se la rete lo supporta, è anche possibile specificare un percorso di rete nella forma seguente:

\\Server\Nome condiviso\Cartella\Nomefile.adp
password Nota   Se non viene indicata l'estensione del nomefile, viene assegnata l'estensione adp. Per aprire i file ADP, è possibile utilizzare questo metodo o il metodo OpenCurrentDatabase.

Osservazioni

Il metodo OpenConnection è simile al metodo Open di un oggetto ADO Connection. Questo metodo stabilisce la connessione fisica con l'origine dati. Una volta che questo metodo è stato svolto con esito positivo, viene stabilita la connessione, vengono impostate le proprietà Connection e BaseConnectionString e i dati della nuova connessione dovrebbero essere raccolti nella finestra del database o nella pagina di accesso ai dati. Tutti i parametri di questo metodo sono opzionali. Se non viene fornita alcuna stringa di connessione di base, la connessione viene stabilita nuovamente utilizzando la stringa di connessione di base precedente, ma in questo caso l'utente deve chiamare CloseConnection prima di chiamare di nuovo OpenConnection. Nel caso di un progetto Access, la proprietà BaseConnectionString può specificare solo il provider DB OLE di SQL Server.
