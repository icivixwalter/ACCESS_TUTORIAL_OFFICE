Esportare una tabella o una query in una tabella di database SQL o in un'altra origine dati ODBC

Importante   Non è possibile esportare una tabella o una query in un database SQL o in un'altra origine datiODBC a meno che non sia installato il driver appropriato.

  1. Nella finestra del database fare clic sul nome della tabella o della query (solo per i database di Microsoft Access) da esportare, quindi scegliere Esporta dal menu File.

  2. Nella finestra di dialogo Esporta nome oggetto "nome Oggetto" in selezionare ODBC Databases dalla casella di riepilogoTipo file.

  3. Nella finestra di dialogo Esporta inserire un nome per il file oppure utilizzare il nome suggerito, quindi scegliere OK.

    Verrà visualizzata la finestra di dialogo Seleziona fonte dati con l'elenco delle origini dati definite per tutti i driver ODBC installati nel computer. Fare clic sulla scheda Fonte dati su file o sulla scheda Fonte dati macchina, quindi fare doppio clic sull'origine dati ODBC in cui esportare. Per definire una nuova origine dati per ogni driver ODBC installato, fare clic sul pulsante Nuovo, quindi seguire le istruzioni della finestra di dialogo Crea nuova fonte dati e delle successive prima di continuare. Fare clic qui per informazioni sulle origini dati.

    Le origini dati ODBC specifiche hanno requisiti diversi di accesso ai dati. Per la maggior parte, ad esempio, viene richiesto di inserire un ID di connessione e una password. È possibile ottenere tali informazioni dal proprio amministratore di sistema. Inserire le informazioni appropriate nella finestra di dialogo, quindi scegliere OK.

    Si effettua in tal modo la connessione all'origine dati ODBC e viene creata la nuova tabella.

Note

  • Dopo aver esportato una tabella da un database di Microsoft Access a un database SQL, è possibile utilizzare una query SQL di tipo pass-through per aggiungere un indice alla tabella. Per ulteriori informazioni, fare clic su .

  • Se si sta esportando da un progetto di Microsoft Access a un database di Microsoft SQL Server, è consigliabile utilizzare Data Transformation Services (DTS), il modo più veloce e flessibile di esportare e trasformare i dati. Per ulteriori informazioni consultare la documentazione relativa a Microsoft SQL Server.
