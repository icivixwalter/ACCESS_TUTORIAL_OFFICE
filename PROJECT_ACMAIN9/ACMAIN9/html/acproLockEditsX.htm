Esempio di proprietà LockEdits (MDB)

Nell'esempio riportato di seguito si apre un oggetto Recordset di tipo dynaset e si imposta la relativa proprietà LockEdits a False (0). In tal modo, viene attuato un blocco ottimistico in modo che sia possibile ad altri utenti modificare i record del database in qualsiasi momento. Microsoft Access attiva un errore intercettabile se i dati vengono modificati prima di utilizzare il metodo Update.

Sub MostraStatoBlocco()
    Dim dbs As Database, rst As Recordset

    ' Restituisce il riferimento al database corrente.
    Set dbs = CurrentDb
    ' Apri oggetto Recordset di tipo dynaset.
    Set rst = dbs.OpenRecordset("Clienti", dbOpenDynaset)
    rst.LockEdits = False
    On Error Goto ErroreModificheBlocco
    ' Consente la modifica.
    rst.Edit
   .                        ' Modifica i record.
   .
   .
    rst.Update            ' Tenta di aggiornare le modifiche.

EsciModificheBlocco:
    rst.Close
    Set dbs = Nothing
    Exit Sub

ErroreModificheBlocco:
        If Err.Number = 3197 Then
            MsgBox "Dati modificati da un altro utente."
            Resume Next
        Else
            MsgBox "Altro errore."
            Resume EsciModificheBlocco
        End If
End Sub
