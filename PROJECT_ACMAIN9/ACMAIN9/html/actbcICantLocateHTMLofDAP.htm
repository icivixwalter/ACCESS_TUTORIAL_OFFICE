Impossibile localizzare il file HTML corrispondente a una pagina di accesso ai dati

Il file HTML associato alla pagina di accesso ai dati è stato rinominato, spostato o eliminato.

Anche se una pagina di accesso ai dati è un oggetto di database, la maggior parte delle informazioni relative alla pagina sono memorizzate in un file HTML corrispondente. Il nome e la posizione di questo file HTML vengono specificati quando si crea la pagina di accesso ai dati. Nella finestra Database di Microsoft Access viene visualizzata un'icona di collegamento alla posizione nel file system del corrispondente file HTML. È possibile vedere la posizione del file HTML nella finestra Database posizionando il puntatore del mouse sul nome della pagina.

Se il file HTML è stato rinominato o spostato, è possibile ripristinarne il nome originale, riportarlo alla posizione di origine o modificare il collegamento mantenuto da Microsoft Access specificando il nuovo nome e la nuova posizione. Fare clic sul pulsante Individua nella finestra di messaggio visualizzata quando non è possibile localizzare il file HTML.

Attenzione   È possibile avere due o più pagine in uno o più database di Microsoft Access o in progetti di Microsoft Access che puntano allo stesso file HTML, se ad esempio la pagina è stata copiata o importata. È consigliabile modificare la pagina in visualizzazione Struttura in un solo database o progetto di Access per volta per evitare problemi di condivisione ed eventuali perdite delle modifiche alla struttura.

Attenzione   Se il file HTML viene eliminato, non è possibile ripristinare la pagina se non recuperando il file HTML da una copia di backup. Per ulteriori informazioni, vedere Proteggere pagine di accesso ai dati.
