Azione RieseguiQuery

        

È possibile utilizzare l'azione RieseguiQuery per aggiornare i dati contenuti in un determinato controllo nell'oggetto attivo tramite la riesecuzione di query sull'origine del controllo. Se non è stato specificato alcun controllo, l'azione rieseguirà la query sull'origine dell'oggetto stesso. Utilizzare tale azione per essere certi che l'oggetto attivo o uno dei relativi controlli visualizzi i dati più recenti.

Impostazione

L'azione RieseguiQuery utilizza l'argomento riportato di seguito.

Argomento azione Descrizione
Nome controllo Il nome del controllo che si desidera aggiornare. Immettere il nome del controllo nella casella Nome controllo della sezione Argomenti azione nella finestra della macro. Si consiglia di utilizzare solo il nome del controllo, non l'identificatore completo, come Forms!nomemaschera!nomecontrollo. Per rieseguire la query sull'origine dell'oggetto attivo, non immettere alcun valore per quest'argomento. Se l'oggetto attivo è un foglio dati o un set di risultati di query, è necessario non immettere alcun valore per l'argomento.

Osservazioni

L'azione RieseguiQuery:

  • Riesegue la query su cui è basato il controllo o l'oggetto.

  • Visualizza tutti i record nuovi o modificati e rimuove i record eliminati dalla tabella su cui è basato il controllo o l'oggetto.

Nota   L'azione RieseguiQuery non influenza la posizione del puntatore del record.

I controlli basati su una query o su una tabella comprendono:

  • Caselle di riepilogo e caselle combinate

  • Controlli di tipo sottomaschera

  • Oggetti OLE, come grafici

  • Controlli contenenti funzioni di aggregazione sui domini, come DSum.

Se il controllo specificato non è basato su una query o tabella, l'azione forza un ricalcolo del controllo.

Se non si immette alcun valore per l'argomento Nome controllo, l'azione RieseguiQuery equivale a premere MAIUSC+F9 quando l'oggetto possiede lo stato attivo. Se un controllo di tipo sottomaschera possiede lo stato attivo, l'azione richiede solo l'origine della sottomaschera, così come succede premendo MAIUSC+F9.

Nota   L'azione RieseguiQuery sottopone nuovamente a query l'origine del controllo o dell'oggetto. L'azione AggiornaOggetto invece aggiorna i controlli nell'oggetto specificato ma non riesegue una query sul database, né visualizza nuovi record. L'azione MostraOgniRecord non solo sottopone nuovamente a query l'oggetto attivo, ma rimuove anche qualsiasi filtro applicato, come non farebbe l'azione RieseguiQuery.

Se si desidera rieseguire una query su un controllo non contenuto nell'oggetto attivo, è necessario utilizzare il metodo Requery in Visual Basic, non l'azione RieseguiQuery o il corrispondente metodo Requery dell'oggetto DoCmd. Il metodo Requery in Visual Basic è più veloce dell'azione RieseguiQuery o del metodo DoCmd.Requery. Quando si utilizza l'azione RieseguiQuery o il metodo DoCmd.Requery, la query inoltre viene chiusa e ricaricata dal database. Quando invece si utilizza il metodo Requery, la query viene rieseguita senza essere né chiusa né ricaricata. Notare che il metodo ADO (ActiveX Data Object) Requery funziona come il metodo Requery di Microsoft Access.
