Ottenere prestazioni ottimali con le tabelle collegate in un database di Microsoft Access

Nota   È possibile collegare una tabella solo in un database di Microsoft Access e non in un progetto di Microsoft Access.

Sebbene sia possibile utilizzare delle tabelle collegate come delle normali tabelle di Microsoft Access, si noti che queste non si trovano realmente nel database in uso. Ogni volta che si visualizzano dei dati di una tabella collegata, vengono recuperati dei record da un altro file. L'operazione può richiedere del tempo, soprattutto se la tabella collegata si trova su una rete o in un database SQL.

Se si sta utilizzando una tabella collegata su una rete o in un database SQL, seguire le istruzioni riportate di seguito:

  • È possibile migliorare notevolmente le prestazioni quando si aprono il database principale e le tabelle e le maschere forzando il database collegato in modo che rimanga aperto. A tal fine, creare una tabella vuota nel database collegato e collegarla al database principale, quindi utilizzare il metodo OpenRecordset per aprire la tabella collegata. Ciò impedisce che il modulo di gestione di database Microsoft Jet apra e chiuda più volte il database collegato e crei e distrugga il file con estensione ldb associato.

  • Visualizzare esclusivamente i dati necessari. Non spostare verso l'alto o verso il basso la pagina del foglio dati se non è necessario. Evitare di andare all'ultimo record di una tabella di grandi dimensioni. Se si vogliono aggiungere dei nuovi record ad una tabella di grandi dimensioni, utilizzare il comando Immissione dati del menu Record, per evitare il caricamento in memoria dei record esistenti.

  • Utilizzare dei filtri o delle query per limitare il numero di record visualizzati in una maschera o in un foglio dati. Un numero inferiore di dati verrà trasferito in rete.

  • Nelle query che comprendono delle tabelle collegate, evitare l'utilizzo di funzioni nei criteri della query. Non utilizzare, in particolare, le funzioni di aggregazione sui domini come, ad esempio, Dsum. Quando si utilizza una funzione di aggregazione sui domini, per eseguire la query vengono recuperati tutti i dati della tabella collegata.

  • Se le aggiunte di record a una tabella collegata sono frequenti, creare una maschera per l'aggiunta di record impostando la proprietà Immissione dati a Sì. Quando si apre la maschera per immettere dei nuovi dati, nessuno dei record esistenti viene visualizzato. Questa impostazione di proprietà consente un risparmio di tempo, poiché non è necessario recuperare tutti i record della tabella collegata.

  • È bene tener presente che più utenti contemporaneamente possono tentare di utilizzare una tabella esterna, Occorre, quindi, evitare di bloccare dei record più del tempo necessario.
