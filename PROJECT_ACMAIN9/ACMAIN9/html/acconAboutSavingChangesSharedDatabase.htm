Salvataggio delle modifiche alla struttura in un database di Microsoft Access condiviso

Non è possibile salvare le modifiche alla struttura di un database di Access se tale database è condiviso con altri utenti. L'unico modo per salvare le modifiche è aprire il database di Access in modalità esclusiva.

Quando si tenta di apportare una modifica alla struttura di un oggetto di database ad eccezione di tabelle e query o di un elemento in modalità condivisa, in genere viene attivata temporaneamente la modalità esclusiva per il database di Access se si è il solo utente che lo sta utilizzando al momento. Dopo il salvataggio di tutte le modifiche alla struttura e la chiusura di tutte le finestre in visualizzazione Struttura, viene riattivata la modalità condivisa. Nel frattempo gli altri utenti non potranno aprire il database di Access.

Se un altro utente ha aperto il database di Access in modalità condivisa e si tenta di apportare una modifica principale alla struttura, ad esempio si modifica una maschera, un messaggio avverte che potrebbe non essere possibile salvare le modifiche. Se in una situazione analoga si tenta invece di apportare una modifica secondaria alla struttura, ad esempio si modificano le impostazioni di stampa, non viene visualizzato alcun messaggio di avviso. In entrambi i casi è opportuno attendere che nessun altro utente utilizzi il database di Access in modo da poter salvare le modifiche principali e secondarie apportate alla struttura.

Una pagina di accesso ai dati viene gestita in modo diverso. Sebbene per creare, rinominare, spostare ed eliminare una pagina di accesso ai dati sia comunque necessaria la modalità esclusiva perché queste operazioni comportano la modifica di informazioni nel database di Access, tale modalità non è richiesta per la modifica della pagina perché il file HTML corrispondente si trova nel file system all'esterno del database.

Differenze tra le modifiche principali e secondarie alla struttura

Se non si dispone dell'accesso esclusivo a un database di Access, viene visualizzato un messaggio di avviso quando potrebbe non essere possibile salvare le seguenti modifiche principali alla struttura:

  • Modifiche agli oggetti di database, ad eccezione di tabelle e query, apportate in visualizzazione Struttura

  • Modifiche alla finestra delle proprietà di una maschera in visualizzazione Maschera

  • Compilazione del progetto, modifica delle proprietà del progetto oppure aggiunta o rimozione di un riferimento in Visual Basic Editor

  • Operazioni per rinominare, incollare o eliminare un oggetto di database

  • Salvataggio di un oggetto di database come altro tipo

  • Aggiunta o modifica dei controlli su una barra dei comandi

  • Modifica dei gruppi personalizzati nella finestra del database

  • Operazioni per creare, rinominare, spostare ed eliminare una pagina di accesso ai dati.

Se non si dispone dell'accesso esclusivo a un database di Access, non viene visualizzato alcun messaggio di avviso quando si tenta di salvare le seguenti modifiche secondarie alla struttura:

  • Modifiche alle proprietà di formattazione del foglio dati, quali stili di linea e tipi di carattere

  • Operazioni per bloccare, sbloccare, nascondere e visualizzare le colonne del foglio dati

  • Regolazione della larghezza delle colonne e delle dimensioni delle righe del foglio dati

  • Modifiche al filtro o al tipo di ordinamento per una maschera o un foglio dati

  • Modifiche allo stato di un foglio dati secondario, espanso o compresso

  • Modifiche al collegamento OLE/DDE di un oggetto OLE o modifiche al contenuto di una cornice di oggetto non associato

  • Modifiche alle impostazioni di stampa

  • Modifiche a una barra dei comandi, ad esempio la posizione e la visualizzazione

  • Modifiche al layout della finestra del database.

Strategie per la condivisione della progettazione e dello sviluppo di un database di Access

Se si intende condividere la progettazione di un database di Access, è opportuno considerare l'utilizzo delle seguenti strategie:

  • Assegnare oggetti o gruppi di oggetti specifici a sviluppatori specifici. Uno sviluppatore ad esempio può essere addetto alla creazione delle maschere e un altro a quella dei report. Ogni sviluppatore può utilizzare una propria copia del database di Access. Quando il team di sviluppo è pronto per la verifica, l'assemblaggio e la produzione, ogni sviluppatore può esportare gli oggetti di database dalla propria copia in un database di Access principale.

  • Utilizzare un programma di controllo del codice sorgente. È possibile ad esempio utilizzare in combinazione Microsoft Visual SourceSafe versione 6.0 e il componente aggiuntivo Controllo del codice sorgente che attiva il supporto Visual Source Safe Code in Access. Questo componente aggiuntivo è disponibile in Microsoft Office 2000 Developer (MOD). Se si utilizza Visual SourceSafe, è opportuno eseguire regolarmente l'Analizzatore prestazioni (scegliendo Analizza dal menu Strumenti, quindi Prestazioni) per garantire sempre prestazioni ottimali.
