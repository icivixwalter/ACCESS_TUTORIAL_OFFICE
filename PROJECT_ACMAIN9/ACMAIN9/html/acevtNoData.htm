Evento ReportVuoto

                                   

L'evento ReportVuoto si verifica dopo che Microsoft Access formatta un report vuoto per la stampa (il report è associato a un set di record vuoto), ma prima che il report venga stampato. È possibile utilizzare questo evento per annullare la stampa di un report vuoto.

Osservazioni

Per eseguire una macro o una routine evento quando si verifica questo evento, impostare la proprietà Su report vuoto al nome della macro o a [Routine evento].

Se il report non viene associato a una tabella o a una query tramite la proprietà Origine record del report, l'evento ReportVuoto non si verifica.

Questo evento si verifica dopo gli eventi Formattazione relativi al report ma prima del primo evento Stampa.

Questo evento non si verifica per i sottoreport. Per nascondere i controlli in un sottoreport vuoto in modo che non vengano stampati, è possibile utilizzare la proprietà HasData in una macro o routine evento in esecuzione quando si verifica l'evento Formattazione o Stampa.

L'evento ReportVuoto si verifica prima del primo evento Pagina del report.
