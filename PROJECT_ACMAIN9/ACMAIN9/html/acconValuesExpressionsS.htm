Utilizzare i valori nelle espressioni

È possibile specificare un valore in un'espressione utilizzando un valore letterale, una costante, una funzione o un identificatore.

  • Un valore letterale rappresenta un valore, ad esempio un numero, una stringa o una data valutato dal programma nella maniera esatta in cui è scritto. "New York", 100 e #1-gen-94# sono esempi di valori letterali. Le date sono racchiuse tra segni di cancelletto (#) e le stringhe tra virgolette doppie (").

  • Una costante rappresenta un valore invariabile; True, False e Null sono esempi di costanti definite automaticamente dal programma. È inoltre possibile definire costanti personalizzate in Visual Basic, Applications Edition da utilizzare nelle routine Visual Basic.

  • Una funzione restituisce un valore basato sui risultati di un calcolo o di un'altra operazione. Sono incluse molte funzioni incorporate, ad esempio:
    • La funzione Data, che restituisce la data corrente.

    • La funzione Somma, che restituisce la somma di un set di valori di campo.

    • La funzione DLookup, che restituisce un valore di campo specifico.

    È inoltre possibile creare funzioni personalizzate utilizzando Visual Basic.

  • Un identificatore fa riferimento al valore di un campo, di un controllo o di una proprietà. L'identificatore riportato di seguito ad esempio fa riferimento al valore della proprietà DefaultValue del controllo DataOrdine nella maschera Ordini:

    Forms![Ordini]![DataOrdine].DefaultValue

È possibile combinare il valore di un campo, di un controllo o di una proprietà con un letterale di stringa utilizzando l'operatore di concatenazione E commerciale (&). L'espressione riportata di seguito ad esempio combina il letterale di stringa [IDCategoria] = " con il valore del controllo IDCategoria nella maschera Prodotti:

"[IDCategoria] = " & Forms![Prodotti]![IDCategoria]

In alcune circostanze, ad esempio in una funzione di aggregazione sui domini come DLookup, il valore di un campo, controllo o proprietà deve apparire tra apici (') o virgolette doppie ("). È possibile ottenere tale risultato in modo semplice aggiungendo un apice (') al letterale di stringa, quindi combinando l'espressione con un altro letterale di stringa, costituito da un apice dopo il valore del campo, del controllo o della proprietà, nel seguente modo:

"[IDCategoria] = ' " & Forms![Prodotti]![IDCategoria] & " ' "

È possibile che le applicazioni di Microsoft Access esistenti utilizzino gli operatori barre verticali (| |) al posto di una serie di operatori di concatenazione come virgolette doppie aperte e chiuse ed E commerciale (&), nel seguente modo:

"[IDCategoria] = '|Forms![Prodotti]![IDCategoria]|' "

Non è consigliabile tuttavia utilizzare le barre verticali poiché possono produrre in determinate circostanze risultati non previsti.

Se si desidera che l'espressione produca una stringa racchiusa tra virgolette doppie, è possibile racchiudere la stringa nidificata sia fra gli apici ('), sia fra tre gruppi di virgolette doppie. Le espressioni che seguono ad esempio sono equivalenti:

Forms![Contatti]![Città].DefaultValue = ' "Parigi" '

Forms![Contatti]![Città].DefaultValue = " " "Parigi" " "
