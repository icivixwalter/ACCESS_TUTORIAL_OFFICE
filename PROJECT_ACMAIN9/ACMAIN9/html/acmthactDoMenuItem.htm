Metodo DoMenuItem

                 

Nota   In Microsoft Access 97 il metodo DoMenuItem era stato sostituito con il metodo RunCommand. Il metodo DoMenuItem è incluso in questa versione di Microsoft Access solo per compatibilità con le versioni precedenti. Quando viene eseguito un codice Visual Basic esistente che contiene il metodo DoMenuItem, Microsoft Access visualizza il comando del menu o della barra degli strumenti appropriato per Microsoft Access 2000. Diversamente dall'azione EseguiVoceMenu in una macro, tuttavia, il metodo DoMenuItem nel codice di Visual Basic non viene convertito nel metodo RunCommand quando viene convertito un database creato con una versione precedente di Microsoft Access.

Alcuni comandi delle versioni precedenti di Microsoft Access non sono disponibili in Microsoft Access 2000 e i metodi DoMenuItem che eseguono questi comandi causano un errore quando vengono eseguiti in Visual Basic. È necessario modificare il codice di Visual Basic per sostituire o eliminare le occorrenze di questi metodi DoMenuItem.

Sintassi

DoCmd.DoMenuItem barramenu, nomemenu, comando[, sottocomando][, versione]

Il metodo DoMenuItem utilizza gli argomenti riportati di seguito.

Argomento Descrizione
barramenu Utilizzare la costante intrinseca acFormBar per la barra dei menu in visualizzazione Maschera.
  Per le altre visualizzazioni, come indicato nella finestra delle macro delle versioni precedenti di Microsoft Access, utilizzare il numero di visualizzazione presente nell'elenco dell'argomento Barra dei menu; l'elenco ha una numerazione progressiva a partire da zero.
nomemenu È possibile utilizzare una delle seguenti costanti intrinseche:
  acFile
acEditMenu
acRecordsMenu
  È possibile utilizzare acRecordsMenu solo per la barra dei menu in visualizzazione Maschera nei database di Microsoft Access 2.0 e di Microsoft Access per Windows 95.
  Per gli altri menu, come indicato nella finestra delle macro delle versioni precedenti di Microsoft Access, utilizzare il numero di menu presente nell'elenco dell'argomento Nome menu; l'elenco ha una numerazione progressiva a partire da zero.
comando È possibile utilizzare una delle seguenti costanti intrinseche:
  acNew
acSaveForm
acSaveFormAs
acSaveRecord
acUndo
acCut
acCopy
acPaste
acDelete
acSelectRecord
acSelectAllRecords
acObject
acRefresh
  Per gli altri comandi, passare alla finestra delle macro delle versioni precedenti di Microsoft Access ed utilizzare il numero relativo al comando nell'elenco di argomenti Comando, scorrendo l'elenco ed assegnando alla prima voce il numero zero, alla seconda il numero uno e così via.
sottocomando È possibile utilizzare una delle seguenti costanti intrinseche:
  acObjectVerb
acObjectUpdate
  La costante acObjectVerb rappresenta il primo comando del sottomenu del comando Oggetto del menu Modifica. Il tipo di oggetto determina il primo comando del sottomenu. Questo comando, ad esempio, è Modifica per un oggetto Paintbrush modificabile.
  Per gli altri comandi dei sottomenu, passare alla finestra delle macro delle versioni precedenti di Microsoft Access ed utilizzare il numero del sottocomando nell'elenco di argomenti Sottocomando, scorrendo l'elenco ed assegnando alla prima voce il numero zero, alla seconda il numero uno e così via.
versione Utilizzare la costante intrinseca acMenuVer70 per il codice scritto per database di Microsoft Access 95, la costante intrinseca acMenuVer20 per il codice scritto per database di Microsoft Access 2.0 e la costante intrinseca acMenuVer1X per il codice scritto per database di Microsoft Access versione 1.x. Questo argomento è disponibile solo in Visual Basic.
  Nota   L'impostazione predefinita per questo argomento è acMenuVer1X, di conseguenza il codice scritto per i database di Microsoft Access versione 1.x viene eseguito senza modifiche. Se si scrive del codice per un database di Microsoft Access 95 o versione 2.0 e si desidera utilizzare i comandi di menu di Microsoft Access 95 o di Microsoft Access 2.0 con il metodo DoMenuItem, è necessario impostare questo argomento ad acMenuVer70 oppure ad acMenuVer20.
  Inoltre quando si scorrono gli elenchi degli argomenti delle azioni relative a Barra menu, Nome menu, Comando e Sottocomando nella finestra delle macro per ricavare i numeri da utilizzare per gli argomenti del metodo DoMenuItem, è necessario utilizzare gli elenchi di Microsoft Access 95 se l'argomento versione è acMenuVer70, gli elenchi di Microsoft Access 2.0 se l'argomento versione è acMenuVer20 e gli elenchi di Microsoft Access 1.x se l'argomento versione è acMenuVer1X o se viene omesso.
  Nota   Per questo argomento non esiste alcuna impostazione di acMenuVer80. Non è possibile utilizzare il metodo DoMenuItem per visualizzare i comandi di Microsoft Access 97 o Microsoft Access 2000, sebbene i metodi DoMenuItem esistenti nel codice di Visual Basic continuino a funzionare. Utilizzare al suo posto il metodo RunCommand.

Osservazioni

Le selezioni negli elenchi degli argomenti delle azioni relative a Nome menu, Comando e Sottocomando nella finestra della macro dipendono dalle selezioni effettuate per gli argomenti precedenti. È necessario utilizzare numeri o costanti intrinseche appropriate per ogni argomento barramenu, nomemenu, comando e sottocomando.

Se si omette l'argomento sottocomando ma si specifica l'argomento versione, è necessario includere la virgola dell'argomento sottocomando. Se si omettono gli argomenti sottocomando e versione, non utilizzare la virgola dopo l'argomento comando.
