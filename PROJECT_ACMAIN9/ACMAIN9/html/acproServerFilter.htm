Proprietà ServerFilter (Filtro su server)

                 

È possibile utilizzare la proprietà ServerFilter per specificare un sottoinsieme di record da visualizzare quando un filtro su server viene applicato a una maschera o a un report in un progetto di Microsoft Access (adp) o in una pagina di accesso ai dati in un progetto (adp) o in un database (mdb) di Microsoft Access.

Impostazione

La proprietà ServerFilter è un'espressione stringa che consiste in una proposizione WHERE senza la parola chiave WHERE. Ad esempio, il seguente codice di Visual Basic definisce e applica un filtro che consente di visualizzare solo i clienti degli Stati Uniti:

Me.ServerFilter = "Paese = 'USA'"
Me.ServerFilterByForm = True

Il modo più semplice di impostare questa proprietà consiste nell'utilizzo della finestra delle proprietà della maschera o del report. È anche possibile impostare questa proprietà su una maschera o su un report utilizzando Visual Basic.

Per impostare la proprietà ServerFilter occorre innanzitutto effettuare una delle seguenti operazioni:

  • Impostare il valore della proprietà nella finestra delle proprietà della maschera.

  • Impostare la proprietà in Visual Basic digitando
    Forms(0).ServerFilter = "nomecampo = valore"

Nota   L'impostazione della proprietà ServerFilter non ha effetto sulla proprietà ADO Filter.

Osservazioni

È possibile utilizzare la proprietà ServerFilter per salvare un filtro e applicarlo successivamente. I filtri vengono salvati con gli oggetti in cui vengono creati e quando l'oggetto viene aperto vengono caricati automaticamente, sebbene non vengano applicati automaticamente.

Per applicare a una maschera un filtro salvato, è possibile fare clic sul pulsante Applica filtro su server sulla barra degli strumenti, scegliere Applica filtro/ordina dal menu Record oppure utilizzare una macro o Visual Basic per impostare la proprietà ServerFilterByForm a True (–1).

Il pulsante Applica filtro su server indica lo stato delle proprietà ServerFilter e ServerFilterByForm. Tale pulsante rimane disattivato finché non c'è un filtro da applicare. Se un filtro esistente è applicato, il pulsante Applica filtro su server appare premuto.

Per applicare un filtro automaticamente all'apertura di una maschera, specificare nell'impostazione della proprietà evento OnOpen della maschera una macro che utilizzi l'azione ApplicaFiltro o una routine evento che utilizzi il metodo ApplyFilter dell'oggetto DoCmd. In ogni caso, la maschera viene visualizzata nella finestra Filtro su server in base a maschera.

È possibile rimuovere un filtro su server solo utilizzando Visual Basic per impostare la proprietà ServerFilterByForm a False (0) o cancellando tutti i criteri filtro nella finestra Filtro su server in base a maschera e facendo quindi clic su Applica filtro su server.

Quando la proprietà ServerFilter è impostata in visualizzazione Struttura maschera, Microsoft Access non tenta di convalidare l'espressione SQL. Se tale espressione non è valida, quando il filtro viene applicato si verificherà un errore.

Note

  • Quando viene creato un nuovo oggetto, esso eredita le proprietà RecordSource, Filter, ServerFilter, OrderBy e OrderByOn della tabella o della query dalla quale è stato creato.

  • L'impostazione della proprietà ServerFilter viene ignorata se l'origine record della maschera è una stored procedure.
