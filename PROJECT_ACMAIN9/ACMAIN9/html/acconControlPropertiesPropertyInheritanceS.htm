Correlazione tra le proprietà dei controlli di una maschera o di un report e le proprietà nei relativi campi sottostanti in un database di Access

Quando viene creato un controllo associato nelle maschere o nei report trascinando un campo dall'elenco campi, alcune proprietà vengono copiate dal campo della tabella o della query sottostante nel controllo. Se ad esempio nel campo Prezzo unitario di una tabella Prodotti la proprietà Formato è impostata a Valuta e la proprietà Posizioni decimali è impostata ad Automatiche, quando viene creata una casella di testo associata in una maschera trascinando il campo Prezzo unitario dall'elenco campi, le proprietà Formato e Posizioni decimali per la casella di testo vengono impostate automaticamente agli stessi valori. Si presuppone che la proprietà Visualizza controllo del campo sia impostata a Casella di testo o non abbia alcuna impostazione.

È possibile modificare le impostazioni di tutte le proprietà ereditate (copiate) nella finestra delle proprietà del controllo. Se si modifica l'impostazione della proprietà di un controllo, la modifica non influisce sull'impostazione di quella proprietà per il campo della tabella o della query sottostante. Analogamente se si modifica l'impostazione della proprietà per un campo di una tabella o di una query dopo aver creato una maschera utilizzando quel campo, l'impostazione della proprietà per il controllo non viene aggiornata. È necessario eseguire questo aggiornamento manualmente. Se però si modificano le impostazioni delle proprietà dei campi per le proprietà Valore predefinito, Valido se e Messaggio di errore, le modifiche vengono apportate a tutti i controlli che si basano su questi campi, anche se sono stati creati prima che fossero modificate le proprietà dei campi.

Suggerimento   È consigliabile impostare le proprietà Formato, Posizioni decimali, Maschera di input, Valido se, Messaggio di errore e Valore predefinito nel campo sottostante piuttosto che in un controllo. In questo modo si è certi di avere impostazioni coerenti ogni volta che si aggiungono campi a una maschera o a un report.

Nella tabella riportata di seguito sono elencate le proprietà ereditate per ogni tipo di controllo associato quando si trascina un campo dall'elenco campi nelle maschere e nei report.

Controllo associato Proprietà ereditate
Casella di testo Formato, Posizioni decimali, Maschera di input e Testo barra stato (dalla proprietà Descrizione)
Casella di riepilogo Tutte le proprietà specificate nella scheda Ricerca in visualizzazione Struttura della tabella e la proprietà Testo barra stato (dalla proprietà Descrizione)
Casella combinata Tutte le proprietà specificate nella scheda Ricerca in visualizzazione Struttura della tabella, la proprietà Maschera di input e la proprietà Testo barra stato (dalla proprietà Descrizione)
Casella di controllo, pulsante di opzione, gruppo di opzioni, interruttore,
cornice di oggetto associato		 Testo barra stato (dalla proprietà Descrizione)

Note

  • Le proprietà Valore predefinito, Valido se e Messaggio di errore non vengono ereditate. In Access infatti le proprietà dei controlli non vengono impostate ai valori delle stesse proprietà del campo sottostante. Queste impostazioni tuttavia vengono applicate quando si crea un controllo basato su un campo con queste impostazioni. Se si imposta la proprietà Valido se in un campo e anche in un controllo basato su quel campo, verranno applicate entrambe le regole. Se si imposta la proprietà Valore predefinito per un campo e anche in un controllo basato su quel campo, la proprietà del controllo avrà la precedenza sulla proprietà del campo.

  • Se si crea una casella di testo basata su un campo contenente dati di tipo Memo, la proprietà Funzione tasto INVIO verrà impostata automaticamente a Nuova riga del campo e la proprietà Barre scorrimento a Verticale. Queste proprietà non vengono ereditate dalle proprietà del campo ma sono proprietà del controllo.
