Esempi dell'utilizzo di funzioni di aggregazione sui domini delle maschere e dei report

Nella seguente tabella vengono elencati esempi di espressioni che possono essere utilizzate nei controlli calcolati delle maschere e dei report.

Espressione Descrizione
=DLookup("[Contatto]", "[Fornitori]", "[IDFornitore] = Forms![IDFornitore]") Utilizza la funzione DLookup per visualizzare il valore del campo Contatto nella tabella Fornitori quando il valore del campo IDFornitore della tabella corrisponde al valore del controllo IDFornitore nella maschera attiva.
=DLookup("[Contatto]", "[Fornitori]", "[IDFornitore] = Forms![Nuovi fornitori]![IDFornitore]") Utilizza la funzione DLookup per visualizzare il valore del campo Contatto della tabella Fornitori quando il valore del campo IDFornitore della tabella corrisponde al valore del controllo IDFornitore nella maschera Nuovi fornitori.
=DSum("[ImportoOrdine]", "[Ordini]", "[IDCliente] = 'RATTC'") Utilizza la funzione DSum per visualizzare il totale dei valori del campo ImportoOrdine nella tabella Ordini in cui l'IDCliente è RATTC.

Note  

  • In un controllo calcolato l'espressione deve essere preceduta dall'operatore =.

  • Quando si imposta la proprietà Nome elemento di un controllo calcolato, assicurarsi di utilizzare un nome univoco. Non utilizzare il nome di uno dei controlli utilizzati nell'espressione.

  • Non è possibile utilizzare il nome di un controllo in un'espressione che utilizza una funzione di aggregazione sui domini; è necessario utilizzare solo i nomi dei campi di una tabella, di una query o di un'istruzione SQL.
