Definizione degli interruttori nelle maschere e nei report

È possibile utilizzare un interruttore in una maschera come controllo autonomo per visualizzare un valore Sì/No da un'origine record sottostante. L'interruttore della figura che segue ad esempio è associato al campo Sospeso della tabella Prodotti di un database. Il tipo di dati del campo Sospeso è Sì/No. Quando il pulsante è premuto, il valore nella tabella Prodotti è Sì. Quando il pulsante non è premuto, il valore della tabella Prodotti è No.

Quando si fa clic su un interruttore associato a un campo Sì/No in un database di Microsoft Access, il valore nella tabella sottostante viene visualizzato in base alla proprietà Formato (Sì/No, Vero/Falso o On/Off) del campo. In un progetto di Microsoft Access il pulsante di opzione è associato a una colonna definita come tipo di dati Bit. Quando il valore di questa colonna è 1, indica condizioni Sì, On o Vero. Quando il valore è 0, indica condizioni No, Off o Falso.

Gli interruttori risultano particolarmente utili quando vengono utilizzati in un gruppo di opzioni insieme ad altri pulsanti. In un gruppo di opzioni è possibile stabilire con facilità se un pulsante è selezionato. Per ulteriori informazioni sui gruppi di opzioni, fare clic su .

Sugli interruttori è possibile utilizzare immagini invece che testo. Invece di visualizzare la parola "Sospeso" sull'interruttore del primo esempio, si può visualizzare l'immagine di un cestino.

È inoltre possibile utilizzare un interruttore in una finestra di dialogo personalizzata per accettare l'input dell'utente.
