Creare lo snapshot di un report utilizzando l'azione OutputSu e pubblicarlo sul Web

  1. Creare una macro con una o più azioni OutputSu, una per ciascun report di cui si desidera ottenere l'output in formato snapshot. Ad esempio:

    Macro Crea snapshot

    Azione Argomento Valore
    OutputSu Tipo oggetto Report
      Nome oggetto Catalogo
      Formato output Formato snapshot
      File di output \\Website\Inetpub\wwwroot\Webreports\Catalog.snp *
      Avvio automatico No

    * Output salvato direttamente nella cartella del server World Wide Web.

  2. Nella home page del sito Web effettuare una delle seguenti operazioni:
    • creare un collegamento a ciascun snapshot di report utilizzando un tag HTML di ancoraggio. Ad esempio: 
      <A HREF="http://SitoWeb/ReportWeb/catalogo.snp">report Catalogo</A>
    • Incorporare il controllo Snapshot Viewer in un tag HTML Object mediante l'applicazione Microsoft ActiveX Control Pad e impostare i parametri di controllo con Control Pad Object Editor, metodo che può essere utilizzato solo con Microsoft Internet Explorer 3.0 e versioni successive. Ad esempio: 
      <OBJECT ID="SnapshotViewer" WIDTH=640 HEIGHT=480
 CLASSID="CLSID:F0E42D60-368C-11D0-AD81-00A0C90DC8D9">
    <PARAM NAME="_ExtentX" VALUE="16722">
    <PARAM NAME="_ExtentY" VALUE="11774">
    <PARAM NAME="_Version" VALUE="65536">
    <PARAM NAME="SnapshotPath"
 VALUE="http://SitoWeb/ReportWeb/catalogo.snp">
    <PARAM NAME="Zoom" VALUE="0">
    <PARAM NAME="AllowContextMenu" VALUE="-1">
    <PARAM NAME="ShowNavigationButtons" VALUE="-1">
</OBJECT>

Note

  • Se il report si basa su una query con parametri, sarà necessario modificare la macro per l'esecuzione in un file batch. È possibile utilizzare l'azione InviaTasti per passare le informazioni sui parametri oppure è possibile modificare il report rimuovendo la query con parametri e utilizzando criteri specifici.

  • È anche possibile creare un programma di Visual Basic, Applications Edition che svolga la stessa operazione.
