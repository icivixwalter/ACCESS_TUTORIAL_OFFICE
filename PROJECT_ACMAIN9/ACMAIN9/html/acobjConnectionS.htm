Oggetto Connection, insieme Connection

Quando si lavora con oggetti ADO (ActiveX Data Objects) in Microsoft Access, è spesso richiesto l'uso di una variabile oggetto Connection che rappresenta il collegamento al provider del database corrente. Usare la proprietà Connection dell'oggetto CurrentProject per restituire un oggetto Connection del database in cui si sta lavorando. Questo oggetto Connection viene accodato automaticamente all'insieme Connections.

Ad esempio, si supponga di lavorare in Microsoft Access con il database di esempio Northwind. È possibile creare un oggetto Connection che effettua il collegamento al provider di quel database dichiarando innanzitutto una variabile oggetto Connection, quindi facendo in modo che faccia riferimento all'oggetto Connection restituito dalla proprietà Connection dell'oggetto CurrentProject.

Dim cnn As ADODB.Connection
Set cnn = CurrentProject.Connection

Non è necessario conoscere il nome del provider per utilizzare il database corrente. Per conoscere il nome del provider corrente, controllare la proprietà ConnectionString dell'oggetto Connection, che contiene il nome del provider e la stringa di collegamento al database. Per trovarne la posizione nell'insieme Connections, esaminare l'insieme.

Dall'interno di un codice di Visual Basic è possibile creare più variabili oggetto Connection indipendenti per rappresentare i collegamenti a più database. Ciò consente di manipolare a livello del codice più database contemporaneamente. È possibile anche creare più variabili di oggetto Connection e fare in modo che facciano riferimento al database corrente.

Nota   Ogni riferimento alla proprietà Connection restituirà un nuovo oggetto collegamento che fa riferimento al collegamento attivo. Per eseguire più azioni su quel collegamento, ad esempio transazioni, assegnare Connection ad una variabile come descritto in precedenza.
