Informazioni sulla conversione di un database di Access 2000 nel formato Access 97

È possibile convertire un database di Microsoft Access 2000 in un database di Access 97 se non appartiene a un set di repliche. Quando si esegue la conversione nel formato Access 97, si perdono le funzionalità specifiche di Access 2000.

In seguito alla conversione di un database di Access 2000 nel formato Access 97, il database ottenuto subisce le modifiche di seguito elencate:

  • I collegamenti alle pagine di accesso ai dati vengono persi.

  • È possibile che i dati basati sul supporto Unicode in Access 2000 non vengano convertiti in modo corretto. Access 97 utilizza i caratteri di un'unica tabella codici, vale a dire un set numerato di 256 caratteri. È possibile ad esempio che un database di Access 97 utilizzi unicamente i caratteri ANSI. Durante la conversione la sequenza di ordinamento del database di Access 2000 originale determina il set di caratteri che il nuovo database di Access 97 potrà utilizzare. Tutti i caratteri non appartenenti a questo set non verranno convertiti in modo corretto.

  • Un database di Access 2000 contenente una tabella con una proprietà Dimensione campo impostata a Decimale non può essere convertito nel formato Access 97. È necessario impostare la proprietà Dimensione campo a un altro valore, ad esempio Precisione singola o Precisione doppia, oppure cambiare il tipo di dati del campo in Valuta prima di procedere alla conversione del database.

Per convertire un database di Access protetto, l'account utente utilizzato per accedere deve disporre delle seguenti autorizzazioni:

  • Autorizzazioni di Apertura/esecuzione e Apertura esclusiva per il database stesso. Autorizzazione di Lettura struttura per tutti gli oggetti nel database.

  • Per proteggere il database convertito, aprirlo con Microsoft Access 97 e applicare la protezione a livello utente.

Se il database di Access è protetto da password, è possibile convertirlo senza rimuovere la password. Se il codice Visual Basic, Applications Edition (VBA) è protetto da password, è necessario fornire la password in ambiente Visual Basic.
