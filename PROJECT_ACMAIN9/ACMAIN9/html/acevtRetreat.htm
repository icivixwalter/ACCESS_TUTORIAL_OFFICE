Evento Riesame

                                   

L'evento Riesame si verifica quando Microsoft Access ritorna a una sezione di report precedente durante la formattazione del report.

Nota   L'evento Riesame si applica a tutte le sezioni di report tranne le intestazioni di pagina e i piè di pagina.

Osservazioni

Per eseguire una macro o una routine evento quando si verifica tale evento, impostare la proprietà Su riesame al nome della macro o a [Routine evento].

In certi casi si rende necessario ritornare a una sezione di report precedente per determinare la posizione di determinati controlli e determinate sezioni in un report e se questi si adattano allo spazio consentito. Vengono riportati di seguito alcuni esempi:

  • Gruppi (tranne le intestazioni e i piè di pagina di pagina) per cui la proprietà Stampa sezione unita è impostata a Gruppo intero o Con primo corpo.

  • Sottoreport o sottomaschere per cui la proprietà Espandibile e/o Riducibile è impostata a Sì.

  • Sezioni dell'ultima pagina di un report.

In tali situazioni l'evento Formattazione si verifica appena Microsoft Access determina come vengono sistemate le sezioni sulla pagina stampata. Non si verifica tuttavia un evento Stampa, poiché le sezioni non sono ancora stampate. Se non è possibile stampare le sezioni, Microsoft Access ritorna alla posizione che rende possibile la stampa delle sezioni nella pagina successiva. L'evento Riesame si verifica per ciascuna sezione già trattata da Microsoft Access. L'evento Formattazione quindi si verifica di nuovo per ciascuna sezione mentre Microsoft Access si prepara a stampare realmente le sezioni.

Ogni volta che Microsoft Access raggiunge la fine dell'ultima pagina durante la formattazione di un report, si verifica ad esempio l'evento Riesame per ciascuna sezione precedente fino a quando Microsoft Access raggiunge la prima sezione posta in cima all'ultima pagina. Quindi si verifica di nuovo l'evento Formattazione per ciascuna sezione della pagina, seguito dall'evento Stampa.

È possibile eseguire una routine evento o una macro quando si verifica l'evento Riesame per annullare qualsiasi modifica effettuata quando si è verificato l'evento Formattazione per la sezione. Ciò risulta utile quando la routine evento o la macro Formattazione esegue azioni che si desidera effettuare una sola volta per ciascuna sezione, come ad esempio il calcolo del totale delle pagine o il controllo della dimensione di una sezione.

L'evento Riesame risulta inoltre utile per conservare la posizione degli elementi di un report. Il report Vendite per anno nel database di esempio Northwind ad esempio utilizza tale evento per determinare se un'intestazione di pagina sia o meno stampata. L'intestazione di pagina viene stampata sulle pagine successive a quella dove è stata stampata l'intestazione di gruppo e non viene stampata sulla pagina successiva a quella dove è stato stampato il piè di pagina di gruppo.
