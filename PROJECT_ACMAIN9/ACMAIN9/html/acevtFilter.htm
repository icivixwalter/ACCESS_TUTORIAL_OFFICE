Evento Filtro

Evento Filtro

                                   

L'evento Filtro può verificarsi in un progetto di Microsoft Access (adp) o in un database di Microsoft Access (mdb).

In un database di Access l'evento Filtro si verifica quando si effettua una delle seguenti operazioni:

  • In visualizzazione Maschera si sceglie Filtro dal menu Record, quindi Filtro in base a maschera oppure si fa clic su Filtro in base a maschera sulla barra degli strumenti. Ciò consente di aprire la finestra Filtro in base a maschera che permette di creare un filtro in base ai campi nella maschera.

  • In visualizzazione Maschera si sceglie Filtro dal menu Record, quindi Ordinamento/filtro avanzato. Ciò consente di aprire la finestra Ordinamento/filtro avanzato che permette di creare filtri complessi per la maschera.

  • Si sceglie Ordinamento/filtro avanzato dal menu Filtro quando la finestra Filtro in base a maschera è aperta oppure Filtro in base a maschera dal menu Filtro quando la finestra Ordinamento/filtro avanzato è aperta. Tale operazione provoca l'esecuzione dell'evento ApplicazioneFiltro quando viene chiusa la finestra di filtro aperta e l'esecuzione dell'evento Filtro quando viene aperta l'altra finestra del filtro.

In un progetto di Access un evento Filtro si verifica quando si effettua una delle seguenti operazioni:

  • In visualizzazione Maschera si sceglie Filtro dal menu Record, quindi Filtro in base a maschera oppure si fa clic su Filtro in base a maschera sulla barra degli strumenti. In questo modo viene aperta la finestra Filtro in base a maschera in cui è possibile creare un filtro basato sui campi della maschera.

  • In visualizzazione Maschera si sceglie Filtro dal menu Record, quindi si fa clic su Filtro su server in base a maschera sulla barra degli strumenti. In questo modo viene aperta la finestra Filtro su server in base a maschera in cui è possibile creare velocemente un filtro su server basato sui campi della maschera.

  • La finestra Ordinamento/filtro avanzato non è disponibile per i progetti di Access.

Osservazioni

Per eseguire una macro o una routine evento quando si verifica questo evento, impostare la proprietà Su filtro al nome della macro o su [Routine Evento].

L'evento Filtro può essere utilizzato per:

  • Rimuovere i precedenti filtri per la maschera. Per fare ciò, impostare la proprietà Filtro o Filtro su server della maschera a una stringa a lunghezza zero (" ") nella macro Filtro o nella routine evento. Ciò risulta particolarmente utile per accertarsi che il nuovo filtro non contenga criteri estranei. Quando ad esempio si utilizza la funzione Filtro in base a selezione, il criterio utilizzato (il testo selezionato nella maschera) viene aggiunto all'espressione della proposizione WHERE della proprietà Filtro o Filtro su server e visualizzato nelle finestre Filtro in base a maschera e Ordinamento/filtro avanzato. È possibile rimuovere i vecchi criteri tramite l'evento Filtro.

  • Immettere le impostazioni predefinite per il nuovo filtro. Per fare ciò, impostare la proprietà Filtro o Filtro su server in modo che includa tali criteri. È possibile ad esempio fare in modo che tutti i filtri di una maschera Prodotti visualizzino soltanto i prodotti correnti (quelli per i quali non è selezionato il controllo Sospeso nella maschera Prodotti).

  • Utilizzare la finestra del filtro personalizzata anziché quelle di Microsoft Access. Quando si verifica l'evento Filtro, è possibile aprire la maschera personalizzata e utilizzare le relative immissioni per impostare la proprietà Filtro o Filtro su server e filtrare la maschera originale. Per applicare il filtro, quando si chiude la maschera personalizzata, impostare la proprietà Applica filtro o Filtro su server da maschera della maschera originale a Vero (-1). Annullando l'evento Filtro, si impedisce l'apertura della finestra del filtro di Microsoft Access.

  • Impedire a determinati controlli della maschera di apparire o essere utilizzati nella finestra Filtro in base a maschera o Filtro su server in base a maschera. Se si nasconde o si disattiva un controllo nella macro Filtro o nella routine evento, il controllo viene nascosto o disattivato nella finestra Filtro in base a maschera o Filtro su server in base a maschera e non può essere utilizzato per impostare i criteri del filtro. È possibile utilizzare l'evento ApplicazioneFiltro per mostrare o attivare il controllo dopo che il filtro è stato applicato o rimosso dalla maschera.
