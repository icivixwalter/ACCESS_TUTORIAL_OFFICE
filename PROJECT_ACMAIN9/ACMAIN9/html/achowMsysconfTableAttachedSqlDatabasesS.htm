Utilizzare la tabella MSysConf in un database di Microsoft Access con dei database SQL collegati

Nota   È possibile collegare una tabella solo in un database di Microsoft Access, non in un progetto di Microsoft Access.

Se si amministra un database SQL che utilizza Microsoft Access come front-end, è possibile creare nel database SQL una tabella MSysConf, che consente di controllare la comunicazione fra le due applicazioni. La tabella MSysConf ha due possibili funzioni:

  • Consente di disattivare la funzione di salvataggio dell'ID e della password di accesso per un database SQL collegato nel front-end di Microsoft Access.

  • Consente di ottimizzare la funzione di riempimento dei record in background nei tempi di attesa impostando il numero di righe di dati che vengono recuperate in una sola volta e il numero di secondi di intervallo fra un recupero e l'altro. Se non si crea la tabella MSysConf, viene utilizzata un'impostazione predefinita di 100 record restituiti ogni 10 secondi.

Struttura della tabella MSysConf

La tabella MSysConf del database SQL presenta in genere la struttura riportata di seguito.

Nome di colonna Tipo di dati Valori Null
Config Tipo di dati che corrisponde a un valore intero di 2 byte No
chValue VARCHAR(255) Si
nValue Tipo di dati che corrisponde a un valore intero di 4 byte Si
Comments VARCHAR(255) Si

Nota   Se nell'origine dati utilizzata viene fatta distinzione fra maiuscole e minuscole, utilizzare i nomi di tabella e di colonna esattamente come indicato.

Tutti gli utenti devono essere autorizzati a utilizzare l'istruzione Select in questa tabella e soltanto l'amministratore di sistema ha in genere l'autorizzazione per l'inserimento, l'aggiornamento e l'eliminazione.

Dati della tabella MsysConf

Esistono tre record validi nella tabella MSysConf. Nella tabella riportata di seguito sono mostrati i valori da immettere nei campi Config e nValue. Le altre colonne sono riservate a un utilizzo futuro e i relativi contenuti non sono noti.

Config nValue Significato
101 0 Non consentono la memorizzazione locale dell'ID e della password di accesso in tabelle collegate.
101 1 Consentono la memorizzazione locale dell'ID e della password di accesso in tabelle collegate.
102 D D è l'intervallo espresso in secondi fra un recupero e l'altro.
103 N N è il numero di righe recuperate.

Note

  • L'impostazione di un intervallo di tempo più lungo consente di diminuire il traffico sulla rete, ma aumenta l'intervallo di tempo durante il quale sono attivi dei meccanismi che impediscono la lettura dei dati, sempre che il server utilizzi tali meccanismi.

  • Se si crea una tabella MSysConf, deve essere impostata correttamente, altrimenti la connessione al database SQL non sarà possibile.
