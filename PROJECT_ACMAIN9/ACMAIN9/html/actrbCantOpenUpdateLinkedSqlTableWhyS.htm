Impossibile aprire o aggiornare una tabella SQL collegata

  • Un indice univoco è richiesto per aggiornare, eliminare o inserire dei dati in una tabella o una visualizzazione SQL collegata. Quando si collega una visualizzazione aggiornabile di Microsoft SQL Server o una tabella di un database SQL che non dispone di un indice univoco, viene richiesto di selezionare uno o più campi per generare un indice univoco. Il programma, tuttavia, non consente di verificare se il campo o i campi selezionati identifichino ciascun record in modo univoco. Se nell'indice generato vi sono dei duplicati, non sarà possibile aggiornare quei record.

    Se si pensa che il campo o i campi selezionati non abbiano creato un indice univoco, eliminare il collegamento alla tabella, effettuarlo nuovamente, quindi selezionare il campo o i campi appropriati per stabilire un indice univoco.

    In alternativa, è possibile generare un indice all'interno di Microsoft Access, creando una query di definizione dati utilizzando l'istruzione Create Index. Assicurarsi, quindi, che il campo o la combinazione di campi scelti identifichino ciascun record in modo univoco. Se l'indice generato contiene dei valori duplicati, non sarà possibile alcun aggiornamento della tabella. Per eliminare l'indice, utilizzare un'altra query di definizione dati. Per ulteriori informazioni sulla creazione di questo tipo di query, fare clic su .

  • Se si collega una tabella SQL che dispone di un proprio indice univoco definito per un campo rappresentato da un numero a virgola mobile, la tabella potrebbe non essere aggiornabile. Poiché il livello di precisione con cui i server trattano dei dati a virgola mobile varia, può verificarsi, a volte, una minore precisione. La differenza è generalmente abbastanza lieve da non avere conseguenze: se, però, i dati costituiscono parte del segnalibro della tabella SQL, i record interessati potrebbero sembrare, per Microsoft Access, eliminati.

  • Se si incontrano altri errori mentre si importa, si collega o si utilizza una tabella di database SQL, potrebbe sorgere un problema con l'account sul server di database SQL o con lo stesso database. Se l'accesso ad una tabella di database SQL è negato, contattare l'amministratore di database SQL.
