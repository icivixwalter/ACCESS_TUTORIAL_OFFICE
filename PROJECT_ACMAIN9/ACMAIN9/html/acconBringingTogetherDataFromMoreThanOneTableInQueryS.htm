Metodi per collegare dati in una query da più tabelle o query

L'efficacia delle query consiste nella capacità di collegare o di eseguire un comando su dati di più tabelle o query. È possibile ad esempio visualizzare informazioni su un cliente e sui relativi ordini effettuati. Per visualizzare tali informazioni sono necessari i dati delle tabelle Clienti e Ordini.

Quando si aggiungono a una query più tabelle o query, è necessario assicurarsi che i relativi elenchi di campi siano collegati l'uno all'altro con una linea di join, in modo che in Microsoft Access sia possibile connettere le informazioni.

Se le tabelle in una query non sono collegate l'una all'altra con join, direttamente o indirettamente, i record a esse associate non vengono riconosciuti, pertanto vengono visualizzate tutte le combinazioni di record (operazione definita "tra prodotti" o "prodotto cartesiano") tra le due tabelle. Se ciascuna tabella contenesse quindi 10 record, i risultati della query conterrebbero 100 record (10X10). Ciò significa inoltre che l'esecuzione della query potrebbe richiedere molto tempo e in definitiva produrre dei risultati meno significativi.

Se sono state create in precedenza delle relazioni tra tabelle nella finestra Relazioni, vengono visualizzate automaticamente delle linee di join quando si aggiungono tabelle correlate in visualizzazione Struttura della query. Se viene applicata l'integrità referenziale, viene visualizzato anche un "1" sulla linea di join per mostrare quale tabella si trova sul lato "uno" di una relazione uno-a-molti e un simbolo di infinito per mostrare quale tabella si trovi sul lato "molti".

Anche se non sono state create relazioni tra tabelle, vengono creati automaticamente dei join se si aggiungono due tabelle a una query, se le tabelle contengono ciascuna un campo con il tipo di dati uguale o compatibile e se uno dei campi di join è una chiave primaria. In questo caso i simboli "uno" e "molti" non vengono visualizzati, in quanto non viene applicata l'integrità referenziale.

A volte le tabelle aggiunte alla query non includono campi che possono essere collegati con join. In tal caso è necessario aggiungere una o più tabelle o query che servano soltanto da ponte tra le tabelle di cui si desidera utilizzare i dati. Se ad esempio si aggiungono le tabelle Clienti e Dettagli ordini a una query, queste non avranno tra di loro una linea di join perché non contengono campi che possono essere collegati con join. La tabella Ordini è invece correlata a entrambe le tabelle, pertanto è possibile includerla nella query per fornire una connessione tra le altre due.

Dopo aver collegato con join tabelle e query e aver aggiunto i campi da entrambe le tabelle o query alla griglia di struttura in visualizzazione Struttura della query, il join predefinito indica alla query di ricercare i valori corrispondenti nei campi di join. Questa operazione viene definita join interno nella terminologia del database. Quando si trovano corrispondenze, questi due record vengono combinati e visualizzati come un unico record nei risultati della query. Se una tabella o una query non ha un record corrispondente nell'altra tabella o query, nei risultati della query non viene visualizzato nessuno dei due record. Se si desidera che la query selezioni tutti i record da una tabella o da una query sia che abbia o meno dei record corrispondenti nell'altra tabella o query, è possibile modificare il tipo di join.

Esempi di diversi tipi di join e della relativa influenza sui risultati delle query Se si desidera che la query selezioni dei record basati sul valore nel campo di join maggiore di, minore di, diverso da, maggiore o uguale a oppure minore o uguale al valore nell'altro campo di join, è necessario creare un'istruzione SQL in visualizzazione SQL.
