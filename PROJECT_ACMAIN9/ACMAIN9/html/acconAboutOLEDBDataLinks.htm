Microsoft Access e l'architettura OLE DB

OLE DB è un'architettura di database che consente l'accesso in rete o tramite Internet a numerosi tipi di origini dati, inclusi i dati relazionali, i file di posta, i file flat e i fogli di calcolo. Nell'architettura OLE DB l'applicazione che accede ai database viene definita "consumatore di dati", ad esempio Microsoft Access 2000 o un programma Microsoft Visual Basic 6.0, mentre il programma che consente l'accesso nativo ai dati viene definito "provider di database", ad esempio Microsoft OLE DB Provider for SQL Server o Microsoft Jet 4.0 OLE DB Provider.

Definizione delle informazioni di connessione

Un Data Link è l'insieme delle informazioni di connessione che un "consumatore di dati" utilizza per accedere a un'origine dati mediante il provider OLE DB di quell'origine. Nell'interfaccia utente di Microsoft Access si utilizza la finestra di dialogo Proprietà di Data Link per definire un Data Link per un provider OLE DB con le seguenti modalità:

  • Per connettersi a un database di Microsoft SQL Server quando si crea un progetto di Microsoft Access utilizzando il comando Nuovo del menu File o la Creazione guidata Database. Per ulteriori informazioni, vedere l'argomento relativo alla creazione di un progetto di Access.

  • Per ridefinire le informazioni di connessione per un progetto di Access utilizzando il comando Connessione del menu File. Ciò consente di passare con facilità da un database di prova a un database di produzione o da un server primario a uno secondario. Per ulteriori informazioni, vedere l'argomento relativo alla modifica della connessione a un progetto di Access.

  • Per ridefinire le informazioni di connessione di una pagina di accesso ai dati esistente in visualizzazione Struttura della pagina facendo clic con il pulsante destro del mouse sull'Elenco campi e scegliendo Connessione dal menu di scelta rapida. Quando si crea una pagina di accesso ai dati, Microsoft Access definisce automaticamente le informazioni di connessione connettendosi al database o al progetto di Access al momento aperto. Per ulteriori informazioni, vedere l'argomento relativo alla modifica della connessione a una pagina.

  • Per connettere una pagina Web aperta mediante la finestra di dialogo Apri (disponibile scegliendo Apri dal menu File) dalla finestra di un database vuoto.

  • Per creare un elenco di tabella pivot o modificare la connessione a un elenco di tabella pivot.

Ciascun provider OLE DB definisce informazioni di connessione specifiche. Microsoft OLE DB Provider for SQL Server ad esempio richiede il nome e il percorso di un server e un nome utente, ma è anche possibile definire informazioni aggiuntive, quale una password o se utilizzare o meno la protezione integrata di Microsoft Windows NT.

In generale per definire le informazioni di connessione nella finestra di dialogo Proprietà di Data Link:

  • Fare clic sulla scheda Provider, selezionare il provider OLE DB, quindi scegliere il pulsante Avanti per visualizzare la scheda Connessione e immettere informazioni di connessione specifiche per quel provider OLE DB. A seconda di come si accede alla finestra di dialogo Proprietà di Data Link, è possibile che la scheda Provider non sia disponibile. Access definisce automaticamente il provider come il database di Access al momento aperto o come database di SQL Server connesso al progetto di Access aperto.

  • Fare clic sulla scheda Avanzate per fornire ulteriori informazioni, ad esempio le impostazioni di rete e le autorizzazioni di accesso.

  • Fare clic sulla scheda Tutti per definire le proprietà di inizializzazione per il provider OLE DB.

Nel file della Guida relativo alla finestra di dialogo Proprietà di Data Link (msdasc.hlp) sono contenute ulteriori informazioni sull'utilizzo della finestra di dialogo e sulla definizione delle informazioni di connessione per ciascun provider OLE DB.

Creazione e accesso ai file Data Link Microsoft

Anche se in genere ci si connette a un provider OLE DB utilizzando la finestra di dialogo Proprietà di Data Link, è possibile definire le informazioni di connessione per un provider OLE DB in un file Data Link Microsoft (udl) da utilizzare in un programma Visual Basic e per organizzare e gestire le informazioni di connessione sul proprio computer.

Per creare le informazioni di connessione in questo modo, aprire Esplora risorse (Gestione risorse in Windows 95 e Windows NT 4.0), scegliere Microsoft Data Link dal menu Nuovo, creare un file Data Link Microsoft, selezionare il nome del file, farvi clic con il pulsante destro del mouse, quindi scegliere Proprietà dal menu di scelta rapida.

Installazione del provider OLE DB

Microsoft Jet 4.0 OLE DB Provider e Microsoft OLE DB Provider per SQL Server vengono automaticamente installati con Microsoft Office 2000 o Microsoft Access 2000 su computer autonomi. Vengono inoltre installati provider OLE DB aggiuntivi quali OLE DB Provider for ODBC Drivers e OLE DB Provider for Oracle. Per un elenco completo dei provider OLE DB installati nel computer, visualizzare la finestra di dialogo Proprietà - Data Link da un file Data Link e fare clic sulla scheda Provider.

Quando si visualizza una pagina con Microsoft Internet Explorer 5 o versioni successive e si dispone di una licenza Microsoft Office 2000 sul computer, vengono scaricati automaticamente i provider OLE DB necessari e altri file correlati installando Microsoft Office Web Components.

Programmazione OLE DB

È anche possibile accedere ai dati a livello di programmazione mediante un provider OLE DB in Visual Basic, Applications Edition in Microsoft ActiveX Data Objects (ADO) definendo nella proprietà ConnectionString del metodo Open dell'oggetto Connection una stringa di connessione che passi le informazioni di connessione al provider OLE DB. Per ulteriori informazioni, consultare il Manuale del programmatore di Microsoft Office 2000/Visual Basic.
