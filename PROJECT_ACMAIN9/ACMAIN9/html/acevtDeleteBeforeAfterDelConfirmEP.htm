Eventi Eliminazione, PrimaDiConfermaEliminazione, DopoConfermaEliminazione - Routine evento

Eventi Eliminazione, PrimaDiConfermaEliminazione, DopoConfermaEliminazione - Routine evento

        

Per creare una routine evento che viene eseguita quando si verifica l'evento Eliminazione, PrimaDiConfermaEliminazione o DopoConfermaEliminazione, impostare la proprietà Su eliminazione, Prima di conferma eliminazione o Dopo conferma eliminazione a [Routine evento] e fare clic sul pulsante dei generatori .

Sintassi

Private Sub Form_Delete(Cancel As Integer)

Private Sub Form_BeforeDelConfirm(Cancel As Integer, Response As Integer)

Private Sub Form_AfterDelConfirm(Status As Integer)

La routine evento Eliminazione utilizza l'argomento riportato di seguito.

Argomento Descrizione
Cancel L'impostazione determina l'eventuale verificarsi dell'evento Eliminazione. L'impostazione dell'argomento Cancel a True (-1) annulla l'evento Eliminazione.

La routine evento PrimaDiConfermaEliminazione utilizza gli argomenti riportati di seguito.

Argomento Descrizione
Cancel L'impostazione determina l'eventuale verificarsi dell'evento PrimaDiConfermaEliminazione. L'impostazione dell'argomento Cancel a True annulla l'evento PrimaDiConfermaEliminazione e impedisce la visualizzazione della finestra di dialogo Conferma eliminazione. Se l'evento viene annullato, vengono ripristinati i record iniziali, ma si verifica ancora l'evento DopoConfermaEliminazione. Se Cancel è impostato a True, l'argomento Response viene ignorato.
  Se Cancel è impostato a False (0), cioè l'impostazione predefinita, il valore dell'argomento Response viene utilizzato da Microsoft Access per determinare il tipo di risposta all'evento di eliminazione.
Response L'impostazione determina l'eventuale visualizzazione della finestra di dialogo Conferma eliminazione che richiede se è necessario eliminare il record. L'argomento Response può essere impostato a una delle costanti intrinseche riportate di seguito:
  Costante Descrizione
  acDataErrContinue Continua senza la visualizzazione della finestra di dialogo Conferma eliminazione. L'impostazione dell'argomento Cancel a False e dell'argomento Response a acDataErrContinue consente a Microsoft Access di eliminare i record senza effettuare richiesta all'utente.
  AcDataErrDisplay Impostazione predefinita. Visualizza la finestra di dialogo Conferma eliminazione.

La routine evento DopoConfermaEliminazione utilizza l'argomento riportato di seguito.

Argomento Descrizione
Status L'impostazione indica l'eventuale avvenuta eliminazione di un record. L'argomento Status può corrispondere a una delle costanti intrinseche riportate di seguito:
  Costante Descrizione
  acDeleteOK Indica l'avvenuta eliminazione.
  AcDeleteCancel Indica l'annullamento dell'eliminazione in Visual Basic.
  acDeleteUserCancel Indica l'annullamento dell'eliminazione da parte dell'utente.

Osservazioni

Non è possibile annullare l'evento DopoConfermaEliminazione.

È possibile utilizzare questi eventi per personalizzare le operazioni di eliminazione di record. Quando si elimina un record, ad esempio il record relativo a un impiegato, si verifica l'evento Eliminazione. È possibile immettere del codice nella routine evento Eliminazione per salvare i dati dei campi ID impiegato e Nome in un set di variabili temporanee. Quando si verifica l'evento PrimaDiConfermaEliminazione, è possibile impedire la visualizzazione della finestra di dialogo Conferma eliminazione impostando l'argomento Response ad acDataErrContinue. In tal modo si consente la visualizzazione di una finestra di dialogo personalizzata. Quando si verifica l'evento DopoConfermaEliminazione e l'argomento Status è impostato ad acDeleteOK, è possibile effettuare l'eliminazione delle variabili temporanee o salvare le informazioni relative agli impiegati in un'altra tabella.
