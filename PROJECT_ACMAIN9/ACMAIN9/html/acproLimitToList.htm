Proprietà LimitToList (Solo in elenco)

                 

È possibile utilizzare la proprietà LimitToList per limitare ai soli elementi elencati i valori contenuti in una casella combinata.

Impostazione

La proprietà LimitToList viene impostata come riportato di seguito.

Impostazione Visual Basic Descrizione
Sì True (-1) Se si seleziona un elemento dall'elenco contenuto nella casella combinata o si immette del testo corrispondente a un elemento elencato, Microsoft Access lo accetta. Il testo immesso non corrispondente a un elemento elencato non viene accettato. È necessario digitare di nuovo la voce, selezionare uno degli elementi elencati, premere ESC o scegliere Annulla dal menu Modifica.
No False (0) Impostazione predefinita. Microsoft Access accetta qualsiasi tipo di testo conforme alla proprietà ValidationRule.

È possibile impostare la proprietà LimitToList utilizzando la finestra delle proprietà di una casella combinata, una macro o Visual Basic.

Per i campi di tabelle con la proprietà DisplayControl impostata a Casella combinata o a Casella di riepilogo, è possibile impostare tale proprietà nella scheda Ricerca della sezione Proprietà campo della visualizzazione Struttura tabella.

Suggerimento   Microsoft Access imposta automaticamente la proprietà LimitToList quando nella visualizzazione Struttura della tabella si seleziona come tipo di dati per un campo Ricerca guidata.

Osservazioni

Una volta impostata a No la proprietà LimitToList di una casella combinata associata, è possibile immettere nella casella combinata un valore non incluso nell'elenco. Microsoft Access memorizza il nuovo valore nella tabella o nella query sottostante la maschera, nel campo indicato nella proprietà ControlSource della casella combinata e non nella tabella o nella query impostate per la casella combinata tramite la proprietà RowSource. Per visualizzare nella casella combinata i valori appena immessi, è necessario aggiungere il nuovo valore alla tabella o alla query impostata nella proprietà RowSource utilizzando una macro o una routine evento di Visual Basic che viene eseguita quando si verifica un evento NonInElenco.

Nota    Se si imposta la proprietà BoundColumn della casella combinata associata a una qualsiasi colonna che non sia la prima colonna visibile, o se la proprietà BoundColumn è impostata a 0, la proprietà LimitToList viene automaticamente impostata a Sì.

L'impostazione a Sì delle proprietà LimitToList e AutoExpand consente a Microsoft Access di trovare nell'elenco dei valori corrispondenti mentre l'utente immette caratteri nella parte casella di testo della casella combinata e limita le voci solo a tali valori.

Quando la proprietà LimitToList è impostata a Sì e si fa clic sulla freccia accanto alla casella combinata, Microsoft Access seleziona i valori corrispondenti nell'elenco man mano che l'utente immette dei caratteri nella parte casella di testo della casella combinata anche se la proprietà AutoExpand è impostata a No. Se si preme INVIO o ci si sposta a un altro record o a un altro controllo, il valore selezionato viene visualizzato nella casella combinata.

Le caselle combinate accettano valori di tipo Null quando la proprietà LimitToList è impostata a Sì o True, sia che l'elenco contenga valori di tipo Null o meno. Se si desidera evitare che gli utenti immettano un valore di tipo Null in una casella combinata, impostare a Sì la proprietà Required del campo della tabella al quale è associata la casella combinata.
