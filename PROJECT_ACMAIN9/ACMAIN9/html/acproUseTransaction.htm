Proprietà UseTransaction (Utilizza transazione)

                 

È possibile utilizzare la proprietà UseTransaction per specificare o determinare se una query di comando debba venire eseguita come transazione singola. In alcuni casi, impostando a No tae proprietà, è possibile accelerare l'esecuzione di alcune query.

Impostazione

La proprietà UseTransaction viene impostata come riportato di seguito.

Impostazione Visual Basic Descrizione
Sì True (-1) Impostazione predefinita. La query viene eseguita come transazione singola.
No False (0) La query non viene eseguita come transazione singola.

È possibile impostare la proprietà UseTransaction utilizzando la finestra delle proprietà della query o Visual Basic. Per impostare tale proprietà in un database di Microsoft Access (MDB) con Visual Basic, è prima necessario aggiungerla all'insieme Properties di un oggetto QueryDef utilizzando il metodo CreateProperty.

Osservazioni

Quando una query di creazione tabella, eliminazione, accodamento o aggiornamento ha la proprietà UseTransaction impostata a Sì, il modulo di gestione Microsoft Jet memorizza i record restituiti in una cache e, se necessario, i risultati in un database temporaneo su disco. Quando la query è terminata, il modulo di gestione Jet legge i record nel database temporaneo e li riscrive nel database di origine.

In alcuni casi è possibile ottenere delle prestazioni migliori impostando la proprietà UseTransaction a No. Quando la proprietà UseTransaction è impostata a No, non si verificano mai problemi relativi a un numero troppo alto di richieste di blocco. Inoltre il modulo di gestione del database Jet non memorizza i risultati delle query in un database temporaneo, migliorando sostanzialmente le prestazioni.

Se deve essere scritto nel database temporaneo un numero elevato di record, le prestazioni possono peggiorare a causa dell'esecuzione di una query di comando come transazione singola. Quando si eseguono le query di eliminazione o aggiornamento in un database condiviso, viene generato un numero elevato di richieste di blocco. Di conseguenza le prestazioni possono peggiorare e in alcuni sistemi operativi di rete, come Netware, l'esecuzione delle query può non avere esito positivo se il numero delle richieste di blocco è superiore a 10.000.

Se una transazione non può essere completata, viene visualizzato un messaggio di errore e viene chiesto se si desidera salvare eventuali modifiche effettuate.

Quando la proprietà UseTransaction è impostata a No, l'unico modo per annullare un'intera transazione consiste nell'utilizzare il metodo ADO RollbackTrans o DAO Rollback in Visual Basic.
