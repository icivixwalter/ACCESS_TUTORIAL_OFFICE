Pagine di accesso ai dati

La pagina di accesso ai dati viene creata come un oggetto di database che contiene un collegamento alla posizione del file HTML corrispondente. Per ulteriori informazioni, vedere l'argomento relativo alla creazione delle pagine di accesso ai dati.

Utilizzare le pagine per visualizzare, modificare, aggiornare, eliminare, filtrare, raggruppare e ordinare i dati direttamente da un database di Microsoft Access o da un database di Microsoft SQL Server in Microsoft Internet Explorer 5 o versioni successive. Una pagina può anche contenere controlli aggiuntivi, inclusi un foglio di calcolo, un elenco di tabella pivot e un grafico.

Per rendere disponibili le proprie pagine sul World Wide Web, è necessario pubblicarle nelle cartelle Web o su un server Web e rendere accessibile agli utenti delle pagine il database di Access o di SQL Server.

È sufficiente che Internet Explorer scarichi la pagina una sola volta dal server Web per consentire la visualizzazione e l'interazione con i dati in essa contenuti. Poiché la pagina utilizza il linguaggio HTML dinamico, l'accesso al database è particolarmente efficiente in ambienti client/server.

HTML generato dal server

È possibile creare file con codice HTML generato dal server, ASP o IDC/HTX da tabelle, query e maschere. I file con codice HTML generato dal server vengono visualizzati sotto forma di tabella nel browser Web. È possibile utilizzare questo tipo di file quando si desidera utilizzare un qualunque browser Web, quando i dati vengono modificati spesso, quando è necessario visualizzare i dati di una tabella connessa a un'origine dati ODBC, ma è sufficiente avere accesso ai dati in sola lettura.

Una volta inviati i file ASP o IDC/HTX come output, è necessario pubblicarli per poterli elaborare su un prodotto e su una piattaforma per server Web supportati.

Ogni volta che un utente apre o aggiorna un file ASP o HTX da un browser Web, il server Web crea un file HTML in modo dinamico, quindi lo invia al browser.

HTML statico

È possibile creare file con codice HTML statico da tabelle, query, maschere e report. In un browser Web i report vengono visualizzati nel formato report, mentre tabelle, query e maschere vengono visualizzate nel formato di foglio dati. Utilizzare i file con codice HTML statico quando si desidera utilizzare un browser Web che supporti il linguaggio HTML 3.2 o versioni successive e quando i dati non vengono modificati spesso.

Per rendere disponibili i file HTML sul World Wide Web, è necessario pubblicarli nelle cartelle Web o su un server Web.

Quando si accede ai dati mediante un browser Web, è sufficiente che il browser scarichi una sola volta il file HTML statico dal server Web per consentire la visualizzazione dei dati. I file HTML risultanti sono tuttavia uno snapshot dei dati al momento della pubblicazione. Nessuna origine dati ODBC è connessa al file HTML statico e, in caso di modifiche, è necessario effettuare di nuovo l'esportazione per poter visualizzare i nuovi dati in un browser Web.
