Proprietà ReturnsRecords (Restituisci record)

                 

È possibile utilizzare la proprietà ReturnsRecord in una query SQL di tipo pass-through per specificare se la query restituisce record. Se ad esempio la query SQL di tipo pass-through è SELECT * FROM IMPIEGATI, è possibile impostare la proprietà ReturnsRecord a Sì per visualizzare tutti i record relativi agli impiegati contenuti nel foglio dati della query.

Nota   La proprietà ReturnsRecord si applica solo a query di tipo pass-through.

Impostazione

La proprietà ReturnsRecord viene impostata come riportato di seguito.

Impostazione Visual Basic Descrizione
Sì True (-1) Impostazione predefinita. La query restituisce record, come una query di selezione.
No False (0) La query non restituisce record, come una query di comando.

È possibile impostare tale proprietà utilizzando la finestra delle proprietà della query o Visual Basic.

Suggerimento   È possibile accedere alla proprietà ReturnsRecords di una query di tipo pass-through in Visual Basic, ma il metodo migliore consiste nell'utilizzare la proprietà DAO ReturnsRecords.

Osservazioni

Una query di tipo pass-through è in grado di restituire record, oppure può essere utilizzata per modificare dati, creare un oggetto database, o eseguire un'azione, come una query di comando.

Quando si imposta a Sì la proprietà ReturnsRecords, è possibile utilizzare la query di tipo pass-through in un'altra query o come base per una casella combinata, per una casella di riepilogo, per una maschera o per un report. Se si imposta a No la proprietà ReturnsRecords, non è possibile utilizzare la query come base per un oggetto o per un controllo poiché non viene restituito alcun record.
