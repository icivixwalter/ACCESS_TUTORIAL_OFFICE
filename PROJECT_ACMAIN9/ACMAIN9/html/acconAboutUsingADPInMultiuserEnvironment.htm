Utilizzo di progetti di Access in un ambiente multiutente

In un ambiente multiutente ogni utente deve avere a disposizione una propria copia del file del progetto di Access connesso allo stesso database di Microsoft SQL Server. Non è possibile condividere lo stesso file del progetto di Access tra due o più utenti.

Quando si apre un progetto di Access, si attiva sempre la modalità esclusiva. Se un altro utente ha già aperto un progetto di Access e si tenta di aprire lo stesso progetto, Access avverte che il file è stato aperto in modalità esclusiva e dà la possibilità di aprirne una copia in sola lettura.

In pratica, quando si esegue un'applicazione condivisa, si fornisce semplicemente una copia del file del progetto di Access a ogni utente dell'applicazione. In questo caso ogni utente dispone di una copia di maschere, report, pagine di accesso ai dati, macro e moduli. Gli utenti stanno di fatto condividendo un database di SQL Server e le tabelle, le visualizzazioni, le stored procedure e i diagrammi di database che risiedono in quel database. Pertanto, in un ambiente multiutente, è importante fornire una protezione adeguata per questi oggetti di database condivisi in modo che la loro integrità non venga compromessa. Per ulteriori informazioni, vedere l'argomento relativo alla protezione di un progetto di Access.

Gestione di aggiornamenti concomitanti

In un ambiente multiutente è possibile che nello stesso momento più utenti lavorino sullo stesso record. Poiché alcuni utenti possono modificare o eliminare gli stessi dati che un altro sta utilizzando, è possibile che di tanto in tanto si verifichino dei conflitti. I progetti di Access utilizzano una tecnica chiamata "blocco dei record ottimistico" per gestire le priorità relative ai record.

Se un utente modifica un record e un altro aggiorna lo stesso record prima che venga salvato dal primo utente, un messaggio avverte che se si salva il record, le modifiche apportate dall'altro utente verranno sovrascritte. Procedere in uno dei seguenti modi:

  • Scegliere il pulsante Salva il record per salvare il record e sovrascrivere le modifiche dell'altro utente.

  • Scegliere il pulsante Copia negli Appunti per esaminare le modifiche apportate dall'altro utente in modo da poterle allineare adeguatamente con le proprie.

  • Scegliere il pulsante Non salvare le modifiche per ignorare le proprie modifiche e accettare quelle dell'altro utente.

Poiché è possibile che altri utenti aggiungano, modifichino o eliminino i record a cui si sta accedendo, è opportuno aggiornare periodicamente la visualizzazione dei dati in visualizzazione Foglio dati o Maschera utilizzando il comando Aggiorna del menu Record per eseguire di nuovo le query e vedere i dati aggiornati.

Condivisione di un database di Microsoft SQL su Internet

È possibile creare pagine di accesso ai dati oppure inviare come output uno o più oggetti di database in codice HTML statico o generato dal server, visualizzare queste pagine Web in un browser, ad esempio Microsoft Internet Explorer, e accedere ai dati tramite queste pagine Web. Per ulteriori informazioni, vedere l'argomento relativo ai tre tipi di pagine Web create da Access.

Replica di un database di SQL Server

Se si utilizzano due computer, ad esempio un computer desktop e uno portatile, è possibile eseguire delle repliche del database di SQL Server e mantenerle in sincronia. Molti utenti in diverse postazioni possono lavorare contemporaneamente sulle proprie copie di un database master e quindi sincronizzarle sulla rete, tramite una connessione di accesso remoto o Internet. Per ulteriori informazioni, vedere l'argomento relativo alla replica di un progetto di Access.
