Proprietà BoundColumn (Colonna associata)

                 

Quando si effettua una selezione da una casella di riepilogo o da una casella combinata, la proprietà BoundColumn indica a Microsoft Access quali tra i valori della colonna utilizzare come valore del controllo. Se il controllo è associato a un campo, nel campo indicato nella proprietà ControlSource viene memorizzato il valore contenuto nella colonna specificata dalla proprietà BoundColumn.

Impostazione

La proprietà BoundColumn viene impostata come riportato di seguito.

Impostazione Descrizione
0 Nel record corrente viene memorizzato il valore della proprietà ListIndex e non il valore contenuto nella colonna. Il valore della proprietà ListIndex della prima riga è zero, quello della seconda è 1 e così via. Microsoft Access imposta la proprietà ListIndex quando si seleziona un elemento da una casella di riepilogo o dalla parte casella di riepilogo di una casella combinata. L'impostazione a zero della proprietà BoundColumn e l'utilizzo del valore della proprietà ListIndex del controllo può risultare utile se si desidera solo memorizzare una sequenza di numeri.
1 o maggiore L'impostazione predefinita è 1. Il valore della colonna specificata diventa il valore del controllo. Se il controllo è associato ad un campo, l'impostazione viene memorizzata in quel campo nel record corrente. La proprietà BoundColumn non può essere impostata a un valore maggiore di quello dell'impostazione della proprietà ColumnCount.

È possibile impostare la proprietà BoundColumn utilizzando la finestra delle proprietà del controllo, una macro o Visual Basic.

Per i campi di tabelle, con la proprietà DisplayControl impostata a Casella combinata o Casella di Riepilogo, è possibile impostare tale proprietà nella scheda Ricerca della sezione Proprietà campo della visualizzazione Struttura della tabella.

Suggerimento   La proprietà BoundColumn viene impostata automaticamente quando in visualizzazione Struttura della tabella si seleziona come tipo di dati per un campo Ricerca guidata.

In Visual Basic impostare la proprietà BoundColumn utilizzando un numero o un'espressione numerica uguale ad un valore compreso tra zero e il numero impostato per la proprietà ColumnCount.

Osservazioni

La colonna visibile all'estrema sinistra di una casella combinata, cioè la colonna all'estrema sinistra la cui impostazione nella casella combinata della proprietà ColumnWidths non sia 0, contiene i dati che, nella visualizzazione Maschera o in un report, vengono visualizzati nella parte casella di testo di una casella combinata. La proprietà BoundColumn determina quale valore della colonna di una casella di testo o di un elenco della casella combinata viene memorizzato quando si effettua una selezione. Ciò consente di visualizzare dati diversi rispetto a quelli memorizzati come valore del controllo.

Nota   Se la colonna associata non contiene gli stessi valori di quella visibile all'estrema sinistra del controllo, o se imposta la proprietà BoundColumn a 0, la proprietà LimitToList è impostata a Sì.

In Microsoft Access si utilizzano numeri che iniziano da zero per fare riferimento alle colonne della proprietà Column. Ciò significa che per fare riferimento alla prima colonna viene utilizzata l'espressione Column(0), per fare riferimento alla seconda colonna Column(1)e così via. La proprietà BoundColumn, tuttavia, per fare riferimento alle colonne utilizza numeri che iniziano da uno. Ciò significa che, se la proprietà BoundColumn è impostata a uno, è possibile accedere al valore memorizzato in tale colonna utilizzando l'espressione Column(0).

Se la proprietà AutoExpand è impostata a Sì, Microsoft Access inserisce automaticamente nella parte casella di testo della casella combinata un valore che, mentre viene digitato, corrisponde a un valore presente.
