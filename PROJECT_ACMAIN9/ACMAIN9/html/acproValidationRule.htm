Proprietà ValidationRule, ValidationText (Valido se, Messaggio errore)

                 

È possibile utilizzare la proprietà ValidationRule per specificare i requisiti relativi a dati immessi in un record, campo o controllo. Quando si immettono dati che violano l'impostazione della proprietà ValidationRule, è possibile specificare il messaggio da visualizzare all'utente impostando la proprietà ValidationText.

Nota   Le proprietà ValidationRule e ValidationText non vengono applicate a controlli casella di controllo, pulsante di opzione o interruttore, quando questi fanno parte di un gruppo di opzioni. Tali proprietà si applicano solo al gruppo di opzioni stesso.

Impostazione

Immettere un'espressione per l'impostazione della proprietà ValidationRule e il testo per l'impostazione della proprietà ValidationText. La lunghezza massima per l'impostazione della proprietà ValidationRule è di 2048 caratteri, quella della proprietà ValidationText è di 255 caratteri.

Per i controlli è possibile impostare la proprietà ValidationRule a una qualsiasi espressione valida. Per le regole di convalida relative a campi e record l'espressione non può includere funzioni definite dall'utente, funzioni di aggregazione o di aggregazione sui domini, la funzione Eval o il metodo CurrentUser o riferimenti a maschere, query o tabelle. Le espressioni relative alle regole di convalida dei campi non possono inoltre includere riferimenti ad altri campi. Per i record, le espressioni possono includere riferimenti a campi presenti nella tabella.

È possibile impostare le proprietà ValidationRule e ValidationText:

  • Nella sezione Proprietà campo della visualizzazione Struttura della tabella (per una regola di convalida di campi).

  • Nella finestra delle proprietà di una tabella scegliendo Proprietà dal menu Visualizza in visualizzazione Struttura della tabella (per una regola di convalida di record).

  • Nella finestra della proprietà per un controllo di una maschera.

  • Utilizzando una macro o Visual Basic. Per impostare tali proprietà in Visual Basic, utilizzare un'espressione stringa.

Per campi di tabella e record è inoltre possibile impostare tali proprietà con Visual Basic utilizzando la proprietà DAO ValidationRule.

Osservazioni

I valori vengono automaticamente convalidati sulla base del tipo di dati di un campo. Non è consentito, ad esempio, immettere testo in un campo numerico. Utilizzando la proprietà ValidationRule è possibile impostare più regole specifiche.

Se viene impostata la proprietà ValidationRule, ma non la proprietà ValidationText, nel caso venga violata la regola di convalida viene visualizzato un messaggio di errore standard. Se viene impostata la proprietà ValidationText, il messaggio di errore è rappresentato dal testo immesso.

Quando si aggiunge ad esempio un record relativo a un nuovo impiegato, è possibile immettere una proprietà ValidationRule che richiede che il valore del campo Data di inizio relativo all'impiegato sia compreso tra la data di fondazione della società e la data corrente. Se la data immessa non è compresa nell'intervallo, è possibile visualizzare il messaggio della proprietà ValidationText: "Data di inizio inesatta."

Se viene creato un controllo trascinando un campo dall'elenco dei campi, la regola di convalida del campo rimane attiva, pur non essendo visualizzata nella casella della proprietà ValidationRule del controllo contenuta nella finestra delle proprietà. Ciò è dovuto al fatto che una regola di convalida del campo proviene da un controllo associato a tale campo.

Le regole di convalida di controllo, campo e record vengono applicate come riportato di seguito:

  • Le regole di convalida impostate per campi e controlli vengono applicate quando si modificano i dati e il campo oppure il controllo non ha più lo stato attivo.

  • Le regole di convalida per i record vengono applicate quando si effettua uno spostamento a un altro record.

  • Se vengono create regole di convalida sia per un campo che per un controllo associato a tale campo, queste verranno entrambe applicate quando si modificano i dati e il controllo non possiede più lo stato attivo.

La tabella riportata di seguito include esempi di espressioni relative alle proprietà ValidationRule e ValidationText.

Proprietà ValidationRule Proprietà ValidationText
<> 0 Il valore immesso deve essere un valore diverso da zero.
> 1000 o Is Null Non deve esservi alcun valore o il valore immesso deve essere maggiore di 1000.
Like "A????" Il valore immesso deve essere di cinque caratteri e cominciare con la lettera A.
>= #1/1/96# And <#1/1/97# Il valore immesso deve essere una data del 1996.
DLookup("IDCliente", "Clienti", "IDCliente = Forms!Clienti!IDCliente") Is Null Il valore immesso deve essere un IDCliente univoco. Le funzioni di aggregazione o di aggregazione sui domini sono consentite solo per le convalide a livello di maschera.

Se viene creata una regola di convalida per un campo, non viene in genere consentita la memorizzazione nel campo di un valore Null. Se si desidera consentire un valore Null, aggiungere "Is Null" alla regola di convalida, come in "<> 8 Or Is Null" e assicurarsi che la proprietà Required sia impostata a No.

Non è possibile impostare le regole di convalida relative a campi o record per tabelle create con programmi diversi da Microsoft Access, ad esempio dBASE, Paradox e Server SQL. Per tale tipo di tabelle è possibile creare regole di convalida esclusive per controlli.
