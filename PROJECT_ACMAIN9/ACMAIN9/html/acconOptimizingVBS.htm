Ottimizzazione delle prestazioni del codice Visual Basic

Il modo migliore per ottimizzare la velocità di esecuzione del codice Visual Basic consiste spesso nello scegliere un approccio più efficace. Alcune tecniche possono essere tuttavia di aiuto nella velocizzazione dell'esecuzione del codice.

  • Verificare che il codice presente nel database sia compilato. Nella finestra Codice di Visual Basic Editor scegliere Compila dal menu Debug.

    Nota   Poiché assegnando un nuovo nome a un database il codice viene decompilato, accertarsi di ricompilare il codice dopo aver rinominato un database.

  • Dichiarare sempre in modo esplicito le variabili. È possibile richiedere che le variabili siano dichiarate in modo esplicito prima che vengano utilizzate in una routine selezionando la casella di controllo Dichiarazione di variabili obbligatoria nella scheda Editor della finestra di dialogo Opzioni. Per ulteriori informazioni, fare clic su .

  • Utilizzare il tipo più specifico possibile quando si dichiarano le variabili. Dichiarare ad esempio una variabile che verrà utilizzata per rappresentare una maschera come Form anziché come Object o Variant.

  • Utilizzare le variabili per fare riferimento a proprietà, controlli e oggetti DAO. Se si fa riferimento più volte al valore di una proprietà o di un controllo in una maschera oppure a un oggetto DAO o alla relativa proprietà, creare variabili di oggetto e fare riferimento alle variabili anziché utilizzare gli identificatori completi.

  • Utilizzare la parola chiave Me per riferimenti alle maschere all'interno di una routine evento.

  • Utilizzare la funzione Ilf solo quando è necessario. Evitare di utilizzare questa funzione se è necessario molto tempo per valutare una delle espressioni restituite.

  • Utilizzare le funzioni stringa solo quando è necessario. Utilizzare le funzioni $ (ad esempio Str$) quando si utilizzano le stringhe e le funzioni non di tipo $ quando si utilizzano tipi di dati Variant o altri tipi di dati.

  • Utilizzare il tipo di dati Intero o Lungo per operazioni matematiche, quando è possibile.

  • Utilizzare le matrici dinamiche e l'istruzione Erase o ReDim per liberare memoria.

  • Utilizzare le costanti ogni volta risulti possibile.

  • Utilizzare i segnalibri per velocizzare il riposizionamento. Utilizzare i segnalibri anziché il metodo FindNext o altri mezzi per tornare a un determinato record.

  • Utilizzare i metodi FindRecord e FindNext sui campi indicizzati. Tali metodi sono molto più efficaci quando vengono utilizzati su un campo indicizzato.

  • Organizzare i moduli. In Visual Basic i moduli vengono caricati su richiesta, ossia vengono caricati in memoria solo quando il codice chiama una delle routine in tale modulo. Se le routine correlate vengono posizionate nello stesso modulo, i moduli aggiuntivi vengono caricati solo quando è necessario.

  • Eliminare le routine e le variabili che non vengono più utilizzate. Quando si sviluppano e si modificano le applicazioni, si potrebbero tralasciare le routine che non vengono più chiamate nel codice o le variabili dichiarate che non vengono più utilizzate. È consigliabile rivedere il codice in modo da rimuovere le routine e le variabili inutilizzate.

Per ulteriori informazioni, vedere Creazione di applicazioni con maschere e report.
