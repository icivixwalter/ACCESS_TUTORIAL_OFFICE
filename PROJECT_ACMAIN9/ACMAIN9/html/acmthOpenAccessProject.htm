Metodo OpenAccessProject

                 

È possibile utilizzare il metodo OpenAccessProject per aprire un progetto di Microsoft Access (adp) esistente come progetto corrente nella finestra di Microsoft Access.

Sintassi

oggetto.OpenAccessProject nomeprogetto

Il metodo OpenAccessProject utilizza gli argomenti riportati di seguito.

Argomento Descrizione
oggetto L'oggetto Application.
nomeprogetto Un'espressione stringa che rappresenta il nome del progetto esistente di Access, compreso il nome del percorso e l'estensione del nome file. Se la rete lo supporta, è anche possibile specificare un percorso di rete nella forma seguente:

\\Server\Nome condiviso\Cartella\Nomefile.adp
  Nota   Se non viene indicata l'estensione del nomefile, viene assegnata l'estensione adp. Per aprire i file ADP, è possibile utilizzare questo metodo o il metodo OpenCurrentDatabase.

Osservazioni

Il metodo OpenAccessProject consente di aprire un progetto Access esistente da Microsoft Access o da un'altra applicazione tramite l'automazione, precedentemente nota come Automazione OLE. Ad esempio, è possibile utilizzare il metodo OpenAccessProject da Microsoft Excel per aprire il database di esempio Northwind.adp nella finestra di Microsoft Access. Dopo aver creato un'istanza di Microsoft Access in un'altra applicazione, occorre anche creare un nuovo progetto di Access o specificare un particolare progetto di Access da aprire; quest'ultimo viene aperto nella finestra di Microsoft Access.

Se è già stato aperto un progetto e si desidera aprirne un altro nella finestra di Microsoft Access, è possibile utilizzare il metodo CloseCurrentDatabase per chiudere il progetto di Access corrente prima di aprirne un altro.

Nota   Per aprire un database di Access (mdb), utilizzare il metodo OpenCurrentDatabase dell'oggetto Application.
