Azione TrasferisciFoglioCalcolo

        

Utilizzare l'azione TrasferisciFoglioCalcolo per importare o esportare dati tra il database di Microsoft Access (MDB) o il progetto di Microsoft Access (ADP) corrente e un file di foglio di calcolo. È inoltre possibile collegare i dati di un foglio di calcolo di Microsoft Excel al database corrente di Microsoft Access. È possibile visualizzare e modificare da Microsoft Access i dati contenuti in un foglio di calcolo collegato e nello stesso tempo accedere a tali dati da Microsoft Excel. È anche possibile collegarsi ai dati di un file di foglio di calcolo di Lotus 1-2-3, ma tali dati sono disponibili in sola lettura in Microsoft Access.

Impostazione

L'azione TrasferisciFoglioCalcolo utilizza gli argomenti riportati di seguito.

Argomento azione Descrizione
Tipo trasferimento Il tipo di trasferimento che si desidera effettuare. Selezionare Importa, Esporta o Collega nella casella Tipo trasferimento della sezione Argomenti azione della finestra della macro. L'impostazione predefinita è Importa.
  Nota   Il tipo di trasferimento Collega non è supportato nei progetti di Microsoft Access (ADP).
Tipo foglio calcolo Il tipo di foglio di calcolo da cui importare, in cui esportare o a cui collegarsi. È possibile selezionare un tipo di foglio di calcolo tra quelli riportati nella casella. Il tipo predefinito è Microsoft Excel 8.
  Nota   È possibile importare e collegare (sola lettura) i dati dei file WK4 di Lotus ma non è possibile esportare i dati di Microsoft Access in questo formato di foglio di calcolo. In Microsoft Access inoltre non è più supportato l'utilizzo di questa azione per l'importazione, l'esportazione o il collegamento dei dati di fogli elettronici WKS di Lotus o della versione 2.0 di Microsoft Excel. Se si desidera importare o collegare dati di foglio di calcolo da questi formati in Microsoft Access, è necessario prima convertire tali dati in una versione successiva alla 2.0 di Microsoft Excel o al formato WKS di Lotus.
Nome tabella Il nome della tabella di Microsoft Access in cui importare, da cui esportare o a cui collegare i dati foglio di calcolo. È anche possibile digitare il nome della query di selezione di Microsoft Access dalla quale si desidera esportare i dati. Questo argomento è richiesto.
  Se si seleziona Importa nell'argomento Tipo trasferimento e la tabella esiste già, i dati del foglio di calcolo vengono accodati a questa tabella. Altrimenti, viene creata una nuova tabella contenente i dati del foglio di calcolo.
  In Microsoft Access 2000 non è possibile utilizzare un'istruzione SQL per specificare i dati da esportare quando si utilizza l'azione TrasferisciFoglioCalcolo ma è necessario prima creare una query e quindi specificarne il nome nell'argomento Nome tabella.
Nome file Il nome del file di foglio di calcolo da cui importare, in cui esportare o a cui collegarsi. Includere il percorso completo. Questo argomento è richiesto.
  Quando si esportano dati da Microsoft Access, viene creato un nuovo foglio di calcolo. Se il nome del file è uguale a quello di un foglio di calcolo esistente, quest'ultimo viene sostituito, a meno che non si stia esportando in una cartella di lavoro di Microsoft Excel versione 5.0, 7.0, 8.0 oppure di Excel 2000. In tal caso, i dati esportati vengono copiati nel successivo foglio di lavoro disponibile nella cartella di lavoro.
  Se viene importato o collegato un foglio di calcolo di Microsoft Excel versione 5.0, 7.0, 8.0 oppure di Excel 2000, è possibile specificare un determinato foglio di lavoro utilizzando l'argomento Intervallo.
Nomi campo Specifica se la prima riga del foglio di calcolo contiene i nomi dei campi. Se si seleziona Sì, i nomi di questa riga vengono utilizzati come nomi dei campi della tabella di Microsoft Access quando vengono importati o collegati i dati del foglio di calcolo. Se si seleziona No, la prima riga viene considerata come una normale riga di dati. L'impostazione predefinita è No.
  Quando si esporta una tabella o una query di selezione di Microsoft Access in un foglio di calcolo, in nomi dei campi vengono inseriti nella prima riga del foglio di calcolo, indipendentemente dalla selezione effettuata per questo argomento.
Intervallo L'intervallo di celle da importare o collegare. Lasciare vuoto questo argomento per importare o collegare l'intero foglio di calcolo. È possibile digitare il nome di un intervallo nel foglio di calcolo oppure specificare l'intervallo di celle da importare o collegare, ad esempio A1:E25 (notare che la sintassi A1..E25 non è valida in Microsoft Access 97 o in Microsoft Access 2000). Se viene importato o collegato un foglio di lavoro di Microsoft Excel versione 5.0, 7.0, 8.0 oppure di Excel 2000, è possibile far precedere l'intervallo dal nome del foglio di lavoro e da un punto esclamativo, ad esempio Budget!A1:C7.
  Nota   Quando si esportano dati in un foglio di calcolo, è necessario lasciare vuoto questo argomento. Se si immette un intervallo, l'esportazione non riesce.

Osservazioni

È possibile esportare i dati delle query di selezione di Microsoft Access in fogli di calcolo. Il set di risultati della query viene esportato come se fosse una tabella.

I dati del foglio di calcolo accodati a una tabella esistente di Microsoft Access devono essere compatibili con la struttura della tabella.

  • Ogni campo del foglio di calcolo deve essere dello stesso tipo di dati del campo corrispondente della tabella.

  • I campi devono essere nello stesso ordine, a meno che l'argomento Nomi campo non sia stato impostato a Sì, nel qual caso i nomi dei campi del foglio di calcolo devono corrispondere a quelli della tabella.

Questa azione equivale a scegliere Carica dati esterni o Esporta dal menu File della finestra del database e quindi Importa o Collega tabelle. È possibile utilizzare questi comandi per selezionare un'origine dati, ad esempio Microsoft Access oppure un tipo di database, di foglio di calcolo o di file di testo. Se si seleziona un foglio di calcolo, vengono visualizzate alcune finestre di dialogo oppure viene eseguita una creazione guidata di Microsoft Access nella quale è possibile selezionare il nome del foglio di calcolo e altre opzioni. Gli argomenti dell'azione TrasferisciFoglioCalcolo riflettono le opzioni presenti in queste finestre di dialogo o nelle creazioni guidate.

Nota   Se si esegue una query o si applica un filtro al foglio di calcolo collegato, la query o il filtro distinguono tra maiuscole e minuscole.

Se ci si collega a un foglio di calcolo di Excel aperto in modalità Modifica, il collegamento non viene completato finché non si esce da tale modalità. Non si verifica alcun timeout.

Per eseguire l'azione TrasferisciFoglioCalcolo in Visual Basic, utilizzare il metodo TransferSpreadsheet dell'oggetto DoCmd.
