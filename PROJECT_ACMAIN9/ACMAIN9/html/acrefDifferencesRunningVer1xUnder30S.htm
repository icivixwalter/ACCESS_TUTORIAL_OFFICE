Quando effettuare la conversione di un database di Access

Nella maggioranza dei casi conviene convertire un database di Microsoft Access di una versione precedente nel formato di Microsoft Access 2000. È comunque possibile abilitare in Microsoft Access 2000 un database creato con una versione precedente del programma senza convertirlo. In tal caso però non sarà possibile modificare la struttura degli oggetti né sfruttare le molte nuove funzioni di Microsoft Access 2000 senza prima convertire il database. Una volta effettuata la conversione, non sarà più possibile aprire il database con la versione precedente. Un database di Access 2000 non può essere convertito in formato Access 2.0 o 95, ma è possibile convertire un database di Access 2000 in formato Access 97. Condivisione di un database di Access con utenti di versioni diverse di Microsoft Access

Se il database di Access è un database multiutente (condiviso) e non tutti gli utenti possono effettuare l'aggiornamento a Microsoft Access 2000 contemporaneamente, è possibile aggiornare parti del database in modo che supporti le nuove funzioni mantenendo al contempo inalterato il database di Access originale. Per ulteriori informazioni sulla condivisione di un database con utenti di versioni diverse di Microsoft Access, fare clic su .

Creazione e modifica di oggetti

Se si lavora con Microsoft Access 2000 su un database abilitato di una versione precedente di Access, è possibile visualizzare e modificare i dati utilizzando gli oggetti del database. Non è tuttavia possibile salvare le modifiche apportate agli oggetti del database. Per modificare la struttura di un oggetto o creare un nuovo oggetto nel database di Access, è necessario aprirlo nella versione precedente del programma oppure convertire il database in Microsoft Access 2000.

Modifica delle autorizzazioni di un database

Per modificare o aggiungere autorizzazioni in un database di Access creato con una versione precedente del programma è necessario aprirlo nella versione precedente oppure convertirlo in Microsoft Access 2000. Per sfruttare al meglio le funzioni di protezione di Microsoft Access 2000 e per conservare inalterata la protezione del database originale, è necessario ricreare gli account e le password di utenti e gruppi utilizzando un file di informazioni sul gruppo di lavoro di Microsoft Access 2000. Per ulteriori informazioni, vedere Convertire un database protetto da una versione precedente di Microsoft Access.

Successivamente non sarà più possibile proteggere i moduli con protezione a livello utente, anche se sarà possibile proteggerli proteggendo il codice di Visual Basic, Applications Edition con una password.

Nuove barre degli strumenti e dei menu

Microsoft Access 97 e versioni successive supportano barre degli strumenti e dei menu ridisegnate. Quando si effettua la conversione di un database di Microsoft Access 2.0 o 95 in formato Access 2000, le barre strumenti personalizzate e le barre strumenti incorporate vengono convertite automaticamente secondo il nuovo disegno. Quando si apre un database di Access convertito, le barre dei menu personalizzate create in Microsoft Access 95 con la procedura di creazione guidata di menu o tramite macro vengono interpretate come barre del nuovo tipo. Esse non vengono però convertite in modo automatico e quindi non possono essere modificate tramite la finestra di dialogo Personalizza. Per creare una barra dei menu di tipo nuovo tramite macro che usano azioni AggiungiMenu, utilizzare i sottocomandi Crea menu da macro e Crea menu di scelta rapida da macro del comando Macro, accessibile dal menu Strumenti. Il tal modo sarà possibile modificarle tramite la finestra di dialogo Personalizza. Ulteriori informazioni sulla conversione di barre degli strumenti e dei menu.

Compatibilità del codice

In Microsoft Access 2000 non è supportata la libreria di compatibilità DAO 2.5/3.x. Se si tenta di convertire un database di Access in cui il codice contiene versioni precedenti di oggetti, metodi e proprietà DAO dipendenti dalla libreria di compatibilità DAO 2.5/3.x, verrà visualizzato un messaggio indicante che si sono verificati errori di compilazione durante la conversione. Prima di convertire un database di Access, aggiornare il codice in modo che non dipenda più dalla libreria di compatibilità DAO 2.5/3.x. Se si continua a ricevere un messaggio indicante errori di compilazione durante la conversione, aprire il database convertito in Microsoft Access 2000, rimuovere il riferimento alla libreria di compatibilità DAO 2.5/3.x mancante e impostare un riferimento alla libreria di oggetti Microsoft DAO 3.6.

Se il database di Access utilizza componenti aggiuntivi o database di librerie creati con una versione precedente di Microsoft Access, è necessario convertire anche quelli. Azione EseguiVoceMenu sostituita da EseguiComando

L'azione EseguiVoceMenu viene sostituita in Microsoft Access 2000 dall'azione EseguiComando ma continua a essere supportata per ragioni di compatibilità con le versioni precedenti. Quando si converte un database, l'azione EseguiVoceMenu viene automaticamente convertita nell'azione EseguiComando.
