Informazioni sul collegamento di un report principale a un sottoreport

Quando un sottoreport che contiene informazioni correlate a dei dati viene inserito nel report principale, è necessario che il controllo di sottoreport sia collegato al report principale in modo che i record stampati nel sottoreport corrispondano esattamente a quelli stampati nel report principale.

Quando si crea un sottoreport utilizzando una creazione guidata oppure trascinando un report o un foglio dati dalla finestra del database su un altro report, il report principale e il sottoreport vengono sincronizzati automaticamente nei seguenti casi:

  • Sono state definite relazioni per le tabelle selezionate o per quelle sottostanti le query selezionate.

  • Il report principale è basato su una tabella con una chiave primaria e il sottoreport è basato su una tabella che contiene un campo con lo stesso nome della chiave primaria e con un tipo di dati uguale o compatibile. Se ad esempio la chiave primaria della tabella sottostante la maschera principale è un campo Contatore e la proprietà Dimensione campo è impostata a Intero lungo, il campo corrispondente della tabella sottostante la sottomaschera dovrebbe essere un campo di tipo Numerico con la proprietà Dimensione campo impostata a Intero lungo. Se vengono selezionate una o più query, è necessario che le tabelle sottostanti contenute nella query o nelle query soddisfino le stesse condizioni.

Nota   Le proprietà Collega campi master e Collega campi secondari del controllo di sottoreport vengono utilizzate per collegare il report principale al sottoreport. Se per qualche motivo il report e il sottoreport non vengono collegati, è possibile impostare direttamente queste proprietà.
