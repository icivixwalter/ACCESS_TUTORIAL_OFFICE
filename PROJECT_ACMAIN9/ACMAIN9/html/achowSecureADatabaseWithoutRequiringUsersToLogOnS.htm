Proteggere un database senza richiedere l'accesso agli utenti

  1. Proteggere il database utilizzando l'Impostazione guidata Protezione.

    Procedura

  2. Assegnare le autorizzazioni appropriate all'account utente Amministratore per ogni tabella, query, maschera, report e macro che si desidera rendere disponibili per gli utenti.

    Procedura

  3. Disattivare la finestra di dialogo Accesso.

    Procedura

Quando gli utenti avviano Microsoft Access e aprono un database senza collegarsi, vengono automaticamente collegati con l'account utente Amministratore, pertanto dispongono solo delle autorizzazioni assegnate al passaggio 2. Questo vale per qualsiasi gruppo di lavoro, dato che l'account Amministratore è lo stesso in qualsiasi file di informazioni sul gruppo di lavoro. Solo l'account Amministratore e i membri del gruppo Amministratori del file di informazioni sul gruppo di lavoro corrente al momento dell'esecuzione della creazione guidata al passaggio 1 disporranno di autorizzazioni complete per le tabelle, le query, le maschere, i report e le macro del database.

Per eseguire funzioni relative all'amministrazione, è possibile riattivare la procedura di accesso e collegarsi come membro del gruppo Amministratori oppure utilizzare le opzioni della riga di comando /pwd e /user per specificare la password e il nome utente all'avvio di Microsoft Access.
