Importare o collegare delle tabelle o dei dati di database SQL da altre origini dati ODBC

Nota   È possibile collegare una tabella solo in un database di Microsoft Access, non in un progetto di Microsoft Access.

Importante   È possibile importare o collegare un'origine dati SQL o un'altra origine dati ODBC solo se è installato il driver appropriato.

  1. Aprire un database oppure passare alla finestra del database aperto.

  2. Per importare delle tabelle, scegliere Carica dati esterni dal menu File, quindi Importa.

    Per collegare delle tabelle, scegliere Carica dati esterni dal menu File, quindi Collega tabelle.

  3. Dalla casella Tipo file della finestra di dialogo Importa o Collega selezionare Database ODBC ().

    Importante   Se ci si collega a un file in una rete locale, assicurarsi di utilizzare un percorso UNC (Universal Naming Convention) invece di utilizzare la lettera identificativa di un'unità di rete mappata in Esplora risorse di Windows (Gestione risorse in Windows 95 e Windows NT 4.0). La lettera di un'unità può variare da computer a computer o non essere definita, mentre un percorso UNC è un modo affidabile e coerente per individuare l'origine dati che contiene la tabella collegata.

    Verrà visualizzata la finestra di dialogo Seleziona fonte dati con l'elenco delle origini dati definite per tutti i driver ODBC installati nel computer. Fare clic sulla scheda Fonte dati su file o Fonte dati macchina, quindi fare doppio clic sull'origine dati ODBC che si desidera importare. Per definire una nuova origine dati per un driver ODBC installato, scegliere il pulsante Nuova, quindi seguire le istruzioni della finestra di dialogo Crea nuova fonte dati e delle finestre successive prima di continuare. Per ulteriori informazioni, vedere l'argomento relativo alle origini dati.

  4. Se per l'origine dati ODBC selezionata è necessario effettuare l'accesso, immettere l'ID e la password di accesso, e, se non sono richieste ulteriori informazioni, scegliere OK.

    Microsoft Access effettua l'accesso all'origine dati ODBC e visualizza l'elenco di tabelle che è possibile importare o collegare.

  5. Se si sta collegando una tabella, selezionare la casella di controllo Usa connessione a cui si concede fiducia se si desidera salvare queste informazioni relative alla tabella nel database corrente per evitare di reimmetterle a ogni apertura. Se la casella di controllo non è selezionata, tutti gli utenti dovranno immettere l'ID e la password di accesso ogni volta che aprono quella tabella in una nuova sessione di Microsoft Access. L'amministratore del database SQL può anche disattivare questa casella di controllo per richiedere a tutti gli utenti di immettere l'ID e la password di accesso ogni volta che si connettono al database SQL. Per ulteriori informazioni sulla disattivazione della casella di controllo Usa connessione a cui si concede fiducia, fare clic su .

  6. Selezionare ogni tabella da importare o collegare, quindi scegliere OK. Se la tabella che si sta collegando non dispone di un indice che identifichi ciascun record in modo univoco, viene visualizzato un elenco dei campi che appartengono alla tabella collegata. Selezionare un campo o una combinazione di campi che identificheranno ciascun record in modo univoco, quindi scegliere OK.

Note

  • Dopo aver importato o collegato una tabella di database SQL, è possibile impostare delle proprietà di campo della tabella. Se si importa una tabella, è possibile impostare una chiave primaria per la tabella. Per ulteriori informazioni sull'impostazione delle proprietà dei campi di una tabella, fare clic su   .

  • Se si è verificato un errore nell'importazione, nel collegamento o nell'utilizzo di una tabella di database SQL, potrebbe essere causato da un problema con l'account sul server di database SQL o con lo stesso database. Se non è possibile accedere alla tabella di database SQL, contattare l'amministratore di database SQL.

  • Per modificare una tabella di database SQL collegata, questa deve generalmente disporre di un indice univoco sul server. Se si desidera modificare una tabella che non dispone di un indice univoco o modificare una visualizzazione SQL, è possibile creare un indice all'interno di Microsoft Access senza influire sul database SQL. A questo scopo si crea una query di definizione dati utilizzando l'istruzione Create Index. Si tenga presente, comunque, che l'indice deve essere creato in un campo o in una combinazione di campi in cui ciascun valore è univoco. Se il campo contiene dei valori duplicati, tutti gli aggiornamenti della tabella avranno esito negativo. Per eliminare l'indice, utilizzare un'altra query di definizione dati. Per ulteriori informazioni sulla creazione di una query di definizione dati, fare clic su .

  • Se la struttura di una tabella di database SQL viene modificata dopo essere stata collegata, utilizzare il Gestore tabelle collegate per aggiornare il collegamento. Per ulteriori informazioni, fare clic su .

  • Se si sta effettuando un'importazione da un database di Microsoft SQL Server in un progetto di Access, è consigliabile l'utilizzo di DTS (Data Transformation Services), un metodo più rapido e flessibile per l'importazione e la trasformazione dei dati. Per ulteriori informazioni, consultare la documentazione di Microsoft SQL Server.
