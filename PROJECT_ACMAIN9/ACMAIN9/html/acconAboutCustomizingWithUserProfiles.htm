Informazioni sulla personalizzazione dell'ambiente dell'applicazione con i profili utente di Microsoft Access

Microsoft Access fornisce proprietà di database e opzioni di avvio per soddisfare le esigenze di personalizzazione più varie. Gli sviluppatori di applicazioni possono però avvalersi degli ulteriori vantaggi offerti dai profili utente di Access per personalizzare l'ambiente dell'applicazione.

Un profilo utente di Access è uno speciale set di chiavi del registro di configurazione di Windows che è possibile creare per sostituire le impostazioni standard di Access e del modulo di gestione di database Microsoft Jet e per specificare altre opzioni di run-time. È quindi possibile utilizzare l'opzione della riga di comando /profile per specificare il profilo utente di Access desiderato all'avvio dell'applicazione.

È possibile utilizzare un profilo utente di Access per:

  • Impedire che le impostazioni standard di Access modificabili dagli utenti nelle proprietà di database e nelle opzioni di avvio siano in conflitto con le esigenze dell'applicazione.

  • Assicurarsi che al momento della distribuzione delle applicazioni le impostazioni personalizzate non vengano sostituite da altri programmi che utilizzano Microsoft Jet.

  • Eseguire test dell'applicazione con diverse configurazioni.

Nota   Il profilo utente utilizzato per avviare Access dalla riga di comando non è il profilo utente definito per l'accesso al sistema operativo. Il primo è applicabile solo ad Access ed esclusivamente quando si avvia il programma dalla riga di comando. Un profilo utente definito per il sistema operativo invece si applica a tutte le applicazioni sul sistema operativo e viene utilizzato per gestire i dati di sistema per i singoli utenti.
