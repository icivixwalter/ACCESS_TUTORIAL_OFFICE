Esempio di metodo Clone

Nell'esempio seguente, un oggetto Recordset viene creato utilizzando un'istruzione SQL su una tabella Dipendenti, quindi viene utilizzato il metodo Clone per creare un clone dell'oggetto Recordset in modo che i due oggetti possano condividere gli stessi segnalibri. Questa tecnica risulta particolarmente utile quando occorre confrontare i risultati di una query da più punti contemporaneamente.

La funzione legge e memorizza in una variabile il valore della proprietà Bookmark del record corrente, in questo caso il secondo record dell'oggetto Recordset originale. La proprietà Bookmark dell'oggetto Recordset duplicato viene impostata a questa stringa, in modo che il secondo record diventi il record corrente anche in questo oggetto Recordset. I valori correnti del campo Cognome per i due recordset sono uguali, come si può vedere quando vengono stampati nella finestra Debug.

Sub CreaClone()
    Dim cnn As ADODB.Connection
    Dim rstEmployees As New ADODB.Recordset, rstDuplicate As ADODB.Recordset
    Dim fldName As ADODB.Field, varBook As Variant

    ' Restituisce il riferimento al database corrente.
    Set cnn = CurrentProject.Connection
    ' Apre un oggetto Recordset di tipo dynaset.
        rstEmployees.Open "SELECT * FROM Dipendenti ORDER BY Cognome", _ cnn, adOpenKeyset, adLockOptimistic
    ' Clona oggetto Recordset.
    Set rstDuplicate = rstEmployees.Clone
    Set fldName = rstEmployees.Fields!Cognome
    ' Imposta record corrente.
    rstEmployees.MoveFirst
    ' Passa al secondo record.
    rstEmployees.MoveNext
    ' Richiama il valore della proprietà Bookmark e stampa il valore di campo corrente.
    If rstEmployees.Properties("Con segnalibro") Then
        varBook = rstEmployees.Bookmark
        Debug.Print fldName.Value
        ' Imposta la proprietà Bookmark del clone al valore ottenuto.
            rstDuplicate.Bookmark = varBook
        Debug.Print fldName.Value
    End If    

    rstEmployees.Close
    rstDuplicate.Close
    Set cnn = Nothing
End Sub
