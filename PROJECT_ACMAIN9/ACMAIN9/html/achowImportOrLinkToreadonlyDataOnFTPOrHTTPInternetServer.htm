Importare o collegare (sola lettura) i dati in un server Internet FTP oppure HTTP

Nota   Non è possibile utilizzare come origine o come destinazione un database di Microsoft Access, un file di testo o un'origine dati ODBC quando si importa o ci si collega a un server FTP o HTTP. Per un riepilogo generale dei dati che è possibile importare o collegare, fare clic su .

  1. Aprire un database o passare alla finestra del database di un database già aperto.

  2. Per importare dei dati, scegliere Carica dati esterni dal menu File, quindi Importa.

    Per collegare dei dati, scegliere Carica dati esterni dal menu File, quindi Collega tabelle.

  3. Per importare o collegare dati da un server HTTP, digitare l'indirizzo Internet nella casella Nome file. Per collegare ad esempio il file Vendgenn.html del server HTTP //webpub, digitare http://webpub/venditegenn.html.

    Per importare o collegare dati da un server FTP, digitare l'indirizzo Internet nella casella Nome file oppure fare clic sulla freccia a destra della casella Cerca in e selezionare Percorsi FTP. Nell'elenco dei percorsi FTP fare doppio clic sul percorso desiderato e poi sulla voce desiderata.

Note

  • Sebbene non sia possibile importare o collegare un file di testo tramite il comando Importa, è consentito utilizzare l'azione TrasferisciTesto o il metodo TransferText.

  • Microsoft Access copia l'intero file nella cartella della cache di Microsoft Internet Explorer nel computer locale. Questo processo quindi può richiedere del tempo, poiché da Internet possono venire trasferite grandi quantità di dati.

  • Una tabella collegata è uno snapshot dei dati dell'origine remota originaria al momento della creazione del collegamento. Mentre la tabella è aperta, gli aggiornamenti dei dati eseguiti presso la postazione remota non vengono visualizzati, ma alla successiva apertura della tabella, Microsoft Access aggiornerà la copia locale nella cartella della cache.
