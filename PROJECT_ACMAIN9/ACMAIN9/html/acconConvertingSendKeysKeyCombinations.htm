Convertire combinazioni di tasti SendKeys

Se un database della versione 1.x o 2.0 viene convertito in Microsoft Access 2000, in alcune finestre di dialogo o in alcuni menu potrebbero verificarsi delle modifiche che richiedono la ricodifica dell'istruzione SendKeys o dell'azione InviaTasti. Il sottomenu Componenti aggiuntivi ad esempio è stato spostato dal menu File al menu Strumenti, mentre i precedenti sottomenu Importa, Esporta e Collega del menu File sono stati trasformati nei sottomenu Carica dati esterni e Salva con nome/esporta. Dal momento che modifiche di questo tipo è probabile che di verifichino per tutte le nuove versioni di Microsoft Access, si consiglia di evitare, quando possibile, di utilizzare l'istruzione SendKeys o l'azione InviaTasti per eseguire comandi o immettere dati nelle finestre di dialogo.

Sono riportati di seguito alcuni suggerimenti per evitare di utilizzare l'istruzione SendKeys o l'azione InviaTasti.

  • Per eseguire un comando di menu, prima di utilizzare l'istruzione SendKeys o l'azione InviaTasti, ricercare un'azione macro o un metodo di Visual Basic equivalenti. Per la maggior parte dei comandi normalmente eseguibili mediante un'opzione o un comando di menu esiste un'azione o un metodo equivalente. È possibile ad esempio utilizzare il metodo Dropdown di una casella combinata anziché il codice SendKeys "{F4}".

  • Se per un comando di menu non esiste un'azione o un metodo equivalente, per eseguire il comando utilizzare l'azione EseguiComando anziché l'istruzione SendKeys o l'azione InviaTasti.

  • Evitare di utilizzare l'istruzione SendKeys o l'azione InviaTasti per impostare le opzioni nella finestra di dialogo Opzioni. Le nuove versioni di Microsoft Access avranno opzioni nuove e modificate, quindi il codice contenuto nell'istruzione SendKeys o nell'azione InviaTasti potrebbe facilmente subire interruzioni. Utilizzare invece i metodi GetOption e SetOption.

  • Quando si utilizza l'istruzione SendKeys per eseguire un'azione, è consigliabile fare riferimento alle costanti incorporate dei codici dei tasti oppure definire nell'istruzione SendKeys le costanti per i valori. La definizione dei tasti come costanti renderà più semplice il futuro aggiornamento del codice.

  • In Microsoft Access versione 2.0 è possibile, ma non obbligatorio, racchiudere tra virgolette l'argomento Sequenza tasti. Nelle versioni successive di Microsoft Access, se si racchiude tra virgolette questo argomento, si verifica un errore. Per utilizzare le virgolette nell'argomento Sequenza tasti è necessario inserire due serie di virgolette, come in Giuseppe ""Pino"" Rossi. Si verifica un errore anche quando l'argomento Sequenza tasti contiene una sintassi non corretta, errori di battitura o altri valori non appropriati per la finestra a cui è diretta la sequenza di tasti.
