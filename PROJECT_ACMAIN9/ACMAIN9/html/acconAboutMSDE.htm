Definizione di MSDE

MSDE è una nuova tecnologia che fornisce un sistema di memorizzazione locale dei dati compatibile con Microsoft SQL Server 7.0. È anche possibile utilizzare MSDE come soluzione per la memorizzazione remota dei dati. Si pensi a MSDE come a un modulo di gestione dati client/server alternativo al modulo di gestione di database Microsoft Jet. MSDE è eseguibile in Microsoft Windows NT 4.0 o versioni successive e in Windows 95 o versioni successive ed è stato progettato e ottimizzato per l'utilizzo su piccoli sistemi, ad esempio un computer per un singolo utente o un server di un gruppo di lavoro di piccole dimensioni.

Poiché MSDE è basato sullo stesso modulo di gestione dati di SQL Server, la maggior parte dei progetti di Microsoft Access o delle applicazioni client/server può essere eseguita su entrambe le versioni. A differenza di SQL Server 7.0, MSDE però limita le dimensioni del database a 2 GB, non supporta Symmetrical Multiprocessing (SMP) in Windows 95 o versioni successive e non può essere utilizzato come editore di repliche (ma solo come sottoscrittore di repliche) con la replica transazionale.

Quando utilizzare MSDE

MSDE può essere considerato come un database desktop alternativo al database di Microsoft Access con le seguenti modalità:

  • Come database di server per gruppi di lavoro di piccole dimensioni. Le applicazioni vengono sviluppate utilizzando un progetto di Access connesso a MSDE. È possibile prevedere che il gruppo di lavoro e le relative necessità si sviluppino nel tempo fino a richiedere le funzionalità complete di SQL Server 7.0 in esecuzione su un server di rete di maggiori dimensioni.

  • Come sottoscrittore di repliche in un ambiente di database replicato di SQL Server. È possibile ad esempio avere dipendenti che lavorano in modalità remota lontano dall'ufficio principale e che accedono e aggiornano MSDE su laptop o computer non connessi per poi connettersi periodicamente al database master di SQL Server per sincronizzare le modifiche.

  • Per sviluppare e verificare con facilità un progetto di Access o un'applicazione client/server su un personal computer o una workstation, quindi modificare le informazioni di connessione del progetto per connettersi a un database di SQL Server su un server remoto per la verifica e la produzione finale.

Installazione di MSDE

Sebbene MSDE non sia incluso nell'installazione di Microsoft Office 2000 o Microsoft Access 2000 su computer autonomi, è possibile installare MSDE dal CD-ROM eseguendo il file Setupsql.exe contenuto nella cartella \Sql\x86\Setup.

La versione Microsoft Office 2000 di MSDE contiene inoltre nella cartella \Windows\Help dei file della Guida aggiuntivi, SQL Server Enterprise Manager Help (entmgr.chm) e Replication Wizard Help (replwiz.chm), che è possibile utilizzare con le utilità di database SQL Server accessibili dal menu Strumenti di Access.

Dopo avere installato MSDE nel computer, in Microsoft Windows 95 o versioni successive, può essere necessario avviare SQL Server Service Manager (fare doppio clic sull'icona MSSQLServer sulla barra delle applicazioni e scegliere Start/Continue). MSDE si avvia automaticamente in Windows NT 4.0 o versioni successivi. Per avviare automaticamente MSDE in Windows 95 o versioni successive, è necessario selezionare la casella di controllo Auto-start service when OS starts nella finestra di SQL Server Service Manager.
