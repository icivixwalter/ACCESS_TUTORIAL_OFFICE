Esportare un foglio dati in formato HTML generato da server

  1. Nella finestra del database fare clic sul nome della tabella, della query o della maschera che si desidera esportare, quindi scegliere Esporta dal menu File.

  2. Nella casella Tipo file selezionare Microsoft IIS 1-2 (*.htx;*.idc) o su Microsoft Active Server Pages (*.asp) a seconda del formato di output HTML generato da server desiderato. Fare clic qui per informazioni sul formato HTML generato da server.

  3. Fare clic sul collegamento nella barra percorsi o sulla freccia a destra della casella di riepilogo Salva in e selezionare l'unità o la cartella di destinazione.

  4. Nella casella Nome file immettere il nome del file.

  5. Fare clic sul pulsante Salva.

  6. Immettere le informazioni appropriate nella finestra di dialogo Opzioni di output per Microsoft Active Server Pages o Opzioni di output per HTX/IDC:

    Nella casella di testo Modello HTML immettere la posizione del modello HTML. Fare clic qui per informazioni sul file modello HTML.

    Nella casella di testo Nome origine dati immettere il nome dell'origine dati ODBC alla quale si verrà connessi quando i file HTML generati da server verranno elaborati sul server Web. Fare clic qui per informazioni sulle origini dati ODBC.

    In un database di Microsoft Access nelle caselle Utente connesso come e Password utente immettere un nome utente e una password di protezione a livello utente che consentano agli utenti di accedere al database di Access dalla pagina Web. Se non si immette un nome utente e una password, verrà utilizzato il nome utente predefinito Amministratore senza password. Fare clic qui per informazioni sulla protezione a livello utente.

    In un progetto di Microsoft Access nelle caselle Nome utente e Password immettere un nome utente e una password per consentire agli utenti di accedere al database di Microsoft SQL Server dalla pagina Web. Se non si immettono i dati richiesti, verrà utilizzato il nome utente predefinito Sa senza password. Fare clic qui per informazioni sulla protezione del database SQL Server.

Una volta esportato un foglio dati in un formato output ASP o IDC/HTX, è necessario pubblicare i file per renderli disponibili su World Wide Web. Fare clic qui per informazioni sulla pubblicazione dei file HTML generati da server su un server Web.

Note  

  • Se il foglio dati contiene una query con parametri, Microsoft Access simula la finestra di dialogo Immissione valore parametro creando una pagina dei parametri HTML aggiuntiva che contiene una casella di testo di una maschera HTML per immettere i valori dei parametri e un pulsante per eseguire la query. È necessario definire esplicitamente i tipi di dati dei parametri quando si crea una query con parametri, nonché visualizzare questa pagina Web con i parametri HTML prima di visualizzare la pagina Web del foglio dati. Durante l'operazione di esportazione, Microsoft Access esegue la query e visualizza la finestra di dialogo Immissione valore parametro, ma è possibile solo fare clic su OK perché l'input viene ignorato.

  • I campi con tipo di dati Collegamento ipertestuale sono esportati come stringhe che utilizzano il seguente formato di collegamento ipertestuale: testovisualizzato#indirizzo#sottoindirizzo#descrizione. Sebbene non sia possibile fare clic sulla stringa in un browser Web per seguire l'indirizzo del collegamento ipertestuale, è possibile copiare e incollare la parte di indirizzo sulla barra degli indirizzi del browser Web.

  • L'invio come output e la visualizzazione di un foglio dati di grandi dimensioni in un browser Web può richiedere molto tempo. Si consiglia di ridurre le dimensioni del foglio dati dividendolo in fogli dati più piccoli in base a criteri come, ad esempio, il campo di tipo Data oppure utilizzando un report o una maschera per visualizzare i dati.
