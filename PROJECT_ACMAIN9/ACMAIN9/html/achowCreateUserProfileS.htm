Creare un profilo utente di Microsoft Access

Per creare un profilo utente di Microsoft Access

  1. Avviare l'Editor del registro di configurazione.

    Procedura

  2. Spostarsi alla sottochiave \HKEY_LOCAL_MACHINE\SOFTWARE.

  3. Aggiungere tre sottochiavi nidificate utilizzando il seguente formato per garantirne l'univocità: \Società\Applicazione\Numero versione applicazione.

  4. Nella sottochiave \Numero versione applicazione ricreare una delle sottochiavi posizionate nella sottochiave \HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Office\9.0\Access che contengono i valori da modificare (a eccezione della sottochiave \Profiles che verrà creata al passo 7), quindi copiare i valori appropriati in ciascuna sottochiave.

  5. Modificare i valori desiderati.

  6. Spostarsi alla sottochiave \HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Office\9.0\Access.

  7. Aggiungere una nuova sottochiave e assegnarle il nome \Profiles.

  8. Aggiungere una stringa alla sottochiave \Profiles e assegnarle il nome che si desidera utilizzare sulla riga di comando.

  9. Per i dati del valore, immettere una stringa che punti alla posizione delle sottochiavi create nel passo 3: \SOFTWARE\Società\Applicazione\Numero versione applicazione

  10. Creare una riga di comando per l'applicazione che specifichi il nome immesso nel passo 8 racchiudendolo tra virgolette ("). Ad esempio: MSAccess.exe /profile "Profilo" "Applicazione.mdb"

Per avere la precedenza sulle impostazioni predefinite del modulo di gestione di database Jet 4.0 Microsoft per Access

  1. Spostarsi alla sottochiave del profilo \HKEY_LOCAL_MACHINE\SOFTWARE\Società\Applicazione\Numero versione applicazione.

  2. Creare una sottochiave denominata \Jet\4.0\Engines.

  3. Per ciascuna impostazione del driver interno al sistema che si desidera venga ignorata, effettuare le seguenti operazioni:

    Aggiungere una sottochiave con un nome appropriato sotto la sottochiave creata nel passo 2, ad esempio, per ignorare le impostazioni di Jet Paradox aggiungere una sottochiave chiamata "Paradox".

    Aggiungere a questa sottochiave i valori di nome e tipo appropriato e quindi specificare le impostazioni.

    • È possibile utilizzare come modello per le proprie sottochiavi e i propri valori quelli contenuti in \HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Jet\4.0\Engines.

Note

  • Per determinare se l'applicazione ha caricato correttamente il profilo utente di Access, è possibile utilizzare l'argomento acSysCmdProfile del metodo SysCmd. Gli utenti potrebbero, ad esempio, aprire accidentalmente l'applicazione tramite Esplora risorse (Gestione risorse in Windows 95 o Windows NT 4.0) e ignorare l'opzione della riga di comando /profile. Per ulteriori informazioni, fare clic su .

  • È anche possibile utilizzare il metodo Data Access Object (DAO), DBEngine.SetOption, per modificare dinamicamente i valori del registro di configurazione di Microsoft Jet all'esecuzione di una singola sessione, senza modificare in modo permanente i valori del registro di configurazione di Windows. Per ulteriori informazioni, fare clic su .

  • In un database di Microsoft Access o in un progetto di Microsoft Access è possibile avere lo stesso set di chiavi del registro di configurazione in tre modi differenti, ma è possibile utilizzare solo un valore. In questo caso, un profilo utente ha la precedenza su un'impostazione di un database o di un progetto di Microsoft Access, che a sua volta ha la precedenza su un'impostazione di Microsoft Jet. La seguente tabella mostra questo ordine di precedenza utilizzando l'impostazione ODBC ConnectionTimeout come esempio.
    Ordine Posizione Impostazione del registro sotto \HKEY_LOCAL_MACHINE\SOFTWARE...
    1 profilo utente \Società\Applicazione\Numero versione applicazione\ODBC\ConnectionTimeout=100
    2 Microsoft Access \Microsoft\Office\9.0\Access\Jet\4.0\Engines
    \ODBC\ConnectionTimeout=300
    3 Microsoft Jet \Microsoft\Jet\4.0\Engines
    \ODBC\ConnectionTimeout=600

    Se si avvia l'applicazione utilizzando una riga di comando che specifica il profilo utente, verrà utilizzato il valore di ConnectionTimeout 100. Se invece si avvia l'applicazione senza utilizzare il profilo utente, verrà utilizzato il valore di ConnectionTimeout 300.
