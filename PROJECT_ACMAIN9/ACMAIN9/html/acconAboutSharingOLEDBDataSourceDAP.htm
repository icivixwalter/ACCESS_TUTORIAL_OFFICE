Informazioni sulla condivisione del database connesso a una pagina di accesso ai dati

È necessario collocare il database di Microsoft Access o di Microsoft SQL Server che funge da origine dati OLE DB per una pagina di accesso ai dati su un server o un computer condiviso per consentire ad altri utenti di visualizzare e interagire con la pagina da un browser Web o in un messaggio di posta elettronica.

Può essere utile collocare il database su un server o su un computer condiviso prima di creare la pagina. Se si sposta il database a cui è connessa la pagina dopo averla creata, è necessario modificare la connessione dell'origine dati OLE DB della pagina. Per eseguire questa operazione, nella visualizzazione Struttura della pagina fare clic con il pulsante destro del mouse sull'icona del database nell'Elenco campi, quindi scegliere Connessione dal menu di scelta rapida.

Importante   Accertarsi di utilizzare un percorso UNC (Universal Naming Convention) invece della lettera di identificazione dell'unità di rete mappata in Esplora risorse (Gestione risorse in Windows 95 e Windows NT 4.0). La lettera dell'unità può essere diversa su un altro computer o può non essere definita, mentre un percorso UNC è un metodo affidabile e uniforme per individuare l'origine dati della pagina.

Sebbene l'origine dati di una pagina possa essere solo un database di Access o di SQL Server, è possibile rendere accessibili dalla pagina altri formati di dati mediante le tabelle collegate da un database di Access.

Per proteggere al meglio le pagine di accesso ai dati, è necessario collocare il database di Access o di SQL Server sullo stesso server Web della pagina di accesso ai dati.
