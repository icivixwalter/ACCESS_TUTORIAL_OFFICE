Proprietà UniqueTable (Tabella univoca)

                 

È possibile utilizzare la proprietà UniqueTable per specificare o determinare la possibilità di aggiornare la tabella quando:

  • Una maschera o una pagina di accesso ai dati è associata a una visualizzazione o a una stored procedure di tabella multipla in un progetto di Microsoft Access (ADP).

  • Una pagina di accesso ai dati è associata a una query o a una macro di tabella multipla in un progetto di Microsoft Access (ADP).

Impostazione

La proprietà UniqueTable è un'espressione stringa che indica che la tabella univoca è aggiornabile.

È possibile impostare questa proprietà utilizzando la finestra delle proprietà o Visual Basic.

Osservazioni

La proprietà UniqueTable identifica la tabella di un join sul lato molti della relazione. Se non si imposta la proprietà UniqueTable, una maschera associata a una visualizzazione, a una stored procedure o a una stringa SQL contenente un join sarà di sola lettura. Il foglio dati prodotto da View.Open o StoredProc.Run è di sola lettura nel caso di un join, poiché non c'è modo di impostare la proprietà UniqueTable. Dopo aver aggiornato la proprietà UniqueTable, risultano aggiornabili solo i campi di quella tabella e gli inserimenti e le eliminazioni possono essere eseguiti solo in quella tabella.

Una maschera o una pagina di accesso ai dati basata su un cursore join deve avere una stringa di proprietà UniqueTable perché il set di record sia uno snapshot aggiornabile. La tabella univoca è la tabella della query sottostante le cui righe hanno una corrispondenza uno-a-uno con le righe nel cursore. In un join semplice Pazienti - Medici, la tabella Pazienti è la tabella univoca poiché ogni riga del cursore corrisponde a una riga della tabella Pazienti. Si noti che un join molti-a-molti non ha una proprietà UniqueTable ed è quindi di sola lettura. La proprietà UniqueTable verrà esposta come oggetto RecordsetDef nel caso di una pagina di accesso ai dati. La proprietà UniqueTable ha lo scopo seguente:

Applicare la corretta semantica di aggiornabilità:

  • Le colonne chiave della tabella univoca devono essere presenti nell'elenco di selezione della query che forma il cursore, anche per il server SQL. Per le altre origini dati, vedere la sezione Osservazioni dell'argomento della Guida in linea relativo alla proprietà ResyncCommand.

  • Eliminando una riga in un cursore di join la riga viene eliminata solo dalla tabella univoca.

  • L'inserimento di una riga in un cursore di join è consentito solo per la tabella univoca.

  • L'aggiornamento di una riga in un cursore di join è consentito solo per i campi della tabella univoca.

Per fornire i corretti valori di parametro per la query Resync. La proprietà UniqueTable di una maschera o di un RecordsetDef supporta la notazione catalogo.proprietario.nometabella in modo da distinguere perfettamente una tabella di base dalle altre nello stesso cursore, se necessario. Ad esempio, se si esegue un join tra dbo.autori e utente1.autori in un cursore, la proprietà UniqueTable dovrà essere specificata come dbo.autori o utente1.autori.

Per un cursore di join, se la proprietà UniqueTable è vuota il set di record riprende lo stato di sola lettura e qualsiasi tentativo di modifica produce un segnale acustico e il messaggio di stato "Recordset non aggiornabile perché la proprietà UniqueTable non è impostata". Nel caso di una proprietà UniqueTable non vuota, impostare la proprietà UniqueTable, e se necessario le proprietà UniqueSchema e UniqueCatalog, sul set di record o sul rowset sottostante. Contrassegnare quindi ogni colonna che non corrisponde alla proprietà UniqueTable come di sola lettura.

Durante le operazioni di inserimento e aggiornamento possono essere modificati solo i campi della tabella univoca. Quando si tenta di inserire dei dati in questi campi, viene prodotto un segnale acustico e viene visualizzato il messaggio "È possibile modificare solo i campi di una tabella univoca". Se la proprietà UniqueTable è stata impostata in modo errato, ciò avverrà per tutte le colonne.

Impostare la tabella univoca nel caso di una tabella singola

Nel caso di un set di record basato su una visualizzazione o su una stringa SQL, il comando ResyncCommand gestisce le colonne di espressioni server. Nel caso di una stored procedure è inoltre possibile che l'utente desideri fornire una proprietà personalizzata ResyncCommand anche nel caso di tabella singola. Per questa ragione, si consiglia di impostare sempre la proprietà di provider "Unique Table" nel caso di tabella singola. QBU consente l'impostazione di un comando ResyncCommand solo se la tabella univoca è stata impostata.
