Azione ApriReport

        

L'azione ApriReport apre un report in visualizzazione Struttura o in Anteprima di stampa oppure stampa direttamente il report. È inoltre possibile limitare i record stampati nel report.

Impostazione

L'azione ApriReport utilizza gli argomenti riportati di seguito.

Argomenti azione Descrizione
Nome report Il nome del report da aprire. La casella Nome report della sezione Argomenti azione della finestra della macro visualizza tutti i record del database corrente. L'argomento è richiesto.

Se viene eseguita una macro contenente l'azione ApriReport in un database di libreria, Microsoft Access cerca il report contraddistinto da tale nome innanzitutto nel database di libreria, quindi nel database corrente.
Visualizzazione La visualizzazione in cui viene aperto il report. Selezionare Stampa, se si desidera stampare il report immediatamente, Struttura o Anteprima di stampa nella casella Visualizzazione. L'impostazione predefinita è Stampa.
Nome filtro Un filtro che limita i record contenuti nel report. È possibile immettere il nome di una query esistente o di un filtro salvato come query. È necessario tuttavia che la query includa tutti i campi del report da aprire o che abbia la proprietà Mostra tutti i campi impostata a Sì.
Condizione WHERE Una proposizione SQL WHERE valida, cioè priva della parola WHERE o un'espressione utilizzata da Microsoft Access per selezionare record dalla tabella o dalla query sottostante al report. Se si seleziona un filtro tramite l'argomento Nome filtro, Microsoft Access applica la proposizione WHERE ai risultati del filtro.

Per aprire un report e limitare i record in esso contenuti a quelli specificati dal valore di un controllo di una maschera, utilizzare la seguente espressione:

[nomecampo] = Forms![nomemaschera]![nomecontrollo di maschera]

L'argomento nomecampo è il nome di un campo contenuto nella tabella o query sottostante al report che si desidera aprire. L'argomento nomecontrollo di maschera è il nome del controllo della maschera contenente il valore al quale si desidera far corrispondere i record contenuti nel report.
  Nota   La lunghezza massima dell'argomento condizione WHERE è di 256 caratteri. Se si rende necessario immettere una proposizione SQL WHERE più complessa e più lunga, utilizzare invece il metodo OpenReport dell'oggetto DoCmd in Visual Basic. È possibile immettere istruzioni con proposizioni SQL WHERE contenenti fino a 32.768 caratteri in Visual Basic.

Osservazioni

L'impostazione Stampa per l'argomento Visualizzazione stampa immediatamente il report, utilizzando le impostazioni correnti della stampante senza far apparire la finestra di dialogo Stampa. È inoltre possibile utilizzare l'azione ApriReport per aprire e impostare un report, quindi utilizzare quindi l'azione Stampa per stamparlo. È possibile ad esempio modificare il report o utilizzare l'azione Stampa per modificare le impostazioni della stampante prima delle operazioni di stampa.

Il filtro e la condizione WHERE applicati diventano l'impostazione della proprietà Filtro del report.

L'azione ApriReport equivale a fare clic sul pulsante Struttura o Anteprima oppure a scegliere il comando Stampa dal menu File nella finestra del database dopo aver selezionato un report.

Suggerimenti

  • Per stampare report simili per diversi set di dati, utilizzare un filtro o una proposizione WHERE per limitare i record stampati nel report. Modificare quindi la macro per applicare un filtro diverso o cambiare l'argomento Condizione WHERE.

  • È possibile selezionare un report nella finestra del database e trascinarlo in una riga di azione della macro. In tal modo viene automaticamente creata un'azione ApriReport che apre il report in Anteprima di stampa.
