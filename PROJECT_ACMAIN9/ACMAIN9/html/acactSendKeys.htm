Azione InviaTasti

        

È possibile utilizzare l'azione InviaTasti per inviare direttamente sequenze di tasti a Microsoft Access o a un'applicazione attiva per Windows.

Impostazione

L'azione InviaTasti utilizza gli argomenti riportati di seguito.

Argomento azione Descrizione
Sequenza tasti Le sequenze di tasti che Microsoft Access o dell'applicazione deve elaborare. Immettere le sequenze di tasti nella casella Sequenza tasti della sezione Argomenti azione nella finestra della macro. È possibile digitare fino a 255 caratteri. L'argomento è obbligatorio.
Attesa Specifica se la macro debba o meno effettuare una pausa fino a che l'elaborazione delle sequenze di tasti sia stata completata. Selezionare Sì per effettuare la pausa oppure No per non effettuarla. L'impostazione predefinita è No.

Osservazioni

Microsoft Access elabora le sequenze di tasti ricevute tramite l'azione InviaTasti, come se venissero digitate direttamente in una finestra di Microsoft Access.

Per specificare le sequenze di tasti, utilizzare la stessa sintassi dell'istruzione SendKeys.

Nota   È possibile che si verifichi un errore se l'argomento Sequenza tasti contiene una sintassi scorretta, un testo errato o altri valori non appropriati alla finestra cui le sequenze di tasti sono destinate.

È possibile utilizzare tale azione per immettere informazioni in una finestra di dialogo, in modo particolare se non si desidera interrompere la macro per rispondere manualmente alla finestra di dialogo. Alcune azioni di Microsoft Access, come Stampa e TrovaRecord, selezionano automaticamente le opzioni in alcune finestre di dialogo utilizzate di frequente. È possibile utilizzare l'azione InviaTasti per selezionare le opzioni in finestre di dialogo utilizzate meno di frequente.

Note

  • La finestra di dialogo sospende la macro, è necessario perciò inserire l'azione InviaTasti prima dell'azione che fa in modo che la finestra di dialogo venga aperta e impostare a No l'argomento Attesa.

  • È possibile che il tempo impiegato nell'invio delle sequenze di tasti a Microsoft Access o a un'altra applicazione provochi delle imprecisioni. Si raccomanda, se possibile, di utilizzare qualche altro metodo, come l'azione TrovaRecord, in modo da completare l'attività desiderata. Utilizzare tale metodo alternativo all'azione InviaTasti per completare le opzioni contenute in una finestra di dialogo.

Se si desidera inviare più di 255 caratteri a Microsoft Access o a un'altra applicazione per Windows, è possibile utilizzare in una macro più azioni InviaTasti in sequenza.

L'utilizzo dell'azione InviaTasti per inviare delle sequenze di tasti attiva gli eventi TastoGiù, TastoSu e Pressione appropriati. L'invio di tasti diversi dallo standard ANSI, quale un tasto funzione, non attiva l'evento Pressione.

Tale azione non è disponibile da Visual Basic. Utilizzare invece l'istruzione SendKeys.
