Metodo OpenReport

                 

Il metodo OpenReport esegue l'azione ApriReport in Visual Basic. Per ulteriori informazioni sul funzionamento dell'azione e dei relativi argomenti, vedere l'argomento della Guida in linea relativo all'azione.

Sintassi

DoCmd.OpenReport nomereport[, visualizzazione][, nomefiltro][, condizioneWHERE]

Il metodo OpenReport utilizza gli argomenti riportati di seguito.

Argomento Descrizione
nomereport Espressione stringa che è il nome valido di un report contenuto nel database corrente.
  Se si esegue del codice di Visual Basic contenente il metodo OpenReport in un database di libreria, Microsoft Access cerca il report con questo nome prima nel database di libreria, quindi nel database corrente.
visualizzazione Una delle seguenti costanti intrinseche:
  acViewDesign
acViewNormal (impostazione predefinita)
acViewPreview
  acViewNormal stampa il report immediatamente. Se si omette questo argomento, viene utilizzata la costante predefinita (acViewNormal).
nomefiltro Espressione stringa che è il nome valido di una query contenuta nel database corrente.
condizioneWHERE Espressione stringa che è una proposizione SQL WHERE valida senza la parola WHERE.

Osservazioni

L'argomento condizioneWHERE può avere una lunghezza massima di 32.768 caratteri, a differenza dell'argomento dell'azione condizione WHERE nella finestra della macro, che può avere una lunghezza massima di 256 caratteri.

È possibile omettere un argomento facoltativo all'interno della sintassi, ma è necessario includere la virgola dell'argomento stesso . Se si omettono uno o più argomenti finali, non utilizzare la virgola dopo l'ultimo argomento specificato.
