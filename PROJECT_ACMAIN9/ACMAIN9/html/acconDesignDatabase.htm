Informazioni sulla strutturazione di un database

Prima di utilizzare Microsoft Access per la creazione delle tabelle, delle maschere e degli altri oggetti che andranno a costituire il database, è importante dedicare del tempo alla strutturazione di quest'ultimo. Sia che si utilizzi un database di Microsoft Access o un progetto di Microsoft Access, una struttura efficiente è fondamentale per la creazione di un database efficiente, accurato e funzionale.

Per ulteriori informazioni, vedere l'argomento relativo a definizione e funzionamento dei database di Access.

1  Determinare lo scopo del database

Il primo passo per la strutturazione di un database consiste nel determinarne lo scopo e l'utilizzo. È necessario stabilire quali informazioni si desidera ottenere dal database, in modo da poter determinare gli argomenti per cui si desidera memorizzare i dati (le tabelle) e i dati che si desidera memorizzare per ciascun argomento (i campi delle tabelle).

Consultare gli utenti che utilizzeranno il database per individuare le domande alle quali il database dovrà rispondere. Abbozzare i report che si desidera produrre. Raccogliere le maschere correntemente utilizzate per registrare i dati. Esaminare altri database ben strutturati e simili a quello che si intende strutturare.

2  Determinare le tabelle necessarie nel database

La determinazione delle tabelle può rappresentare il passo più importante di tutto il processo di strutturazione del database in quanto i risultati desiderati, ossia i report da stampare, le maschere da utilizzare, le domande a cui si richiede una risposta, non forniscono necessariamente le indicazioni sulla struttura delle tabelle che dovranno produrli.

Non è necessario strutturare le tabelle con Microsoft Access. In realtà può risultare più efficace abbozzare ed elaborare la struttura prima su carta. Durante questa operazione è consigliabile suddividere le informazioni tenendo presente i principi di strutturazione fondamentali riportati di seguito.

  • Una tabella non dovrebbe contenere informazioni duplicate e le stesse informazioni non dovrebbero comparire in più tabelle. È questa la differenza tra una tabella di un database relazionale e una tabella di un'applicazione non relazionale, come un foglio di calcolo.

    Se ogni informazione è memorizzata in una sola tabella, è possibile aggiornarla in un'unica posizione. Ciò risulta più funzionale e inoltre impedisce che voci duplicate contengano informazioni diverse. È preferibile ad esempio memorizzare l'indirizzo e il numero telefonico di ciascun cliente una sola volta e in un'unica tabella.

  • Ciascuna tabella deve contenere informazioni relative a un solo argomento.

    Se ciascuna tabella contiene dati relativi a un solo argomento, è possibile conservare le informazioni relative a un argomento separatamente da quelle degli altri. Se ad esempio gli indirizzi dei clienti sono memorizzati in una tabella diversa rispetto agli ordini, è possibile eliminare un ordine ma continuare a conservare le informazioni sul cliente.

3  Determinare i campi necessari nelle tabelle

Se ogni tabella contiene informazioni relative allo stesso argomento, ciascun campo di una tabella contiene singoli dati relativi all'argomento della tabella. Una tabella di clienti ad esempio può comprendere i campi per il nome della società, l'indirizzo, la città, lo stato e il numero telefonico. Durante la progettazione dei campi per ciascuna tabella, considerare i seguenti suggerimenti:

  • Correlare ciascun campo direttamente all'argomento della tabella.

  • Non includere dati derivati o calcolati (dati che siano il risultato di un'espressione).

  • Includere tutte le informazioni necessarie.

  • Memorizzare le informazioni specificando le componenti logiche minime (ad esempio Nome e Cognome, piuttosto che solo Nome).

4  Identificare i campi con valori univoci in ciascun record

Per fare in modo che le informazioni memorizzate in diverse tabelle vengano collegate, ad esempio per collegare un cliente a tutti i suoi ordini, è necessario che ciascuna tabella del database comprenda un campo o un set di campi che identifichi in modo univoco ogni singolo record della tabella. Tale campo o set di campi viene definito chiave primaria.

Per ulteriori informazioni, vedere l'argomento relativo alle chiavi primarie in un database di Access.

5  Determinare le relazioni tra le tabelle

Dopo avere suddiviso le informazioni nelle tabelle e avere identificato i campi chiave primaria, è necessario indicare a Microsoft Access in che modo riunire le informazioni correlate in maniera significativa. A tale scopo è necessario definire le relazioni tra le tabelle in un database di Microsoft Access.

Per ulteriori informazioni, vedere l'argomento relativo alle relazioni in un database di Microsoft SQL Server.

Può risultare utile visualizzare le relazioni in un database esistente con ottima strutturazione. Ad esempio aprire il database di esempio Northwind e scegliere Relazioni nel menu Strumenti per vedere le relazioni tra le tabelle. È anche possibile visualizzare le relazioni nel progetto di Access di esempio Northwind e nel database di Microsoft SQL Server.

Per ulteriori informazioni, vedere l'argomento relativo all'integrità referenziale.

6  Perfezionare la struttura

Dopo aver strutturato le tabelle, i campi e le relazioni necessarie, esaminare la struttura e individuare le eventuali imperfezioni rimaste. È più semplice modificare la struttura del database in questa fase, piuttosto che dopo aver inserito i dati nelle tabelle.

Utilizzare Microsoft Access per creare tabelle, specificare le relazioni tra queste ultime e aggiungervi dati di esempi sufficienti che consentano di verificare la struttura. Per verificare le relazioni nel database, controllare se è possibile creare delle query per ottenere le risposte desiderate. Creare bozze delle maschere e dei report e verificare se visualizzano i dati desiderati. Ricercare eventuali dati ripetuti ed eliminarli. Se si rilevano dei problemi, perfezionare la struttura.

7  Aggiungere dati e creare altri oggetti di database

Una volta constatato che la struttura della tabella risponde ai principi illustrati, è necessario procedere all'aggiunta di tutti i dati esistenti nelle tabelle. È possibile creare tutte le query, le maschere, i report, le pagine di accesso ai dati, le macro e i moduli desiderati.

8  Utilizzare gli strumenti di analisi di Microsoft Access

In Microsoft Access sono disponibili due strumenti che consentono di perfezionare la struttura del database di Microsoft Access. La Creazione guidata Analizzatore tabelle è in grado di analizzare una tabella alla volta, di proporre nuove strutture o relazioni, se appropriate, e di dividere una tabella in nuove tabelle correlate, se opportuno. Per informazioni sull'esecuzione della Creazione guidata Analizzatore tabelle, fare clic su .

L'Analizzatore di prestazioni è in grado di analizzare l'intero database e di fornire consigli e suggerimenti per migliorarlo, consigli e suggerimenti che in seguito possono essere anche implementati dalla creazione guidata. Per informazioni sull'utilizzo dell'Analizzatore di prestazioni, fare clic su .

Per ulteriori informazioni e suggerimenti sulla strutturazione di un database di Microsoft Access, esaminare la struttura del database di esempio Northwind o uno dei database che è possibile creare con la Creazione guidata Database. Per informazioni sull'utilizzo della Creazione guidata Database, fare clic su .

Per ulteriori informazioni su come analizzare il database di Microsoft SQL Server per migliorarne le prestazioni, vedere l'argomento relativo alla documentazione di Microsoft SQL Server.
