Proprietà InputParameters (Parametri di input)

                 

È possibile utilizzare la proprietà InputParameters per specificare o determinare i parametri di input da passare a un'istruzione SQL nella proprietà RecordSource di una maschera, di un report o di una stored procedure quando utilizzata come origine record all'interno di un progetto di Microsoft Access (adp).

Impostazione

<nome> <tipo> = <espressione> [,<nome> <tipo> = <espressione> ]..

La proprietà InputParameters ha i seguenti argomenti.

Argomento Descrizione
nome Il nome del parametro. Si tratta di un parametro di stored procedure che richiede il simbolo iniziale @. Nei nomi di parametro non sono ammessi spazi.
tipo Tipo dati SQL Server del parametro.
espressione Un'espressione di Microsoft Access passata alla funzione per espressioni perché venga risolta. Come le altre espressioni, essa può fare riferimento a un altro controllo della maschera, può fornire un testo di richiesta di immissione da parte dell'utente o può essere una stringa letterale.
  Se il segno di uguale (=) è l'ultimo o è seguito dalla virgola, la routine viene eseguita utilizzando il valore di parametro predefinito indicato nella query o nel comando CREATE PROC. Questo è vero anche se il parametro non è incluso nella stringa.

È possibile impostare questa proprietà utilizzando la finestra delle proprietà o Visual Basic.

Osservazioni

Utilizzo con la proprietà RecordSource:

Di seguito viene riportata una stringa di esempio della proprietà InputParameter utilizzata con un'istruzione SQL nella proprietà RecordSource.

stato char=[Maschera]![ElencoStati], annodivendita smallint=[Immettere l'anno di interesse]

Il parametro stato viene impostato al valore corrente del controllo ElencoStati e all'utente viene richiesto il parametro annodivendita. Ai valori dei parametri eventualmente non presenti nell'elenco vengono assegnati i valori predefiniti.

È necessario eseguire la query con un segno ? per ciascun parametro non predefinito presente nell'elenco di InputParameter.

Un comando di aggiornamento o requery eseguito tramite menu, tastiera o barra di spostamento in Access dovrebbe attivare una riesecuzione della query. Questa operazione può essere effettuata da codice chiamando il metodo Recordset.Requery standard. Se il valore di un parametro è associato a un controllo sulla maschera, il valore corrente del controllo verrà utilizzato alla riesecuzione della query. La query non viene rieseguita automaticamente alla modifica del valore del controllo.

Utilizzo con le stored procedure:

Di seguito viene riportata una stringa di esempio della proprietà InputParameter utilizzata con una stored procedure:

@stato char=[Maschera]![ElencoStati], @annodivendita smallint=[Immettere l'anno di interesse]

In questo modo si otterrebbe l'impostazione del parametro @stato al valore corrente del controllo ElencoStati e all'utente verrebbe richiesto il parametro @annodivendita. In presenza di altri parametri della stored procedure non inclusi in questo elenco, essi otterrebbero i valori predefiniti assegnati.

La stored procedure dovrà essere eseguita usando una stringa di comando contenente la sintassi {call } con un indicatore ? per ogni parametro non predefinito inlcuso nell'elenco InputParameter.

Un comando di aggiornamento o requery eseguito tramite menu, tastiera o barra di spostamento in Access dovrebbe attivare la riesecuzione della stored procedure. Gli utenti potrebbero ottenere la stessa azione con il codice, chiamando il metodo standard Recordset.Requery. Se il valore di un parametro è associato a un controllo sulla maschera, al momento della requery viene utilizzato il valore corrente del controllo. La stored procedure non viene rieseguita automaticamente al variare del valore del controllo.

Questa finestra di dialogo di generatore viene richiamata la prima volta che una stored procedure viene selezionata come origine record di un maschera se la stored procedure presenta dei parametri. Dopo la creazione iniziale della stringa InputParameters, la stessa finestra di dialogo viene utilizzata come generatore per modificare la stringa. In questo caso, però, l'elenco dei parametri deriva dal contenuto già presente nella stringa.

È anche possibile impostare i valori dei parametri con il codice, utilizzando gli oggetti ADO Command e Parameter. Se il risultato restituisce un set di risultati, è possibile associare a quest'ultimo una maschera impostando la proprietà Recordset della maschera. Il codice ADO rappresenta l'unico modo di gestire stored procedure che non restituiscono set di risultati come query di azione, quelle che restituiscono parametri di output o quelle che restituiscono più set di risultati.
