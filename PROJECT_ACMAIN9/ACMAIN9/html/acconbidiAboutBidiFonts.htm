About bidirectional fonts

Informazioni specifiche sulla lingua

To enable the right-to-left features in Microsoft Access 2000, you must be running a 32-bit Microsoft Windows operating system that has right-to-left support — for example, the Arabic version of Microsoft Windows 95. Learn about system or special requirements for right-to-left languages.

Any version of Access 2000 with left-to-right support is designed to display and print both right-to-left and left-to-right text. To do this, the program should use a font that contains both right-to-left and left-to-right language characters. Most fonts available with bidirectional Windows operating systems contain both right-to-left and left-to-right characters. If you choose a font that does not contain both right-to-left and left-to-right characters, existing text in one or both languages might be displayed incorrectly by using the default secondary font, Tahoma. To correct this problem, choose a different font.
