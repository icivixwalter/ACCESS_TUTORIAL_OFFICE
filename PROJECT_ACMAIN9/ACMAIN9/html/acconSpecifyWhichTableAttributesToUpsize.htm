Scegliere gli attributi di tabella su cui eseguire l'upsize

È possibile selezionare gli attributi di tabella su cui eseguire l'upsize nel database di Microsoft SQL Server. L'Upsize guidato converte come minimo i nomi di campo di Access in nomi di campo di SQL Server validi e i tipi di dati di Access nei tipi di dati equivalenti di SQL Server. In base all'impostazione predefinita, vengono selezionati tutti gli attributi per l'upsize. In questo argomento verranno fornite informazioni di riferimento relative alle seguenti operazioni:

Eseguire l'upsize degli indici

Eseguire l'upsize delle regole di convalida

Eseguire l'upsize delle impostazioni predefinite

Eseguire l'upsize delle relazioni tra le tabelle

Aggiungere campi data e ora alle tabelle

Scegliere di non eseguire l'upsize dei dati

Eseguire l'upsize degli indici

Se si seleziona la casella di controllo Indici, l'Upsize guidato esegue l'upsize di tutti gli indici.

L'Upsize guidato converte le chiavi primarie di Microsoft Access in indici univoci non raggruppati di Microsoft SQL Server e li contrassegna come chiavi primarie di SQL Server. Se si desidera collegare la tabella di SQL Server di cui è stato eseguito l'upsize al database di Access, l'Upsize guidato aggiunge anche il prefisso "aaaaa". Poiché viene scelto come chiave primaria il primo indice in ordine alfabetico nell'elenco degli indici disponibili, il prefisso "aaaaa" assicura la scelta dell'indice corretto.

Tutti gli altri indici conservano il proprio nome, a eccezione di quelli contenenti caratteri non validi, che vengono sostituiti dal carattere "_". Gli indici di Microsoft Access univoci o non univoci diventano indici di SQL Server univoci o non univoci. È opportuno tenere presente che in SQL Server non sono supportati gli indici in ordine crescente o decrescente.

Una tabella collegata deve disporre di un indice univoco aggiornabile in Microsoft Access. L'Upsize guidato è in grado di eseguire l'upsize di un indice univoco esistente, ma non di crearne uno. Per poter aggiornare le tabelle, accertarsi di aggiungere un indice univoco a ciascuna tabella di Access prima di eseguire l'upsize.

Eseguire l'upsize delle regole di convalida

Se si seleziona la casella di controllo Regole di convalida, l'Upsize guidato esegue l'upsize di tutte le regole di convalida di tabelle, record e campi e delle proprietà Richiesto dei campi come trigger di aggiornamento e inserimento.

Un trigger è costituito da una serie di istruzioni Transact-SQL associate a una tabella SQL Server. Una tabella può avere tre trigger, uno per ciascuno dei comandi che possono modificare i dati della tabella: UPDATE, INSERT e DELETE. Il trigger viene eseguito automaticamente con l'esecuzione del comando. L'Upsize guidato utilizza i trigger invece delle regole di SQL Server per applicare la convalida a livello di campo poiché le regole di SQL Server non consentono di visualizzare i messaggi di errore personalizzati.

Le regole di convalida non hanno necessariamente una corrispondenza uno-a-uno con un trigger. È possibile che una regola di convalida diventi parte di più trigger oppure che ciascun trigger contenga codice per l'emulazione delle funzionalità di più regole di convalida.

Quando si imposta la proprietà Richiesto di un campo di Microsoft Access a Vero, l'utente non può inserire un record e lasciare il campo richiesto nullo se non esiste alcuna associazione predefinita al campo, né può assegnare un valore nullo quando si aggiorna un record. I campi richiesti vengono sottoposti all'upsize e trasformati in campi che non accettano valori nulli (Null) su SQL Server. L'Upsize guidato crea inoltre un blocco di codice Transact-SQL che visualizza un messaggio simile a quello visualizzato da Microsoft Access se viene rilevato un campo nullo quando si aggiunge o si aggiorna un record.

Eseguire l'upsize delle impostazioni predefinite

Se si seleziona la casella di controllo Predefiniti, l'Upsize guidato esegue l'upsize di tutti i valori predefiniti come valori predefiniti ANSI (American National Standards Institute), non come oggetti predefiniti associati al campo SQL Server appropriato. Le impostazioni predefinite di Microsoft SQL Server, a differenza di quelle di Microsoft Access, sono indipendenti da campi e tabelle. Una volta creata un'impostazione predefinita, è possibile associarla a un numero indefinito di campi diversi.

Eseguire l'upsize delle relazioni tra le tabelle

È possibile decidere come eseguire l'upsize delle relazioni tra le tabelle e dell'integrità referenziale utilizzando i trigger di aggiornamento, inserimento o eliminazione oppure l'Integrità referenziale integrativa (DRI, Declarative Referential Integrity).

L'Integrità referenziale integrativa funziona allo stesso modo dell'integrità referenziale di Microsoft Access in quanto definisce i vincoli della chiave primaria per le tabelle di base (la parte "uno" di una relazione uno-a-molti) e i vincoli delle chiavi esterne per le tabelle esterne (generalmente la parte "molti" di una relazione uno-a-molti). L'Integrità referenziale dichiarativa di SQL Server non dispone tuttavia delle funzioni di aggiornamento o eliminazione a catena di Microsoft Access.

  • Usa trigger   Se per le relazioni tra le tabelle di Microsoft Access sono stati definiti aggiornamenti o eliminazioni a catena e si desidera conservare questa funzione dopo l'upsize delle tabelle, selezionare le caselle di controllo Relazioni della tabella e Usa trigger per eseguire l'upsize degli aggiornamenti o eliminazioni a catena come trigger per l'applicazione dell'integrità referenziale.

    Una relazione tra tabelle non deve necessariamente avere una corrispondenza uno-a-uno con un trigger. Ciascuna relazione può diventare parte di più trigger oppure ciascun trigger può contenere codice per l'emulazione delle funzionalità di più regole di integrità referenziale. I trigger di inserimento vengono utilizzati nelle tabelle secondarie e quelli di eliminazione nelle tabelle principali.

    Nota   Lo strumento di progettazione di Microsoft SQL Server necessita dell'Integrità referenziale dichiarativa (DRI) per leggere il diagramma di database di un database di SQL Server. Per consentire la lettura dei diagrammi del database da parte degli strumenti di progettazione di SQL Server, contemporaneamente all'applicazione dell'integrità referenziale mediante i trigger, l'Upsize guidato colloca la DRI sulle relazioni disattivando però il controllo dei vincoli delle chiavi esterne.

  • Usa DRI   Se non sono state definite operazioni di aggiornamento o eliminazione a catena nel database oppure non si desidera conservare queste funzioni nel database di cui si esegue l'upsize, selezionare le caselle di controllo Relazioni della tabella e Usa DRI per utilizzare la DRI per l'applicazione dell'integrità referenziale. In questo caso un messaggio di avvertimento informa che si perderanno le funzionalità di aggiornamento ed eliminazione a catena.

Aggiungere campi data e ora alle tabelle

In Microsoft SQL Server vengono utilizzati i campi data e ora per indicare che un record è stato modificato (ma non quando sono state apportate le modifiche) creando un campo di valore univoco e aggiornando tale campo ogni volta che viene aggiornato un record. Per una tabella collegata, il valore dei campi data e ora viene utilizzato da Access prima di aggiornare un record per determinare se è stato modificato. Un campo data e ora in genere garantisce ottime prestazioni e affidabilità. Senza un campo di questo tipo SQL Server deve controllare tutti i campi del record per determinare se sono state apportate modifiche, con conseguente rallentamento delle prestazioni.

  • Se si seleziona il valore predefinito Sì, determinazione automatica, l'Upsize guidato crea nuovi campi con tipo di dati data e ora nelle tabelle SQL Server di cui è stato eseguito l'upsize dalle tabelle di Microsoft Access contenenti campi con virgola mobile (semplice o doppia), Memo o Oggetto OLE.

  • È anche possibile fare in modo che l'Upsize guidato crei un campo data e ora per tutte le tabelle di cui si è eseguito l'upsize indipendentemente dai tipi di campi in esse contenuti selezionando Sì, sempre. Questa opzione ottimizza le prestazioni delle tabelle di Access di cui si è eseguito l'upsize che non contengono campi Memo, Oggetto OLE o con virgola mobile, bensì campi di altro tipo.

Nota   Nelle tabelle SQL Server collegate non viene eseguito alcun controllo da Microsoft Access per determinare se i campi Memo o Oggetto OLE sono stati modificati perché questi campi possono essere di dimensioni molto elevate e il confronto potrebbe impiegare troppe risorse di rete e troppo tempo. Se è stato modificato solo un campo di testo o immagine e non è presente alcun campo data e ora, la modifica verrà sovrascritta. Il valore di un campo con virgola mobile inoltre potrebbe apparire modificato anche se non lo è e, in assenza di un campo data e ora, il record potrebbe essere rilevato come modificato anche se non è stata apportata alcuna modifica.

Scegliere di non eseguire l'upsize dei dati

In base all'impostazione predefinita, l'Upsize guidato esegue l'upsize di tutti i dati su SQL Server. Se si seleziona la casella di controllo Crea solo la struttura della tabella e non eseguire l'upsize dei dati, viene eseguito l'upsize solo della struttura dei dati.
