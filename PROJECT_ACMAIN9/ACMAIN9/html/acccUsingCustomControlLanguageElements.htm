Informazioni su proprietà, metodi ed eventi di un controllo ActiveX

                 

In questo argomento vengono fornite informazioni su come impostare e utilizzare gli elementi di linguaggio dei controlli ActiveX in Microsoft Access 2000 e vengono indicate alcune caratteristiche dei controlli ActiveX che sono stati aggiunti in Microsoft Access 95 e Microsoft Access 97.

Nota   Queste informazioni non si applicano ai controlli ActiveX in una pagina di accesso ai dati.

Considerazioni generali

L'utilizzo dei controlli ActiveX con Microsoft Access 95 ha eliminato la necessità di utilizzare la proprietà Object per fare riferimento a proprietà e metodi nelle espressioni di sintassi. Le due righe di sintassi riportate di seguito ad esempio sono divenute equivalenti.

ActiveXCtl0.Object.MonthLength = 0
ActiveXCtl0.MonthLength = 0

Se tuttavia il nome dell'elemento di linguaggio del controllo ActiveX è uguale a quello di un elemento di linguaggio di Microsoft Access esistente che si applica anche a questo controllo ActiveX, è necessario utilizzare la sintassi .Object.

Le variabili dei controlli ActiveX possono avere dimensioni solo di tipo Controllo e non di tipo più specifico, come ad esempio Calendario.

Proprietà

Le proprietà dei controlli ActiveX vengono visualizzate nella finestra delle proprietà di Microsoft Access. La maggior parte di queste proprietà può essere impostata sia nella finestra di dialogo delle proprietà personalizzata del controllo ActiveX, sia nella finestra delle proprietà di Microsoft Access.

Nota   Le proprietà del controllo ActiveX non sono elencate nell'insieme Properties ADO o Properties DAO. L'unico modo per accedere a queste proprietà è utilizzare la sintassi riportata nella sezione precedente.

Eventi

Per gli eventi di Microsoft Access, come ad esempio Invio, che si applicano a controlli ActiveX, è possibile eseguire una macro o una routine evento quando si verifica l'evento impostando la relativa proprietà al nome della macro oppure a [Routine evento]. Per gli eventi specifici dei controlli ActiveX, quali l'evento NewMonth del Controllo calendario, è tuttavia necessario passare al modulo della maschera in visualizzazione Struttura, selezionare il nome del controllo ActiveX dalla casella Oggetto, quindi selezionare l'evento dalla casella Routine. In tal modo si crea una routine evento che viene eseguita quando si verifica l'evento. Se inoltre si desidera creare manualmente la sintassi creata in questo caso per la routine evento, è possibile duplicarla.

Se un evento di Microsoft Access e un evento di un controllo ActiveX con lo stesso nome si applicano entrambi a un oggetto di un controllo ActiveX, Microsoft Access accoda "Oggetto" all'evento del controllo ActiveX nell'elenco a discesa della casella Routine. In tal modo ogni volta che si verifica un evento è possibile eseguire una routine differente.
