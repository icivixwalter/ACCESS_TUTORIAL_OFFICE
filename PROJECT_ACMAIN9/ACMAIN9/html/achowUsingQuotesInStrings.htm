Virgolette nelle stringhe

        

Quando è necessario costruire delle stringhe da concatenare, è possibile che si debba incorporare una stringa o una variabile in una stringa. È possibile dover nidificare una stringa in un'altra nei casi riportati di seguito:

  • quando si specificano criteri per le funzioni di aggregazione sui domini

  • quando si specificano criteri per i metodi Find

  • quando si specificano criteri per la proprietà Filtro o Filtro su server di una maschera

  • quando si costruiscono stringhe SQL

In tutti questi casi Microsoft Access deve passare una stringa al modulo di gestione di database Microsoft Jet. Quando ad esempio viene specificato un argomento criteri per una funzione di aggregazione sui domini, Microsoft Access deve valutare tutte le variabili, concatenarle in una stringa e passare l'intera stringa al modulo di gestione di database Microsoft Jet.

Se viene incorporata una variabile numerica, questa viene valutata e il relativo valore viene concatenato nella stringa. Se la variabile è una stringa di testo, la stringa criteri risultante contiene una stringa incorporata in un'altra. Una stringa all'interno di una stringa deve essere identificata da delimitatori di stringa, altrimenti il modulo di gestione di database Microsoft Jet non può determinare quale parte della stringa è il valore da utilizzare.

I delimitatori di stringa non fanno parte della variabile, ma devono essere inclusi nella stringa nell'argomento criteri. Sono disponibili tre metodi per costruire la stringa nell'argomento criteri, ciascuno dei quali produce un argomento simile a uno dei seguenti esempi:

"[Cognome] = 'Rossi'"

– oppure –

"[Cognome] = ""Rossi"""

Inclusione di apici

Nell'argomento criteri l'inclusione di apici deve essere effettuata in modo che quando il valore della variabile viene concatenato alla stringa è racchiuso tra apici. Se ad esempio l'argomento criteri deve contenere una variabile stringa denominata strNome, è possibile costruire l'argomento criteri come nell'esempio seguente:

"[Cognome] = '" & strNome & "'"

Quando la variabile strNome viene valutata e concatenata nella stringa criteri, quest'ultima diventa:

"[Cognome] = 'Rossi'"

Nota   Tale sintassi non consente l'utilizzo di apostrofi all'interno del valore della variabile. Se il valore della variabile di stringa comprende un apostrofo ('), viene generato un errore di run-time. Se la variabile può rappresentare valori contenenti apostrofi, tenere in considerazione l'utilizzo di una delle forme di sintassi descritte di seguito.

Inclusione di virgolette doppie

Nell'argomento criteri l'inclusione di virgolette doppie deve essere effettuata in modo che, quando il valore della variabile viene valutato, viene racchiuso tra virgolette. All'interno di una stringa è necessario utilizzare due set di virgolette doppie per rappresentare un set singolo di virgolette doppie. È possibile costruire l'argomento criteri come nell'esempio seguente:

"[Cognome] = """ & strNome & """"

Quando viene valutata la variabile strNome e concatenata nell'argomento criteri, ciascun set di due virgolette doppie viene sostituito da un solo segno di virgolette doppie. L'argomento criteri diventa:

"[Cognome] = Rossi"

Tale sintassi potrebbe risultare più complessa dell'utilizzo dell'apice, ma consente di incorporare una stringa contenente un apostrofo all'interno di una argomento criteri e permette inoltre di nidificare una o più stringhe all'interno di una stringa incorporata.

Inclusione di una variabile che rappresenta le virgolette doppie

È possibile costruire una variabile di stringa che rappresenta le virgolette doppie e concatenare tale variabile nell'argomento criteri insieme al valore della variabile. La rappresentazione ANSI per le virgolette doppie e Chr$(34). È possibile assegnare tale valore a una variabile di stringa denominata strVirgolette. È quindi possibile costruire l'argomento criteri come nell'esempio seguente:

"[Cognome] = " & strVirgolette & strNome & strVirgolette

Quando le variabili vengono valutate e concatenate nell'argomento criteri, questo appare come nell'esempio seguente:

[Cognome] = "Rossi"
