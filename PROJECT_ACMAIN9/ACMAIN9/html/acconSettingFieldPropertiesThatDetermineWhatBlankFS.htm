Proprietà che controllano la gestione dei campi vuoti

È possibile controllare la modalità di gestione dei campi vuoti impostando combinazioni diverse delle proprietà Richiesto e Consenti lunghezza zero. La proprietà Consenti lunghezza zero è disponibile solo per i campi Testo, Memo o Collegamento ipertestuale. La proprietà Richiesto determina se deve essere effettuata un'immissione. Se la proprietà Consenti lunghezza zero è impostata a Sì, verranno differenziati due tipi di valore vuoti: valori Null e stringhe a lunghezza zero.

Differenza tra valori Null e stringhe a lunghezza zero

In Microsoft Access è possibile distinguere tra due tipi di valori vuoti. In alcuni casi un campo viene lasciato vuoto perché non si conoscono le informazioni a esso relative o perché non è valido per il record. Se ad esempio una tabella ha un campo Numero, è possibile lasciare questo campo vuoto se non si conosce con sicurezza il numero di fax di un cliente o se ne possiede uno. In questo caso, se si lascia il campo vuoto, verrà immesso un valore Null che indica la non conoscenza delle informazioni. Se si immette una stringa a lunghezza zero digitando (" "), per il campo non esiste alcun valore.

Quando consentire valori Null in un campo

Se si desidera lasciare un campo vuoto senza determinarne la ragione, impostare sia la proprietà Richiesto che la proprietà Consenti lunghezza zero a No. Questa è l'impostazione predefinita quando si crea un nuovo campo Testo, Memo o Collegamento ipertestuale.

Quando non consentire né valori Null né stringhe a lunghezza zero in un campo

Se si desidera che non sia mai possibile lasciare un campo vuoto, impostare la proprietà Richiesto a Sì e la proprietà Consenti lunghezza zero a No.

Quando consentire sia valori Null che stringhe a lunghezza zero in un campo

Se si desidera poter fare distinzione tra un campo vuoto perché non si conoscono le informazioni e un campo vuoto perché non si applica al record, impostare la proprietà Richiesto a No e la proprietà Consenti lunghezza zero a Sì.

In questo caso, quando si aggiunge un record, se non si conoscono le informazioni il campo viene lasciato vuoto (quindi viene immesso un valore Null). D'altro canto, se il campo non si applica al record corrente, vengono digitate le virgolette (" ") non separate da alcuno spazio per immettere una stringa a lunghezza zero.

Quando consentire solo stringhe a lunghezza zero o alcuni valori in un campo

Se si desidera solo un campo vuoto se si sa che non è applicabile a un record, impostare sia la proprietà Richiesto che la proprietà Consenti lunghezza zero a Sì.

In questo caso l'unico modo per lasciare un campo vuoto è digitare le virgolette non separate da alcuno spazio oppure premere la barra spaziatrice per immettere una stringa a lunghezza zero.

Distinzione tra i valori Null e le stringhe a lunghezza zero

Quando si visualizzano i dati in un campo contenente sia valori Null che stringhe a lunghezza zero, i campi hanno lo stesso aspetto, ossia non contengono alcun valore. È possibile distinguere i valori Null dalle stringhe a lunghezza zero utilizzando un'espressione in un campo query o nell'origine di un controllo non associato in una maschera, un report o una pagina di accesso ai dati. L'espressione seguente restituisce "Sconosciuto" se il campo contiene Null e "ZLS" se il campo contiene una stringa a lunghezza zero. Altrimenti l'espressione restituisce il valore contenuto in quel campo.

=IIf(IsNull([nomecampo]),”Sconosciuto”,Format([nomecampo],”@;\ZLS”))

Ricerca di stringhe a lunghezza zero o valori Null

Per ricercare stringhe a lunghezza zero e valori Null è possibile utilizzare il comando Trova del menu Modifica. In visualizzazione Foglio dati o visualizzazione Maschera selezionare il campo in cui si desidera effettuare la ricerca. Nella casella Trova nel campo digitare Null per ricercare i valori Null oppure digitare le virgolette (" ") non separate da alcuno spazio per ricercare le stringhe a lunghezza zero. Dalla casella Confronta selezionare Campo intero e assicurarsi che la casella di controllo Cerca in campi come formattati non sia selezionata.

Per ulteriori informazioni sull'utilizzo del comando Trova, fare clic su .
