Proprietà LinkChildFields, LinkMasterFields (Collega campi secondari, Collega campi master)

                 

È possibile utilizzare insieme le proprietà LinkChildFields e LinkMasterFields per specificare il modo in cui Microsoft Access collega i record di una maschera o di un report ai record di una sottomaschera, di un sottoreport o di un oggetto incorporato, come un grafico. Se si impostano tali proprietà, Microsoft Access aggiorna automaticamente i record correlati della sottomaschera quando si passa a un nuovo record di una maschera principale.

Impostazione

È possibile impostare le proprietà LinkChildFields e LinkMasterFields per la sottomaschera, il sottoreport o l'oggetto incorporato, come indicato di seguito:

  • Proprietà LinkChildFields. Immettere il nome di uno o più campi di collegamento nella sottomaschera, nel sottoreport o nell'oggetto incorporato.

  • Proprietà LinkMasterFields. Immettere il nome di uno o più campi di collegamento o di controlli nella maschera principale o nel report.

Per impostare tali proprietà, è possibile utilizzare Collegamento campi sottomaschera o Collegamento campi sottorecord facendo clic sul pulsante dei generatori a destra della proprietà nella finestra delle proprietà.

È possibile impostare queste proprietà anche utilizzando un'espressione stringa in una macro o Visual Basic.

È possibile impostare tali proprietà solo in visualizzazione Struttura o durante l'evento Apertura di una maschera o di un report.

Osservazioni

Non è necessario che i campi o i controlli utilizzati per impostare tali proprietà abbiano gli stessi nomi, ma è necessario che contengano lo stesso tipo di dati e che abbiano un tipo di dati e la dimensione del campo uguali o compatibili. Un campo Contatore è compatibile con un campo Numerico se la proprietà FieldSize relativa al campo Numerico è impostata a Intero lungo.

È possibile utilizzare il nome del controllo, compreso il nome di un controllo calcolato, per impostare la proprietà LinkMasterFields, ma non è possibile utilizzare il nome di un controllo per impostare la proprietà LinkChildFields. Se si desidera utilizzare un valore calcolato come collegamento per una sottomaschera, un sottoreport o un oggetto incorporato, definire un campo calcolato nella query sottostante dell'oggetto secondario e impostare al campo la proprietà LinkChildFields.

Quando si specifica più di un nome di campo o di controllo per le impostazioni di tali proprietà, è necessario immettere lo stesso numero di campi o di controlli per ciascuna impostazione della proprietà e separare i nomi con un punto e virgola (;).

Quando si crea una sottomaschera o un sottoreport trascinando una maschera o un report dalla finestra del database su un'altra maschera o su un altro report oppure utilizzando la Creazione guidata Maschera, Microsoft Access imposta automaticamente le proprietà LinkChildFields e LinkMasterFields nel rispetto delle condizioni seguenti:

  • sia la maschera che il report principali che l'oggetto secondario sono basati su tabelle tra le quali è stata definita una relazione utilizzando il comando Relazioni. Microsoft Access utilizza come campi di collegamento i campi relativi alle due tabelle.

  • la maschera o il report principali sono basati su una tabella contenente una chiave primaria e la sottomaschera o il sottoreport sono basati su una tabella o una query contenente un campo con lo stesso nome e un tipo di dati uguale o compatibile come chiave primaria. Microsoft Access utilizza la chiave primaria della tabella sottostante dell'oggetto principale e il campo con lo stesso nome della tabella o query sottostante dell'oggetto secondario come campi di collegamento.

Nota   Non è necessario includere i campi di collegamento nell'oggetto principale o nell'oggetto secondario. Se i campi sono contenuti nelle tabelle o nelle query sottostanti degli oggetti, è possibile utilizzarli per collegare gli oggetti. Quando si utilizza una creazione guidata, Microsoft Access include automaticamente i campi di collegamento.
