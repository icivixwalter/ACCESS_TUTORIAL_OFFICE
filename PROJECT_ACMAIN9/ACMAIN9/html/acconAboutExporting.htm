Definizione di esportazione

L'esportazione è un sistema per inviare i dati e gli oggetti di database come output a un altro database, foglio di calcolo o formato di file per permetterne l'utilizzo da parte di un altro database, applicazione o programma. L'esportazione è simile alle operazioni Copia e Incolla. Per esportare i dati o gli oggetti di database si utilizza generalmente il comando Esporta del menu File, ma sono disponibili anche altri comandi, come descritto di seguito. È possibile esportare i dati in numerosi database, programmi e formati di file supportati.

Invio di oggetti di database tramite posta elettronica

Il comando Destinatario posta elettronica (come allegato) del sottomenu Invia a del menu File consente di inviare l'output di un oggetto di database allegandolo a un messaggio di posta elettronica in vari formati di file, tra cui Microsoft Excel (xls), Rich Text Format (rtf), testo MS-DOS (txt), HTML (html), IDC/HTX, ASP o come pagina di accesso ai dati.

Il comando Destinatario posta elettronica del sottomenu Invia a del menu File consente inoltre di inviare una pagina come corpo di un messaggio di posta elettronica.

Per inviare un oggetto di database tramite posta elettronica, è necessario installare Microsoft Outlook, Microsoft Exchange, Microsoft Mail o un'altra applicazione per la posta elettronica che supporti l'interfaccia MAPI (Messaging Application Programming Interface).

Esportazione in una pagina Web

Vi sono tre modi per utilizzare i dati o gli oggetti di database di Microsoft Access per la creazione di una pagina Web:

  • In un browser Web che supporti l'HTML dinamico 4.0 o versioni successive. In questo caso le pagine non vengono inviate come output, ma vengono create come oggetti di database che contengono un collegamento alla posizione del file HTML corrispondente. È possibile utilizzare le pagine di accesso ai dati per visualizzare, modificare, aggiornare, eliminare, filtrare, raggruppare e ordinare i dati direttamente da un database di Microsoft Access o di Microsoft SQL Server. Per ulteriori informazioni, vedere l'argomento relativo alla creazione delle pagine di accesso ai dati.

  • È possibile inviare come output i file con codice HTML generato dal server, ASP o IDC/HTX da tabelle, query e maschere. I file con codice HTML generato dal server vengono visualizzati sotto forma di tabella in un browser Web. È possibile utilizzare questo tipo di file quando si desidera utilizzare un qualsiasi browser Web, quando i dati vengono modificati spesso e quando è necessario visualizzare i dati di una tabella connessa a un'origine dati ODBC, ma non sono richieste operazioni di aggiornamento o interazione.

  • È possibile creare i file HTML di output da tabelle, query, maschere e report. In un browser Web i report vengono visualizzati sotto forma di report, mentre tabelle, query e maschere vengono visualizzate come un foglio di calcolo. Utilizzare file con codice HTML statico quando è necessario utilizzare un browser Web che supporti l'HTML 3.2 o versioni successive e quando i dati non vengono modificati spesso.

Esportazione o scambio di dati con Microsoft Word

Vi sono quattro modi per utilizzare i dati di Microsoft Access in Microsoft Word:

  • In un database di Access è possibile utilizzare la Creazione guidata Stampa unione di Microsoft Word per creare un documento stampa unione in Microsoft Word 2000 per il collegamento ai dati di Microsoft Access 2000 mediante il comando Unisci a MS Word del sottomenu Collegamenti di Office del menu Strumenti. Una volta stabilito il collegamento, è possibile aprire in qualsiasi momento il documento in Microsoft Word per stampare una nuova serie di lettere tipo o di etichette con i dati correnti di Microsoft Access.

  • Per qualsiasi versione di Microsoft Word è possibile esportare i dati di Microsoft Access in un file di origine dati stampa unione che può essere utilizzato con la funzione di stampa unione di Word.

  • È possibile salvare l'output di un foglio dati, di una maschera o di un report come file Rich Text Format (rtf). Un file Rich Text Format conserva la formattazione, ad esempio il tipo di carattere e gli stili, e può essere aperto in Microsoft Word e in altri programmi di elaborazione testi e desktop publishing di Windows.

  • È possibile salvare l'output di un foglio dati, di una maschera o di un report come file rtf e caricarlo automaticamente in Microsoft Word mediante il comando Pubblica con MS Word del sottomenu Collegamenti di Office del menu Strumenti.

Esportazione in Microsoft Excel o in un altro programma per fogli di calcolo

Vi sono tre modi per utilizzare i dati di Microsoft Access in Microsoft Excel o in un altro programma per fogli di calcolo:

  • È possibile esportare i fogli dati come dati non formattati in Microsoft Excel o in un altro programma per fogli di calcolo.

  • È possibile salvare l'output di un foglio dati, di una maschera o di un report direttamente come foglio di lavoro o file di Excel (xls).

  • È possibile salvare l'output di un foglio dati, di una maschera o di un report come file di Excel (xls) e caricare automaticamente il file in Microsoft Excel mediante il comando Analizza con MS Excel del sottomenu Collegamenti di Office del menu Strumenti.

Nei due casi precedenti è possibile conservare la maggior parte della formattazione, come i tipi di carattere e i colori. I livelli di gruppo dei report vengono salvati come livelli di struttura di Microsoft Excel, mentre la maschera viene salvata come tabella dati.

Automatizzazione delle operazioni di esportazione

Le operazioni di esportazione eseguite con frequenza possono essere semplificate e automatizzate mediante la creazione di una macro o la creazione di una procedura Visual Basic, Applications Edition. Ciò può essere utile ad esempio per l'esportazione dei dati a scadenze regolari o per la creazione di un sito Web con più oggetti di database.
