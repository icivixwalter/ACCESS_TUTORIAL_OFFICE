Proprietà LogMessages (Registra messaggi)

                 

È possibile utilizzare la proprietà LogMessages in una query di tipo pass-through SQL per specificare se i messaggi restituiti da un database SQL siano stati memorizzati in una tabella messaggi del database Microsoft Access corrente.

Nota   La proprietà LogMessages si applica solo a query di tipo pass-through.

Impostazione

La proprietà LogMessages viene impostata come riportato di seguito.

Impostazione Visual Basic Descrizione
Sì True (-1) Microsoft Access memorizza i messaggi restituiti dal database SQL in una tabella messaggi.
No False (0) Impostazione predefinita. Microsoft Access non memorizza i messaggi restituiti dal database SQL.

È possibile impostare tale proprietà utilizzando la finestra delle proprietà della query o Visual Basic.

Osservazioni

Il nome della tabella messaggi in cui vengono memorizzati i messaggi restituiti è nomeutente - nn, dove nomeutente è il nome di accesso dell'utente che esegue la query di tipo pass-through e nn è un valore intero che aumenta di una unità a partire da 00. Se, ad esempio, l'utente SaraG imposta a Sì la proprietà LogMessages e riceve messaggi da un database SQL, la tabella messaggi viene chiamata SaraG - 00. Se SaraG riceve messaggi in una diversa sessione Microsoft Access e la prima tabella non è stata cancellata, viene creata una nuova tabella chiamata SaraG - 01.

Nota   I messaggi di errore provenienti da SQL Server non vengono memorizzati nella tabella messaggi.
