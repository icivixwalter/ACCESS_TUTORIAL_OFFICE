Proprietà FailOnError (Errore)

                 

È possibile utilizzare la proprietà FailOnError per specificare di interrompere una query di aggiornamento o di eliminazione eseguita su una origine dati ODBC se si verifica un errore. La proprietà FailOnError consente di ottimizzare le query di aggiornamento collettive eseguite su origini dati ODBC in modo che vengano eseguite sul server se in caso di errore parziale non si desidera il comportamente del modulo di gestione di database Microsoft Jet.

Nota   La proprietà FailOnError si applica solo alle query di aggiornamento e di eliminazione.

Impostazione

La proprietà FailOnError utilizza le seguenti impostazioni.

Impostazione Visual Basic Descrizione
Sì True (–1) In caso di errore la query viene interrotta.
No False (0) Impostazione predefinita. In caso di errore la query non viene interrotta.

È possibile impostare la proprietà FailOnError utilizzando la finestra delle proprietà della query o Visual Basic. Per impostare questa proprietà in un database di Microsoft Access (MDB) con Visual Basic, occorre prima aggiungerla a un insieme Properties dell'oggetto QueryDef utilizzando il metodo CreateProperty.

Osservazioni

Quando la proprietà FailOnError è impostata a Sì e la query viene ripetuta su un'unica tabella in un database ODBC e non contiene espressioni che non possono essere valutate sul server, la query verrà inviata al server per essere eseguita. Quando la proprietà FailOnError viene impostata a Sì, se si verifica un errore durante un'eliminazione o un aggiornamento, l'esecuzione della query si interrompe e la tabella torna allo stato originario. In tutti gli altri casi, la proprietà FailOnError verrà considerata come se fosse impostata a No. Quando la proprietà FailOnError è impostata a No e si verifica un errore, è possibile accettare i risultati della query o annullare le eventuali modifiche.

Quando la proprietà FailOnError è impostata a No e la query viene eseguita su dati locali, le query di aggiornamento o di eliminazione funzionano esattamente come nelle precedenti versioni di Microsoft Access; quando la query interessa più righe, alcune righe possono essere aggiornate e altre non essere eseguite correttamente a causa di errori. Se si verifica un errore, viene notificata la causa e il numero di record interessati ed è possibile accettare i risultati della query o annullare le eventuali modifiche.

Impostare questa proprietà su Sì equivale a usare la costante intrinseca dbFailOnError del metodo DAO Execute.
