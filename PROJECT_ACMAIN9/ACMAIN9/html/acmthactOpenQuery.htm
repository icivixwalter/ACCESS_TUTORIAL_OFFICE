Metodo OpenQuery

                 

Il metodo OpenQuery esegue l'azione ApriQuery in Visual Basic. Per ulteriori informazioni sul funzionamento dell'azione e dei relativi argomenti, vedere l'argomento della Guida in linea relativo all'azione.

Nota   Questo metodo è disponibile soltanto con database di Microsoft Access (mdb). Vedere i metodi OpenView o OpenStoredProcedure se si lavora con un progetto di Microsoft Access Project (adp).

Sintassi

DoCmd.OpenQuery nomequery[, visualizzazione][, modalitàdati]

Il metodo OpenQuery utilizza gli argomenti riportati di seguito.

Argomento Descrizione
nomequery Espressione stringa che è il nome valido di una query contenuta nel database corrente.
  Se si esegue del codice di Visual Basic contenente il metodo OpenQuery in un database di libreria, Microsoft Access cerca la query con questo nome prima nel database di libreria, quindi nel database corrente.
visualizzazione Una delle seguenti costanti intrinseche:
  acViewDesign
acViewNormal (impostazione predefinita)
acViewPreview
  Se l'argomento nomequery è il nome di una query di selezione, a campi incrociati, di unione o di tipo pass-through la cui proprietà Restituisci Record è impostata a -1, acViewNormal visualizza il set di risultati della query. Se l'argomento nomequery si riferisce a una query di azione, di definizione dati o di tipo pass-through la cui proprietà Restituisci Record è impostata a 0, acViewNormal esegue la query.
  Se si omette questo argomento, viene utilizzata la costante predefinita (acViewNormal).
modalitàdati Una delle seguenti costanti intrinseche:
  acAdd
acEdit (impostazione predefinita)
acReadOnly
  Se si omette questo argomento, viene utilizzata la costante predefinita (acEdit).

Osservazioni

Se si specifica l'argomento modalitàdati e si omette l'argomento visualizzazione, è necessario includere la virgola dell'argomento visualizzazione. Se si omette un argomento finale, non utilizzare la virgola dopo l'ultimo argomento specificato.
